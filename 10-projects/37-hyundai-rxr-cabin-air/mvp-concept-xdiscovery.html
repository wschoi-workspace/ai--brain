Executive Summary

피처 정의가 안 돼 있어도,
고객 언어에서 Core Experience를 뽑아낼 수 있다

X-Discovery의 핵심 전제이자, RXR 파이프라인이 이미 증명한 것

핵심 제안

이미 현대차 고객 데이터(cabin-air 30,398건)로 검증된 RXR 분석 파이프라인을 X-Discovery 엔진으로 확장하여, 3개월 내 MVP를 구현합니다.

What

X-Discovery Engine 아키텍처

4개 레이어 + 향후 확장(Navigator). MVP 범위는 Layer 1~4로 한정합니다
LAYER 1 · 데이터 인제스트
  • 정성 데이터 업로드 (인터뷰/VoC/고객센터)
  • SNS 크롤 (RXR crawl 모듈 재사용)
  • 전처리 + DB 적재
LAYER 2 · RXR 분석 엔진 핵심 차별화
  • Gate 1~4 진정성 필터
  • 토픽 분류 + 감성 분석
  • Latent Desire 도출 (언어분석 + 심리게이지)
  • 정성/정량 교차검증 (Say-Do Gap)
LAYER 3 · 경험 구조화
  • Core Experience 정의
  • 경험 가설 생성
  • User Story 초안 (As a / I want / So that)
LAYER 4 · Human-in-the-Loop
  • 대시보드 (현황 뷰)
  • 채팅/질의 (분석 결과 탐색)
  • 검증/수정 포인트
향후 · Experience Navigator ('27년) MVP 범위 제외
Node Map 시각화 PRD 자동생성 Flow/R&R Chart

As a 출퇴근 운전자 (동승 가족 포함)
I want 시동 직후 공기질이 자동으로 최적화되기를
