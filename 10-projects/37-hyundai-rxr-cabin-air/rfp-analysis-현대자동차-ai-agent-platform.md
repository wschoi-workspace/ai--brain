## Executive Summary

이 프로젝트의 진짜 문제는 "AI 도구 도입"이 아니라 **기획-개발 조직 간 번역 비용을 줄이는 내부 설득 도구 구축**이다. 3개월 MVP가 곧 2027년 풀 프로젝트의 티켓이자 경영진 IR 자료다.

**Winning Point**: cabin-air RXR 분석 파이프라인(정성 인터뷰→테마 추출→구조화)이 X-Discovery 방법론과 구조적으로 동형. "우리는 현대차 고객의 언어를 이미 분석해본 팀"이라는 레퍼런스가 결정적 차별화.

## 5. 핵심 문제 정의

### 표면 vs 근본

```
표면: "AI 에이전트로 User Story를 자동 생성해달라"

실제: 기획자가 만든 아이디어가 개발팀에 도달하기까지
      너무 많은 것이 소실된다.

      기획자 → [번역 비용] → 개발자 → [해석 오류] → 출시 지연

근본: "기획 언어와 개발 언어 사이의 번역기"가 필요하다.
      AI 에이전트는 그 번역기의 형태일 뿐.
```

### 성공 조건
- 기획자가 "이제 개발팀 설득이 쉬워졌다"고 느낄 것
- 도구 성능이 아닌 **워크플로우 마찰 감소**가 실제 가치

### 잠재 모순

| 모순 | 상충 지점 | 대응 |
|------|---------|------|
| "전사 도구" vs "단일 주제 PoC" 🔍P7 | 범위 정의 필요 | kick-off 시 스코프 협상 선제 |
| "재현성" vs LLM 비결정성 🔍P3 | 기술 제약 | Temperature=0 + 캐싱 + 검증레이어 |
| "3개월" vs "프로토타입 2개 + X-Discovery 전체" | 일정 극히 타이트 | Sprint별 점진 MVP 제안 |

## 6. 전략적 차별화 & Winning Point

### 핵심 연계: RXR ↔ X-Discovery

```
cabin-air 프로젝트 (이미 수행):
  현대차 고객 정성 데이터 → 진정성 필터(Gate4) → 테마 추출 → 욕구 5종 → 구조화

X-Discovery 요구:
  고객 인터뷰 102건 → 잠재욕구 이해모델 → Core Experience 도출

= 방법론 동형 (같은 구조 + 이미 현대차 데이터로 검증됨)
```

### 포지셔닝

| 경쟁사 예상 포지션 | 프로젝트렌트+RXR |
|-------------------|----------------|
| 대형 SI (삼성SDS류) — 플랫폼 위주 | 방법론 + 구현 이원화 |
| AI 스타트업 — 기술 위주 | 현대차 컨텍스트 사전 이해 |
| 컨설팅 펌 — 프레임워크 위주 | 실행 가능 프로토타입까지 |

### Winning Message
> "우리는 현대차 고객의 언어를 이미 분석해본 팀이다. X-Discovery는 그 경험의 스케일업이다."

### cabin-air 시너지 구체항목
- **데이터 처리**: Gate4 광고필터 → VoC 전처리 로직에 즉시 활용
- **Latent Desire 모델**: 욕구 5카테고리 도출 방법론 → 언어분석 기반 기초
- **교차검증**: 정성(3,586건)→정량(합성) 교차분석 → 동일 패턴
- **User Story**: 인사이트→제언 구조 → As a/I want/So that 변환 자연스러움
