베드체어 / 검진센터 프로젝트 진행 회의 - 2026년 4월 26일
============================================

[핵심 주제: 평가 지표 개발, 2차 연도 과업, 시뮬레이션/관제 시스템, 액션 아이템]

## 1. 경험 가치 평가 지표 개발

### 서비스 디자인 지표 관련
- 삼성기평(삼기평) RFP에 서비스 디자인 성과 지표가 요구되고 있음
- 기존 지표: NPS, CST, CES, SUS 등
- 방향: 기존 병원 서비스 평가 지표 기준점을 가져와서 커스텀하는 게 가장 클리어
  - "예전에는 이 기준이었는데 → 우리 새로운 서비스에 맞춰 업그레이드"
- 서비스 디자인 쪽은 지표가 필수

## 2. 2차 연도 개발 범위 확인

### BDCI(베드체어) 측 과업
- 완제품 디자인 및 시연
- 테스트 검증용 시작품 제작
- AI 및 센서 기술을 활용한 주행 기능 탑재
  - 단, "자율주행" 표현은 의료기기 등급(2등급) 인증 문제로 주의 필요
- 2등급 인증 신청 준비

### 경로 계획 시나리오
- 시나리오 기반: 대기 → 검사실A → 검사실B → 대기 순으로 자동 이동
- 장애물 회피 기능
- 와이파이 기반 중앙 관제 시스템 연동
- 복수 베드체어의 위치/상태 실시간 모니터링 및 이동 명령 송수신

## 3. 시뮬레이션 vs 피지컬 테스트

### 시뮬레이션
- 5대 동시 운영 시뮬레이션: 가상으로 교차 이동 시각화
- Python 기반 시뮬레이션 (이전에 파이썬으로 짜놓은 것 활용)
- 검사실 배치도 위에 차량 이동 경로를 그림으로 보여주는 방식
- 인더스트리얼 시뮬레이션: 마찰력/모터 스펙 등 넣고 가상 검증 (듀얼 시뮬레이션)

### 피지컬 테스트
- 현재 시제품 1대로 테스트
- 최소 공간: 대기실 + 검사실 3~4개 (테이블로 모의 구성도 가능)
- 관제: 키보드로 1번 → 1번 방, 2번 → 2번 방 이동 명령
- 긴급 명령 변경 테스트: 3번 가던 중 → 4번으로 변경 시 반응 확인

### 합의 사항
- 2차 연도: 하드웨어는 개별 성능 검사, 시뮬레이션은 멀티플 확인 → 각자 분리 검증
- 시스템 연동(하드웨어 + 소프트웨어 통합)은 2단계 1차 연도 이후

## 4. 관제 시스템 및 위치 파악

### 베드체어 위치 파악 방법
- 라이더(LiDAR) + 투디 라이더 보유 → 위치 오차 거의 없음
- 와이파이 기반 위치 전달 가능
- 비콘: 서울대 연구에 포함되어 있으나, 이번 과제에서는 별도 언급 안 함
- 대안: 바닥 QR코드, 벽면 QR(비전 기반), 가드레인 등

### 관제에서 필요한 것
- 관제 입장에서 베드체어가 어디 있는지 위치 정보 필수
- 방 안에 들어가면 보이지 않으므로 통신 기반 위치 전달 필요
- 구동체 회사의 기존 소프트웨어 활용 검토 → 미팅 예정

## 5. 서비스 디자인 평가 지표 검증 계획

### 1단계: 전문가 자문단 평가 (델파이 방식)
- 작년에 만든 평가 지표(약 60가지 항목)를 전문가 10곳에 발송
- 피드백 수집 → 분석 → 수정 → 재발송 (왕복 검증)
- 의료진 + 간호직 피드백 포함
- 결과물로 KCI 논문 작성 가능 (1타 2피)

### 2단계: 공인기관 평가 (확인 필요)
- 국민건강보험공단 평가 등 공인 인증이 올해인지 내년인지 확인 필요
- 국가기관 공증이면 서둘러야 함, 의료진 공증이면 비교적 여유

### 비교 대조군 설계
- 기존 검진센터(서울대 강남센터, 한신 등) = 대조군
- 우리가 제시하는 건진 센터 = 실험군
- 동일 10개 검사 항목 기준으로 시간/효율/만족도 비교

## 6. 기타 프로젝트 현황

### 디자인 변경
- 베드체어 디자인을 새로 바꿈 (업데이트된 형태)
- 별도 사업(요양/가정용)에서는 라이더 등 고성능 센서 제거하여 성능 다운 버전 제작
- 7월부터 테스트베드 진행 예정

### 바이오 컨퍼런스 (6월, 샌디에고)
- 자료 업데이트 중, 메디컬 분야 전시/발표

### 문진 디바이스 개발
- AI 기반 문진 시스템: 아이패드 등 디스플레이에서 인터랙티브 문진
- 소프트웨어적으로 구현 가능 (ChatGPT 연결도 가능)
