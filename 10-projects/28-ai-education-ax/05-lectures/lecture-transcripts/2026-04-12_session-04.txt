## 1. 서브에이전트 병렬 워크플로우

### 서브에이전트의 핵심 = 병렬 처리
- 혼자(메인 에이전트)가 순차적으로 하면 30분 → 팀(서브에이전트)으로 동시에 하면 10분
- 메인 컨텍스트의 토큰을 쓰지 않음 → 컨텍스트 오염 방지
- 각 에이전트가 별도 컨텍스트에서 작업 → 결과만 가져옴

### 실전 예시: 경쟁사 분석
- "경쟁사 3곳 비교 분석 보고서 만들어줘. 에이전트로 병렬로 조사해줘"
- 리서치워커 3개가 백그라운드에서 동시 실행
- 각각 독립적으로 웹 검색 → 정보 수집 → 파일 정리 → 결과 취합

### 워커 시리즈 (5종)
1. Research Worker: 조사 전문 (한국어/영어 병행, 최소 3개 독립 소스 검증)
2. Analysis Worker: 분석 전문
3. Content Worker: 글쓰기 전문 (톤/페르소나 다양화 가능)
4. Design Worker: 디자인 관련
5. Development Worker: 개발 관련

### 캐스케이드 워크플로우: Decompose → Execute → Integrate
1. /decompose: 큰 미션을 병렬 실행 가능한 작업으로 분해
   - 최종 산출물 정의 → 필요 정보 파악 → 의존성 분석 → 작업 분류 → 매니페스트 생성
2. /execute: 매니페스트 기반으로 워커 투입, 병렬 실행
3. /integrate: 결과 합치기, 최종 산출물 생성

### 스킬 vs 서브에이전트
- 스킬: 절차가 고정된 반복 업무 (에스프레소 추출 매뉴얼)
- 서브에이전트: 매번 다른 내용의 작업 (리서치, 분석, 글쓰기)
- 같이 쓰면 강력: 크롤링 스킬로 데이터 수집 → 분석 에이전트가 처리

## 2. PRD (Product Requirements Document)

### PRD란?
- 제품 요구사항 명세서: 무엇을 왜 만드는가를 정의하는 기획 문서
- 핵심 구성: 해결할 문제 → 타겟 유저 → 목표/측정 기준 → 기능 요구사항 → 범위 → 일정

### 대시보드 PRD 스킬
- 대화형으로 PRD를 생성하는 도구
- 사용자가 아는 것에서 시작: "엑셀 3개 합치는 데 2시간 걸려"
- 핵심 원칙: Decision Specificity - "이 대시보드를 보고 어떤 결정을 내리고 싶은가"
- 4단계 질문 프로세스:
  1. 의사결정과 고민 (존재 이유)
  2. 성능 요구사항 (누가 얼마나 자주 보는가)
  3. 데이터 소스 파악 (실제 파일 분석)
  4. 화면 설계 공동 작업

### 기술 스택 자동 추천
- 1만 행 미만 + 소수 열람 → Google Sheets로 충분
- 5만 행 이상 + 다수 열람 → 데이터베이스 필요

## 3. Stitch (Google AI Studio 디자인 도구)

### 사용법
1. stitch.withgoogle.com 접속 → 구글 로그인
2. 모델: Gemini 3.1% Thinking으로 변경
3. PRD 파일 업로드 + 디자인 시스템 선택
4. "PRD 문서대로 디자인 만들어줘"

### 주요 기능
- 프로토타이핑: 페이지 간 연결, 클릭 동작 시뮬레이션
- 다크모드 변환, 모바일 버전 생성
- 개별 컴포넌트 수정 (자연어로)
- 디자인 시스템 자동 생성 (웹사이트 코드 분석)

### 디자인 레퍼런스
- Dribbble(dribbble.com): 대시보드, 랜딩페이지, 이커머스 등 레퍼런스
- 스크린샷 찍어서 Stitch에 주면 분석 후 유사 디자인 생성

## 4. 웹앱 개발 워크플로우 (전체 프로세스)

1. PRD 작성 (/dashboard-prd 또는 /webapp-prd)
2. Stitch에서 프론트엔드 디자인 생성
3. 디자인 파일 다운로드 (ZIP) → 프로젝트 폴더에 배치
4. Claude Code에서:
   - PRD + design.md 읽고 CLAUDE.md 생성
   - 플랜 모드로 개발 계획 수립
   - (바이패스 모드 권장) 개발 실행
5. Vercel로 배포 → 웹에서 접근 가능

### 바이패스 모드
- `claude --dangerously-skip-permissions`
- 모든 퍼미션 자동 허용 → 개발 시 1번/2번 누르기 불필요
- 개발 전용 터미널에서 사용 권장

## 5. 참여자 프로젝트 사례

- CEO 대시보드: 임대료 날짜 알림, 미회신 메일 체크, 주간 스케줄, 매장별 VOC, 프로젝트 현황, 근태 관리
- 투자 회사 관리 대시보드
- 업무 분배 대시보드: 메일 기반 업무량 파악, 누구에게 일을 시킬 수 있는지 체크
- 주류 업체: 월별 매출/주세 계산, 캐시플로우 파악, 거래처별 매출채권 관리
- 숙박업 관리: 매니저 스케줄 배정, 근태/급여 자동 계산, 데일리 로그 기반
- 숙박 CS 자동화: 멀티 플랫폼(네이버/야놀자/인스타/DM) 통합 인박스, AI 자동 응답

## 6. 참여자들이 만든 대시보드/웹앱 사례 (일룸 등)

- 매출 대시보드 + 해외 일정 이벤트 트래킹
- 마케팅 매체 성과 + 골프 선수 관리 (SNS 팔로워/댓글 분석)
- 발주/입고 관리 대시보드
- 화장품 수출 라벨 검수 도구 (국가별 규정 자동 체크)
- SCM 재고 관리: /morning 치면 ERP 데이터 → 발주 리스트 자동 생성
