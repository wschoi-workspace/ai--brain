# 일일 업무보고 봇 v2 — GPTs 설정 가이드

## v1 → v2 변경 요약

| 항목 | v1 | v2 |
|------|----|----|
| 수집 방식 | 업무 하나씩 드릴다운 | **전체 나열 → 핵심만 드릴다운** |
| 시니어(8개 업무) 대응 | 24턴 (시간 초과) | **7~10턴 유지** |
| 카테고리 분류 | 없음 | **자동 태깅 7종** |
| 출력 | 마크다운 보고서만 | **보고서 + JSON 데이터** |
| 데이터 연동 | 없음 | **구글시트 저장용 JSON** |

## 대화 흐름 요약

```
STEP 0: 인사
STEP 1: 기본 정보 (이름/팀/역할)
STEP 2: "오늘 한 일 쭉 나열해주세요" → 한꺼번에 수집    ← 변경
STEP 3: 카테고리 자동 태깅 + 정리 확인                   ← 신규
STEP 4: "핵심 1~2개만 자세히" → 선택적 드릴다운          ← 변경
STEP 5: 이슈 & 블로커
STEP 6: 내일 계획
STEP 7: 숨겨진 니즈 (선택)
STEP 8: 보고서 + JSON 동시 출력                          ← 변경
```

## 업무 카테고리 (7종)

| 카테고리 | 예시 |
|---------|------|
| 기획 | 전략 수립, 기획서, 콘셉트 설계 |
| 실행 | 디자인, 개발, 콘텐츠 제작, 현장 작업 |
| 커뮤니케이션 | 미팅, 메일, 보고, 클라이언트 대응 |
| 관리 | 일정, 인력, 예산, 문서 정리, 승인 |
| 리서치 | 시장 조사, 경쟁사 분석, 자료 수집 |
| 교육 | 교육, 멘토링, OJT, 세미나 |
| 기타 | 위에 해당하지 않는 업무 |

## 출력 포맷

### 1. 보고서 (마크다운)
- 전체 업무 테이블 (카테고리 포함)
- 핵심 업무 상세 (드릴다운 한 것만)
- 이슈 & 블로커
- 내일 우선순위
- 개선 포인트

### 2. JSON 데이터 (구글시트 저장용)
- core_tasks: 드릴다운한 핵심 업무 (프로세스, 도구, 산출물, 이슈 포함)
- sub_tasks: 나머지 업무 (업무명, 카테고리, 상태만)
- blockers, tomorrow, improvement: 메타 정보

---

## 구글시트 연동 구조

```
GPT 봇 → JSON 출력 → 구글시트 3시트에 분산 저장

시트1 "핵심업무": core_tasks 배열 → 업무당 1행
시트2 "서브업무": sub_tasks 배열 → 업무당 1행
시트3 "메타":    blockers/tomorrow/improvement → 사람당 1행
```
