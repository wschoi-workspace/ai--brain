## 1. 전역 폴더 동기화 — Git으로 `.claude` 관리

> "저는 전역 폴더만 관리하는 Git 레포를 따로 만들어놓은 거예요. 이 레포지토리가 바로 저의 전역 스킬, 전역 폴더만 왔다 갔다 하는 거야."
— 강사

- `claude-config`라는 별도 레포지토리로 `.claude/` 전역 폴더만 Push/Pull
- 회사 PC ↔ 집 PC 전역 설정 동기화 용도
- do-better 워크스페이스와는 별개의 레포지토리

## 3. Permission 시스템 — 1번/2번 줄이기

### 3-1. 왜 매번 물어보나

> "기본적으로 얘는 안전 모드에요. 함부로 건드리지 않겠다는 거. 이거를 건드려도 되라고 하는 게 settings.json의 Permission 구조입니다."
— 강사

### 3-2. 대화로 Permission 설정하기

> "말로 합니다. '나 이 퍼미션 때문에 힘드니까, 웬만한 걸 허가하고 싶어. 근데 네가 생각했을 때 허가하지 않아야 할 게 뭐라고 생각해?' 라고 물어보세요."
— 강사

**settings.json에서 설정 가능한 권한:**

| 항목 | 설명 | 권장 |
|---|---|---|
| Read | 파일 읽기 | Allow |
| Write | 파일 생성 | Allow |
| Edit | 파일 수정 | 판단 필요 |
| Delete | 파일 삭제 | 주의 |
| Grep | 파일 검색 | Allow |
| Bash | 터미널 명령 | Allow |
| WebSearch | 웹 검색 | Allow |
| Skill 실행 | 스킬 자동 실행 | Allow |

### 3-3. 전역 settings.json 설정 권장

> "settings.json도 전역 세팅을 해달라고 하는 게 더 좋습니다."
— 강사

## 4. Bypass 모드 — 모든 권한 허용

> "클로드 다음에 `--dangerously-skip-permissions`라고 하면 모든 게 허용된 모드입니다."
— 강사

```bash
claude --dangerously-skip-permissions
```

- **Bypass 모드 5** 활성화 → 절대 1번/2번 안 물어봄
- 메인 작업에서는 사용하지 않는 것이 좋음
- **개발/빌드** 시 별도 터미널에서 Bypass 모드로 실행 권장

### 4-1. Vibe Coding / 루프와의 관계

> "목표를 하나 주고 이걸 위해서 계속 돌아. 이걸 하려면 AI한테 퍼미션 모드를 두지 않으면 허락을 구하면서 중간에 멈추거든요. 그래서 Bypass 모드 켜고 목표를 하나 주고 끝까지 가능한 루프 구조를 만들어서 시행시키는 게 Vibe Coding입니다."
— 강사

## 5. 에센셜 커맨드 — 실전 활용

### 5-1. `/context` — 컨텍스트 사용량 확인

> "컨텍스트가 오염되지 않게 하기 위해서 `/clear`를 자주 쓰시라고 했어요."
— 강사

- `/context` → 현재 토큰 사용량, 세션 정보 확인
- 100만 토큰(1M context) 시대 — 예전보다 훨씬 여유

### 5-2. `/clear` — 세션 초기화

- 대화가 길어지면 컨텍스트 오염 → `/clear`로 초기화
- 초기화 전 중요 내용은 md로 저장하거나 `/export`

### 5-3. `/resume` — 이전 세션 복원

> "/clear를 해도, /resume 하면 내가 했던 세션들이 다 나옵니다."
— 강사

### 5-4. `/rename` — 세션 이름 지정

> "세션을 명확하게 나중에 다시 하고 싶으면, 이 세션은 '매드업'이다. 이렇게 표시해놓으면 나중에 /resume에서 쓰기가 좋아요."
— 강사

- `/rename` + `/resume`은 세트
- 이름 없으면 "이게 뭐였지?" → 항목 열어봐야 함

### 5-5. `/copy` — 마지막 답변 복사

> "/copy 하면 마지막 세션에 있는 텍스트를 복사해줍니다. 심지어 항목이 여러 가지면 어떤 내용을 복사할까요, 선택도 해줍니다."
— 강사

- 줄바꿈/띄어쓰기 깨짐 없이 깨끗하게 복사
- Bash 명령도 `/copy`로 복사 가능

### 5-6. `/export` — 대화 전체 저장

> "/export라고 하면 여기 있는 대화 내용이 전부 MD 파일로 빠져요. 대화 내용 자체를 공유 가능합니다."
— 강사

- 아이콘부터 모든 대화 내용을 md로 저장
- 다른 사람과 대화 공유 용도

### 5-7. `/simplify` — 코드 자동 리팩토링

> "/simplify라고 하면 최근에 변경된 파일을 자동으로 찾아서 에이전트 3개가 동시에 코드 품질을 점검해서 중복 코드 줄이고 비효율적인 부분을 개선합니다."
— 강사

### 5-8. 기타

| 커맨드 | 용도 |
|---|---|
| `/model` | 대화 중 모델 변경 (Opus/Sonnet/Haiku) |
| `/config` | 오토컴팩트, 씽킹모드 등 설정 |
| `/cost` | 토큰 사용량 확인 |
| `/usage` → `/status` | 남은 사용량, 사용 통계 |

### 5-9. 모델 선택 팁

> "개인적으로 Opus 쓰세요. 가능하다면 정말 다릅니다."
— 강사

## 핵심 요약

1. **전역 `.claude` 폴더**: 별도 Git 레포로 관리하면 PC 간 동기화 가능
2. **CLAUDE.md**: 팀 구성원 정보 포함, 150줄 이내, 변경 시 즉시 업데이트
3. **Permission**: 대화로 설정 ("웬만한 걸 허가하고 싶어, 위험한 것만 알려줘")
4. **Bypass 모드**: `--dangerously-skip-permissions` — 개발 시에만 별도 터미널에서
5. **에센셜 커맨드**: `/context`, `/clear`, `/resume`, `/rename`, `/copy`, `/export`, `/simplify`
6. **`/rename` + `/resume` = 세트**: 세션 이름 지정 후 나중에 복원
7. **`/copy`**: 줄바꿈 깨짐 없이 깨끗한 복사
8. **Memory**: 클로드가 자동으로 학습 포인트 저장 (최근 기능)
9. **모델 선택**: Opus 강력 권장
10. **스킬 설계**: 잘게 쪼개고 오케스트레이션 스킬로 결합
