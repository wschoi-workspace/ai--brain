## 1. 외부 협업 툴 연동

> "컨플루언스든 노션이든, API 연결하면 클로드 코드에서 말하는 내용이 바로 거기에 다 작성이 되는 거예요."
— 강사

- 클로드 코드는 외부 SaaS 도구(Confluence, Jira, Notion, Slack)와 API로 연결 가능
- 연결하면 클로드 코드 안에서 직접 해당 도구에 문서 작성/관리 가능
- 수강생 회사에서 Confluence 사용 중 → 연동 가능성 확인

---

## 2. 플랜 모드 활용법

> "얘랑 플랜 모드를 키고, 물론 아직도 헤맬 수 있지만, 차근차근 해내면 된다는 사실을 오늘 익히는 게 제일 핵심이었어요."
— 강사

- 플랜 모드(Plan Mode): 실행 전에 계획을 먼저 세우는 모드
- 외부 툴 연동처럼 복잡한 작업도 플랜 모드로 단계별 접근 가능

---

## 3. SOP 만들기 — 인턴 가르치기 마인드

> "클로드 코드 처음에 스킬 만들 때는 인턴을 가르친다는 마인드만 가지면 다 됩니다. 좀 더 디테일하게, 대신 플랜 모드와 함께 빠짐없이 가르쳐주면 웬만한 게 다 됩니다."
— 강사

- 외부 툴 연동 다음 단계 = **업무 SOP화**
- 스킬 제작의 핵심 비유: "인턴에게 업무를 가르치듯이"
- 사수가 인턴을 잘 가르치는 것과 못 가르치는 것의 차이 = 스킬 품질의 차이
- 실제 사례: 부가세 처리 스킬 — 위 원칙대로 만들어 자동화 성공

---

## 4. 다음 주 예고 — 파일 구조와 서브 에이전트

> "이 1번과 2번을 안 누르고 싶어. 그럼 어떻게 해야 될까를 물어보셔야 되는데, 사실 그거는 클로드 코드의 파일 구조 시스템에 달려있어요. settings.json 파일을 설정하고 그래야 되는데, 물론 이것도 해달라고 하면 해줘요."
— 강사

**다음 회차 커리큘럼 예고:**
1. **클로드 코드 파일 구조** — settings.json 등 내부 설정 파일 이해
2. **서브 에이전트** — 병렬 처리를 위한 전문 에이전트 활용
3. **자동화 고도화** — 매번 수동 실행(1번, 2번 선택) 없이 자동 실행되도록 설정

---

## 핵심 요약

1. **외부 툴 연동**: Confluence, Jira, Notion, Slack 등 API 연결로 클로드 코드 안에서 직접 제어 가능
2. **플랜 모드**: 복잡한 작업은 플랜 모드로 단계별 접근
3. **SOP 마인드**: 스킬 만들기 = 인턴에게 업무 가르치기 (디테일하게, 빠짐없이)
4. **다음 단계**: 파일 구조(settings.json) 이해 → 서브 에이전트 → 자동화 고도화
