## 2. Q&A — 이미지 붙여넣기 & 경로 복사

### 2-1. 클로드 코드에 이미지 직접 붙여넣기

**수강생 질문**: 이미지를 직접 붙여넣기가 안 되어서, Claude 채팅 사이드바를 별도로 열어 사용하고 있었음

**답변**:

## 3. 컨텍스트 관리 — 세션 정리 전략

### 3-1. 세션 관리 3가지 방법

> "첫 번째는 우리는 아무리 그래도 컨텍스트 관리를 해줘야 돼요. 만약 우리가 대화를 하면 앞에 막 이만큼 대화를 했어. 그럼 여기서 한마디를 더 하잖아요. 그러면 얘는 이 앞에 있는 맥락을 그대로 다 갖고 있어요. 그래서 점점 점점 무거워진다."
— 강사

| 방법 | 설명 |
|---|---|
| **1. 워크로그 저장 + /clear** | 진행 내용을 프로젝트에 저장 후 세션 초기화 (권장) |
| **2. /resume** | 세션 이름을 rename 해두고, 필요할 때 resume으로 복원 |
| **3. 그냥 이어서** | 50% 넘기 전까지 맥락 오염 없으면 끌고 써도 됨 |

> "가능하다면 웬만큼의 하나의 프로젝트가 끝났으면, 이거를 md로 저장해 놓고 하는 게 훨씬 더 깔끔하긴 합니다."
— 강사

> "50%가 넘기 전까지 맥락이 오염 안 시키고 한다면 그냥 끌고 써도 돼요."

## 4. MCP vs 스킬 — 핵심 개념 설명

### 4-1. MCP (Model Context Protocol)란?

> "과거에는 하나의 외부 툴을 내가 사용하는 무엇에 연동하는 게 생각보다 코딩을 해서 짜는 게 되게 어려웠어요. 내가 어떤 시스템을 쓰고 있어, 근데 이거에다가 노션을 붙이고 싶어, 슬랙을 붙이고 싶어... 되게 어려웠어요."
— 강사

**USB-C 비유**:
> "핸드폰 충전이랑 똑같은 목표가 있어도 똑같은 규격이 아니고, 쟤는 이런 API 규격, 쟤는 이런 규격... 다 충전기 코드를 다 따로 갖고 다녀야 되잖아요. 근데 이거를 MCP라고 하는 USB-C처럼, MCP로 하면 다 여기로 똑같이 들어와서 활용하면 되라는 하나의 규약을 만들었어요."
— 강사

### 4-2. MCP의 문제점

> "노션의 기능이 뭐가 있을까요? 페이지 만들기, 블록 만들기, 이동... 정말 많은 기능들이 있잖아요. 그 기능들을 MCP는 클로드 코드에 그대로 다 올려놓는 거예요. 그러면 컨텍스트에 정말 많이 들어요."
— 강사

- MCP는 외부 도구의 **모든 기능 설명**을 컨텍스트 윈도우에 항상 올려놓음
- 도구가 많아질수록 컨텍스트가 무거워짐

### 4-3. 스킬이 더 나은 이유

> "스킬을 만들면 클로드 코드에 이 기능이 다 올라가는 게 아니고, 딱 **디스크립션만** 올라가 있어요. 이 줄만 올라가는 거예요. 그러면 누가 이 얘기를 하면, 그제서야 이게 그대로 로드가 되어서 활용이 되는 거예요."
— 강사

| 구분 | MCP | 스킬 |
|---|---|---|
| **로딩 방식** | 모든 기능 설명이 항상 로드 | 디스크립션(한 줄)만 로드 |
| **본문 로드 시점** | 항상 | 호출 시에만 |
| **컨텍스트 소비** | 높음 (수~수십 %) | 극히 낮음 |

### 4-4. 실제 수치 증거

> "여기서 `/context` 치면... 스킬스가 **0.4%**밖에 안 돼요. 제가 스킬이 무려 거의 80개가 넘는데 0.4%라는 건 이 안에 텍스트가 다 들어가 있는 게 아니고 디스크립션만 모아놓은 게 0.4%밖에 안 된다는 거예요."
— 강사

- 강사의 스킬 80개 이상 → 컨텍스트 0.4%만 차지
- 같은 기능을 MCP로 연결했다면 수~수십 % 차지했을 것

### 4-5. 결론: 스킬로 만들어 쓸 것

> "스킬로 만들어서 그 기능만 넣어 쓴다. OCR을 하려고 하는데 이거를 MCP... 왜냐하면 스킬을 만든다고 얘기는 안 했을 거예요. 그럼 MCP를 써야겠네? 라는 생각밖에 안 되는 거예요. 근데 이걸 **스킬로 만든다**고 하면, '아 그래? 그러면 API를 써서 스킬을 만들어야겠구나'라는 생각을 하면서 이제 되는 거죠."
— 강사

## 핵심 요약

1. **이미지 붙여넣기**: Alt+V (Windows) / macOS는 클립보드에서도 가능
2. **컨텍스트 관리**: 작업 단위로 md 저장 + /clear, 50% 넘기면 정리
3. **MCP vs 스킬**: MCP는 모든 기능을 항상 로드(무거움), 스킬은 디스크립션만 로드 후 호출 시 활성화(가벼움)
4. **스킬 80개 = 컨텍스트 0.4%**: 스킬 방식이 압도적으로 효율적
5. **외부 API 연동 시**: MCP 대신 스킬로 만들 것 (예: OCR → Gemini API 스킬)
6. **Skip Permission**: 설정 후 세션 재시작 필요
7. **세션 간 정보 공유**: 환경변수(.env) 저장 확인 필수
