## 3. Git & GitHub — 세이브 포인트 개념

### 3-1. 왜 Git을 쓰는가

> "게임 하다가 세이브 포인트를 찍잖아요. 세이브 포인트를 여러 개 만들 수 있고, 엉망이 됐으면 다시 돌아갈 수 있는 저장소가 Git이라고 생각하시면 됩니다."
— 강사

- Git = 세이브 포인트가 있는 저장소
- 커밋(Commit) = 세이브 포인트 만들기
- 푸시(Push) = 세이브 포인트를 클라우드에 업로드

### 3-2. PC 간 동기화

> "내가 지금 do-better 워크스페이스라는 저장소가 있는데, 이 폴더만 항상 회사 PC와 집 PC가 동기화가 되면 좋겠다."
— 강사

- Git Push/Pull로 회사 ↔ 집 동기화
- 클라우드 동기화(Dropbox 등) 대신 Git 사용 가능

### 3-3. 비개발자의 Git 사용법

> "개발자들은 파트별로 정확하게 커밋하지만, 저희 같은 비개발자들은 한 일들이 그냥 적재되길 원하는 거지. 우리는 커밋하고 푸시를 그냥 다 올리는 게 좋은 겁니다."
— 강사

### 3-4. GitHub CLI (gh)

> "GitHub CLI라는 걸 설치하면, 굳이 웹에 들어가지 않아도 레포지토리 만드는 것도 클로드 코드에서 바로 가능합니다."
— 강사

## 4. 슬래시 커맨드, 스킬, 서브 에이전트 — 3대 핵심 개념

### 4-1. 슬래시 커맨드 (Slash Command) = 포스트잇

> "자주 쓰는 지시를 한 줄로 요약해 놓은 것. `/daily-note` 한 줄이면 긴 설명 없이 바로 실행."
— 강사

- 포스트잇처럼 간단한 한 줄 지시
- 사용자가 `/` 입력 시에만 실행 (User-invoked)
- MD 파일 하나에 프롬프트 저장

예시:
- `/daily-note` → 오늘 날짜로 일기 생성
- `/todo` → 할 일 추가
- `/daily-review` → 이번 주 종합 리뷰 생성

### 4-2. 스킬 (Skill) = 작업 지시서 (SOP)

> "카페에서 에스프레소를 추출하는 방법처럼, 순서를 차례대로 넣는 겁니다. 포터필터 무게 제거, 원두 도징, 그램 체크... 이게 바로 스킬입니다."
— 강사

**슬래시 커맨드와의 차이점:**

| 구분 | 슬래시 커맨드 | 스킬 |
|---|---|---|
| 비유 | 포스트잇 | 작업 지시서 (SOP) |
| 실행 방식 | 사용자가 `/` 입력 (User-invoked) | 대화 중 키워드 감지 시 자동 실행 (Model-invoked) |
| 복잡도 | 간단한 한 줄 | 절차 + 코드 + 참조파일 |

> "슬래시 커맨드는 내가 `/`를 쳐야 실행되지만, 스킬은 '노션'이라는 단어만 나오면 '아, 노션 스킬 쓰면 되는구나'라고 알아서 판단해서 자동으로 가져오는 거예요."
— 강사

**스킬의 폴더 구조:**
```
.claude/skills/
  └── [스킬이름]/          ← 폴더 이름 = 스킬 이름
      ├── skill.md         ← 스킬 정의 (언제, 무엇을 하는지)
      ├── scripts/         ← 코드 도구들
      └── templates/       ← 참조 파일/템플릿
```

- `skill.md` 상단의 디스크립션만 클로드에 로드됨 (전체 내용 X)
- 호출 시에만 본문이 활성화

### 4-3. 서브 에이전트 (Sub-agent) = 전문 팀원

> "메인 에이전트 하나가 팀장이라면, 서브 에이전트는 '너는 조사만 해', '너는 글만 써', '너는 검토만 해'라는 전문 영역을 맡는 에이전트."
— 강사

- 각 서브 에이전트는 별도 컨텍스트 윈도우에서 실행 (메인 오염 방지)
- **핵심 용도 = 병렬 처리**
- 경쟁사 3곳 동시 조사 → 결과만 메인에 전달

## 5. Source of Truth — 오류 없는 스킬 만들기

> "AI는 생각보다 룰이 정확하게 없다면 허투를 하게 만들 때가 많다. 제대로 만들려면 신뢰할 만한 근거를 바탕으로 작업을 해라고 하는 게 가장 정석적으로 좋은 방법이다."
— 강사

**Source of Truth 파일 목록:**
- `skills-guide.md` — 스킬 만들기 공식 가이드
- `sub-agent-guide.md` — 서브 에이전트 가이드
- `claude-md-guide.md` — CLAUDE.md 베스트 프랙티스
- `commands-guide.md` — 슬래시 커맨드 가이드
- `plugins-guide.md` — 플러그인 가이드
- `rules-guide.md` — 룰 가이드

> "스킬을 만들 때, 커맨드를 만들 때, 에이전트를 만들 때 오류가 생기지 않도록 이 Source of Truth 파일을 참조해서 만들어라."
— 강사

사용법:
```
"@skills-guide.md를 바탕으로 [스킬이름] 스킬 만들어줘"
```

## 7. GWS CLI — Google Workspace 연동 실습

### 7-1. GWS CLI란

> "구글에서 공식적으로 Google Workspace CLI라고 하는 걸 만들었어요. Gmail, Calendar, Drive, Sheets, Slides, Chat, Tasks... 다 연결을 할 수 있어요."
— 강사

- 이전에는 개인 개발자가 만든 비공식 CLI 사용
- 최근 구글이 공식 GWS CLI 출시

### 7-2. 설치 과정

1. **Node.js 설치** (npm 사용을 위해)
2. **npm install** — GWS CLI 패키지 설치
3. **Google Cloud Console** — OAuth 설정 (가장 어려운 부분)
4. **API 활성화** — Gmail API, Calendar API, Drive API 등 필요한 것만 선택
5. **OAuth 동의 화면** 설정
6. **인증 완료** 후 클로드 코드에서 바로 사용 가능

### 7-3. 주의사항

> "이 구글 OAuth 설정이 어렵습니다. 한 번 하면 2시간 이상 걸리거든요."
— 강사

> "블로그 읽는 법은 읽지 말고 시키세요. 클로드 코드가 설치되어 있으면, 페이지를 긁어서 던져준 다음에 '나 이거 하고 싶은데 아직 하나도 몰라'라고 하면 차근차근 100단계 만들어 줍니다."
— 강사

- 터미널에서 명령어 복사 시 **줄바꿈/빈칸 주의** — 에러의 주요 원인
- Homebrew 설치가 필요할 수 있음

## 핵심 요약

1. **3대 핵심 개념**: 슬래시 커맨드(포스트잇) / 스킬(SOP 작업지시서) / 서브 에이전트(전문 팀원)
2. **슬래시 커맨드 vs 스킬**: 커맨드는 사용자 호출, 스킬은 AI가 키워드 감지 시 자동 호출
3. **스킬 폴더 구조**: `skill.md` + `scripts/` + `templates/`
4. **Source of Truth**: 공식 가이드 문서를 참조하면 오류 없는 스킬/에이전트 생성 가능
5. **Payroll 실무 사례**: 월 20만원 노무사 대행 → `/payroll` 스킬 하나로 대체
6. **Git = 세이브 포인트**: 커밋으로 작업 저장, Push/Pull로 PC 간 동기화
7. **GWS CLI**: Google Workspace 전체를 터미널에서 제어 (OAuth 설정이 진입장벽)
8. **블로그/문서 읽는 법**: 직접 읽지 말고 클로드에 던져서 "같이 해보자"
