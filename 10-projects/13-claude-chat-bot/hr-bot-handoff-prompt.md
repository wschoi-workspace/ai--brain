# HR 업무 매뉴얼 봇 — 클로드코드 핸드오프 프롬프트

아래 내용을 그대로 복사해서 새 세션에서 실행하세요.

---

## 프롬프트

```
# 미션: 텔레그램 업무 매뉴얼 통합 봇 구축

## 배경
- 회사 업무 프로세스 매뉴얼(PDF/워드)을 직원들이 봇으로 질의할 수 있게 만든다
- 카카오워크에 업무보고 봇이 있지만, 새 봇은 텔레그램에 구축한다
- 기존 텔레그램 봇(13-claude-chat-bot)과 별도로 새 봇을 만든다
- 텔레그램 BotFather에서 새 봇 토큰을 발급받아야 한다 (아직 미발급)

## 봇 기능 범위 (단계별)

### 1단계: 매뉴얼 RAG Q&A
- PDF/워드 매뉴얼 파일 → 청킹 → 벡터 임베딩 → 질문 시 검색+답변
- 출처 표기 ("매뉴얼 X조 X항 참고")
- 매뉴얼 파일은 추후 제공 예정 — 먼저 봇 서버 구조를 세팅

### 2단계: 개인 데이터 조회 (구글시트 연동)
- 직원별 연차 잔여일 조회 ("내 남은 연차 며칠?")
- HR포탈 에이전트가 구글시트 기반으로 이미 존재 — 같은 데이터 소스 활용
- 직원 인증: 텔레그램 ID ↔ 직원명 매핑 테이블 (구글시트)

### 3단계: 일정 확인
- Google Calendar API 연동 ("이번 주 회의 일정?")

### 4단계: 업무보고 기능 이관
- 카카오워크 업무보고 봇 → 텔레그램으로 이전

### 5단계: 법인카드 정산 (추후)
- 고위드(Gowid) API 연동 — API Key 미발급 상태

## 기술 스택 추천
- RAG: Dify (노코드, 텔레그램 플러그인 지원) 또는 LangChain + ChromaDB
- LLM: GPT-4o 또는 Claude API
- 봇: Python telegram-bot 또는 Dify 텔레그램 연결
- 벡터DB: ChromaDB 또는 Dify 내장

## 질문 의도 분류 구조
```
직원 질문 (텔레그램)
    ↓
의도 분류 (LLM)
    ├─ 프로세스/규정 질문 → RAG (매뉴얼 검색)
    ├─ 연차 잔여일 조회  → 구글시트 API
    ├─ 일정 확인        → Google Calendar API
    ├─ 정산 현황        → 고위드 API (추후)
    └─ 업무 보고        → 양식 수집 → 저장
```

## 연관 세팅 (대기 중, 별도 진행)
1. 싸인오케이 API — 입사서류 전자서명 자동화 (API Key 미발급)
2. 고위드 API — 법인카드 지출 정산/분석 (총괄관리자 권한 미확보)
```
