병렬 작업 결과를 수집하고 통합하여 최종 산출물을 생성합니다.

**경로**: $ARGUMENTS (생략 시 현재 디렉토리에서 `_parallel/` 탐색)

---

## 통합 프로세스

다음 단계를 순서대로 수행하세요:

### 1단계: 매니페스트 확인

`_parallel/_manifest.md` 파일을 읽어 다음 정보 파악:
- 원본 미션
- 최종 산출물
- 작업 목록
- 통합 지침

**매니페스트가 없는 경우:**
- `_parallel/` 폴더 내 모든 `.md` 파일 나열
- 사용자에게 미션과 통합 지침 질문

### 2단계: 결과 파일 수집

매니페스트의 작업 목록에서 각 파일을 읽기:

| 파일 | 존재 여부 | "메인 세션 전달 사항" |
|------|-----------|----------------------|
| 01-xxx.md | ✓ / ✗ | [요약] |
| 02-xxx.md | ✓ / ✗ | [요약] |

### 3단계: 완료 상태 확인

모든 병렬 작업이 완료되었는지 체크:

**모든 파일 존재**: 통합 진행
**일부 파일 누락**:
- 누락 파일 목록 표시
- 계속 진행할지 사용자에게 질문

### 4단계: 전달 사항 종합

각 파일의 "메인 세션 전달 사항" 섹션에서 추출:
- 통합 시 참고할 포인트
- 다른 작업과의 연결점
- 미결정 사항

### 5단계: 통합 실행

매니페스트의 "통합 지침"에 따라:
1. 각 파일의 핵심 내용 종합
2. 연결점 파악 및 분석
3. 미결정 사항 해결
4. 최종 산출물 생성

### 6단계: 저장 및 정리

- 최종 산출물을 지정된 경로에 저장
- 사용자에게 `_parallel/` 폴더 정리 여부 질문:
  - 삭제: 폴더 전체 삭제
  - 아카이브: `_parallel_archive_YYYYMMDD/`로 이동
  - 유지: 그대로 두기

```markdown
| # | 작업 | 상태 | 핵심 전달 사항 |

| 3 | [작업3] | ✓ | [요약] |

---

### 연결점 분석

각 작업 결과 간의 연결점:
- [연결점 1]
- [연결점 2]

### 미결정 사항 해결

| 사항 | 출처 | 결정 |
|------|------|------|

```

## 예시

**입력**: `/integrate pkm/20-operations/21-cafe-operations/21.05-lotte-christmas-market/_parallel`

### 1단계 - 매니페스트 확인

```
_parallel/_manifest.md 읽기:
- 원본 미션: "롯데 마켓 운영 회고 작성"
- 최종 산출물: 47일 운영 회고 문서
- 작업 목록: 3개 (운영로그, 매출분석, 고객피드백)
- 통합 지침: 성공/실패 요인 도출 → 회고 작성
```

### 2단계 - 결과 파일 수집

```
| 파일 | 존재 | 메인 세션 전달 사항 |
|------|------|---------------------|
| 01-운영로그분석.md | ✓ | 주요 이슈 5건, 대응 성공 3건 |
| 02-매출분석.md | ✓ | 일평균 256만, 피크 12/24 |
| 03-고객피드백.md | ✓ | 긍정 70%, 대기시간 불만 |
```

### 3단계 - 완료 확인

```
✓ 모든 파일 존재 - 통합 진행
```

### 4단계 - 전달 사항 종합

```
통합 시 참고:
- 12/24 매출 피크와 운영 이슈 연관성
- 대기시간 불만 → 피크타임 인력 부족 문제

연결점:
- 매출 피크 시간 = 고객 불만 집중 시간
- 성공 대응 사례가 긍정 피드백으로 이어짐

미결정 사항:
- 전년 대비 매출 감소 원인 (매출분석)
- 재방문 의향 데이터 부재 (고객피드백)
```

### 5단계 - 통합 실행

```
통합 지침에 따라:
1. 성공 요인: 직원 응대, 제품 품질
2. 실패 요인: 피크타임 대기, 날씨 영향
3. 개선점: 인력 배치, 예약 시스템
```

### 6단계 - 저장 및 정리

```
최종 산출물: 21.05-lotte-christmas-market/retrospective.md
_parallel/ 폴더: 아카이브 (_parallel_archive_20260107/)
```

## 매니페스트 없이 실행

`_manifest.md`가 없는 경우:

1. `_parallel/` 폴더의 모든 `.md` 파일 나열
2. 사용자에게 질문:
   - "이 병렬 작업의 원본 미션은 무엇인가요?"
   - "최종 산출물은 무엇인가요?"
   - "어떻게 통합해야 하나요?"
3. 답변을 바탕으로 통합 진행

```

    ↓

    ↓

    ↓
워커 에이전트가 _parallel/01-xxx.md, 02-xxx.md, ... 저장
    ↓
/integrate 실행
    ↓
매니페스트 읽기 → 결과 수집 → 통합 → 최종 산출물
    ↓
_parallel/ 정리
```
