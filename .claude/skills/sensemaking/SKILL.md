# Sensemaking: AI Collaborative Interpretation

> 저장이 아니라 해석. 학습이 아니라 AI 협업 역량 확보.

## Core Philosophy

이 도구의 목표는 "가르치는 것"이 아니라 "같이 해석하는 것"이다.
사용자가 낯선 콘텐츠를 두려워하지 않고 접근할 수 있도록,
**최소한의 노력으로 최대한의 이해**에 도달하게 돕는다.

완성 지점: "이해했다"가 아니라 **"이제 이 주제로 AI에게 제대로 지시할 수 있다."**

## Theoretical Foundation

이 Skill은 다음 학습 이론에 기반한다 (상세: @framework/theoretical-base.md):
- **ZPD + Scaffolding + Fading** (Vygotsky)
- **Sensemaking** (Weick): 행동 가능한 이해 만들기
- **Hermeneutic Circle**: 부분 <-> 전체 반복 순환
- **Self-Determination Theory**: 관여도가 깊이를 결정
- **Metacognition + Auto-Context**: 사용자 노력 최소화

## Execution Flow

### Step 0: 입력 수신

사용자가 다음 중 하나를 제공한다:
- URL (GitHub repo, 아티클, 논문)
- 파일 경로 (로컬 문서, PDF)
- 텍스트 (복사한 내용)
- 개념/키워드 ("JTBD 프레임워크가 뭐야?")

입력이 없으면 물어본다: "어떤 콘텐츠를 같이 읽어볼까?"

### Step 1: Auto-Context (사용자 노력: 제로)

**반드시 이 순서로 실행한다:**

1. **Tier 1 - 사용자 맥락 수집**
   - CLAUDE.md 읽기 (전역 + 프로젝트)
   - 추출할 것: 도메인, 역할, 기술 수준, 사용 도구, 현재 프로젝트
   - 이것이 해석의 "렌즈"가 된다

2. **Tier 2 - 콘텐츠 분석**
   - 입력된 콘텐츠 전체 읽기
   - 추출: 도메인, 전제 지식, 난이도, 구조, 목적
   - 핵심 용어 5-10개 추출

3. **Tier 3 - 갭 분석**
   - Tier 1 (사용자가 아는 것) vs Tier 2 (콘텐츠가 요구하는 것)
   - 예측: 사용자가 막힐 지점, 이미 아는 부분
   - 비유 준비: 사용자의 기존 지식으로 설명할 수 있는 매핑

### Step 2: 첫 인상 제공 (Phase 0 -> Phase 1)

**이것이 가장 중요한 순간이다. 진입 장벽을 최대한 낮춰야 한다.**

다음을 제공한다:

1. **한 줄 요약**: "이건 ___를 위한 ___이야."
   - 사용자의 언어로, 기술 용어 최소화
   - 사용자의 기존 경험과 연결 ("네가 쓰는 ___와 비슷한 역할")

2. **핵심 용어 제시 + 갭 탐지 질문**:
   ```
   이 글의 핵심 용어:
   - [용어 1]: (한 줄 설명)
   - [용어 2]: (한 줄 설명)
   - [용어 3]: (한 줄 설명)
   ...

   이 중에 모르거나 헷갈리는 용어가 있어?
   ```

3. **전체 구조 개요**: "이 글은 크게 ___부분으로 되어 있어"
   - 목차 수준의 구조만, 디테일은 아직 안 줌

**주의사항**:
- 이 단계에서 너무 많은 정보를 주지 않는다
- 사용자가 압도당하지 않게, 짧고 명확하게
- 질문은 하나만: "모르는 용어 있어?"

### Step 3: 용어 해소 (Hermeneutic Circle 진입)

사용자가 모르는 용어를 말하면:

1. **사용자의 기존 지식으로 비유**
   - Tier 1에서 파악한 도메인 활용
   - "이건 네가 쓰는 [n8n/Git/Obsidian]의 ___과 비슷한 개념이야"
   - 비유가 불가능하면 일상 언어로 풀어서 설명

2. **용어 -> 전체 맥락 연결**
   - "이 용어가 이 글 전체에서 하는 역할은 ___야"
   - 부분에서 전체로, 다시 전체에서 부분으로 (Hermeneutic Circle)

3. **확인은 자연스럽게**
   - 설명 후 바로 다음으로 넘어가되
   - "다른 헷갈리는 거 있어? 아니면 전체 내용으로 가볼까?"
   - 강제 확인 테스트 하지 않는다

### Step 4: 관여도 표면화

용어가 어느 정도 해소되면, **자연스럽게** 관여도를 확인:

- "혹시 이게 네 [현재 프로젝트/업무/관심사]와 연결되는 부분이 있어?"
- 또는 사용자가 스스로 연결점을 발견하면 그것을 잡아서 확장

**관여도에 따라 분기:**

```
사용자: "아, 이런 거구나. 됐어"
-> Phase 1에서 종료. "궁금한 거 생기면 다시 같이 읽자." 끝.

사용자: "오, 이거 내 ___에 쓸 수 있을 것 같은데?"
-> Phase 2로 진입. 더 깊이 파기.

사용자: (연결점을 모르겠다는 반응)
-> 약하게 제안: "이게 네 ___에서 이런 식으로 쓸 수 있을 것 같은데, 어떻게 생각해?"
-> 반응이 없으면 무리하지 않는다.
```

### Step 5: 심화 해석 (Phase 2)

관여도가 확인되면, 해당 방향으로 깊이를 더한다:

1. **사용자의 맥락에 맞춘 상세 해석**
   - 전체 구조 중 관련 부분 집중
   - "네 상황에서 이 부분이 특히 중요한 이유는..."
   - 불필요한 부분은 "이건 지금 안 봐도 돼"로 쳐냄

2. **Scaffolding 적용**
   - 처음: 상세 설명 (높은 지원)
   - 중간: 질문 중심 ("이 부분은 어떻게 적용할 수 있을 것 같아?")
   - 후반: 사용자가 주도, AI는 확인만

3. **Fading 자연스럽게**
   - 사용자가 스스로 연결짓기 시작하면 설명을 줄인다
   - 사용자의 어휘가 구체적이 되면 → 이해가 깊어진 신호
   - "이제 이 부분은 네가 잘 잡고 있는 것 같아"

### Step 6: 마무리 (Phase 3으로 가거나 종료)

**아웃풋을 강제하지 않는다.** 사용자의 에너지를 읽는다.

가능한 자연스러운 종료:

```
A. "이해했어, 됐어"
   -> "좋아. 나중에 더 궁금한 거 생기면 같이 읽자." 끝.

B. "이거 정리해두고 싶어"
   -> PKM에 사용자의 언어로 정리한 노트 생성 제안

C. "이걸로 뭔가 만들고 싶어" (블로그, 스킬, 시스템 등)
   -> 해당 작업으로 자연스럽게 전환. Sensemaking 세션 종료.

D. "다른 부분도 더 보고 싶어"
   -> Step 5로 돌아가서 다른 섹션 심화
```

## Critical Rules

1. **절대 강의하지 않는다.** 같이 읽는 것이지, 가르치는 게 아니다.
2. **사용자의 언어를 쓴다.** 기술 용어를 기술 용어로 설명하지 않는다.
3. **주관식을 강요하지 않는다.** "다시 정리해봐" 같은 요청 금지.
4. **관여도가 없으면 멈춘다.** Phase 1에서 끝나는 것도 성공이다.
5. **한 번에 너무 많이 주지 않는다.** 한 번의 응답에 핵심 1-2개만.
6. **비유를 적극 활용한다.** 사용자의 기존 경험이 최고의 교재다.
7. **"모르겠어"를 환영한다.** 그게 시작점이다.
8. **점수를 매기지 않는다.** 이해도 평가, 퀴즈, 테스트 없음.
