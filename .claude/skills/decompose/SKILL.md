## 분해 프로세스

다음 단계를 순서대로 수행하세요:

### 1단계: 최종 산출물 정의

이 미션이 완료되면 무엇이 만들어지나요?
- 산출물 형태: (문서 / 코드 / 데이터 / 분석 결과 / 기타)
- 산출물 위치: (파일 경로 또는 "미정")

### 2단계: 필요한 입력 나열

산출물을 만들려면 어떤 정보/자료가 필요한가요?
각 입력을 나열하세요:
- 입력 1: [설명]
- 입력 2: [설명]
- 입력 3: [설명]
- ...

### 3단계: 입력별 획득 방법 분석

각 입력을 얻는 작업을 정의하고 유형과 권장 워커를 분류하세요:

| 입력 | 획득 작업 | 유형 | 권장 워커 |
|------|-----------|------|-----------|
| 입력 1 | [작업 설명] | 리서치 | research-worker |
| 입력 2 | [작업 설명] | 분석 | analysis-worker |
| 입력 3 | [작업 설명] | 콘텐츠 | content-worker |
| 입력 4 | [작업 설명] | 디자인 | design-worker |
| 입력 5 | [작업 설명] | 개발 | development-worker |

**워커 선택 가이드:**
| 유형 | 워커 | 용도 |
|------|------|------|
| 리서치 | research-worker | 정보 수집, 검색, 소스 평가 |
| 분석 | analysis-worker | 패턴 파악, 프레임워크 적용, 인사이트 |
| 콘텐츠 | content-worker | 글쓰기, 문서 생성, 구조화 |
| 디자인 | design-worker | 랜딩페이지/웹 디자인 (Carrd/Framer/Webflow) |
| 개발 | development-worker | 코드 작성, 자동화, API 연동 |

### 4단계: 의존성 분석

각 작업 간의 의존성을 확인하세요:

**질문**: 작업 A의 결과 없이 작업 B를 시작할 수 있나요?
- Yes → 두 작업은 **병렬 가능**
- No → 작업 B는 A에 **의존** (순차 필요)

의존성 맵:
- 작업 1: 독립 (바로 시작 가능)
- 작업 2: 독립 (바로 시작 가능)
- 작업 3: 작업 1, 2 완료 후 가능
- ...

### 5단계: 작업 분류

위 분석을 바탕으로 분류:

**병렬 가능 (독립 작업)**:
- 다른 작업 결과 없이 시작 가능
- 각각 다른 세션에서 동시 진행

**순차 필요 (의존 작업)**:
- 선행 작업 완료 후 시작
- 의존하는 작업 명시

**통합 (메인 세션)**:
- 여러 결과를 합쳐야 하는 작업
- 항상 메인 세션에서 처리

### 6단계: 저장 경로 결정

- 프로젝트 관련 미션 → 해당 프로젝트 폴더 내 `_parallel/`
- 일반 미션 → `pkm/00-inbox/_parallel/`
- 파일명 규칙: `{번호}-{작업명}.md`

### 7단계: 매니페스트 생성

`_parallel/_manifest.md` 파일을 생성하여 통합 시 필요한 정보 저장:

```markdown
# 병렬 작업 매니페스트

## 미션 정보
- **원본 미션**: "[미션]"
- **최종 산출물**: [산출물 설명]
- **생성 시각**: [현재 날짜/시간]
- **저장 경로**: `[프로젝트 경로]`

## 작업 목록

| # | 작업 | 파일 | 워커 | 상태 |
|---|------|------|------|------|
| 1 | [작업1] | `01-xxx.md` | research-worker | pending |
| 2 | [작업2] | `02-xxx.md` | analysis-worker | pending |
| 3 | [작업3] | `03-xxx.md` | content-worker | pending |

## 통합 지침
1. 모든 파일의 "메인 세션 전달 사항" 확인
2. [통합 시 수행할 분석/작업]
3. [최종 산출물 생성 방법]
4. 저장 위치: `[최종 파일 경로]`

## 완료 후 정리
- [ ] _parallel/ 폴더 삭제 또는 아카이브
```

**중요**: 이 매니페스트는 `/integrate` 커맨드가 읽어서 통합 작업을 수행합니다.

```
## 워커 실행

분해 완료 후 `/execute` 커맨드로 워커를 자동 실행할 수 있습니다:
- `/execute all` - 모든 pending 작업 실행
- `/execute 1` - 특정 작업만 실행
```

## 병렬 세션 출력 형식

각 병렬 세션은 반드시 다음 형식으로 결과 저장:

```
## [작업명] 결과

### 요약 (3줄 이내)
- 핵심 발견/결과

### 상세 내용
(본문)

### 메인 세션 전달 사항
- 통합 시 참고할 포인트
- 다른 작업과의 연결점
- 미결정 사항 (메인에서 판단 필요)
```

## 예시

**입력**: `/decompose 롯데 마켓 운영 회고 작성`

### 분해 분석 과정

**1단계 - 최종 산출물**:
- 형태: 회고 문서 (Markdown)
- 위치: `21.05-lotte-christmas-market/retrospective.md`

**2단계 - 필요한 입력**:
- 입력 1: 47일간 운영 로그 (이벤트, 문제, 대응)
- 입력 2: 매출 데이터 (일별, 주별 트렌드)
- 입력 3: 고객 피드백/반응
- 입력 4: 성공/실패 요인 분석

**3단계 - 획득 작업**:

| 입력 | 획득 작업 | 유형 |
|------|-----------|------|
| 운영 로그 | daily-logs 폴더 분석 | 리서치 |
| 매출 데이터 | 대시보드 데이터 분석 | 분석 |
| 고객 피드백 | 운영 메모에서 추출 | 리서치 |
| 성공/실패 요인 | 위 3개 통합 분석 | 분석 |

**4단계 - 의존성**:
- 운영 로그: 독립 ✓
- 매출 데이터: 독립 ✓
- 고객 피드백: 독립 ✓
- 성공/실패 요인: 위 3개에 의존 ✗

**5단계 - 분류**:
- 병렬: 운영 로그, 매출 데이터, 고객 피드백
- 순차: 성공/실패 요인 (1,2,3 완료 후)
- 통합: 최종 회고 문서 작성

### 최종 출력

```
## 미션 분석

**미션**: "롯데 마켓 운영 회고 작성"
**최종 산출물**: 47일 운영 회고 문서
**유형**: 복합 (리서치 + 분석 + 생성)
**저장 경로**: `pkm/20-operations/21-cafe-operations/21.05-lotte-christmas-market/_parallel/`

---

## 분해 결과

### 병렬 가능 (동시 진행)

2. [분석] 매출 데이터 분석 (일별/주별 트렌드)
3. [리서치] 고객 피드백/반응 정리

### 순차 필요 (의존성 있음)
4. [분석] 성공/실패 요인 도출 ← (1,2,3 완료 후)
5. [생성] 회고 문서 작성 ← (4 완료 후)
```
