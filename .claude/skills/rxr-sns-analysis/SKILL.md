# RXR SNS Raw Data Analysis

SNS 데이터를 크롤링하고, **Sincerity Filter**로 노이즈/리그램/협찬/위장 계정을 분류한 후, **2-Layer 분석** + **Sincerity Gate 3단계**를 적용하여 "진짜 반응"만 추출하는 통합 파이프라인입니다.

**핵심 철학:** 기존 도구의 "281건 mentions"이 아닌, **"진짜 유효 반응 204건"**을 찾아내고, **왜 그런 결과인지 해석**까지 제공하는 것.

**실증 케이스:** 새로중앙박물관 팝업 (2026.03~04) — 네이버 281건 + Brand24 비블로그 103건 = 총 384건 통합. Gate 3 적용 시 유효 204건, 긍정률 50%→38% (-12%p, 과장 긍정 77건 탈락). Auth -20점 급락(장기 운영 부작용), 방탈출 체험이 제품(68%)에 묻힘(7%)

| **분석 기간** | 선택 (**기본 규정: 이벤트 전 2주 + 이벤트 기간 + 이벤트 후 4주**) | "2026-02-26 ~ 2026-04-12" |

> **📌 분석 기간 자동 확장 규정 (필수)**

> 사용자가 별도 분석 기간을 명시한 경우에만 그 값을 사용합니다.
| **검색 키워드** | 선택 (자동 생성 가능) | "이클립스 팝업, 이클립스월드, #이클립스성수" |
| **Brand24 데이터** | 선택 | Brand24 엑셀 파일 경로 |

## 실행 프로세스

### STEP 1: 검색 키워드 설계

브랜드명+이벤트명에서 검색 키워드 세트를 자동 생성:

```
기본 키워드:
- "{브랜드명} 팝업"
- "{브랜드명} {장소}"
- "{이벤트명}"
- "{브랜드명} 팝업스토어"
- "{브랜드명} 후기"

해시태그:
- "#{브랜드명}팝업"
- "#{이벤트명약어}"
- "#성수팝업" / "#팝업스토어" (장소+일반 해시태그)

노이즈 사전 체크:
- 브랜드명이 다의어인지 확인 (예: "이클립스" = 껌/게임/일식/IDE)
- 다의어인 경우 제외 키워드 제안
```

사용자에게 키워드 확인 후 진행.

### STEP 2: 네이버 블로그 크롤링

**네이버 검색 API** (`.env`의 NAVER_CLIENT_ID, NAVER_CLIENT_SECRET 사용)

#### 2-1. 이벤트 기간 + 전후 데이터 수집

```python
# 키워드별 최대 1,000건 페이징 (start=1~901, display=100)
# sort=sim (관련도순) + sort=date (최신순) 병행
# 과거 데이터는 sim으로만 접근 가능 (date는 최신 100건만)
```

#### 2-2. 3단계 필터링

```
1차: 날짜 필터 — 분석 기간 내 포스트만
2차: 관련성 필터 — 브랜드명 + 이벤트 맥락 키워드 포함
3차: 정밀 필터 — 제목에 브랜드+이벤트 맥락 동시 포함 OR 설명에 강한 키워드
```

#### 2-3. 기간 분류 (자동 확장 규정 기반)

- **사전 2주** (베이스라인 버즈 — 사전 홍보·인플루언서 시딩·기대감 측정)
- **이벤트 진행** (본 운영 버즈 — 초반/중후반으로 세분화 권장)
- **사후 4주** (롱테일 후기 — 기억 미화·리셀 유입·진정성 변화 감지)

> 사용자가 별도의 분석 기간을 명시하지 않은 경우, **이벤트 시작일 −14일 ~ 이벤트 종료일 +28일**을 기본 적용합니다.

#### 2-4. 본문 크롤링

네이버 블로그 모바일 버전에서 본문 텍스트 추출:
```python
# 패턴 순차 시도:
# 1. se-main-container (스마트에디터)
# 2. post-view (구형 에디터)
# 3. se-component se-text
# 4. body 전체 (최후 수단)
```

**저장:** `{이벤트명}-naver-raw.json`, `{이벤트명}-naver-filtered.json`

### STEP 3: Brand24 데이터 통합 (선택)

Brand24 엑셀 파일 또는 웹 대시보드 스크린샷이 제공된 경우:

**통합 원칙:**
- Brand24 Blogs 건수는 네이버 API 크롤링과 **중복 가능** → 제외하고 비블로그 채널만 합산
- 총 멘션 = 네이버 블로그 N건 + Brand24 비블로그 채널(Instagram + News + X + TikTok + Facebook 등) M건
- 심층 분석(2-Layer + Sincerity Gate)은 텍스트 분석이 가능한 **네이버 블로그 기준**
- Brand24 데이터는 **멀티채널 확산 규모 + Reach** 파악용

**주의: Brand24 엑셀은 B열(column=2)부터 데이터 시작. A열은 빈 열.**

```python
# Mentions 시트: R2=헤더, R3~=데이터
# 컬럼: B=ID, C=Date, D=Hrs, E=Title, F=Content, G=Source, H=Domain, I=Category, J=Sentiment(1/0/-1)
```

- Numerical data 시트 → 날짜별 mentions/reach/sentiment 집계
- Analytics data 시트 → 전체 통계 (총 mentions, reach, likes, UGC 등)
- 채널 자동 분류: domain 기반 (instagram.com, tiktok.com, x.com, blog.naver.com 등)

**Brand24 무료 플랜 제한:** Excel 내보내기 50건 제한. 날짜별 집계(Numerical data)와 전체 통계(Analytics)는 제한 없이 확보 가능. 개별 포스트는 웹 화면 스크린샷으로 보충.

### STEP 4: Sincerity Filter 적용

**모든 raw data에 자동 태깅. 이것이 RXR의 핵심 차별화.**

#### 4-1. Content Class 분류 (A~G)

| 등급 | 분류 | 감지 방법 | 분석 처리 |
|:---:|------|------|------|
| **A** | 진성후기 | 개인 경험 신호 2건+ ("다녀왔", "갔는데", "먹어봤", "줄 서", "웨이팅", "친구랑", "데이트") + 구체적 디테일 3건+ ("분", "시간", "층", "원", "개", "맛이") | 2-Layer 핵심 분석 대상 |
| **B** | 일반포스트 | 특별 플래그 없음, 이벤트 관련 | 2-Layer 분석 대상 |
| **C** | 협찬/체험단 | "제공받", "#광고", "#ad", "체험단", "서포터즈", "원고료", "소정의" | 2-Layer 분석 (**진심 스펙트럼 분석** — 버리지 않음) |
| **D** | 나열형정보 | "총정리", "리스트", "모음", "BEST" 제목 | 버즈량만 카운트 |
| **E** | 리그램/단순공유 | "리그램", "퍼옴", "repost" OR 해시태그만(비해시 텍스트 <30자) OR 극히 짧은 텍스트(<20자) | 버즈량만 카운트 |
| **F** | 비즈니스/보도자료 | 보도자료 핵심 문구 3건+ 매칭 OR 비즈니스 신호 2건+ ("공식", "판매처", "구매링크", "할인코드") | 제외 또는 카운트만 |
| **G** | 노이즈(이벤트 무관) | 이벤트 관련 키워드 미포함 (다의어 문제) | 제외 |

**중요:** C등급(협찬)은 **제외하지 않는다.** 협찬 안에서도 진심 스펙트럼이 있으며(Auth 10~63), 이를 구분하는 것이 RXR의 차별화.

> **📌 Content Class와 eWOM 생성 동기의 매핑 (King et al., 2014)**
> eWOM(전자 입소문) 연구에서 리뷰 작성 동기는 4가지로 분류된다. Content Class는 이 동기를 텍스트 패턴으로 역추정하는 구조:
> | eWOM 동기 | Content Class | 해석 |
> |----------|:---:|------|
> | **이타적 동기** (타인을 돕고자) | A | 가장 높은 Auth. 구체적 디테일과 경험 공유 |
> | **자아 향상** (전문가로 보이고자) | A~B | 분석적 톤, Q4~Q5 비율 높음. Clout도 높을 수 있음 |
> | **경제적 보상** (포인트/협찬) | C | Auth 스펙트럼이 넓음(10~63). 동기가 외재적이므로 진심 여부를 별도 판단 |
> | **감정 해소** (불만 표출) | A~B(부정) | 부정 리뷰이지만 **감정적 진단성**이 높아 정보 가치가 큼 |

#### 4-2. Trust Score 산출 (0~100)

기본 100점에서 감점/가점:

| Flag | 점수 | 감지 |
|------|:---:|------|
| REGRAM | -30 | "리그램", "퍼옴", "repost" |
| HASHTAG_ONLY | -25 | 비해시 텍스트 <30자 & 해시태그 3개+ |
| PRESS_COPY | -35 | 보도자료 문구 3건+ 매칭 |
| PRESS_PARTIAL | -15 | 보도자료 문구 2건 매칭 |
| NO_PERSONAL_OPINION | -20 | 보도자료 문구 2건+ & 전체 200자 미만 |
| BIZ_SUSPECT | -25 | 비즈니스 신호 2건+ |
| LISTING | -20 | 제목에 "총정리/리스트/모음" |
| SPONSORED | -15 | 협찬 마커 감지 |
| OFF_TOPIC | -30 | 이벤트 키워드 미포함 |
| TOO_SHORT | -20 | 전체 텍스트 20자 미만 |
| PERSONAL_EXP | **+15** | 개인 경험 신호 2건+ |
| DETAILED | **+10** | 구체적 디테일 3건+ |

판정: **70+ = 신뢰 가능, 40~69 = 주의, ~39 = 필터 대상**

#### 4-3. Account Trust Level (Brand24 데이터에 적용)

| 등급 | 유형 | 감지 | 가중치 |
|:---:|------|------|:---:|
| T1 | 순수 개인 | 소규모 팔로워, 일상 혼재, 비정기 | ×1.0 |
| T2 | 마이크로 인플루언서 | 1K~10K, 카테고리 집중 | ×0.8 |
| T3 | 매크로 인플루언서 | 10K+, 높은 영향력 점수 | ×0.6 |
| T4 | 비즈니스 (명시) | 프로필에 사업자 표시 | ×0.3 |
| T5 | 비즈니스 위장 | 특정 브랜드만 포스팅, Auth 극저+Clout 극고 | ×0.1 |

**T5 감지 신호:** 특정 브랜드 포스트 80%+, 보도자료 유사도 높음, Authenticity 극저 + Clout 극고 조합

> **📌 Account Trust와 사회적 거리·유대강도 이론**
> - **파라소셜 관계(Parasocial Relationship)**: 팔로워가 인플루언서를 "가까운 지인"처럼 느끼는 일방적 친밀감. 마이크로 인플루언서(T2)는 팔로워와의 심리적 거리가 짧아 추천의 설득력이 높고 Auth도 상대적으로 진정성 있음. 매크로 인플루언서(T3)는 심리적 거리가 멀어 "광고"로 인식될 확률 증가.
> - **설득력 순위**: 가까운 지인 추천(T1) > 마이크로 인플루언서(T2) > 매크로 인플루언서(T3) > 비즈니스(T4). 이 순서가 Account Trust 가중치(×1.0 > ×0.8 > ×0.6 > ×0.3)와 일치하는 이론적 근거.
> - **행동적 증거 > 태도적 증거**: "많은 사람이 실제로 갔다"(행동)가 "많은 사람이 좋아한다"(태도)보다 진정성이 높다고 인식됨. T1(순수 개인)의 방문 후기는 행동적 증거이므로 가중치 최대.

### STEP 5: 2-Layer 분석

Content Class **A + B + C**에 해당하는 포스트 대상:

#### 5-1. Content Layer (무엇을 말했는가)

**토픽 분석:**
- 이벤트별 커스텀 토픽 사전 정의 (예: 공간/분위기, 게임/미션, 제품/맛, 굿즈/가챠, 포토부스, 웨이팅/예약, 가격/혜택)
- 키워드 빈도 기반 1차 토픽 → 상위 3개 추출
- 기간별 토픽 분포 변화 추적

**감성 분석:**
- 긍정/부정/혼합/중립 **4단계** (기존 3단계 대비 "혼합" 추가)
- 긍정+부정 키워드 동시 존재 시 "혼합" = 균형 잡힌 리뷰 신호

> **📌 부정성 편향과 감정적 진단성 (Negativity Bias & Emotional Diagnosticity)**
> - **69개 연구 메타분석 결과(Ismagilova et al., 2020)**: 리뷰의 **논거 품질(argument quality)**이 구매 의도에 가장 강한 영향(★★★★★). 리뷰 양(volume)은 약한 영향(★★☆☆☆). → **RXR의 RQL(품질 등급화)이 단순 건수 집계보다 정확한 이유의 학술적 근거.**
> - **부정 리뷰의 높은 정보 가치**: 부정 감정이 소폭 증가하면 리뷰 유용성(helpfulness)이 대폭 증가(Lee et al., 2026). 시선 추적 연구에서도 부정 댓글 주시 시간 > 긍정 댓글.
> - **해석 원칙**: Gate 3 통과 포스트 중 "혼합" 감성이 높은 비율을 차지하면, 이는 **부정적 신호가 아니라 정보 밀도가 높은 양질 리뷰의 증거**. 혼합 리뷰는 ELM 중심경로 처리의 산물이므로 Auth도 높은 경향.
> - **리포트 적용**: 부정/혼합 리뷰를 "실패 신호"로 해석하지 말 것. 오히려 "브랜드가 귀 기울여야 할 구체적 피드백"으로 프레이밍. 부정 리뷰 1건의 매출 영향 > 긍정 리뷰 1건(비대칭 효과).

**RQL (Review Quality Level):**

| 등급 | 조건 | 가중치 |
|:---:|------|:---:|
| Q5 서사형 | 500자+ & 디테일 3건+ & 개인경험 1건+ | ×2.0 |
| Q4 분석형 | 300자+ & 디테일 2건+ | ×1.5 |
| Q3 경험형 | 150자+ & (디테일 1건+ OR 개인경험 1건+) | ×1.0 |
| Q2 감상형 | 50자+ | ×0.5 |
| Q1 간단형 | 50자 미만 | ×0.2 |

> **📌 RQL 가중 자산값 산출 필수**
> RQL 등급 분포를 보여줄 때, 반드시 **가중합산 자산값**도 함께 산출한다:
> `리뷰 자산값 = Σ(Qn 건수 × 가중치)` (예: Q5 5건×2.0 + Q4 20건×1.5 + Q3 10건×1.0 + Q2 5건×0.5 + Q1 2건×0.2 = 52.9)
> 이 수치가 있어야 비교 리포트(rxr-sns-analysis-compare)에서 "리뷰 깊이" 축을 정량적으로 비교할 수 있다.
> 또한 **raw 건수 대비 자산값 비율** (자산값 / 총건수)도 표기하여 "평균 포스트당 리뷰 밀도"를 보여준다.

#### 5-2. Psyche Layer (어떻게 말했는가)

**대명사 패턴:**
- 1인칭 (나/내/저/제가/난/전) — 개인적 경험 서술
- 복수형 (우리/같이/함께) — 사회적 경험 공유
- 장소 (여기/이곳/이 공간/이 매장) — 공간 몰입

**조사 패턴 → Freshness Index:**
- "이/가" (초점/발견): "분위기**가** 좋았다" = 새로운 발견
- "은/는" (주제/설명): "이클립스**는** 껌 브랜드" = 이미 아는 것
- `Freshness = "이/가" / ("이/가" + "은/는") × 100`

> **📌 Freshness Index의 심리학적 근거**
> - **해석 수준 이론(Construal Level Theory, Trope & Liberman)**: "이/가"는 저수준 해석(구체적, 심리적으로 가까운 대상)을 반영하고, "은/는"은 고수준 해석(추상적, 이미 스키마에 통합된 정보)을 반영한다. Freshness가 높다는 것은 대상을 아직 구체적·감각적으로 처리하고 있다는 의미.
> - **기억 시스템 이론(Memory Systems)**: 시간이 지나면 감각 기억·에피소드 기억이 의미 기억(스키마)으로 전환되면서, "이/가"(새로운 인상)가 "은/는"(이미 아는 것)으로 자연 전환된다. 이것이 Freshness가 시간 경과 시 자연 하락하는 심리학적 메커니즘.
> - **BEI Discovery 축과의 연결**: Freshness 하락은 "나빠졌다"가 아니라 "감각 기억이 의미 기억으로 통합(consolidation) 완료되었다"는 정상적 인지 과정.

**Authenticity 추정 (0~100):**
```
기본 60점
- 협찬: -15
- 과잉긍정어 5개 초과 시: 초과분 × -3
- 느낌표 10개 초과 시: 초과분 × -2
- 표현 모방 의심(다른 리뷰와 유사 문장 구조 3건+): -10  ← 프라이밍 효과 보정
+ 개인경험 2건+: +10
+ 구체적 디테일 3건+: +10
+ 균형 평가(긍+부 공존): +15  ← ELM 중심경로 처리 신호
범위: 10~95
```

> **📌 Auth 산출의 심리학적 근거**
> - **균형 평가 +15점 근거 — 정교화 가능성 모델(ELM, Petty & Cacioppo)**: 장단점을 모두 언급하는 것은 중심 경로 처리(높은 인지적 노력)의 행동적 증거. 주변 경로(감정적 "좋아요!")보다 진정성이 현저히 높으므로 가점을 상향.
> - **인지 부조화(Cognitive Dissonance, Festinger) 주의**: 무료 체험·초대 참여자는 "내 시간을 쓸 만큼 좋은 것이었다"고 합리화하는 부조화 해소 패턴을 보일 수 있음. 협찬 -15점은 이 메커니즘을 부분적으로 보정하지만, 무료 체험(비협찬)에서도 이 패턴이 나타날 수 있으므로 과잉 긍정 감지와 병행 해석.
> - **프라이밍 효과(Priming, Bargh) 보정**: 블로그 리뷰어가 다른 리뷰를 먼저 읽고 쓰면 유사한 표현/감정 톤을 무의식적으로 모방. "비슷비슷한 긍정 리뷰"가 쏟아지는 원인. 표현 모방 의심 시 -10점으로 프라이밍된 모방 리뷰를 보정.

**Clout 추정 (0~100):**
```
기본 40점
+ 추천 표현("추천", "꼭 가", "강추", "가보세요"): 건당 +15
+ 복수형 > 1인칭: +10
+ 제3자 경험 인용("친구가", "같이 간 사람이", "데려갔는데"): +12  ← 사회적 증거
+ 예방 초점 추천("안 가면 후회", "놓치면", "없어지기 전에"): +8  ← 손실 회피 활용
+ 조건부 추천("~라면 강추", "~하는 분들에게", "~좋아하면"): +10  ← 높은 신뢰도
+ 정체성 선언("~덕후", "~팬", "~애호가", "~러버"): +12  ← 사회적 정체성
범위: 10~95
```

> **📌 Clout 산출의 심리학적 근거**
> - **사회적 증거(Social Proof, Cialdini)**: 본인의 추천보다 제3자 경험 인용("친구도 좋아했어")이 실제 설득력이 더 높음. 독립적 경험 확인은 사회적 증거 + 권위의 결합.
> - **조절 초점 이론(Regulatory Focus, Higgins)**: "꼭 가봐요"(촉진 초점)와 "안 가면 후회"(예방 초점)는 동일한 추천이지만 메커니즘이 다름. 예방 초점은 **손실 회피(Loss Aversion)**를 활성화하므로 실제 전환율이 더 높을 수 있음.
> - **사회적 정체성 이론(Social Identity, Tajfel)**: "나는 ___팬/덕후"라는 정체성 수준의 추천은 경험 수준의 추천("이거 괜찮더라")보다 지속성이 현저히 높음(자기 정체성은 쉽게 안 바뀜). Advocacy Decay 해석 시 정체성 선언 비율도 참조.

**과잉 표현 감지:**
- 극단 긍정어: "진짜", "너무너무", "완전", "대박", "미쳤", "최고의", "인생"
- 느낌표 수: 10개 이상이면 과잉 신호

### STEP 6: Sincerity Gate 3단계

| Gate | 필터 | 답하는 질문 | 새로 실증 |
|:---:|------|---------|---------|
| **Gate 1** | G(노이즈) 제거만 | "시장에 얼마나 노출됐는가?" | 281건, 긍정 50% |
| **Gate 2** | + C(협찬) + E(리그램) + F(비즈니스) 제거 | "유료 홍보 빼면 실제 반응은?" | 276건, 긍정 49% (-1%p) |
| **Gate 3** | + Authenticity 60+ 필터 | "진심으로 좋았던 사람은?" | 204건, 긍정 38% (-12%p) |

각 Gate에서 집계:
- 건수, 긍정/부정/중립 비율, Authenticity/Clout/Freshness 평균
- **Gate간 Delta(차이) 강조** — 이것이 핵심 인사이트

**새로중앙박물관 실증 결과:**
- 과장 긍정 77건이 Gate 3에서 탈락 → 긍정률 **12%p 하락** (Gate 1→3)
- 협찬 Auth **37.8** vs 자발적 **58.7** (차이 20.9점)
- Auth 이벤트전 66.4 → 이벤트후 46.2 (**-20점** 급락, 장기 운영 부작용)
- 방탈출(핵심 컨셉)이 토픽의 7%만 차지 — 제품(68%)에 묻힘

> **📌 Auth 60 임계값 — 왜 규모와 무관하게 일률 적용하는가?**
>
> Gate 3의 Auth ≥ 60 기준은 데이터 규모(2일 팝업 26건이든, 16일 팝업 281건이든)에 관계없이 동일하게 적용한다. 이유:
> 1. **텍스트 품질 기준은 모수 크기에 의존하지 않음** — Auth는 개별 포스트의 텍스트 패턴(과잉 표현, 구체성, 경험 신호)을 평가하므로, 전체 건수와 무관
> 2. **소규모 데이터에서 임계값을 낮추면 노이즈가 "진심"으로 오분류되는 위험 증가** — 소수 데이터일수록 엄격한 기준이 더 중요
> 3. **BEI 4축 간 비교 가능성 유지** — 팝업마다 다른 임계값을 적용하면 follow-up/compare 분석에서 기준선이 달라져 교차 비교 불가
>
> **단, 초단기 팝업(3일 이하) 또는 극소 데이터(30건 미만) 시:**
> - 리포트 상단에 "⚠️ 구조적 한계: 데이터 N건으로 통계적 대표성이 제한적" 경고 명시
> - Gate 3 통과 건수가 10건 미만일 경우 "해석 주의" 라벨 추가

  1. **📊 Raw Data 근거 박스:** 실제 포스트 제목, 날짜, Auth/Clout/RQL 수치. 최고점 vs 최저점 실제 사례 대비 (초록 배경 vs 빨강 배경)
  2. **결론 해석 박스 (파란 배경 #eef2ff):** 수치의 의미를 구체적으로 해석. 단순 나열("Auth 차이 20.9점")이 아닌, **왜 그런 차이가 나는지**(원인) + **그래서 뭘 의미하는지**(시사점) + **기존 도구와 뭐가 다른지**(차별화)를 2~3줄로 설명.
  - 예시: "→ 협찬 포스트는 자발적 대비 진정성이 36% 낮음. 같은 '긍정'이라도 의무적 톤 vs 실제 감동의 질적 차이가 크다. RXR은 같은 긍정 안에서도 진심의 농도를 구분한다."

**Sincerity Gate 섹션:**
- Gate 1 → 2 → 3 퍼널 시각화 (너비 줄어드는 바)
- 각 Gate에서 건수/긍정률/Auth 비교

**기간별 심리 변화 섹션:**
- Auth/Clout/Freshness 게이지 + 기간별 인라인 바 차트
- 📊 각 기간의 수치가 다른 이유를 실제 텍스트 패턴으로 설명 (3열 그리드: 사전/이벤트/이후)
- **⚠️ Freshness 해석 필수 규정:** Freshness 게이지마다 반드시 2문장 이상 해석을 포함한다. (1) "이/가 vs 은/는" 비율이 의미하는 바 (2) BEI Discovery 축과의 연계 ("Freshness가 높으면 Discovery 기여 ↑, 낮으면 Discovery 약화 원인을 설명"). Freshness 수치만 나열하고 해석을 생략하면 follow-up 분석(BEI Discovery)의 근거가 약해진다.

**협찬 vs 자발적 섹션:**
- VS 비교 카드 (건수 + Auth 게이지)
- 📊 협찬 포스트 개별 Auth 스펙트럼 (실제 포스트별 Auth, 느낌표 수, 판단 근거 — 초록/노랑/빨강 배경)

**종합 비교표:**
- 9개 영역 BEFORE/AFTER 테이블

**BOTTOM LINE (확장형 — 필수):**
- 상단: 라이트 배경 + 큰 퍼플 텍스트로 핵심 결론 대비 (기존 vs RXR)
- **"왜 이런 결과가 나왔는가?" 4카드 그리드:**
  - 각 카드: 수치 헤드라인(빨강/오렌지/퍼플/시안) + 2~3줄 해석 (원인 → 결과 → 시사점)
  - 예시: "긍정률 -12%p → 과장 긍정 77건이 실제를 부풀림, 기존 도구가 '좋아요!'와 '진심'을 구분 못함"
- **"제언: 다음 팝업에서 무엇을 바꿀 것인가?" 3카드 그리드:**
  - 각 카드: 아이콘 + 제목 + 구체적 액션 (2~3줄)
  - 예시: "체험 콘텐츠 비중 확대 — 미션 클리어 인증샷·스토리 공유 장치로 Q5 서사형 비율 상승"

**부록: Sincerity Gate 3 통과 — 유효 Raw Data 테이블 (필수)**
- Gate 3(Auth 60+, 자발적) 통과한 전체 포스트 리스트
- **테이블 컬럼**: #, 날짜, 감성(색상), Auth(색상: 75+=초록, 60~74=퍼플), Clout, Freshness, RQL, 토픽, 기간, 제목(원본 블로그 링크), 블로거

- 이 부록이 있어야 "왜 이런 결론이 나왔는지" 근거를 즉시 확인 가능

## 주의사항

- Authenticity/Clout 수치는 LIWC 소프트웨어가 아닌 **텍스트 패턴 기반 추정치**
- 협찬 포스트는 **제외하지 않고 진심 스펙트럼으로 분석** — 이것이 기존 도구와의 핵심 차이

- HTML 리포트 상단에 **리서치 개요** (대상/기간/관점/읽는법) 필수 포함
- **BOTTOM LINE은 확장형 필수** — 핵심 수치 대비 + "왜 이런 결과?" 4카드 + "제언" 3카드. 단순 1문장 결론 금지.
- **데이터 소스 통합 시** Brand24 Blogs는 네이버와 중복 제외, 비블로그 채널만 합산. 총 멘션 = 네이버 N + Brand24 비블로그 M.

**비밀번호 잠금 파일 사양:**
- 외부용 HTML 전체를 base64 인코딩 → JS chunks로 분할 저장
- SHA-256 해시로 비밀번호 검증 (기본 비밀번호: `RXR-projectrent`, 사용자 요청 시 변경)
- 잠금 화면: 퍼플 자물쇠 아이콘 + 비밀번호 입력 + 에러 시 흔들림 애니메이션
- 올바른 비밀번호 입력 시 → document.write()로 전체 리포트 렌더링
- **파일 하나로 완결** (외부 서버/CDN 불필요, 오프라인 작동)
- 개발자 도구에서 원문 직접 열람 불가 (base64 chunks만 보임)

**공유 방법 안내 (사용자에게 전달):**
- 파일: `-locked.html` 하나만 전달 (카톡/메일/클라우드 링크)
- 비밀번호: **별도 채널**로 전달 (파일은 메일, 비번은 카톡 등)
- 받는 사람: 다운로드 → 브라우저로 열기 → 비밀번호 입력 → 리포트 확인
