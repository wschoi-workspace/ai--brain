# /brand-analysis — 브랜드 분석 자동화 파이프라인

> `/brand-thinking` 전단계. 브랜드명 하나로 시장 리서치부터 30p 제안서까지 자동 생성.
> 세스크멘슬 프로젝트(2026-06-07~08)에서 검증된 6-Phase 프로세스를 스킬화.

---

## 입력 파라미터

사용자 입력에서 아래 정보를 추출합니다. 누락된 항목은 질문하세요:

| 항목 | 필수 | 예시 |
|------|:---:|------|
| **브랜드명** | 필수 | "세스크멘슬", "블루보틀" |
| **카테고리** | 필수 | "샤퀴테리", "스페셜티 커피" |
| **프로젝트 목적** | 선택 (기본: 브랜드 현황 진단) | "서브 브랜드 스핀오프 전 진단", "리브랜딩 방향성" |
| **경쟁사** | 선택 (자동 리서치) | "메종조, 소금집" |
| **분석 기간** | 선택 (기본: 90일) | "최근 30일", "6개월" |

## 실행 프로세스 — 6 Phase 자동화

### PHASE 0: 킥오프 — 프로젝트 설정
```
입력: 브랜드명, 카테고리, 목적
출력: 프로젝트 폴더 + config + progress.md
```

1. **프로젝트 폴더 생성**
   ```
   10-projects/{번호}-{브랜드}-sns/
     config.json
     data/
     reports/
     research/
     {번호}-progress.md
   ```

2. **config.json 작성** — 브랜드 카테고리에 맞게 커스터마이징
   - 브랜드 aliases, 검색 쿼리
   - 토픽 사전 (카테고리별 맞춤)
   - 감성 키워드 사전
   - 언어학 분석 파라미터
   - 협찬 마커, 경험/디테일 시그널

3. **progress.md 초기화** — 프로젝트 개요, 목적, 배경

### PHASE 1: 시장 리서치 (병렬 에이전트 3개)

| Agent 2 | 브랜드 프로파일 | "{브랜드명}", "{브랜드명} 셰프", "{브랜드명} 매장" |

### PHASE 2: RXR Mention Analysis (크롤링 + 3-Layer 분석)

1. **크롤러 스크립트 생성**
   - `80-r-tech/84-scripts/xylitol/xylitol_crawl.py`를 템플릿으로 복사
   - CONFIG_PATH만 해당 프로젝트로 변경
   - 파이프라인 스크립트도 동일하게 복사+수정

2. **크롤링 실행** (--days 90 기본)
   ```bash
   python3 80-r-tech/84-scripts/{brand}/{brand}_crawl.py --days 90
   ```

3. **Linguistic Layer 파이프라인 실행**
   ```bash
   python3 80-r-tech/84-scripts/{brand}/{brand}_pipeline.py --skip-crawl
   ```

4. **경쟁사 2~3개 동일 실행** (병렬)
   - 경쟁사별 프로젝트 폴더 + config + 크롤러 자동 생성
   - 병렬로 크롤링 + 파이프라인 실행

**산출물:**
- `data/{날짜}/analysis-results-enhanced.json` (브랜드 + 경쟁사 각각)

### PHASE 3: 데이터 분석 & 인사이트 도출

Python 스크립트로 다음 분석 순차 실행:

1. **3-Brand KPI 비교** — 멘션수, 긍정률, Auth, Clout, Freshness, 협찬 비율
2. **토픽×감성 교차분석** — 각 토픽별 긍정률, Auth, Clout, RQL
3. **감성 키워드 매핑** — 구매의사결정 관련 감성 키워드 추출, 독점/약점 pp 격차
4. **포지셔닝 맵 축 산출** — X축(일상↔특별), Y축(감성↔전문성) 점수 계산

### PHASE 4: Thinking Partner — 방향성 논의

**자동 실행하지 않음** — 사용자와 대화형으로 진행.

1. Phase 1~3 결과를 요약하여 공유
2. **패턴 감지 자동 실행** — 아래 5가지 패턴 체크 후 해당 질문 제시
3. **데이터→전략 질문 전환** — 요약이 아니라 선택적 질문으로 전환
4. 사용자의 맥락 정보 수집 (의뢰 맥락, 브랜드 오너 인식, 비즈니스 계획 등)
5. 전략 방향 확정

**★ 패턴 감지 체크리스트 (Phase 3 데이터 기반으로 자동 점검):**

| 패턴 | 감지 조건 | 개입 질문 |
|------|----------|----------|
| **자산-정체성 역설** | 전술적 강점(높은 멘션/긍정률)이 브랜드 본질과 다른 토픽일 때 | "가장 돈 되는 요소가 브랜드 정체성을 해치고 있지 않나?" |
| **아키텍처 갈림길** | 신규 사업/서브 브랜드가 논의될 때 | "모 브랜드 전문성이 충분히 축적됐나? 스핀오프 전에 본체 복구가 선행 필요하지 않은가?" |
| **설명 과잉 경보** | 브랜드 설명에 키워드 3개 이상 필요할 때 | "소비자가 3초 안에 이해 가능한가? 장면 하나로 각인 불가?" |
| **감정 점유 공백** | 경쟁사 분석에서 아무도 점유 안 한 감정이 보일 때 | "이 빈 감정은 점유 가능하면서 브랜드 진정성과 일치하는가?" |
| **공간=미디어** | 매장/공간 설계가 논의될 때 | "이 공간에서 가장 먼저 찍히는 사진은 뭔가? 광각인가 클로즈업인가?" |

**데이터→전략 질문 전환 프로세스:**
```
Raw Data → 패턴 발견 → 비교 격차 → 구조적 원인 → ★ 선택적 질문
```
- "이 데이터에서 결국 물어야 할 질문은 하나야—"
- "[A방향]과 [B방향] 중 뭘 고를 건지에 따라 다음 액션이 완전히 달라져"

**기본 질문 템플릿:**
- "이 분석의 의뢰 맥락이 뭔가요? (클라이언트 직접 요청 vs 제안용)"
- "브랜드 오너는 자기 브랜드를 뭘로 생각하고 있을까요?"
- "데이터가 보여주는 [핵심 발견]에 대해 어떻게 생각하시나요?"
