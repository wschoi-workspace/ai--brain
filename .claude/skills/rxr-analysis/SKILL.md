## 오프라인 퍼널 확장 모드 (funnel_xlsx 제공 시)

사용자가 `funnel.xlsx` 경로를 제공하면 **2-Layer 분석(STEP 1~6) 전에** 다음 4단계를 먼저 수행합니다. 모든 공식·엑셀 스키마·임계값은 `80-r-tech/81-framework/rxr-offline-funnel-spec.md` 확정판을 그대로 따릅니다.

### STAGE A · 오프라인 퍼널 KPI (일자별 집계)
- 시트: `00_meta`, `10_foottraffic`, `20_entry_wait`, `30_dwell_agg`, `40_pos`
- 산출: L2_rate(진입률), L2_wait/L2_churn, **L3_DQS**, L4_rate(구매 전환율), L4_aov(객단가), L5_rate, L6_rate
- DQS 공식: `(평균체류분 / expected_dwell_min) × density_coef × 1.0` (집계 버전이므로 personality_w 고정)
- 결측 규칙: 옵션 컬럼 누락 → 해당 KPI만 `null`, 리포트에 "데이터 미수집" 명시

### STAGE B · 감성품질 스코어링 (기존 command 재사용)
- 시트 `50_survey` → `rxr-survey-analysis` Mode 2 호출 → EDI/VRI/레벨
- 시트 `60_sns_urls` → `rxr-sns-analysis` 호출 (phase 자동 분류 PRE/DURING/POST) → Auth/Clout/Freshness/Sentiment/RQL
- phase 자동 분류: `meta.period_start/end` 기준. PRE=시작 14일 전~, POST=종료+28일
- 정규화: 전 지표 0~1 (spec §4 표 참조)

### STAGE C · PSI (Perception Shift Index)
- 3시점(PRE/DURING/POST) 5축 벡터 — [감성, 진정성, 확신도, 신선함, 깊이]
- 델타: `ΔE→X = DURING − PRE`, `ΔX→A = POST − DURING`, `ΔAll = POST − PRE`
- `PSI_total = 0.4·‖ΔE→X‖ + 0.4·‖ΔX→A‖ + 0.2·‖ΔAll‖`
- `PSI_signed = sign(ΔAll · w_dir) × PSI_total`, `w_dir=[0.3, 0.3, 0.2, 0.1, 0.1]`
- 임계값 라벨: `<−0.25` 경보 / `±0.10` 무변화 / `+0.25` 체험→옹호 / `+0.45` 입소문 촉발 (spec §5-3)
- 경보 플래그: 자기방어적 후기 / 신선함 소진 / 설문 긍정 편향 / 진입-구매 갭 (spec §5-4)

### STAGE D · IEQ (Integrated Experience Quality)
```
FE = 0.15·L2_rate_n + 0.25·L3_DQS_n + 0.30·L4_rate_n + 0.15·L5_rate_n + 0.15·(1 − L2_churn)
EQ = 0.35·EDI_n + 0.25·Auth_post_n + 0.20·Clout_post_n + 0.10·Fresh_post_n + 0.10·RQL_post_n
IEQ = √(FE · EQ) × (1 + 0.2·PSI_signed)
```
- 4사분면 매트릭스(고/저 FE × 고/저 EQ)에 이벤트 배치 → 사분면별 자동 전략 제안(spec §6-2)

### 확장 리포트 섹션 (HTML)
`funnel_xlsx` 제공 시 기존 STEP 5의 HTML 앞단에 다음 섹션 6개를 삽입:
1. 표지 · IEQ 총점 게이지
2. 요약 카드 6 (L2_rate / L3_DQS / L4_rate / L4_aov / EDI / Auth_post + sparkline)
3. 퍼널 시각화 (L0→L5 깔때기 + 드롭 원인 주석)
4. 3시점 감성품질 5축 레이더 (PRE/DURING/POST)
5. PSI 패널 (ΔE→X / ΔX→A / ΔAll + 판정 라벨 + 경보 플래그)
6. IEQ 사분면 매트릭스 (본 이벤트 위치 + 전략 제안)

그 다음 기존 STEP 2~4의 2-Layer 섹션이 이어집니다 (섹션 7 세그먼트 교차, 섹션 8 Sincerity Gate, 섹션 9 전략 제안, 섹션 10 부록).

### 템플릿 배포
사용자에게 빈 엑셀을 요청할 때는 다음 경로의 템플릿을 안내:
```
80-r-tech/81-framework/rxr-offline-funnel-template.xlsx
```
(6시트 · 일자별 집계 · 한국어 헤더 · 필수 컬럼 하늘색 표시 · 각 헤더에 툴팁 주석)

**비교 프레임:**
```
A: 기존 브랜드 인식 (Before)  →  B: 이벤트 후 인식 (During/After)
```

**중요:** A(기존 인식)는 "리뉴얼 전"이 아니라 **소비자가 해당 브랜드에 대해 갖고 있던 누적된 인식**을 의미합니다. 브랜드의 강점뿐 아니라 **약점·논란**도 반드시 수집합니다.

## 실행 프로세스

### STEP 1: 데이터 수집

WebSearch와 WebFetch를 사용하여 리뷰 데이터를 수집합니다.

#### 1-1. 기존 브랜드 인식 (A) 수집

**수집 대상:** 이벤트 이전의 브랜드 전체 인식 — 강점, 약점, 논란 모두 포함
```
검색 쿼리:
- "{브랜드명}" 후기 OR 리뷰 site:blog.naver.com
- "{브랜드명}" 맛 OR 품질 OR 평가 커뮤니티
- "{브랜드명}" 단점 OR 불만 OR 논란    ← 반드시 부정 인식도 수집
- "{브랜드명}" 브랜드 인지도 OR 소비자 인식
- "{브랜드명}" vs {경쟁브랜드}    ← 비교 프레임 파악
```
목표: 15건+
추출 항목: 리뷰 원문 텍스트(핵심 문장), 출처, 감성 톤

#### 1-2. 이벤트 후 인식 (B) 수집

**수집 대상:** 이벤트 체험 리뷰, 현장 르포, SNS 반응
```
검색 쿼리:
- "{이벤트명}" 후기 OR 리뷰 OR 방문
- "{이벤트명}" 체험 블로그 인스타
- "{브랜드명}" 팝업 현장 가보니 기사
```
목표: 20건+

#### 1-3. 기업 데이터 & 뉴스 수집

- 매출, 판매량, 방문객 수 등 정량 데이터
- 수상 이력, 마케팅 캠페인 관련 기사
- 기업 관계자 인용문

### STEP 2: Layer 1 — Content Layer (뭐라고 말했는가)

수집된 리뷰를 **A(기존) vs B(팝업 후)** 비교 분석합니다.

#### 2-1. 감성 분류 (구간별)
각 리뷰를 긍정/중립/부정으로 분류하고 비율 산출.

| 구간 | 긍정 | 중립 | 부정 | 변화 |
|------|------|------|------|------|
| A: 기존 인식 | __% | __% | __% | 기준선 |
| B: 이벤트 후 | __% | __% | __% | ↑↓ |

#### 2-2. 토픽 클러스터 (구간별)
리뷰에서 반복 등장하는 주제를 5~8개 클러스터로 그룹핑.
**핵심 질문:** 이벤트로 어떤 토픽이 새로 등장하고, 어떤 토픽이 사라지는가?

#### 2-3. 핵심 키워드 & Bigram 추출
- A/B 각각 Top 10 단어, Top 5 Bigram
- **신규 등장 키워드** / **소멸 키워드** 별도 표시

#### 2-4. 대표 리뷰 선별
- A: 긍정 대표 1~2건, 부정 대표 1~2건
- B: 긍정 대표 2~3건, 부정 대표 1건, 특이/통찰 1건

| 변수 | A: 기존 | B: 이벤트후 | 변화 | 해석 |
|------|:---:|:---:|:---:|------|

| **Clout** (추천 확신도) | __ | __ | __ | 확언형("꼭"), 청유형("~자"), 1인칭 복수 |

#### 3-3. 기능어 패턴 근거

```
B: "내가 직접 풀어봤는데 두근거렸어요" — 1인칭+체험 서술 = Auth 높음
```

### STEP 4: Layer 3 — 교차 분석 (Synthesis)

#### 4-1. 패턴 분류 (A vs B)

| 패턴 | 조건 | A | B | 변화 | 설명 |
|------|------|---|---|------|------|
| **P-01 진짜 팬** | 긍정 + Auth↑ + Clout↑ | __% | __% | | 진심으로 추천 |
| **P-02 인스타 소비** | 긍정 + Auth↓ + Clout↓ | __% | __% | | 바이럴만, 재방문 약 |
| **P-03 확신 부족** | 긍정 + Auth↑ + Clout↓ | __% | __% | | 좋긴 한데 추천은 안 함 |
| **P-05 탐색만** | Analytical↑ + Clout↓ | __% | __% | | 비교만 함 |
| **체험 전도사** | Auth극높 + Clout극높 + Tone극긍 | — | __% | 신규 | 능동적 체험 후 적극 전파 |
| **Zone C 진심 불만** | 부정 + Auth↑ | __% | __% | | 실제 문제, 개선 필요 |
| **Zone D 노이즈** | 부정 + Auth↓ | __% | __% | | 악의적/필터링 대상 |

**각 패턴에 반드시 포함할 해설:**
- 한 줄 비전문가 설명 (예: "진짜 팬 = 진심으로 좋아하고 남에게 적극 추천하는 최고의 고객")
- 해당 패턴의 대표 리뷰 인용

#### 4-2. Auth × Clout 매트릭스

2×2 매트릭스로 패턴 이동 시각화:
```
              Clout 낮음        Clout 높음
Auth 높음  │ P-03 확신부족  │ P-01 진짜팬    │
Auth 낮음  │ P-05 탐색만    │ P-02 인스타    │
```
**각 칸에 A→B 비율 변화와 이동 방향 표시**

#### 4-3. 기존 약점 해결 테이블

이벤트가 브랜드의 기존 약점을 어떻게 해결했는지 매핑:

| 기존 약점 | 이벤트가 만든 변화 | 수치 근거 |
|----------|-------------------|----------|
| (예: "맛이 밍밍") | 감각축 전환: 미각→시각 | 맛 토픽 28%→15% |
| (예: "추천 안 함") | "꼭 가봐!" 전환 | Clout 35→68 |

#### 4-4. 세그먼트별 전략 제안

비중 높은 상위 3개 패턴에 대해:
- 세그먼트 특성 요약
- 구체적 대응 전략 (공간/제품/마케팅)
- 기대 효과

**비전문가 해설 (card-explain):**
모든 섹션에 노란 배경 해설 박스를 포함합니다:

```html
<div class="card-explain">
  <span class="emoji">💡</span> <strong>이게 뭔가요?</strong><br>
  {비전문가 용어로 설명}
</div>
```

포함 위치:
- 리포트 상단: "이 리포트는 이렇게 읽으세요" (3단계 가이드)
- A/B 비교 프레임 설명
- Content Layer 용어 설명
- 감성 차트 해석 ("이 숫자가 의미하는 것")
- 토픽 전환 의미 ("이게 왜 중요한가요?")
- Psyche Layer 용어 설명 ("왜 이게 중요한가?")
- LIWC 4개 변수 각각: 게이지 아래에 A/B 실제 문장 대비 + "쉽게 말하면"
- Freshness/Distance 직관적 설명 (한국어 조사 예시)
- 교차 분석 소비자 유형 각각 한 줄 설명
- 패턴 차트 해석 ("이 차트에서 가장 중요한 변화")
- 약점 해결 테이블 요약 ("한마디로")

**LIWC Gauge 해설 포맷 (필수):**
```html
<div class="gauge-box">
  <div class="gauge-header">
    <span class="gauge-name">Clout (추천 확신도) <span class="tag tag-up">최대 변화</span></span>
    <span class="gauge-meaning">남에게 추천할 만큼 확신이 있는가?</span>
  </div>
  <!-- gauge track -->
  <div class="gauge-explain">
    <span class="before">"부드러운 편이다"</span> — "편이다"는 조심스러운 표현 = Clout 낮음<br>
    <span class="after">"꼭 가봐야 해!"</span> — "꼭"은 강한 확신 = Clout 높음<br><br>
    <strong>이 +33이 왜 가장 중요한가:</strong> {비즈니스 임팩트 설명}
  </div>
</div>
```

## 주의사항

- LIWC 수치는 **실제 LIWC 소프트웨어가 아닌 텍스트 패턴 기반 추정치**임을 HTML/MD 모두에 명시
- Freshness Index는 **RXR 독자 프레임워크**로 학술적 검증 진행 중임을 명시
- 리뷰 수가 구간별 10건 미만이면 "데이터 부족, 해석 주의" 경고 표시
- 크롤링 시 개인정보(작성자 ID 등)는 수집하지 않음
- **비전문가 해설을 절대 생략하지 않는다** — 모든 수치, 지표, 차트에 "이게 뭔데?", "왜 중요한데?" 해설 필수
- 기존 브랜드 **약점과 논란을 반드시 수집**한다 — 긍정만 수집하면 이벤트 효과를 정확히 측정할 수 없음
