# work-analysis: 업무보고 데이터 분석 → 팀/개인 업무정의서 생성

구글시트에 축적된 일일업무보고 데이터를 분석하여 팀 단위/개인 단위 업무정의서를 자동 생성한다.
"업무 분석", "업무 정의", "팀 분석", "개인 프로필", "work analysis", "업무보고 분석" 등을 언급하면 자동 실행.

- **시트2 서브업무**: 날짜|이름|팀|카테고리|업무명|상태
- **시트3 메타**: 날짜|이름|팀|내일계획|개선포인트|블로커종합

## 실행 흐름

### Step 1: 분석 범위 확인

사용자에게 확인:
- **기간**: 최근 1주 / 2주 / 1개월 / 전체
- **범위**: 팀 단위 / 개인 단위 / 전체
- **대상**: 특정 팀명 또는 특정 이름 (지정 안 하면 전체)

### Step 2: 데이터 읽기

```bash
# 핵심업무 전체 읽기
gws sheets spreadsheets values get \
  --params '{"spreadsheetId": "1izuwjWPiJfbaJZ-Ib-U_NO0_Ln1_5yRr1SIRAum-hi4", "range": "핵심업무!A1:K5000"}'

# 서브업무 전체 읽기
gws sheets spreadsheets values get \
  --params '{"spreadsheetId": "1izuwjWPiJfbaJZ-Ib-U_NO0_Ln1_5yRr1SIRAum-hi4", "range": "서브업무!A1:F5000"}'

# 메타 전체 읽기
gws sheets spreadsheets values get \
  --params '{"spreadsheetId": "1izuwjWPiJfbaJZ-Ib-U_NO0_Ln1_5yRr1SIRAum-hi4", "range": "메타!A1:F5000"}'
```

### Step 3: 데이터 파싱 및 필터링

- GWS CLI 응답을 JSON으로 파싱
- 기간 필터링: 날짜 컬럼 기준
- 팀/개인 필터링: 이름 또는 팀 컬럼 기준
- 핵심업무 + 서브업무 통합 (카테고리별 집계)

### Step 4: 분석 수행

#### 4-1. 팀 단위 분석

핵심업무 + 서브업무 데이터를 팀별로 그룹핑 후:

1. **팀 핵심 업무 TOP 5** — 업무명 유사도 기반 클러스터링, 빈도순 정렬
2. **카테고리 비중** — 기획/실행/커뮤니케이션/관리/리서치/교육/기타 비율 산출
3. **팀 공통 블로커** — 이슈 컬럼 + 블로커종합 컬럼에서 키워드 추출, 빈도순
4. **사용 도구 현황** — 도구 컬럼 파싱, 사용자 수 집계
5. **개선 기회** — 개선포인트 컬럼에서 공통 패턴 추출

#### 4-2. 개인 단위 분석

핵심업무 + 서브업무 데이터를 개인별로 그룹핑 후:

1. **핵심 업무 정의** — 반복 패턴 기반 TOP 3~5 업무, 주당 빈도 추정
2. **카테고리 비중** — 개인별 카테고리 비율
3. **주요 도구** — 사용 빈도순
4. **반복 블로커** — 이슈 패턴
5. **자동화 기회** — 개선포인트 컬럼에서 반복 언급된 항목

### Step 5: HTML 보고서 생성

R스타일 디자인 가이드 적용하여 HTML 보고서 생성.
저장 위치: `10-projects/28-ai-education-ax/reports/`
