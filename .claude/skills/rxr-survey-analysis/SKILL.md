# RXR Survey Analysis

오프라인 방문객의 **마케팅 인게이지먼트 깊이**를 설문으로 측정하고, **표면 만족과 진짜 만족의 격차**를 시각화하는 도구입니다.

**핵심 철학:** 기존 설문의 "추천하시겠어요? → 91% 좋다"로는 알 수 없는 것 — **"진짜 움직일 사람이 몇 %인가?"**를 EDI(Engagement Depth Index)로 답합니다.

**RXR 생태계 내 위치:**
```
RXR SNS Analysis     → 온라인 바이럴 품질 (SNS 데이터)
RXR CCTV Analyzer    → 오프라인 행동 (영상 데이터)
RXR Survey Analysis  → 오프라인 인게이지먼트 깊이 (현장 설문)
```

**교차 검증:** EDI 분포 ↔ SNS 2-Layer 패턴 분포를 집단 수준에서 비교하여, 설문 예측력을 검증합니다.

사용자에게 모드를 확인합니다:

| 모드 | 용도 | Input | Output |

## 공통: 경험 구성요소 프레임워크

모든 오프라인 경험은 아래 **6개 구성요소의 조합**으로 표현됩니다. 경험 유형에 따라 해당 요소를 on/off합니다:

| # | 구성요소 | 설명 | 세부 항목 예시 |
|---|----------|------|---------------|
| **S** | **Space (공간)** | 물리적 공간 경험 | 인테리어, 동선, 조명, 컬러, 분위기 |
| **C** | **Content (콘텐츠)** | 기획/전시/스토리 | 컨셉 기획, 전시 구성, 브랜드 스토리, 테마 |
| **P** | **Product (상품)** | 판매 상품/굿즈 | MD 상품, 한정판, 패키징, 가격 |
| **V** | **Service (서비스)** | 인적 서비스 | 직원 응대, 안내, 대기 관리, 친절도 |
| **A** | **Activity (체험)** | 참여형 활동 | 체험 프로그램, 워크숍, 게임, 미션 |
| **M** | **Community (커뮤니티)** | 사회적 연결 | 네트워킹, 팬 미팅, 동행 경험, 포토존 |

### 유형별 기본 프리셋

| 오프라인 경험 유형 | 기본 ON | 기본 OFF (필요시 추가) |
|-------------------|---------|----------------------|
| **팝업스토어** | S, C, P, M | V, A |
| **플래그십 매장** | S, P, V | C, A, M |
| **전시/갤러리** | S, C, A | P, V, M |
| **페스티벌/이벤트** | C, A, M | S, P, V |
| **F&B 매장** | S, P, V | C, A, M |
| **브랜드 체험관** | S, C, A, V | P, M |
| **커스텀** | 사용자 직접 선택 | — |

프리셋은 시작점이며, 사용자가 자유롭게 on/off 조정 가능.

## 공통: EDI (Engagement Depth Index) 스코어링

### 3단계 측정 구조

```
Stage 1: 매력 포인트 특정 (팩트 체크)         → 카테고리 데이터
Stage 2: 기억 선명도 (구체적 의향/멘탈코스트)  → 최대 3점
Stage 3: 행동 전환 예측 (비용 수반 의향)       → 최대 12점 (3문항 × 4점)
─────────────────────────────────────────────
EDI = Stage 2 + Stage 3 = 3 ~ 15점
```

### Stage 1: 매력 포인트 특정

> "오늘 방문에서 가장 인상적이었던 요소를 1~2개 골라주세요"

카테고리는 선택된 경험 구성요소에서 자동 생성 + 기타(직접 입력).
각 카테고리에는 SNS 매핑 키워드를 내부적으로 연결 (응답자에게 비노출).

### Stage 2: 기억 선명도 (멘탈코스트 측정)

**Q2-1. "[선택한 포인트]에서 가장 기억에 남는 구체적인 순간이 있으세요?"**

| 보기 | 내부 해석 | 점수 |
|------|----------|:---:|
| 지금도 선명하게 떠오르는 장면이 있다 | 강한 기억 각인 = 높은 인게이지먼트 | **3** |
| 전체적으로 좋았는데, 딱 하나를 꼽기는 어렵다 | 분산된 호감 = 중간 | **2** |
| 좋았던 것 같은데 구체적으로는 잘 모르겠다 | 표면적 만족 = 낮음 | **1** |

**Q2-2. (①번 선택 시만) 그 순간을 한 줄로 적어주세요**
> _예: "초콜릿 폭포 앞에서 냄새가 진짜 달콤했다"_

이 자유 서술은 **Micro-Psyche 집단 분석** 대상:
- 감각어 분류 (시각/후각/미각/촉각/청각)
- 감정 강도 (강조어: "진짜", "미쳤다", "역대급")
- 1인칭 비율 ("나", "내가" → Authenticity 신호)
- 구체성 수준 (고유명사, 수치 포함 여부)
- 감각 vs 평가 비율 ("달콤했다" vs "좋았다")

### Stage 3: 행동 전환 예측 (비용 수반 의향)

**Q3-1. "오늘 경험을 SNS(인스타, 블로그 등)에 올리셨거나 올릴 계획이 있으세요?"**

| 보기 | 내부 해석 | 점수 |
|------|----------|:---:|
| 이미 올렸다 / 올리는 중이다 | 행동 완료 = 최고 신뢰도 | **4** |
| 오늘 중으로 올릴 예정이다 | 구체적 시간 계획 = 높음 | **3** |
| 올릴 생각은 있지만 확실하지 않다 | 의향은 있으나 비구속 = 중간 | **2** |
| 올릴 계획 없다 | 바이럴 전환 낮음 | **1** |

**Q3-2. "이 경험을 아직 안 가본 지인에게 어떻게 전달하실 것 같으세요?"**

| 보기 | 내부 해석 | 점수 |
|------|----------|:---:|
| 이미 카톡/DM으로 보냈거나 보낼 것이다 | 능동적 전파 = 최고 | **4** |
| 만나면 직접 얘기할 것이다 | 대면 추천 의향 = 높음 | **3** |
| 대화 중 자연스럽게 나오면 언급할 것이다 | 수동적 전파 = 중간 | **2** |
| 굳이 먼저 얘기하지는 않을 것이다 | 전파 의향 없음 | **1** |

**Q3-3. "이 팝업/매장에 다른 사람을 데리고 다시 오실 의향이 있으세요?"**

| 보기 | 내부 해석 | 점수 |
|------|----------|:---:|
| 이미 누구와 다시 올지 생각해뒀다 | 구체적 계획 = 최고 | **4** |
| 기회가 되면 데리고 오고 싶다 | 조건부 의향 = 높음 | **3** |
| 혼자라면 올 수도 있지만 데려오지는 않을 것 | 개인적 만족 = 중간 | **2** |
| 재방문 의향 없다 | 전환 낮음 | **1** |

### EDI 레벨 분류

| EDI 구간 | 레벨 | 해석 | SNS 교차 예측 |
|---------|------|------|-------------|
| **13~15** | L5: Evangelist | 자발적 전파자 — 이미 행동 중 | P-01 Genuine Fan, Sincerity Gate 통과 |
| **10~12** | L4: Advocate | 적극 추천자 — 구체적 계획 있음 | 높은 Auth, 구체적 콘텐츠 생산 |
| **7~9** | L3: Satisfied | 만족하지만 수동적 | 일반 긍정 리뷰, 중간 Auth |
| **4~6** | L2: Casual | 표면적 호감 — "좋았다"지만 움직이지 않음 | P-02 Instagram Consumer 가능성 |
| **3** | L1: Indifferent | 무관심/기대 이하 | SNS 미언급 또는 중립 |

### Viral Readiness Index (VRI)

단일 지수로 바이럴 파급력을 요약:
```
VRI = (L5인원×4 + L4인원×3 + L3인원×2 + L2인원×1 + L1인원×0) / 총 응답자수 / 4 × 100

해석: 0~100 스케일
- 75+: Very Strong — 자연 바이럴 기대
- 50~74: Strong — 적극적 시딩으로 증폭 가능
- 25~49: Moderate — 바이럴보다 직접 트래픽 유도 필요
- ~24: Weak — 경험 자체 개선 우선
```

## 실행 프로세스

### STEP 1: 경험 구성요소 선택

경험 유형에 따른 기본 프리셋을 제안하고 사용자 확인:

```
[팝업스토어 기본 프리셋]
✅ Space (공간)     — 인테리어, 동선, 조명, 컬러
✅ Content (콘텐츠)  — 컨셉 기획, 전시, 브랜드 스토리
✅ Product (상품)    — MD 상품, 한정판, 굿즈
✅ Community (커뮤니티) — 포토존, 동행 경험
☐ Service (서비스)  — 직원 응대 (필요시 추가)
☐ Activity (체험)   — 체험 프로그램 (필요시 추가)

조정이 필요하시면 말씀해주세요.
```

### STEP 2: Stage 1 카테고리 생성

선택된 구성요소별로 세부 카테고리를 자동 생성:

```
구성요소 → 카테고리 매핑 규칙:

Space    → "공간 디자인/인테리어", "조명/분위기"
Content  → "컨셉/테마 기획", "전시/스토리텔링"
Product  → "상품/굿즈", "F&B(음식/음료)" (해당 시)
Service  → "직원 응대/서비스"
Activity → "체험 프로그램/워크숍"
Community → "포토존/촬영 포인트", "동행/소셜 경험"

+ 항상 마지막에: "기타 (직접 입력: ________)"
```

각 카테고리에 **SNS 매핑 키워드**를 내부적으로 연결:
```
공간 디자인/인테리어  →  [인테리어, 예쁜, 공간, 꾸며, 디자인]
F&B(음식/음료)       →  [맛있, 메뉴, 디저트, 음료, 커피]
포토존/촬영 포인트    →  [사진, 포토, 인생샷, 찍, 촬영]
...
```

사용자에게 카테고리 목록 확인 후 조정.

### STEP 3: 전체 설문지 조립

아래 순서로 설문지를 구성합니다:

#### Part A: 기본 정보 (인구통계 + 방문 경로)

```
A1. 성별
    ○ 남성  ○ 여성  ○ 기타/응답하지 않음

A2. 연령대
    ○ 10대  ○ 20대 초반(20~24)  ○ 20대 후반(25~29)
    ○ 30대 초반(30~34)  ○ 30대 후반(35~39)
    ○ 40대  ○ 50대 이상

A3. 방문 경로 (어떻게 알게 되셨나요?)
    ○ SNS (인스타그램/틱톡 등)
    ○ 인터넷 검색/후기
    ○ 지인 추천
    ○ 우연히 방문 (지나가다)
    ○ 인플루언서/유튜버
    ○ 브랜드 공식 채널
    ○ 기타 (____________)

A4. 동행 형태
    ○ 혼자  ○ 친구/연인  ○ 가족  ○ 단체/모임

A5. 유사 경험 빈도 (최근 1년 기준, 해당 유형의 경험 방문 횟수)
    ○ 처음이다
    ○ 연 1~2회
    ○ 연 3~5회
    ○ 연 6회 이상
```

#### Part B: 매력 포인트 + 인게이지먼트 깊이 (EDI 핵심)

```
B1. [Stage 1] 오늘 방문에서 가장 인상적이었던 요소를 1~2개 골라주세요.
    (STEP 2에서 생성된 카테고리 목록)

B2. [Stage 2-1] 선택하신 요소에서 가장 기억에 남는 구체적인 순간이 있으세요?
    ○ 지금도 선명하게 떠오르는 장면이 있다 → B3으로
    ○ 전체적으로 좋았는데, 딱 하나를 꼽기는 어렵다 → B4로
    ○ 좋았던 것 같은데 구체적으로는 잘 모르겠다 → B4로

B3. [Stage 2-2] 그 순간을 한 줄로 적어주세요.
    ________________________________________________

B4. [Stage 3-1] 오늘 경험을 SNS(인스타, 블로그 등)에 올리셨거나 올릴 계획이 있으세요?
    ○ 이미 올렸다 / 올리는 중이다
    ○ 오늘 중으로 올릴 예정이다
    ○ 올릴 생각은 있지만 확실하지 않다
    ○ 올릴 계획 없다

B5. [Stage 3-2] 이 경험을 아직 안 가본 지인에게 어떻게 전달하실 것 같으세요?
    ○ 이미 카톡/DM으로 보냈거나 보낼 것이다
    ○ 만나면 직접 얘기할 것이다
    ○ 대화 중 자연스럽게 나오면 언급할 것이다
    ○ 굳이 먼저 얘기하지는 않을 것이다

B6. [Stage 3-3] 이 {경험 유형}에 다른 사람을 데리고 다시 오실 의향이 있으세요?
    ○ 이미 누구와 다시 올지 생각해뒀다
    ○ 기회가 되면 데리고 오고 싶다
    ○ 혼자라면 올 수도 있지만 데려오지는 않을 것
    ○ 재방문 의향 없다
```

|------|------|------|

#### 4-2. Micro-Psyche 텍스트 분석 (Q2-2 자유 서술)

**개별 응답이 아닌 전체 응답 집계 분석:**

**Content 측면:**
```
감각어 분류:
- 시각: "예쁜", "색감", "조명", "디자인", "핑크" 등
- 후각: "향", "냄새", "달콤한" 등
- 미각: "맛", "달다", "쓰다", "부드러운" 등
- 촉각: "푹신", "차가운", "질감" 등
- 청각: "음악", "소리", "조용" 등
→ 감각 분포 비율 산출

키워드 빈도:
- Unigram Top 15
- Bigram Top 10
- 매력 포인트 카테고리별 대표 키워드
```

**Psyche 측면 (집단 수준):**
```
감정 강도 = 강조어("진짜", "미쳤다", "역대급", "레알", "대박") 포함 응답 비율
구체성 수준 = 고유명사/수치 포함 응답 비율
1인칭 비율 = "나", "내가", "제가" 포함 응답 비율
감각 vs 평가 = 감각 서술 비율 / 평가 서술("좋다", "괜찮다") 비율

해석:
- 감정 강도↑ + 구체성↑ = 진정한 감동 (SNS Tone↑ 예측)
- 1인칭↑ + 감각↑ = 개인화된 경험 (Auth↑ 예측)
- 평가만 높고 감각 낮음 = 표면적 만족 (Auth↓ 예측)
```

Q3-3 재방문:       ①계획 __% | ②조건부 __% | ③혼자만 __% | ④없음 __%
```

**표면 만족도와의 격차 분석 (핵심):**

- 전반적 만족도 6~7점(표면 긍정) 응답자 중 → Stage 3 실제 행동 전환율

→ 이 격차(Gap)가 리포트의 Executive Summary 핵심 수치
```

### STEP 6: EDI 산출 및 레벨 분류

```
각 응답자별:
EDI = memory_clarity 점수 + sns_share 점수 + recommend 점수 + revisit 점수

레벨 분류 후 전체 분포 산출:
- L5 Evangelist (13~15): __%  (_명)
- L4 Advocate (10~12):   __%  (_명)
- L3 Satisfied (7~9):    __%  (_명)
- L2 Casual (4~6):       __%  (_명)
- L1 Indifferent (3):    __%  (_명)

VRI (Viral Readiness Index) 산출
```

## 기존 데이터 호환 모드

Mode 1으로 설계하지 않은 기존 설문 데이터도 분석 가능합니다.

### 매핑 규칙

기존 설문에서 EDI에 가장 가까운 질문을 찾아 매핑:

| EDI 구성 | 기존 설문에서 찾을 질문 | 매핑 방식 |
|---------|----------------------|----------|
| Stage 1: 매력 포인트 | 만족 요소별 점수 문항 | 최고점 항목을 매력 포인트로 변환 |
| Stage 2: 기억 선명도 | **해당 문항 없는 경우 많음** | 한계로 명시, EDI에서 제외 가능 |
| Stage 3: 행동 전환 | 추천 의향, SNS 공유 의향 | 7점 척도 → 4점 재코딩 |

**7점 → 4점 재코딩 규칙:**
```
7점(매우 동의) → 4점 (이미 행동/확실한 계획)
6점(동의)     → 3점 (적극 의향)
5점(약간 동의) → 2점 (불확실한 의향)
1~4점         → 1점 (의향 없음/보통 이하)
```

**한계 명시:** 기존 설문의 추천 의향(7점)은 Stage 3의 행동 전환 질문과 본질적으로 다릅니다. 기존 데이터의 EDI는 **"표면 EDI(Pseudo-EDI)"**로, 새 설문의 EDI와 직접 비교할 수 없으며, "기존 방식의 한계를 보여주는 시뮬레이션"으로 해석합니다.

## 주의사항

- EDI는 **자가 보고(self-report) 기반**이므로 사회적 바람직성 편향이 존재합니다. 이를 최소화하기 위해 Stage 3를 "의향"이 아닌 "이미 한 행동"에 가중치를 두도록 설계했습니다.
- Micro-Psyche 텍스트 분석은 **단문 집계 분석**으로, 개인 수준의 심리 변수 산출은 통계적으로 불가합니다. 반드시 집단 수준에서 해석하세요.
- VRI는 RXR 독자 지표로 학술적 검증 진행 중임을 리포트에 명시하세요.
- 응답자 300명 미만이면 "소규모 샘플, 해석 주의" 경고를 리포트에 표시하세요.
- 기존 데이터 호환 모드에서 산출된 EDI는 "Pseudo-EDI"임을 반드시 명시하세요.
- **비전문가 해설을 절대 생략하지 않는다** — 브랜드 마케터가 데이터 분석 전문가가 아닌 경우를 기본 가정으로 합니다.
- Stage 1 카테고리의 SNS 매핑 키워드는 응답자에게 노출하지 않습니다.
