# RFP Analysis Framework — 정부사업 · 공공입찰 · 제안사업용

> 단순 요약이 아니라, "발주처의 진짜 의도" → "심사위원이 좋아할 포인트" → "수주 가능성과 GO/PASS 판단"까지 도출하는 프레임워크.

---

## 1. 사업 개요 Summary

### 1-1. 사업 한 줄 정의
- 이 사업이 무엇인지 한 문장으로 설명

### 1-2. 사업 목적
- 발주처가 왜 이 사업을 진행하는가
- 해결하려는 문제는 무엇인가
- 정책적/행정적 목적은 무엇인가

### 1-3. 핵심 키워드
지역 활성화, 관광 콘텐츠, 디지털 전환, ESG, 청년 창업, AI/AX, 문화예술, 데이터 기반 운영, 스마트시티, 도시재생, 탄소중립, 사회적 경제 등

### 1-4. 발주처가 가장 중요하게 보는 포인트
RFP 문맥을 기반으로 추론:
- 가장 강조되는 가치
- 반복적으로 등장하는 표현
- 평가에서 중요하게 보는 관점
- 실질적으로 원하는 결과물

예시 판단:
- 단순 행사 운영보다 "지속 가능성" 중시
- 기술보다 "실행 경험" 중시
- 디자인보다 "지역 파급력" 중시

- 정산/행정 이슈 가능성
- 하도급 제한 사항
- 감리/검수 조건

## 3. 평가 기준 및 가점 요소 분석

### 3-1. 평가 항목 요약
| 평가 항목 | 배점 | 핵심 의미 |
|-----------|------|-----------|
| 수행 경험 | XX | 실적 중심 여부 |
| 운영 계획 | XX | 실행력 중시 여부 |
| 전문 인력 | XX | 조직 역량 중시 여부 |
| 가격 | XX | 가격 경쟁 강도 |
| 지역 연계 | XX | 공공성 강조 여부 |
| 기술 역량 | XX | 기술 차별화 여부 |
| 기타 | XX | |

### 3-2. 실질적 당락 포인트 추론
- 실제 점수 차이가 크게 날 영역
- 형식적 항목 vs 실질적 항목
- 발주처 성향상 중요할 가능성이 높은 부분
- 심사위원이 실질적으로 비교할 포인트

### 3-3. 가점 요소 분석
- 추가 점수 요소
- 우대 기업 조건 (중소기업, 사회적기업, 여성기업 등)
- 정책 연계 포인트
- ESG/청년/지역/기술 가점 여부
- 장애인 고용, 고용 우수 등 정책 가점

### 3-4. 경쟁사 유리 요소 추정
- 어떤 유형의 회사가 유리할 가능성이 있는가
- 대기업형 / 운영형 / 디자인형 / 기술형 / 지역기반형 중 어디가 유리한가
- 기존 수행사 재입찰 가능성
- 우리 포지션에서 부족한 점은 무엇인가

## 4. 제안 전략 및 차별화 포인트 도출

### 4-1. 이 사업에서 중요한 성공 키워드
실행력, 지역성, 데이터 기반, 체험형 콘텐츠, 운영 안정성, 공공 협업 경험, 확장 가능성, 시민 참여, 성과 측정 등

### 4-2. 우리가 가져가야 할 차별화 전략
- 어떤 포지셔닝으로 접근해야 하는가
- 경쟁사 대비 어디를 특화해야 하는가
- 어떤 스토리텔링이 먹히는가
- 심사위원이 기억할 만한 포인트는 무엇인가

### 4-3. 추천 제안 컨셉
예시:
- "데이터 기반 지역 활성화 플랫폼"
- "체류형 시민 경험 콘텐츠"
- "AI 기반 운영 효율화 모델"
- "관광형 리테일 미디어 전략"

### 4-4. 추천 핵심 Feature
제안서에 넣을 만한 요소:
- 공간 경험 설계
- AI/데이터 분석 시스템
- 시민 참여 구조
- 운영 시스템 / 매뉴얼
- SNS 확산 구조
- 성과 측정 체계 (KPI)
- 지속 운영 모델
- 사후 관리 방안

### 4-5. 추천 파트너십 구조
- 디자인 파트너
- 기술 파트너
- 지역 기관 / 대학
- 운영사
- 미디어 협업
- 전문가 자문단

## 5. 최종 전략 메모

### 반드시 체크할 것
- 이 사업에서 가장 중요한 한 가지
- 절대 놓치면 안 되는 조건
- 경쟁 우위 확보 포인트
- 현실적 리스크

### 수주 가능성 분석
- **Low / Medium / High**
- 이유 설명 (자격요건 충족도, 경쟁 강도, 차별화 가능성 기반)

### 추천 액션
- **GO / Conditional GO / PASS**
- 제안 참여 추천 여부 및 이유
- Conditional GO인 경우: 어떤 조건이 충족되면 GO인지

## 분석 원칙

1. 단순 요약이 아니라 **"발주처의 진짜 의도"**를 추론할 것
2. **반복 표현과 강조 문장**을 중요하게 해석할 것
3. 공공사업 특성상 **행정 리스크**도 함께 분석할 것
4. **실제 수주 전략 관점**으로 사고할 것
5. 제안서 방향성과 연결될 수 있도록 **전략적으로** 정리할 것
6. 우리 회사의 강점(공간/콘텐츠/운영/데이터/AX)을 연결 가능한 지점이 있다면 반드시 제안할 것
7. **평가 배점 구조**가 전략의 중심축이 되어야 함 — 배점 높은 곳에 자원을 집중
