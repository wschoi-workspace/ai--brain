# RFP Analysis Framework — 정부사업 · 공공입찰 · 제안사업용

> 단순 요약이 아니라, "발주처의 진짜 의도" → "심사위원이 좋아할 포인트" → "수주 가능성과 GO/PASS 판단"까지 도출하는 프레임워크.

---

## 1. 사업 개요 Summary

### 1-1. 사업 한 줄 정의
- 이 사업이 무엇인지 한 문장으로 설명

### 1-2. 사업 목적
- 발주처가 왜 이 사업을 진행하는가
- 해결하려는 문제는 무엇인가
- 정책적/행정적 목적은 무엇인가

### 1-3. 핵심 키워드
지역 활성화, 관광 콘텐츠, 디지털 전환, ESG, 청년 창업, AI/AX, 문화예술, 데이터 기반 운영, 스마트시티, 도시재생, 탄소중립, 사회적 경제 등

### 1-4. 발주처가 가장 중요하게 보는 포인트
RFP 문맥을 기반으로 추론:
- 가장 강조되는 가치
- 반복적으로 등장하는 표현
- 평가에서 중요하게 보는 관점
- 실질적으로 원하는 결과물

예시 판단:
- 단순 행사 운영보다 "지속 가능성" 중시
- 기술보다 "실행 경험" 중시
- 디자인보다 "지역 파급력" 중시

- 기존 수행사 재입찰 가능성
- 우리 포지션에서 부족한 점은 무엇인가

## 4. 제안 전략 및 차별화 포인트 도출

### 4-1. 이 사업에서 중요한 성공 키워드
실행력, 지역성, 데이터 기반, 체험형 콘텐츠, 운영 안정성, 공공 협업 경험, 확장 가능성, 시민 참여, 성과 측정 등

### 4-2. 마케팅 전략 관점 분석

> 이 사업의 결과물을 "상품"으로 볼 때, 마케팅/세일즈 관점에서 핵심 질문 5가지:

| 질문 | 분석 포인트 |
|------|-----------|
| **Who** — 누가 타겟인가? | 1차 수혜자(시민/참여자), 2차 이해관계자(발주처/정치인), 확산 매개자(미디어) |
| **What** — 어떤 가치를 주는가? | 기능적 가치(편의/해결) + 감정적 가치(경험/자부심) + 사회적 가치(소속/인정) |
| **How** — 어떻게 전달하는가? | 오프라인(공간/행사) + 온라인(SNS/미디어) + 커뮤니티(입소문/추천) |
| **Why different** — 왜 우리인가? | 경쟁사 대비 대체 불가능한 1가지 + 보조 차별점 2가지 |
| **So what** — 비즈니스 효과는? | 정량 KPI 3개 + "이 예산으로 이만큼의 가치가 생긴다" ROI 프레이밍 |

### 4-3. 우리가 가져가야 할 차별화 전략
- 어떤 포지셔닝으로 접근해야 하는가
- 경쟁사 대비 어디를 특화해야 하는가
- 어떤 스토리텔링이 먹히는가
- 심사위원이 기억할 만한 포인트는 무엇인가
- **Rule of Three**: 핵심 전략 3개로 압축 (배점 대응 + 차별화 + 확장성)

### 4-4. 추천 제안 컨셉
예시:
- "데이터 기반 지역 활성화 플랫폼"
- "체류형 시민 경험 콘텐츠"
- "AI 기반 운영 효율화 모델"
- "관광형 리테일 미디어 전략"

### 4-5. 추천 핵심 Feature
제안서에 넣을 만한 요소:
- 공간 경험 설계
- AI/데이터 분석 시스템
- 시민 참여 구조
- 운영 시스템 / 매뉴얼
- SNS 확산 구조
- 성과 측정 체계 (KPI)
- 지속 운영 모델
- 사후 관리 방안

### 4-6. 기대효과 초안 (2단계 전략 제언의 입력이 되는 사전 분석)

> 1단계에서 기대효과의 방향성을 먼저 잡아두고, 2단계에서 구조화한다.

- **직접 효과**: 이 사업으로 즉시 나타나는 결과 (참여자 수, 매출, 콘텐츠 수)
- **행동 변화**: 타겟의 인식/행동이 어떻게 바뀌는가 (인지도, 전환율, 만족도)
- **파급 효과**: 사업 범위를 넘어서는 간접 효과 (경제적/사회적/정책적)
- **지속가능성**: 사업 종료 후에도 효과가 남는 구조
- **핵심 숫자 3개**: 제안서에서 반복할 대표 수치 (앵커링 효과)

### 4-7. 추천 파트너십 구조
- 디자인 파트너
- 기술 파트너
- 지역 기관 / 대학
- 운영사
- 미디어 협업
- 전문가 자문단

## 5. 제안서 양식 심층 분석

> 정부사업 RFP에는 제안서 양식(목차, 작성 지침, 페이지 제한 등)이 함께 제공된다.
> 양식 자체가 **"평가위원이 무엇을 기준으로 점수를 매기는가"**의 직접적 단서다.

### 5-1. 양식 구조 분석
| 섹션 | 요구 항목 | 페이지/분량 제한 | 대응 평가 항목 | 추정 배점 비중 |
|------|-----------|-----------------|---------------|---------------|

- 양식의 각 섹션이 평가표의 어떤 항목과 매핑되는지 연결
- 분량 제한이 있는 경우, 제한 자체가 우선순위 신호 (많이 쓰라는 곳 = 중요)
- 자유 기술 영역 vs 정형화된 영역 구분

### 5-2. 양식에서 읽어내는 발주처의 숨은 요구
- **세부 항목이 많은 섹션**: 발주처가 구체적 답을 원한다 → 추상적 서술 금지, 수치/사례 중심
- **자유 서술 섹션**: 차별화 기회 → 여기서 점수 차이가 난다
- **별도 서식(표, 양식) 요구**: 정량적 비교를 하겠다는 의미 → 빈칸 없이, 경쟁사 대비 우위 수치 필수
- **참고자료/증빙 첨부 요구**: 실적 증명이 당락을 좌우 → 증빙 품질에 투자
- **양식에 없지만 RFP에는 있는 요구사항**: 가산점 또는 정성 평가 영역 → 별도 섹션으로 녹여야 함
- **양식에 있지만 RFP에서 강조 안 한 항목**: 형식 요건 → 기본만 채우고 지면 낭비 금지

### 5-3. 섹션별 작성 전략 가이드
각 양식 섹션에 대해:
| 섹션 | 핵심 메시지 | 작성 방향 | 분량 배분 | 주의사항 |
|------|-----------|-----------|-----------|---------|

- **배점 높은 평가 항목과 연결된 섹션**: 지면의 60% 이상 투자, 구체적 실행 계획 + 차별화 요소
- **실적/경험 섹션**: 유사 용역 실적을 평가 기준 키워드로 재서술, 단순 나열 금지
- **인력 구성 섹션**: 투입 인력의 역할-과업 매핑, 자격 요건 충족 증빙
- **일정/추진 체계 섹션**: 과업별 WBS + 마일스톤, 리스크 대응 포함
- **예산 섹션**: 공종별 산출 근거 명확히, 관급 vs 자체 구분

### 5-4. 양식 vs RFP vs 평가표 교차 분석

```
[양식 섹션] ←→ [RFP 요구사항] ←→ [평가 항목/배점]
```

3자 교차 매핑 테이블:
| 양식 섹션 | RFP 요구사항 | 평가 항목 | 배점 | Gap/불일치 |
|-----------|-------------|-----------|------|-----------|

- **Gap 발견 시**: RFP에서 강조하지만 양식에 전용 섹션이 없는 항목 → 어디에 녹일지 전략 수립
- **불일치 발견 시**: 양식과 RFP 간 용어/범위 차이 → 어느 쪽 기준으로 작성할지 판단

## 6. 제안서 초안 검토 (제안서 제출 후 활용)

> 작성된 제안서 초안을 제공받으면, RFP + 양식 + 평가표 대비 크로스체크한다.

### 6-1. 요구사항 충족도 체크
| RFP 요구사항 | 양식 섹션 | 제안서 반영 여부 | 반영 페이지 | 품질 평가 | 개선 방향 |
|-------------|-----------|----------------|------------|----------|----------|

- ✅ 충족 / ⚠️ 부분 충족 / ❌ 미충족 / 🔍 확인 필요
- 미충족 항목은 즉시 보완 가이드 제공

### 6-2. 배점 대비 지면 균형 분석
| 평가 항목 | 배점 비중(%) | 제안서 지면 비중(%) | 균형 판정 | 조치 |
|-----------|-------------|-------------------|----------|------|

- 배점 비중 대비 지면이 적으면 **Under-invested** → 보강
- 배점 비중 대비 지면이 과하면 **Over-invested** → 압축하여 다른 곳에 재배분

### 6-3. 평가위원 관점 시뮬레이션
평가위원이 제안서를 읽을 때의 관점으로 검토:
- **첫인상 (표지~목차~요약)**: 신뢰감을 주는가, 핵심 메시지가 즉시 전달되는가
- **차별성 인식**: 다른 제안서와 구분되는 포인트가 명확한가
- **실행 가능성**: "이 팀이 진짜 할 수 있겠다"는 확신이 드는가
- **증빙 충분성**: 주장에 대한 근거(실적, 데이터, 사례)가 있는가
- **읽기 편의성**: 구조가 명확한가, 핵심이 눈에 들어오는가
- **감점 요소**: 오탈자, 분량 초과/미달, 양식 미준수, 필수 서류 누락

### 6-4. 강화/보완 우선순위
| 우선순위 | 섹션 | 현재 수준 | 목표 수준 | 구체적 보완 방향 |
|---------|------|----------|----------|----------------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

## 7. 최종 전략 메모

### 반드시 체크할 것
- 이 사업에서 가장 중요한 한 가지
- 절대 놓치면 안 되는 조건
- 경쟁 우위 확보 포인트
- 현실적 리스크

### 수주 가능성 분석
- **Low / Medium / High**
- 이유 설명 (자격요건 충족도, 경쟁 강도, 차별화 가능성 기반)

### 추천 액션
- **GO / Conditional GO / PASS**
- 제안 참여 추천 여부 및 이유
- Conditional GO인 경우: 어떤 조건이 충족되면 GO인지

## 분석 원칙

1. 단순 요약이 아니라 **"발주처의 진짜 의도"**를 추론할 것
2. **반복 표현과 강조 문장**을 중요하게 해석할 것
3. 공공사업 특성상 **행정 리스크**도 함께 분석할 것
4. **실제 수주 전략 관점**으로 사고할 것
5. 제안서 방향성과 연결될 수 있도록 **전략적으로** 정리할 것
6. 우리 회사의 강점(공간/콘텐츠/운영/데이터/AX)을 연결 가능한 지점이 있다면 반드시 제안할 것
7. **평가 배점 구조**가 전략의 중심축이 되어야 함 — 배점 높은 곳에 자원을 집중
8. **제안서 양식은 평가 의도의 거울** — 양식 구조에서 발주처가 진짜 보고 싶은 것을 읽어낼 것
9. **양식 vs RFP vs 평가표 3자 교차 분석** 필수 — Gap과 불일치를 발견해야 경쟁사가 놓치는 포인트를 잡는다
