# RFP Analysis Framework

> 단순 요약이 아니라, "왜 이 프로젝트가 생겼는가" → "진짜 문제는 무엇인가" → "어떻게 이길 것인가"를 구조적으로 도출하는 프레임워크.

---

## 1. PROJECT / CLIENT CONTEXT ANALYSIS (클라이언트 상황 분석)

### 1-1. 기업 기본 분석
- 기업 소개
- 현재 핵심 사업
- 최근 사업 방향성
- 최근 투자/신규사업/브랜드 전략
- 주요 경쟁사
- 현재 시장 내 포지셔닝

### 1-2. 최근 이슈 분석 (6개월~2년)

**조사 대상**: 뉴스, 인터뷰, 보도자료, SNS, 블로그, 커뮤니티, 유튜브

**분석 포인트**:
- 최근 이슈가 있었는가
- 브랜드 방향성이 바뀌고 있는가
- 새로운 고객층 확보가 필요한가
- 매출/브랜드 이미지/시장 경쟁 관련 압박이 있는가
- 기존 방식의 한계가 있는가
- 내부적으로 강조하는 키워드는 무엇인가

### 1-3. 프로젝트 발생 배경 추론
- 이 프로젝트를 진행하는 **진짜 이유**
- 클라이언트 내부에서 중요하게 생각하는 **KPI**
- 표면적 요구사항 뒤에 숨겨진 **목적**
- 클라이언트가 두려워하는 **리스크**
- 클라이언트가 기대하는 **결과**

## 2. RFP REQUIREMENT SUMMARY (요구사항 요약)

### 2-1. 프로젝트 기본 정보
| 항목 | 내용 |
|------|------|
| 프로젝트명 | |
| 발주사 | |
| 프로젝트 기간 | |
| 예산 | |
| 제출 마감일 | |
| 발표/PT 일정 | |
| 운영 기간 | |
| 주요 일정 | |

### 2-2. 핵심 요구사항 정리
- 클라이언트가 **명시적으로** 요구한 사항
- **암묵적으로** 중요해 보이는 요구사항
- **반복적으로** 언급되는 키워드
- **우선순위**가 높아 보이는 항목

### 2-3. 핵심 문제 정의
- 클라이언트는 지금 무엇이 **부족**한가
- 무엇을 **해결**하고 싶은가
- 무엇이 **실패**하고 있는가
- 무엇을 **바꾸고** 싶은가
- 무엇을 시장에 **보여주고** 싶은가

### 2-4. 최종 산출물 정의
프로젝트 종료 시 반드시 나와야 하는 결과물:
공간, 캠페인, 콘텐츠, 운영 시스템, 브랜딩, 영상, SNS 확산, 고객 경험 설계, 데이터 리포트, PR 이슈화, 매출 성과, 방문객 수치, 바이럴 등

---

## 3. TARGET / CUSTOMER ANALYSIS (타겟 분석)

### 3-1. 핵심 타겟 정의
성별, 연령, 라이프스타일, 소비 성향, 관심사, 방문 동기, SNS 사용 패턴

### 3-2. 고객 경험 관점 분석
- 이 고객은 왜 **반응**해야 하는가
- 어떤 경험에서 **감동**하는가
- 무엇을 **공유**하고 싶어 하는가
- 어떤 콘텐츠가 **바이럴** 되는가
- 무엇이 **"가볼 이유"**가 되는가

### 3-3. 현재 고객 경험의 문제점
- 기존 브랜드 경험의 한계
- 기존 오프라인 경험의 문제
- 경쟁사 대비 부족한 점
- 고객과의 관계성 부족 요소

---

## 4. STRATEGIC DIFFERENTIATION ANALYSIS (전략적 차별화)

### 4-1. 디자인 관점
시각적 차별점, 공간 경험, 콘텐츠 연출, 브랜드 톤앤매너, SNS 확산 가능성, 사진 찍고 싶은 요소, 오프라인 경험의 밀도

### 4-2. 고객 경험(CX) 관점
고객 동선, 체류 경험, 인터랙션, 참여 유도, 공유 유도, 재방문 구조, 감정 설계, 커뮤니티화 가능성

### 4-3. 비즈니스 관점
실제 매출 기여, 브랜드 자산 축적, 데이터 확보, 장기적 확장성, 운영 효율성, ROI 관점 설득력, 후속 프로젝트 연결 가능성

### 4-4. 클라이언트 시너지 관점
클라이언트 브랜드 연결성, 기존 자산 활용, 기존 사업 시너지, 내부 조직 협업 용이성, 향후 확장성, 장기 파트너십 가능성

---

## 5. WINNING POINT SUMMARY (제안 승리 포인트)

### 5-1. 클라이언트 핵심 니즈 요약
- 가장 중요한 문제
- 가장 원하는 결과
- 가장 민감한 리스크

### 5-2. 전략 방향
- 어떤 방향성이 맞는가
- 어떤 톤앤매너가 맞는가
- 어떤 메시지가 중요한가

### 5-3. 반드시 강조해야 할 차별점
디자인, 경험, 운영, 데이터, 브랜딩, 실행력, 네트워크, 미디어화 가능성

---

## 6. 핵심 원칙

이 분석의 핵심은 **단순 요약이 아니다.**

도출해야 하는 것:
1. **"왜 이 프로젝트가 생겼는가"** — 배경과 맥락
2. **"클라이언트의 진짜 고민은 무엇인가"** — 표면 아래의 문제
3. **"경쟁 PT에서 무엇으로 차별화해야 하는가"** — 승리 전략
4. **"왜 우리가 가장 적합한 팀인가"** — 설득 논리
