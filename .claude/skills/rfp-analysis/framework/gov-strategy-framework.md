# RFP 2단계: 전략 제언 Framework — 정부사업/공공입찰

> 1단계 RFP 분석(Phase A 진단) 결과를 입력으로, **프로젝트 방향성 3안을 설계하고 비즈니스 효과를 구조화**하는 전략 제언 프레임워크.
> v3.0 · 2026-05 · 전제훈 8단계 프레임 + 마케팅/세일즈 전략 통합 진화판
>
> **핵심 원칙:**
> - "~할 것으로 기대됨"이 아니라, **"이 메커니즘으로 이 가치가 생성된다"**를 증명
> - 핵심 컨셉은 **최소 3개 안**으로 — 의사결정자에게 선택지를 준다
> - 진단 없는 설계, 설계 없는 검증은 흔들린다 — **순서가 곧 의존 관계**

| **검증·전달** | 10. 자체평가 | 배점표로 보수적 채점 | 점수표 + 보완 우선순위 |
| **검증·전달** | 11. 의사결정 리뷰 | 코어만 추려 선택 가능하게 | 3안 비교 + 결정 포인트 |
| **검증·전달** | 12. 제안서 반영 가이드 | 어디에 어떻게 녹일지 | 배치·지면·비주얼 가이드 |

## 1단계 → 2단계 입력 매핑

| 1단계 섹션 | 2단계에서의 역할 |
|-----------|----------------|
| 사업 개요 (목적, 키워드) | JTBD 정의의 기초, Pain Point 발굴 출발점 |
| 발주 의도 분석 (반복 표현, 진짜 의도) | 관통 메시지의 소재, Why Us의 표적 |
| 요구조건 (예산, 기간) | 실현가능성 필터, 예산 설계의 분모, ROI 산출 기준 |
| 평가 기준 (배점표, 당락 포인트) | **전략 제언의 우선순위 결정 축**, 자체평가 기준 |
| 제안 전략 (컨셉, Feature, 차별화) | 2단계의 출발점 — 프레임워크로 "증명" |
| 제안서 양식 분석 (Gap/불일치) | 기대효과를 어느 섹션에 녹일지 가이드 |
| 최종 전략 메모 (GO/PASS) | **GO일 때만 2단계 진행** |

## Step 1: Pain Point 발굴 〔진단 심화〕

> 발주처가 과거에 같은/유사 사업을 했다면 그 성과·한계를 딥리서치한다.
> **작년의 약점이 올해 제안의 'Why us' 표적이 된다.**

### 1-1. 과거 사업 딥리서치

- **받는 것:** 1단계의 발주처 정보 + WebSearch 결과
- **하는 일:** 과거 동종 사업을 딥리서치해 '반복되는 구조적 한계'를 포착
- **넘기는 것:** '심사위원이 가려운 지점' 정의 = Why Us의 표적

**검색 쿼리:**
```
"[발주처명] [사업명] 성과"
"[발주처명] [사업명] 결과보고"
"[발주처명] [유사사업] 평가"
"[발주처명] 과거 수행사"
```

### 1-2. 구조적 한계 포착

| 과거 사업 | 성과 | 한계 | 반복 패턴 |
|----------|------|------|----------|
| | | | |

**포착 기준:**
- 성과 미공개 · 일회성 · 측정 불가 — 매년 반복되는 빈틈
- 미디어 노출은 있으나 실질 전환(매출/바이어/재참여) 미증명
- 운영은 무사히 했으나 "그래서 뭐가 남았는가"에 답 부재
- 수치는 교차검증 — 딥리서치 숫자를 그대로 믿지 않는다. 미검증값은 추정치로 표기

### 1-3. Pain 정의

> **건너뛰면:** 우리 자랑만 늘어놓는 제안서가 된다. 심사위원의 문제의식과 어긋나, '잘 만들었지만 와닿지 않는' 제안이 된다.

**출력 형식:**
```
🎯 심사위원이 가려운 지점 (Pain Point 정의):
1. [Pain 1]: [구체적 설명] — 근거: [🔍/💡]
2. [Pain 2]: [구체적 설명] — 근거: [🔍/💡]
3. [Pain 3]: [구체적 설명] — 근거: [🔍/💡]
```

## Step 2: 관통 메시지 〔진단 심화〕

> Pain Point를 정반대로 뒤집은 한 문장.
> 모든 슬라이드·항목이 이 메시지로 수렴해야 한다.

### 2-1. Pain → 메시지 전환

- **받는 것:** Step 1의 Pain Point 정의
- **하는 일:** Pain을 뒤집어 "작년에 없던 그것을 올해 우리는 한다"를 짧고 선명한 한 문장으로 압축
- **넘기는 것:** 제안서 전체가 수렴할 관통 메시지 한 줄

### 2-2. 관통 메시지 설계 원칙

1. **Pain의 정반대를 한 문장으로** — 진단이 곧 약속으로 바뀐다
2. **짧고 선명하게** — 심사위원이 발표가 끝나도 기억할 만큼
3. **반복해 각인** — 표지·핵심·결정 지점에 같은 메시지를 다시 놓는다

**출력 형식:**
```
📌 관통 메시지:
"[한 줄 메시지]"

Pain 전환 로직:
- Pain: [작년의 한계] → Message: [올해의 약속]
```

> **건너뛰면:** 슬라이드가 따로 논다. 개별 내용은 좋아도 '무엇을 말하는 제안인지' 한 줄로 안 남아 기억되지 않는다.

## Step 3: Why Us 〔진단 심화〕

> 관통 메시지를 우리가 실제로 해낼 수 있는 이유로 분해한다.
> Pain의 각 갈래마다 그것을 푸는 우리 강점을 1:1로 짝짓고, 증거로 뒷받침한다.

### 3-1. Pain ↔ 강점 1:1 매칭

| Pain (심사위원이 가려운 곳) | Why Us (우리가 긁어주는 방법) | 증거 |
|--------------------------|---------------------------|------|
| Pain 1: | 강점 1: | 🔍 실적/특허/데이터 |
| Pain 2: | 강점 2: | 🔍 실적/특허/데이터 |
| Pain 3: | 강점 3: | 🔍 실적/특허/데이터 |

### 3-2. 경쟁사 대비 대체 불가능성

- **결합 차별화:** 단독형 사업자(광고대행·MICE·패션에이전시)가 못 하는 조합을 보여준다
- **증거 기반:** 주장이 아니라 근거로 말한다 — 특허, 실적 데이터, 유사사업 수행 증빙
- **컨소시엄이면:** 각 사의 강점이 Pain의 각 갈래에 정확히 매칭되는 구조

> **건너뛰면:** 메시지는 그럴듯한데 '왜 하필 우리냐'에 답이 없다. 경쟁사도 똑같이 할 수 있는 제안으로 보인다.

## Step 4: 사업 가치 재정의 (JTBD) 〔전략 설계〕

> 정부사업 제안서의 가장 흔한 실패는 "발주처가 요구한 것을 그대로 반복"하는 것이다.
> JTBD 재정의를 통해 "발주처가 진짜 고용하려는 것"을 드러내면, 제안서 전체의 관점이 달라진다.

### 4-1. 표면적 사업 목적
- RFP에 명시된 사업 목적 그대로 인용

### 4-2. JTBD 3차원 재정의

| 차원 | 정의 | 정부사업 맥락 |
|------|------|-------------|
| **Functional Job** | 이 사업이 실질적으로 해결해야 할 과제 | 정책 목표 달성, 예산 집행률, 성과지표 충족 |
| **Emotional Job** | 발주처(담당자/기관장)가 느끼고 싶은 것 | "성과 보고서에 넣을 수 있는 사업", "감사에 걸리지 않는 사업" |
| **Social Job** | 이 사업 결과가 대외적으로 보여줘야 하는 것 | "언론 보도될 만한 사업", "타 지자체가 벤치마크할 만한 사업" |

### 4-3. 재정의된 가치 제안
- 1~2문장으로 압축
- 형식: "이 사업은 [Functional]을 통해 [Emotional]을 실현하고, [Social]을 만들어내는 사업이다"
- 이것이 Step 5~12 전체의 방향타

💡 **So What**: 이 재정의가 제안서 전체의 톤을 결정한다.
- ❌ "운영을 잘 하겠습니다" → ✅ "브랜드의 글로벌 가치를 증명하겠습니다"
- ❌ "홍보를 많이 하겠습니다" → ✅ "팔릴 수밖에 없는 구조를 설계하겠습니다"

## Step 5: 마케팅 전략 설계 (Go-to-Market) 〔전략 설계〕

> 사업 결과물을 "상품"으로 보고, 마케팅/세일즈 관점에서 가치 창출 구조를 설계한다.
> 핵심 질문: **"이 사업의 결과물은 누구에게, 왜 가치 있고, 어떻게 전달되고, 어떻게 확산되는가?"**

### 5-0. 마케팅 전략 캔버스 (1-Page 전략 요약)

| 항목 | 질문 | 이 사업에서의 답 |
|------|------|----------------|
| **타겟** | 누가 이 사업의 가치를 받는가? (1차/2차/3차 수혜자) | |
| **가치 제안** | 왜 타겟이 반응해야 하는가? (핵심 가치 1문장) | |
| **채널** | 어떻게 타겟에게 도달하는가? (터치포인트) | |
| **전환 메커니즘** | 인지 → 참여 → 전환을 만드는 구조는? | |
| **확산 구조** | 참여자가 자발적으로 퍼뜨리는 이유는? | |
| **측정** | 성공을 무엇으로 증명하는가? (핵심 KPI 3개) | |
| **차별화** | 경쟁 제안 대비 "이것만은 우리만 가능"한 것은? | |

### 5-1. STP 분석 (Segmentation → Targeting → Positioning)

**Segmentation** — 수혜자/이해관계자 세분화:
| 세그먼트 | 정의 | 니즈 | 사업과의 접점 |
|---------|------|------|-------------|
| 1차 타겟 (직접 수혜자) | | | |
| 2차 타겟 (간접 수혜자) | | | |
| 핵심 이해관계자 | | | |
| 확산 매개자 (미디어/인플루언서) | | | |

**Targeting** — 자원 집중할 핵심 타겟:
- 배점과 연결: 어떤 타겟에 집중해야 평가 점수가 올라가는가?
- 80/20 법칙: 전체 효과의 80%를 만드는 20% 타겟은?

**Positioning** — 타겟의 마음속에 어떤 위치를 차지할 것인가:
- 포지셔닝 스테이트먼트: "우리는 [타겟]에게 [카테고리]에서 [차별화 포인트]를 제공하는 [정체성]이다"

### 5-2. OMO 통합 마케팅 전략 (Online Merges with Offline)

> 오프라인은 '디지털 콘텐츠의 원천(Original Source)'이자 '팬덤을 응축시키는 미디어'로 기능한다.

**A. 채널 역할 정의**

| 채널 | 역할 | 핵심 기능 |
|------|------|----------|
| **온라인** | 효율 + 확산 | 도달, 전환, 데이터 수집, 리타겟팅 |
| **오프라인** | 경험 + 신뢰 | 감각 공유, 관계 형성, 콘텐츠 원천, 팬덤 응축 |
| **통합(OMO)** | 순환 성장 | 오프라인 경험 → 온라인 확산 → 재방문/구매 전환 |

**B. 경험의 자산화 (Experience → Content → Amplification)**
- **콘텐츠화**: 현장 반응, 인터뷰, 숏폼 영상 → SNS 유통
- **바이럴 엔진**: 공간을 "인스타그래머블"하게 설계 → 고객이 스스로 콘텐츠 생산·확산
- **MaaS (Media as a Space)**: 매장 자체가 미디어 채널 → 광고비 없이 도달 생성

**C. 데이터 통합 (옴니채널 추적)**

| 측정 영역 | 오프라인 지표 | 온라인 연결 지표 | 통합 KPI |
|----------|-------------|----------------|---------|
| 인지 | 방문객 수, 5초 시선 멈춤 | 검색량 변화, SNS 멘션 | 브랜드 인지 전환율 |
| 참여 | 체류시간, 인터랙션 | UGC 생성, 해시태그 | 참여 깊이 지수 |
| 전환 | 현장 구매, 명함 교환 | 온라인몰 구매, 회원가입 | 통합 전환율 |
| 옹호 | 재방문, 추천 | 리뷰, 공유, 바이럴 | NPS + EMV |

**D. OMO 적용 체크리스트**
- [ ] 이 사업에 오프라인 접점이 있는가? → OMO 전략 적용 가능
- [ ] 오프라인 경험을 콘텐츠로 변환하는 계획이 있는가?
- [ ] 온·오프라인 데이터를 연결하여 통합 ROI를 증명할 수 있는가?
- [ ] 오프라인 공간이 "미디어"로 기능하는 설계가 포함되었는가?
- [ ] 일시적 이벤트가 끝난 후에도 디지털 자산이 남는 구조인가?

### 5-3. 가치 전달 경로 (Logic Model + CDJ + OMO)

```
[투입]          예산/인력/인프라
    ↓
[활동]          프로그램/콘텐츠/시스템 구축
    ↓
[산출 Output]   직접 결과물 (수, 양, 범위)
    ↓
[성과 Outcome]  타겟 행동 변화 (인지→참여→전환)
    ↓
[영향 Impact]   장기 사회적/경제적 변화
```

각 단계에 CDJ(Consumer Decision Journey) 매핑:
| 단계 | CDJ 관점 | 핵심 질문 |
|------|---------|----------|
| 산출 | 인지(Awareness) | 타겟이 이 사업의 결과물을 어떻게 알게 되는가? |
| 성과 | 고려→참여(Consider→Engage) | 왜 참여하는가? 무엇이 행동을 바꾸는가? |
| 영향 | 옹호(Advocate) | 참여자가 자발적으로 확산하는 구조는? |

### 5-4. 확산 구조 (Growth Flywheel)

```
[핵심 경험] → [참여자 만족] → [자발적 공유/추천] → [인지도 확대] → [신규 참여자 유입] → [핵심 경험]
```

Cialdini 원칙 매핑:
| 플라이휠 단계 | Cialdini 원칙 | 설계 포인트 |
|-------------|-------------|-----------|
| 핵심 경험 → 만족 | 상호성(Reciprocity) | 무료/저비용 체험이 깊은 참여로 전환 |
| 만족 → 공유 | 사회적 증거(Social Proof) | 참여자 수/후기가 신규 유입을 견인 |
| 공유 → 인지 확대 | 권위(Authority) | 언론/전문가/인플루언서 노출 설계 |
| 인지 → 신규 유입 | 희소성(Scarcity) | 한정 프로그램, 시즌 한정, 선착순 |
| 재참여 | 통합/소속감(Unity) | 커뮤니티 형성, 멤버십, 재방문 구조 |

### 5-5. 사업 포트폴리오 (Ansoff/BCG 응용)

**Ansoff Matrix** — 사업 결과물의 확장 경로:
| | 기존 서비스 | 신규 서비스 |
|---|-----------|-----------|
| **기존 타겟** | 시장 침투 (현재 사업) | 서비스 개발 (기대효과 확장) |
| **신규 타겟** | 시장 개발 (대상 확대) | 다각화 (장기 비전) |

**BCG Matrix** — 사업 구성요소별 포지셔닝:
- **Star**: 성장성+점유율 높은 핵심 프로그램 → 집중 투자
- **Cash Cow**: 안정적 성과를 내는 기본 운영 → 효율 유지
- **Question Mark**: 실험적/혁신적 요소 → 선별 투자
- **Dog**: 성과 미미한 관행적 항목 → 축소/제거

## Step 6: 컨셉 3안 (A/B/C) 〔전략 설계 · 핵심 원칙〕

> **아이데이션 단계에서 하나로 좁히지 않는다.**
> 핵심 컨셉은 A·B·C **최소 3개 베리에이션**을 만들어, 의사결정자가 비교하고 고르게 한다.
> 3개 안은 서로 다른 축에서 출발해야 비교 가치가 있다.

### 6-1. 3안 설계 원칙

1. **서로 다른 각도로** — 3안이 명확히 대비되는 축을 가져야 한다
2. **RFP 골격은 공통 전제** — 발주처가 고정한 골격(예산·기간·장소)은 3안 모두 지킨다
3. **'A vs B vs C' 한 줄 비교** — 각 안에 무엇이 더 강한지 명시. 하나가 막히면 즉시 다른 안
4. **하이브리드 허용** — A의 일부 + B의 일부 조합도 가능하도록 모듈형 설계

### 6-2. 3안 분류 축 (권장)

| 안 | 축 | 성격 | 적합 상황 |
|---|---|------|----------|
| **A안: 안정형** | 검증된 방법론 · 실적 기반 | 리스크 최소화, 실행력 증명 | 기존 수행사가 강하고 정면 대결할 때 |
| **B안: 차별형** | 독자적 관점 · 방법론 차별화 | 경쟁사가 못 따라오는 포지셔닝 | 우리만의 IP/기술/네트워크가 있을 때 |
| **C안: 혁신형** | 패러다임 전환 · 확장 비전 | 스케일과 비전으로 승부 | 발주처가 변화를 원하고 배점에 혁신성이 있을 때 |

> ⚠️ 위 분류는 권장이며, 사업 특성에 따라 다른 축(예: 전통↔미래, 누적↔깊이, 로컬↔글로벌)으로 변경 가능.

### 6-3. 컨셉 설계 템플릿

각 안에 대해:

```markdown
### [A/B/C]안: [컨셉명]

**한 줄 요약:** [이 안의 핵심을 한 문장으로]
**축:** [안정/차별/혁신] 또는 [커스텀 축]
**관통 메시지 연결:** [Step 2의 관통 메시지를 이 안이 어떻게 구현하는가]

**핵심 전략 (Rule of Three):**
1. [전략 1]: [배점 최고 영역 대응]
2. [전략 2]: [차별화 영역]
3. [전략 3]: [지속가능성/확장성]

**강점:** 이 안이 특히 강한 영역
**리스크:** 이 안의 약점 또는 전제 조건
**적합 조건:** 어떤 상황에서 이 안이 최적인가
```

### 6-4. 3안 비교 매트릭스

| 비교 축 | A안: [안정형] | B안: [차별형] | C안: [혁신형] |
|---------|-------------|-------------|-------------|
| **핵심 키워드** | | | |
| **배점 30점(수행능력) 대응** | | | |
| **배점 20점(사업이해도) 대응** | | | |
| **차별화 강도** | ★★☆ | ★★★ | ★★★ |
| **실행 리스크** | ★☆☆ | ★★☆ | ★★★ |
| **예상 점수 (보수적)** | /90 | /90 | /90 |
| **필요 전제 조건** | | | |
| **추천 상황** | | | |

### 6-5. 하이브리드 옵션

3안 중 강점을 조합한 하이브리드도 제안:
```
추천 조합: A안의 [요소] + B안의 [요소] + C안의 [요소]
이유: [왜 이 조합이 유효한가]
```

> **건너뛰면:** 컨셉 하나에 갇힌다. 의사결정자에게 선택지가 없고, 그 안이 막히면 아이데이션을 원점에서 다시 시작한다.

## Step 7: 예산 설계 〔전략 설계〕

> 투찰가를 정하고 그 안에서 마진·집행을 배분한다.
> 컨소시엄이면 마진은 출자비율로 갈린다 — '우리가 실제로 챙기는 몫'을 끝까지 따진다.

### 7-1. 투찰가 — 추정가 대비 비율로

- 추정가격 기준으로 투찰률(%) 결정
- 가격평가 배점이 작으면(예: 10점) 무리한 저가투찰은 손해
- 기술 배점이 압도적이면 "승부는 기술에서, 가격은 손해 안 보는 선"

| 투찰 유형 | 투찰률 | 적합 상황 |
|----------|--------|----------|
| 안정·수익형 | 96~98% | 기술 배점 압도적, 경쟁 약할 때 |
| 균형형 | 92~95% | 가격 배점 보통, 경쟁 보통 |
| 공격·저가형 | 88~91% | 가격 배점 크고, 경쟁 치열할 때 |

### 7-2. 마진 2층 구조

- **제출용:** 일반관리비·이윤을 규정 통상선(보통 합산 ~13%) 표기
- **내부 실질:** 직접비 항목의 보수적 산정 버퍼 + 장부상 이윤
- 2층 = 제출용 표기(예: 13%) + 직접비 버퍼(예: 7%) = 내부 실질 목표(예: 20%)

### 7-3. 컨소시엄 마진 배분

- 공동수급(공동이행)은 손익을 출자비율대로 분배
- 대표사 실수령 = 전체 마진% × 대표사 출자비율
- 출자비율 = 사업 기여도(과업·사업비 수행 비중) 반영

### 7-4. 카테고리 배분

- 인건비 / 권역·주제별 사업비 / 공통 운영 / 일반관리비·이윤
- 평가 배점·과업과 정합하게
- 일회성 큰 고정비는 별도 라인으로 떼어 비교 왜곡 방지

### 7-5. 누락 점검

- **해외사업:** 항공·숙박·체재·비자·보험 별도 항목
- **발주처·협력사 부담:** 우리 예산에서 제외 — 과업 범위의 우리 몫만 계상
- **인력 정합:** person-month 환산 → 인건비 카테고리와 일치 확인
- **참여율 규정:** 핵심인력 월 참여율(보통 30%↑), 전 용역 합산 100% 미초과

### 7-6. 예산 설계 산출물

```
📊 예산 설계 요약:
- 추정가: [금액] × [투찰률]% = 투찰 [금액]
- 마진 2층: 제출용 [X]% / 내부 실질 [Y]%
- 컨소시엄: [비율] → 대표사 실수령 ≈[Z]%
- 집행 배분: [카테고리별 비율]
```

> **건너뛰면:** 컨소시엄 마진을 절반밖에 못 챙기거나, 해외출장비를 누락하거나, 발주처 몫을 우리 예산에 잘못 계상한다.

## Step 8: 비즈니스 효과 구조화 (4-Layer Impact + ROI) 〔효과 구조화〕

> **이것이 2단계의 핵심 차별화 포인트.**
> 경쟁 제안서의 "~할 것으로 기대됨" vs 우리의 **정량 근거 기반 비즈니스 임팩트**.
> 핵심 원칙: **"투자 대비 몇 배의 가치를 만드는가"를 숫자로 보여준다.**

### Layer 1: 직접 성과 (Direct Output) — 정량

| 구분 | 지표 | 산출 근거 | 목표치 |
|------|------|---------|--------|
| 참여 규모 | 참여자 수, 방문자 수, 수혜자 수 | 유사 사업 벤치마크 | 구체적 숫자 |
| 콘텐츠 생산 | 제작 수, 배포 수, 프로그램 회차 | 사업 계획 기반 | 구체적 숫자 |
| 매출/수익 | 직접 매출, 부가 수익, 비용 절감 | 시장 데이터 기반 | 구체적 숫자 |
| 인프라 | 구축 시스템, 공간, 플랫폼 | 사업 범위 기반 | 구체적 범위 |

**작성 원칙**: 모든 수치에 산출 근거 필수
- "유사 사업 [사업명]의 실적 [수치]를 기반으로, 본 사업의 규모([수치])를 감안하여 [X%]로 추정"

### Layer 2: 행동 변화 (Behavioral Outcome) — 정량+정성

> 마케팅 관점에서 가장 중요한 층위. 타겟의 실제 행동이 어떻게 바뀌는가.

| 구분 | 지표 | 측정 방법 | 기대 수준 |
|------|------|---------|----------|
| 인지도 | 브랜드/사업 인지율 변화 | 사전-사후 설문 | Before OO% → After OO% |
| 참여 깊이 | 체류시간, 재방문율, NPS | 현장 데이터 + 설문 | NPS OO점 이상 |
| 전환 | 참여→구매/가입/재방문 전환율 | 퍼널 분석 | 전환율 OO% |
| 확산 | SNS 공유, UGC 생성, 언론 노출 | 미디어 모니터링 | UGC OO건, 노출 OO회 |
| 만족도 | 참여자 만족도, 추천 의향 | 설문 (5점/10점 척도) | 만족도 OO점 이상 |

### Layer 3: 파급 효과 (Ripple Impact) — 정성+추정 정량

> 사업 범위를 넘어서는 간접 효과. 평가위원이 "이 팀은 스케일이 다르다"고 느끼는 부분.

**경제적 파급**: 지역경제 활성화, 직·간접 고용, 관련 산업 매출, 세수 증가
**사회적 파급**: 커뮤니티 형성, 시민 역량 강화, 지역 브랜드 가치, 사회적 포용
**정책적 파급**: 타 지자체 벤치마크, 후속 사업 연결, 정책 모델화, 상위 정책 기여

### Layer 4: 지속가능성 (Sustainability) — 구조적

> 사업 종료 후에도 효과가 지속되는 메커니즘. 정부사업에서 가장 취약한 부분이자 차별화 기회.

**자체 수익 모델**: 사업 종료 후 자립 가능한 구조, 수익원 다각화, 손익분기 시점
**인프라 잔존가치**: 시스템/데이터/네트워크 지속 활용, 디지털 자산, 물리적 자산
**조직 역량 내재화**: 발주처 조직 내 역량 이전, 운영 매뉴얼, 담당자 역량 강화

### ROI 산출 구조

> 정부사업에서 "ROI"를 명시적으로 쓰는 제안서는 거의 없다. 이것만으로 차별화.

**3가지 ROI 프레임**:

| ROI 유형 | 산출식 | 용도 |
|---------|--------|------|
| **직접 ROI** | (직접 매출 + 비용절감) ÷ 사업 예산 | 가장 보수적. 매출이 있는 사업에 적합 |
| **종합 ROI** | (직접효과 + 간접경제효과 + 미디어가치) ÷ 사업 예산 | 파급효과까지 포함. 공공사업 설득력 |
| **사회적 ROI (SROI)** | (경제적 가치 + 사회적 가치 화폐환산) ÷ 투입 비용 | 정부사업의 공공성 정당화 |

**EMV (Earned Media Value) 산출** (해당 시):
```
EMV = Σ (노출 × CPM) + (인게이지먼트 × CPE) + (전환 × CPA)
```

**ROI 산출 시 주의사항**:
- 보수적 추정 원칙 (최소~최대 범위로 제시)
- 산출 근거를 1:1 매핑 (각 숫자마다 출처 명시)
- 과대 추정 시 신뢰도 하락 → 차라리 보수적이 낫다

### 비즈니스 임팩트 시나리오 (Best / Base / Worst)

| 시나리오 | 조건 | L1 매출 | L2 행동 | 종합 ROI |
|---------|------|---------|---------|---------|
| **Best** | 모든 전략 성공 + 외부 환경 우호 | | | |
| **Base** | 핵심 전략 성공, 일부 변수 미달 | | | |
| **Worst** | 핵심 리스크 현실화 | | | |

- 제안서에는 **Base 시나리오** 기준으로 작성
- Best는 "잠재 가능성"으로 언급
- Worst는 "리스크 대응 계획"의 근거로 활용

### 기대효과 종합 1-Pager

```
┌─────────────────────────────────────────────────────────┐
│            [사업명] 기대효과 종합                          │

│  L1 직접 성과:  참여자 OO명 / 매출 OO억 / 콘텐츠 OO건    │
│  L2 행동 변화:  인지도 OO%→OO% / NPS OO점 / 전환율 OO%   │
│  L3 파급 효과:  지역경제 OO억 파급 / 고용 OO명            │
│  L4 지속가능성: 자체수익 OO억/년 / 손익분기 O년차          │
```

## Step 9: 평가위원 설득 전략 〔검증·전달〕

> 평가위원은 하루에 3~5개 제안서를 읽는 "고관여이지만 시간이 제한된" 독자.
> 중심 경로(논리/증거)와 주변 경로(신뢰/인상)를 동시에 설계해야 한다.

### 9-1. 전략 제언 SCQA 구조

```
S (Situation):   이 사업의 현재 맥락과 배경
C (Complication): 기존 접근 방식의 한계 / Pain Point (Step 1에서 도출)
Q (Question):    어떻게 하면 [JTBD에서 도출한 핵심 가치]를 극대화할 수 있는가?
A (Answer):      우리의 3가지 전략 (Rule of Three) — 관통 메시지 + Why Us 축
```

### 9-2. Cialdini 원칙의 제안서 매핑

| Cialdini 원칙 | 제안서 적용 | 구체적 방법 |
|-------------|-----------|-----------|
| 권위 (Authority) | 유사 사업 실적, 전문가 자문단 | 실적 데이터를 "평가 키워드"로 재서술 |
| 사회적 증거 (Social Proof) | 벤치마크 사례, 유사 사업 성과 | "OO 사업에서 참여자 만족도 95%" |
| 일관성 (Commitment) | 관통 메시지가 전체에 일관 | 키워드 반복, 비주얼 통일 |
| 호감 (Liking) | 제안서 디자인 품질 | 표지~목차~요약의 첫인상 |
| 희소성 (Scarcity) | 우리만 할 수 있는 차별점 | "이 접근은 국내 최초/유일" |
| 통합 (Unity) | 발주처와 "함께" 만드는 느낌 | "귀 기관의 비전과 우리의 전문성이..." |

### 9-3. 기대효과 설득력 증폭 기법

| 심리 원리 | 적용 방법 | Before → After |
|----------|---------|---------------|
| 앵커링 (Anchoring) | 상위 벤치마크를 앵커로 먼저 제시 | "글로벌 사례 OO 대비 본 사업은..." |
| 프레이밍 (Framing) | 동일 효과를 이득 프레임으로 표현 | "비용 절감 30%" → "효율 130% 달성" |
| 대비 효과 (Contrast) | 기존 방식 vs 제안 방식 비교 | Before/After 비교 테이블 |
| 구체성 (Specificity) | 모호한 표현 → 수치화 | "향상될 것임" → "연간 OO명, OO억" |
| 가용성 휴리스틱 | 생생한 사례/비주얼 | 사진, 인포그래픽, 다이어그램 |

## Step 10: 자체평가 〔검증·전달〕

> 공고 배점표로 제안서 초안(또는 전략안)을 보수적으로 채점하고,
> 만점이 아닌 항목은 '왜 깎였나'를 명시한다.

### 10-1. 배점 기반 채점

| 평가 항목 | 배점 | A안 점수 | B안 점수 | C안 점수 | 감점 이유 |
|-----------|------|---------|---------|---------|----------|
| | | | | | |
| **합계** | /90 | | | | |

**채점 원칙:**
1. **보수적으로** — 근거가 약하면 배점의 60~70%로 채점
2. **점수 + 감점 이유를 함께** — "−4점, KPI 미확정"처럼 사유까지
3. **[확인] 미확정 정보 표시** — 채우면 몇 점 올라가는지 추정

### 10-2. 협상적격선 확인

- 보통 배점의 85% (기관마다 다름)
- 각 안별 통과 여부 확인
- 미달 시 어떤 항목을 보강해야 통과하는지 명시

### 10-3. 보완 우선순위

| 우선순위 | 항목 | 현재 점수 | 보완 후 기대 | 필요 조치 |
|---------|------|---------|------------|----------|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

> **건너뛰면:** 자기 약점을 모른 채 제출한다. 협상적격선 미달 위험을 발견하지 못하고, 보완할 시간도 놓친다.

## Step 11: 의사결정 리뷰 〔검증·전달〕

> 제안서 전체가 아니라, **의사결정자가 판단할 코어만** 추려 리뷰 자료로 만든다.
> 의사결정자에겐 코어만, 익숙한 포맷으로.

### 11-1. 코어 추출

의사결정자가 봐야 할 6가지:
1. **Pain Point** — 심사위원이 가려운 곳 (Step 1)
2. **관통 메시지** — 우리의 약속 한 줄 (Step 2)
3. **Why Us** — 왜 우리인가 (Step 3)
4. **3안 비교** — A/B/C 나란히 (Step 6)
5. **예산** — 투찰가와 마진 (Step 7)
6. **자체평가** — 점수와 보완 포인트 (Step 10)

### 11-2. 3안 나란히 비교

3안을 같은 구조로 나란히 놓아 의사결정자가 비교 가능하게:

```
┌──────────┬──────────┬──────────┐
│  A안     │  B안     │  C안     │
│ [안정형]  │ [차별형]  │ [혁신형]  │
├──────────┼──────────┼──────────┤
│ 핵심:    │ 핵심:    │ 핵심:    │
│ 강점:    │ 강점:    │ 강점:    │
│ 리스크:  │ 리스크:  │ 리스크:  │
│ 예상점수: │ 예상점수: │ 예상점수: │
├──────────┼──────────┼──────────┤
│       추천: [B안] 또는 [A+B 하이브리드]       │
└──────────┴──────────┴──────────┘
```

### 11-3. 결정 포인트 명시

의사결정자가 정해야 할 것을 명확히:
1. **GO / PASS** — 이 사업에 응찰하는가
2. **컨셉 택일** — A / B / C / 하이브리드 중 어느 안
3. **컨소시엄** — 파트너사 구성 확정
4. **예산** — 투찰가 확정
5. **보완** — [확인] 항목 중 즉시 조치할 것

> **건너뛰면:** 의사결정자가 두꺼운 제안서를 통째로 읽어야 한다. 핵심이 묻혀 결정이 늦어진다.

## Step 12: 제안서 반영 가이드 〔검증·전달〕

> 1단계의 양식 분석 결과와 연결하여, 전략 제언을 제안서의 어디에 어떻게 녹일지 가이드.

### 12-1. 기대효과 배치 전략

| 기대효과 Layer | 제안서 배치 위치 | 표현 방식 |
|-------------|--------------|---------|
| L1 직접 성과 | 사업 계획 본문 + 기대효과 섹션 | KPI 테이블 |
| L2 행동 변화 | 기대효과 섹션 중심 | CDJ 다이어그램 + 수치 |
| L3 파급 효과 | 기대효과 섹션 + 사업 비전 | 파급 효과 시각화 |
| L4 지속가능성 | 사후 관리/지속 운영 섹션 | 수익 모델 + 로드맵 |

### 12-2. 배점 기반 지면 재배분

1단계 배점표를 기반으로:
- 전략 제언 내용을 배점 높은 섹션에 우선 배치
- **배점 큰 항목에 분량을 싣는다** — 배점표에 대응 슬라이드가 없으면 그 항목은 0점
- 기대효과 1-Pager는 제안서 앞부분(총괄 요약)에 반드시 포함

### 12-3. 비주얼 가이드

추천 비주얼 요소:
| 비주얼 | 목적 |
|--------|------|
| 4-Layer Impact 피라미드 | 기대효과 한눈에 |
| Pain ↔ Why Us 매칭 테이블 | 1장으로 기억시키기 |
| Growth Flywheel 순환 다이어그램 | 확산 구조 시각적 증명 |
| Before/After 비교 | 기존 방식 vs 우리 방식 |
| 3안 비교 매트릭스 | 의사결정자용 비교 |
| KPI 대시보드 | ROI + 핵심 수치 종합 |
| CDJ 여정 맵 | 타겟 행동 변화 경로 |
| 컨소시엄 조직도 | 역할 한눈에 |

## 8대 원칙 — 단계 무관 공통 규칙

1. **공고의 사실은 추측하지 않는다** — 모르면 [확인]으로 남긴다. 추측은 평가에서 깨진다.
2. **발주처의 작년 약점이 올해의 무기다** — 심사위원의 문제의식을 정면으로 다룬다.
3. **제안서는 한 줄로 수렴한다** — 모든 항목이 관통 메시지를 가리켜야 기억된다.
4. **배점 큰 항목에 분량을 싣는다** — 배점표에 대응 슬라이드가 없으면 그 항목은 0점.
5. **핵심 컨셉은 최소 3개 안으로** — A·B·C 베리에이션으로 선택지를 남긴다.
6. **예산은 2층, 마진은 출자비율만큼** — 제출용/내부 실질을 나누고, 컨소시엄 마진은 지분만큼이다.
7. **자체평가는 보수적으로, 감점 이유까지** — 근거가 약하면 배점의 60~70%로 채점한다.
8. **의사결정자에겐 코어만, 익숙한 포맷으로** — 두꺼운 제안서가 아니라 결정에 필요한 것만.

## 분석 원칙

1. **프레임워크는 도구, 목적 아님** — "BCG Matrix 적용했습니다"가 아니라 자연스럽게 녹아든 논리
2. **배점이 여전히 중심축** — 기대효과가 아무리 좋아도 배점 낮은 영역이면 비중 줄임
3. **정량 수치는 근거 필수** — "유사사업 OO 실적 기반, 본 사업 규모 감안 OO%로 추정"
4. **1단계와 톤 차이** — 분석적("~이다") → 전략적("~해야 한다")
5. **마케팅/세일즈 관점 유지** — 사업 결과물을 "상품"으로 봄
6. **4-Layer는 빠짐없이** — 모든 Layer를 채우되, 해당 없는 항목은 명시적으로 "해당 없음"
7. **So What → Now What** — 각 분석 결과마다 "그래서 뭘 해야 하는가"로 연결
8. **Pain 기반** — 모든 전략은 Step 1의 Pain Point에서 출발. 우리 자랑이 아니라 심사위원의 문제의식에 답한다

### 더셀렉츠에서 배운 것

1. **과업 범위를 가르면 약점이 사라진다** — "현지 판매운영은 우리 몫이 아니다"는 발견이 GO 판정의 결정적 근거
2. **작년 약점은 공개 자료에 있다** — 뉴스·블로그에서 "성과 미공개·일회성"이라는 Pain을 찾아냄
3. **관통 메시지가 있으면 슬라이드가 알아서 모인다** — "From Buzz to Business" 한 줄이 14슬라이드를 관통
4. **2개 안은 보험이다** — A안이 안 끌리면 즉시 B안으로 전환. 하나에 갇히지 않음
5. **배점표 채점은 보수적으로** — [확인] 미확정을 채우면 점수가 올라가는 구조가 보완 동기를 만듦
