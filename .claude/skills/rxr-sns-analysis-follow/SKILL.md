## BEI란 무엇인가

### 한 줄 정의

> **BEI(Brand Engagement Index)** = 사람들이 브랜드를 얼마나 진심으로 좋아하는지를 SNS 데이터에서 읽어내는 지수 (0~100점)

### 왜 필요한가

> **📌 학술적 배경 — 행동적 증거 > 태도적 증거**
> eWOM 연구(Ismagilova et al., 2020; 69개 연구 메타분석)에서 리뷰의 **논거 품질**이 구매 의도에 가장 강한 영향을, 리뷰 **양**은 약한 영향을 미침. BEI는 이 원리를 수치화한 것 — "많은 사람이 좋다고 말했다"(태도적 증거)가 아니라 "시간과 노력을 들여 상세히 추천했다"(행동적 증거)를 측정.

기존 SNS 분석은 **"몇 건 언급됐고, 긍정이 몇 %"**까지만 알려줍니다. 하지만 마케터가 정말 알고 싶은 건:

- "이 캠페인으로 브랜드를 **진짜 좋아하게 된** 사람이 생겼나?"
- "그 효과가 **얼마나 오래 가나**?"
- "숫자 **하나**로 말해줄 수 있나?"

BEI는 이 질문에 답합니다.

### 측정 원리

설문으로 "좋아하세요?"라고 물으면 대부분 "네"라고 답합니다. 이건 쉬운 행동이라 신뢰하기 어렵습니다.

하지만 누군가 **500자짜리 후기를 정성껏 쓰고, 구체적인 경험을 공유하고, 친구한테 "꼭 가봐"라고 추천**했다면 — 이건 상당한 시간과 노력을 들인 행동입니다. 이런 행동은 거짓이기 어렵습니다.

BEI는 이 원리를 활용합니다:

> **"좋다고 말했으니까 좋아하는 거다" (X)**
> **"시간과 노력을 들여 진심으로 추천까지 했다 → 이것은 좋아한다는 행동적 증거다" (O)**

### 4축이 측정하는 것

| 축 | 쉬운 설명 | 측정 질문 |
|---|---|---|
| **Depth** (관여 깊이) | 얼마나 정성껏 리뷰를 썼나 | "대충 쓴 건가, 진심을 담은 건가?" |
| **Advocacy** (추천 강도) | 남에게 추천까지 했나 | "좋았다고만 한 건가, 가보라고 권한 건가?" |
| **Sincerity** (진심 비율) | 전체 반응 중 진심이 몇 %인가 | "노이즈를 빼면 진짜 목소리는 얼마나 되나?" |
| **Discovery** (발견 신선함) | 새로운 경험이었나 | "예상대로였나, 의외의 발견이었나?" |

### 2가지 관점으로 읽기

같은 BEI 점수도 성격이 다를 수 있습니다:

- **인게이지먼트 관점** — "깊이 있게 좋아하는 사람이 얼마나 되나" (Depth/Discovery 비중↑)
- **이슈성 관점** — "얼마나 퍼지고 화제가 되었나" (Advocacy/Sincerity 비중↑)

예시:
- 인게이지먼트 78 / 이슈성 52 → "조용하지만 깊은 팬을 만든 캠페인"
- 인게이지먼트 48 / 이슈성 75 → "화제는 됐지만 깊은 팬은 못 만든 캠페인"

### 등급

| 점수 | 등급 | 한마디 |
|:---:|------|------|
| 90+ | Exceptional | "모두가 진심으로 좋아한다" |
| 75~89 | Strong | "확실한 팬이 만들어졌다" |
| 60~74 | Solid | "관심은 있지만 팬까지는 아직" |
| 45~59 | Moderate | "화제는 됐지만 더 필요하다" |
| 30~44 | Emerging | "알긴 하는데 관심은 약하다" |
| ~29 | Weak | "아직 인식 자체가 부족하다" |

### 4주 추적 — 캠페인 효과는 얼마나 오래 가는가

BEI를 캠페인 직후(T+0)와 이후 4주간(T+1w ~ T+4w) 반복 측정하면 **효과의 지속성**을 볼 수 있습니다.

- **유지율 80%+** → Strong Retention: 오래 기억에 남는 캠페인
- **유지율 60~79%** → Moderate Decay: 보통 수준의 감쇠
- **유지율 60% 미만** → Rapid Decay: 빠르게 잊히는 캠페인

또한, 4축을 각각 추적하면 **"무엇이 살아남고 무엇이 먼저 죽는지"**를 알 수 있습니다. 예를 들어 "체험의 기억(Depth)은 유지되지만 추천 의지(Advocacy)는 빠르게 사라진다"같은 진단이 가능합니다.

## BEI 산출 공식

### 4축 (Axis)

#### [축1] Depth Score — 관여 깊이 (0~100)

**"얼마나 깊이 관여했는가"**

```
Depth = Auth평균(Gate3) × 0.6 + RQL정규화 × 0.4

RQL 정규화:
  가중평균 = (Q5건수×2.0 + Q4건수×1.5 + Q3건수×1.0 + Q2건수×0.5 + Q1건수×0.2) / 전체건수
  정규화 = (가중평균 - 0.2) / (2.0 - 0.2) × 100
```

- Auth가 높을수록 진심으로 관여한 것
- RQL이 높을수록 깊이 있는 리뷰를 쓴 것 (높은 멘탈코스트)

#### [축2] Advocacy Score — 추천 강도 (0~100)

**"얼마나 적극적으로 추천하는가"**

```
Advocacy = Clout평균(Gate3) × 0.6 + P-01비율정규화 × 0.4

P-01 비율:
  P-01(진짜팬) = Gate 3 내 긍정 + Clout 55+ 건수
  P-01율 = P-01건수 / Gate3전체건수 × 100
  정규화 = min(P-01율 / 40 × 100, 100)  ← 벤치마크 40% = 만점
```

- Clout이 높을수록 추천 의지가 강한 것
- P-01 비율이 높을수록 실제 옹호자가 많은 것

#### [축3] Sincerity Rate — 진심 비율 (0~100)

**"전체 중 진심의 비중은"**

```
Sincerity = Gate 3 통과건수 / Gate 1 전체건수 × 100
```

- 높을수록 노이즈가 적고 건강한 인게이지먼트

#### [축4] Discovery Score — 발견 신선함 (0~100)

**"새로운 발견이었나, 예상 범위였나"**

```
Discovery = Freshness Index (기존 산출값 그대로 사용)
         = "이/가" / ("이/가" + "은/는") × 100
```

- 높을수록 "처음 발견한 놀라움"이 많은 것
- 시간 경과 시 자연 감쇠 (정상적 현상)
- **심리학적 근거**: 해석 수준 이론(CLT) — "이/가"는 저수준 해석(구체적·감각적 처리), "은/는"은 고수준 해석(추상적·스키마 통합). 시간 경과 시 감각 기억→의미 기억 전환으로 Freshness 자연 하락(쾌락 적응, Hedonic Treadmill)

> **📌 원본 분석 Freshness 검증 규정**
> Discovery 축은 원본 리포트(rxr-sns-analysis)의 Freshness Index를 그대로 사용한다.
> 원본 리포트에서 Freshness 해석이 누락된 경우(수치만 있고 해석 없음), follow-up 분석에서 Discovery 해석의 근거가 약해진다.
> - **확인 사항**: 원본 리포트의 "기간별 심리 변화 섹션"에서 Freshness 해석이 2문장 이상 존재하는지 체크
> - **누락 시 조치**: follow-up 리포트 작성 전에 원본 데이터의 Freshness 패턴을 보충 분석하고, Discovery 해석에 "원본 분석 보충" 라벨 표기

### BEI 종합 + 2관점

#### BEI 종합

```
BEI = Depth × 0.35 + Advocacy × 0.35 + Sincerity × 0.15 + Discovery × 0.15
```

> **📌 BEI 가중치의 심리학적 근거 및 대안 공식**
>
> **현행 가중합 공식의 근거:**
> - Depth·Advocacy 각 35%: **계획된 행동 이론(TPB, Ajzen)** — 행동 비용이 높은 실제 행동(깊은 리뷰 작성, 적극적 추천)이 진짜 의도를 가장 강하게 예측
> - Discovery 15%: **단순 노출 효과(Mere Exposure, Zajonc)** — 첫 노출에서만 강하고 빠르게 소멸하므로 장기 지표로서의 가중은 낮음
> - Sincerity 15%: 전체 반응 중 진심 비율
>
> **대안 검토 — 곱셈-가산 하이브리드:**
> ```
> BEI_alt = (Depth × 0.40 + Advocacy × 0.35 + Discovery × 0.25) × (Sincerity / 100)^0.5
> ```
> - **근거**: Sincerity는 **신뢰성 휴리스틱(Credibility Heuristic)**의 기반 변수. 노이즈 비율이 높으면(Sincerity↓) 다른 축이 아무리 높아도 전체 신뢰가 무너지므로, 가산이 아니라 승산(곱셈)으로 반영하는 것이 "노이즈가 많으면 아무리 좋은 반응도 의미 없다"는 직관에 부합.
> - **현행 유지 이유**: 기존 리포트 간 비교 가능성(소급 적용 시 과거 BEI 재산출 필요), 클라이언트 커뮤니케이션의 단순성. 충분한 사례(N=20+)가 축적된 후 대안 공식으로 전환 검토.

#### BEI 인게이지먼트 관점

**"깊이 있게 좋아하는 사람이 얼마나 되나"** — Depth/Discovery 비중 높음

```
BEI(E) = Depth × 0.40 + Discovery × 0.25 + Advocacy × 0.20 + Sincerity × 0.15
```

#### BEI 이슈성 관점

**"얼마나 퍼지고 화제가 되었나"** — Advocacy/Sincerity 비중 높음

```
BEI(I) = Advocacy × 0.40 + Sincerity × 0.25 + Depth × 0.20 + Discovery × 0.15
```

### Decay 판정 (4주 추적 시)

```
유지율 = T+4w BEI / T+0 BEI × 100

80%+ → Strong Retention (강한 지속)
60~79% → Moderate Decay (보통 감쇠)
60% 미만 → Rapid Decay (급속 소멸)
```

> **📌 Decay 패턴의 심리학적 메커니즘 (리포트 "Decay 해석 카드"에 반영 필수)**
>
> | 축 | 감쇠 패턴 | 심리학적 메커니즘 |
> |---|---|---|
> | **Depth** | Slow Decay | **절정-종결 규칙(Peak-End Rule, Kahneman)**: 강렬한 체험의 절정 기억은 에피소드 기억으로 깊이 인코딩되어 천천히 감쇠. 체험이 "절정(peak moment)"을 만들면 장기 기억에 단단히 고정됨 |
> | **Advocacy** | Rapid Decay | **FOMO의 시간 의존성**: 추천 의지는 "지금 핫한" 사회적 맥락에 의존. 시간이 지나면 FOMO가 소멸하면서 추천 동기 급감. 단, **정체성 수준의 추천**("나는 ___팬")은 감쇠가 현저히 느림(사회적 정체성 이론, Tajfel) |
> | **Sincerity** | Inverse(상승) | **가용성 휴리스틱의 역전(Tversky & Kahneman)**: 시간이 지나면 의무적·피상적 리뷰어는 더 이상 안 쓰고, "진짜 좋았던 사람"만 기억에서 꺼내 작성. 노이즈가 먼저 소멸하고 진심이 잔존. **인지 부조화(Festinger)**: 진심이 아니었던 사람은 시간이 지나면 부조화 해소를 위해 태도 수정 |
> | **Discovery** | Sharp Decay | **쾌락 적응(Hedonic Treadmill, Brickman)**: 새로운 자극에 대한 놀라움은 가장 빠르게 적응. **해석 수준 이론(CLT, Trope)**: 시간이 지나면 구체적·감각적 기억(저수준)이 추상적·개념적 기억(고수준)으로 전환 → Freshness 자연 하락 |
>
> **리포트 적용:** Decay 해석 카드(섹션 5)에서 단순히 "유지율 ___%"만 쓰지 말고, 위 심리학적 메커니즘을 1~2문장으로 포함하여 "왜 이런 감쇠 패턴인가"를 설명할 것.

## 주의사항

- BEI는 **SNS 데이터 기반 행동적 프록시**이며, 실제 선호도를 직접 측정한 것이 아님을 리포트에 명시
- Discovery는 시간 경과 시 자연 하락이 정상 — "나빠졌다"가 아니라 "신선함이 소진됐다"로 해석
- Sincerity는 시간 경과 시 상승할 수 있음 — 노이즈가 먼저 사라지고 진심이 남기 때문
- 주차별 데이터 10건 미만 시 "데이터 부족" 경고 표시
- T+0 데이터는 기존 분석에서 추출 (재수집 아님)
- 외부 KPI는 사용자가 수동 입력 — 자동 수집 아님
- 등급 표기 형식: "Moderate (57.6)" — 등급 텍스트 먼저, 숫자 병기
- **참고 시뮬레이션:** `30-knowledge/bei-simulation-saero.html` — 디자인/구조 기준
