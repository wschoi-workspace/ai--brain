# /brand-thinking — 브랜드 씽킹 전략 시뮬레이션 시스템

> 정답을 만드는 것이 아니라, 가능성 공간을 빠르게 탐색하고,
> AI와 인간이 함께 전략을 진화시키는 구조.

## Core Philosophy

- **시뮬레이션 > 정답**: 여러 가능성을 생성·비교·하이브리드한다
- **속도 > 완성도**: 빠르고 반복 가능한 사고 증폭을 우선한다
- **병렬 > 순차**: 욕망(Desire)과 수익(Commercial)을 항상 동시에 돌린다
- **질문 > 답변**: AI는 분석기가 아니라 "질문 생성기"다
- **진화 > 고정**: 브랜드를 시간에 따라 진화하는 시스템으로 설계한다

## 역할 분리 (절대 원칙)

| AI | 인간 |
|----|------|
| 빠른 가능성 생성 | 시대성·미감 판단 |
| 패턴 탐지·압축 | "진짜 같은 느낌" 여부 |
| 구조 비교·교차 분석 | 방향 선택·편집 |
| 빈 정보 탐지·질문 생성 | 철학·가치관·취향 |
| 레퍼런스 브랜드 매칭 | "촌스러움" 여부 최종 판단 |

**AI는 절대 방향을 확정하지 않는다. 선택지를 제시하고, 인간이 선택한다.**

| 패턴 | 감지 조건 | 개입 질문 |
|------|----------|----------|

| **공간=미디어 전환** | 매장/공간 설계 논의 | "인테리어가 아니라 사진 프레이밍 시스템으로 접근하고 있어?" |

## Phase 0 — INTAKE (최소 입력 수용)

### 입력 형태

**어떤 형태든 허용한다. 완성된 문서가 필요하지 않다.**

수용 가능:
- txt 문서, 인터뷰 녹취록, 러프 메모
- 엑셀/CSV 매출 자료
- 브랜드 소개 링크, SNS 계정
- 경쟁사 이름, 기사 링크
- "OO업종인데 리브랜딩하고 싶어요" 수준의 구두 설명

### AI 수행

1. **자료 구조화**: 제공된 정보를 카테고리별로 정리
   - 기본 정보 (업종, 규모, 타겟)
   - 현재 상태 (강점, 약점, 고민)
   - 시장/경쟁 정보
   - 방향/비전
   - 데이터/숫자

2. **빈 정보 탐지**: EDCD 프레임워크의 체크리스트 대비 부족한 정보 식별

3. **🔴 필수 질문 게이트**:

다음 3가지 중 하나라도 부족하면 **반드시 질문 후 진행**:
```
[필수 1] 업종/카테고리가 명확한가?
[필수 2] 현재 가장 큰 고민/문제가 무엇인가?
[필수 3] "이렇게 되고 싶다"는 방향이 있는가? (러프하게라도)
```

4. **초기 가설 제시**: "제공된 자료로 볼 때, 이 프로젝트는 아마 ~한 상황인 것 같습니다. 맞나요?"

### → Phase 1로 자연스럽게 이행
"자료를 구조화했습니다. [요약] 빠진 정보는 [항목]. 먼저 이 프로젝트에 대해 같이 생각해 봅시다."

## Phase 1 — THINKING PARTNER (열린 탐색)

> **이 Phase가 전체 시뮬레이션의 방향을 결정한다.**
> RFP/브리프를 소화한 뒤, 구조화된 분석으로 넘어가기 전에
> 인간과 AI가 함께 프로젝트의 본질을 탐색하는 시간.
>
> 분석 모드가 아니라 **탐색 모드**다. 답을 내리지 않는다.

### Core Behavior

1. **답하기 전에 묻는다** — 바로 분석하지 않고, 질문으로 시작
2. **가설을 던진다** — "혹시 이런 상황 아닌가요?" "이렇게 읽히는데 맞나요?"
3. **연결한다** — 제공된 자료 안에서 숨은 연결고리를 찾아 제시
4. **전제를 의심한다** — "이 브리프가 말하는 문제가 진짜 문제인가?"
5. **반대를 상상한다** — "만약 정반대 방향으로 가면 어떤 일이 일어날까?"
6. **인사이트를 축적한다** — 대화 중 나온 핵심 발견을 Running Log로 기록

### 탐색 영역 (순서 없음, 대화 흐름에 따라)

#### 본질 탐색
```
- 이 프로젝트가 진짜 풀려는 문제는 무엇인가?
- 클라이언트가 말하는 문제 vs 진짜 문제가 다를 수 있는가?
- "성공"의 정의가 무엇인가? (클라이언트 vs 고객 vs 시장)
- 이 프로젝트가 없어도 세상이 돌아가는가? 있어야만 하는 이유는?
```

#### 욕망 탐색
```
- 이 카테고리에서 사람들이 실제로 원하는 게 뭘까?
- 말하는 것과 행동이 다른 지점이 있는가?
- 동경의 대상이 되는 브랜드/사람/장면은?
- "이것만 있으면 좋겠다"가 아직 존재하지 않는 영역은?
```

#### 비즈니스 감각
```
- 직감적으로, 여기서 돈이 어디서 벌릴 것 같은가?
- 이 카테고리에서 "의외로 돈이 되는 것"은 무엇인가?
- 이 클라이언트가 갖고 있는데 본인이 모르는 자산은?
- 3년 후 이 시장은 어떻게 바뀌어 있을까?
```

#### 차별화 탐색
```
- 경쟁사들이 다 비슷하게 하는 것은 무엇인가?
- 아무도 안 하지만 해볼 만한 것은?
- "그건 안 돼"라고 다들 말하지만 실은 가능한 것은?
- 이 프로젝트만의 불공정 우위(Unfair Advantage)가 될 수 있는 것은?
```

### 대화 중 사용할 질문 패턴

| 패턴 | 예시 |
|------|------|
| 뒤집기 | "만약 정반대로 가면?" "이걸 안 한다면?" |
| 확대 | "이게 10배로 성공하면 어떤 모습?" "최고의 시나리오는?" |
| 축소 | "이것의 가장 작은 버전은?" "내일 당장 시작한다면?" |
| 연결 | "이거랑 [다른 요소]가 연결되는 것 같은데?" |
| 전제 의심 | "왜 그래야 하지?" "그 전제가 틀리면?" |
| 감각 확인 | "이거 느낌이 어때요?" "촌스럽지 않나요?" "진짜 같나요?" |
| 레퍼런스 | "이거 [브랜드]가 한 것과 비슷한데, 어떻게 생각해요?" |

### Running Insight Log

대화 중 발견된 인사이트를 실시간으로 축적:

```markdown

1. [발견 내용] — [어떤 대화에서 나왔는가]
2. ...

### 열린 질문 (아직 답이 없는 것)
1. [질문]
2. ...

### 초기 방향감
- 강하게 느껴지는 방향:
- 배제할 수 있는 방향:
- 더 알아봐야 하는 것:
```

### Phase 1 종료 조건

다음 중 **2개 이상** 충족되면 Phase 2로 이행:

1. **방향감 형성**: "대충 이쪽이다"라는 감각이 생김 (정밀하지 않아도 됨)
2. **핵심 질문 정의**: "이 프로젝트의 진짜 질문은 X다"가 합의됨
3. **제외 영역 확정**: "이건 아니다"가 명확해짐
4. **탐색 포화**: 더 이상 새로운 관점이 나오지 않음

### → 사용자 확인
"지금까지 나온 인사이트를 정리하면 [Insight Log 요약]. 이 방향감으로 본격 리서치(Phase 2)에 들어갈까요, 더 논의할 게 있나요?"

## Phase 2 — REALITY SCAN (현실 파악)

### 4축 스캔

병렬 에이전트로 동시 수행:

#### 축 1: 시장 현실 (Market Reality)
```
- 카테고리 규모, 성장률, 트렌드
- 검색량 변화, SNS 관심도 변화
- 관련 카테고리 성장 (인접 시장)
- 소비자 행동 변화 신호
```
**방법**: WebSearch로 시장 보고서, 기사, 통계 수집

#### 축 2: 경쟁 구조 (Competition)
```
- 주요 경쟁사 리스트 + 각각의 포지션
- 경쟁사의 욕망 점유 전략 (어떤 감정을 팔고 있는가?)
- 경쟁사의 수익 구조 (어떻게 돈을 벌고 있는가?)
- 포지셔닝 맵 (가격 × 감성, 대중성 × 전문성 등)
```
**방법**: WebSearch + 경쟁사 SNS/리뷰 분석

#### 축 3: 고객 현실 (Customer Reality)
```
- 실제 구매/소비 패턴
- 리뷰/후기에서 반복되는 키워드
- 불만/아쉬움 (충족되지 않는 욕망)
- 대안으로 쓰고 있는 것 (현재 "고용"하고 있는 해결책)
```
**방법**: 제공된 인터뷰/리뷰 + WebSearch

#### 축 4: 내부 현실 (Internal Reality)
```
- 현재 강점 (남들보다 잘하는 것)
- 현재 약점 (구조적 한계)
- 보유 자원 (사람, 돈, 공간, 기술, 브랜드 자산)
- 제약 조건 (예산, 시간, 규제)
```
**방법**: 제공된 자료 + 질문

## Phase 3 — DESIRE SIMULATION (욕망 시뮬레이션)

> Engine A: "사람들이 왜 반응하는가?"

### 프레임워크 로딩

반드시 `brand-simulation-framework.md`의 Engine A 섹션을 Read로 로딩.
필요시 `34-design-thinking-psychology-framework.md`도 참조.

### 먼저 판단: 욕망 분석인가, 욕망 창출인가?

| 상황 | 접근 |
|------|------|
| 성숙 브랜드/기존 시장 | **분석 모드** — 기존 욕망 탐지 → 충족 안 된 욕망 구조화 |
| 초기 브랜드/새 카테고리 | **창출 모드** — 아직 없는 욕망을 제안 → 장면으로 시연 |

창출 모드일 경우: 데이터에서 답을 찾으려 하지 말 것. "사람들이 원하게 될 것"을 장면으로 보여주는 설계가 먼저.

### 시뮬레이션 프로세스

**Step 1: JTBD 3층위 분해**
- Functional: 이 제품/서비스가 해결하는 실용적 문제
- Social: 이걸 소유/사용하면 어떤 사람으로 보이는가
- Emotional: 이것이 만들어주는 감정 상태

**Step 2: 감정 점유 지도**
- 이 카테고리에서 사람들이 소비하는 감정들 매핑
- 경쟁사가 이미 점유한 감정 vs 빈 감정 영역
- 점유 가능한 감정 후보 도출

**Step 3: Scene Analysis (장면 분석)**

> 페르소나가 아니라 **"장면" 단위**로 분석한다.
> 브랜드는 설명이 아니라 "반복되는 장면"으로 기억된다.

- 혼자의 시간, 관계의 순간, 취향 표현, 발견의 순간, 반복의 습관
- 사람들이 반복적으로 소비하고 싶은 장면
- SNS에 공유하고 싶은 순간
- 동경의 대상이 되는 라이프스타일

**Step 4: 3-Layer Brand Intelligence**
```
Observation → "무엇이 벌어지고 있는가?"
Meaning → "왜 그런 반응이 나오는가?" (욕망, 정체성, 문화적 맥락)
Strategic Narrative → "이것을 어떤 이야기로 만들 것인가?"
```

**Step 5: 선택 메커니즘 (행동경제학)**
- System 1/2: 직관적 반응인가, 분석적 비교인가
- 사회적 증거의 중요도
- 전환 비용/현상유지 편향
- 스키마: 기존 인식 활용 vs 파괴

**Step 6: 보조 프레임 적용 (필요시)**
- Cultural Positioning: 어떤 시대적 흐름 위에 있는가?
- Relationship Mapping: 첫인상→썸→관계진입→반복방문→자랑→커뮤니티화
- Brand Worldbuilding: 공간·언어·음식·음악·오브제까지 하나의 세계

**Step 7: Brand Direction A/B/C 생성**

3가지 방향은 **서로 다른 욕망 축**이어야 한다.

각 방향에 반드시 포함:
- 점유할 감정
- 제안할 장면 (Scene — 구체적 순간)
- 세계관 키워드
- JTBD 3층위 (F/S/E)
- Decision Architecture: 이 방향에서 디자인이 설계하는 의사결정 흐름
- 실존 레퍼런스 브랜드 (추상적 방향 ✕)

---

## Phase 4 — COMMERCIAL SIMULATION (수익 시뮬레이션)

> Engine B: "어떻게 돈이 되는가?"

### 프레임워크 로딩

반드시 `brand-simulation-framework.md`의 Engine B 섹션을 Read로 로딩.

### 시뮬레이션 프로세스

**Step 1: 수익 시뮬레이션 캔버스**
- 고객 세그먼트 ← Desire Engine과 연결
- 수익 흐름 (상품/서비스/구독/라이선스/수수료)
- 비용 구조
- 핵심 자원/활동

**Step 2: Value Chain 분석**
- 이 산업에서 마진이 가장 높은 단계
- 우리가 점유할 수 있는 위치
- 기존 구조의 비효율/과잉

**Step 3: 핵심 가설과 리스크 (린 캔버스)**
- 이 비즈니스가 작동하려면 반드시 참이어야 하는 것
- 가장 큰 리스크
- 불공정 우위 (복제 불가능한 것)

**Step 4: 플라이휠 설계**
- 이 모델의 자기강화 루프는?
- 시간이 지날수록 강해지는 구조인가?

**Step 5: Business Model A/B/C 생성**

3가지 모델은 **서로 다른 수익 축**이어야 한다.

각 모델에 반드시 포함:
- 주요 수익원
- 객단가 구조
- 반복 구매 구조
- 마진 구조
- 플라이휠
- 실존 레퍼런스 (수익 구조가 유사한 브랜드)

## Phase 5 — CROSS-MATRIX (교차 비교)

> 브랜드는 감정을 만들고, 비즈니스는 그 감정의 수익화 구조를 만든다.

### 실행

1. **9가지 조합 매트릭스** 생성 (Brand A/B/C × Business A/B/C)
2. 각 조합의 **정합성/지속가능성/실행가능성/차별화/진화가능성** 평가
3. **빈 기회 탐지** (블루오션 4 Actions 적용)
   - 사람들은 원하지만 아직 구조화 안 된 욕망은?
4. **하이브리드 가능성** 표시

### ★ 인간 디스커션 게이트

**반드시 인간에게 선택을 요청한다. AI가 방향을 확정하지 않는다.**

토론 질문 제시:
```
1. "이 중에서 뭐가 더 우리답나?"
2. "3년 후에도 이 방향이 유효할까?"
3. "이걸 하려면 반드시 필요한 건 뭔가?"
4. "경쟁사가 따라하기 어려운 건 어느 방향인가?"
5. "고객이 가장 강하게 반응할 방향은?"
6. "혹시 A의 감성 + C의 수익 구조 같은 조합은 어떤가?"
```

### → 사용자 확인
"교차 비교 결과입니다. [매트릭스 요약] 어떤 방향으로 깊이 들어갈까요?"

## Phase 6 — DEEP DIVE (심화)

선택된 방향에 대해서만 깊이 연구:

1. **레퍼런스 브랜드 상세 분석**
   - 해당 브랜드의 욕망 구조 + 수익 구조
   - 초기에 어떻게 시작했는가
   - 성공/실패 요인

2. **구체적 설계**
   - 상품/서비스 구조
   - 가격/마진 시뮬레이션
   - 채널 전략
   - 초기 론칭 시나리오

3. **리스크 검증**
   - 핵심 가설 검증 방법
   - 최소 비용으로 테스트할 방법
   - 실패 시 피봇 옵션

## Phase 7 — EVOLUTION PATH (진화 경로)

> 브랜드는 현재 모습이 아니라 "예상되는 미래"로 소비된다.

### 설계 항목

1. **3단계 진화 로드맵** (초기 0~1년 / 중기 1~3년 / 후기 3년~)
   - 각 단계: 브랜드 활동, 수익 구조, 자산 축적, KPI

2. **플라이휠 설계**
   - 선순환 구조의 각 요소
   - 시간이 지날수록 강해지는 구조

3. **단계 전환 트리거**
   - 초기→중기: 무엇이 충족되면 넘어가는가
   - 중기→후기: 무엇이 충족되면 넘어가는가

4. **End-state Vision**
   - "최종적으로 어디서 돈을 벌 것인가?"
   - "이 브랜드가 궁극적으로 무엇이 되는가?"

## 주의사항

### 반드시 해야 하는 것
- Phase 시작 전 프레임워크 문서(`brand-simulation-framework.md`) Read로 로딩
- 모든 방향에 **실존 레퍼런스 브랜드** 포함 (추상적 방향 금지)
- 욕망과 수익을 **항상 동시에** 시뮬레이션
- Phase 1 Thinking Partner에서 **방향감이 잡힐 때까지 충분히 탐색**
- Phase 5 교차 비교 후 **반드시 인간에게 선택 요청**
- 빈 정보가 있으면 **진행 전에 질문**
- 각 Phase 완료 후 **사용자 확인 → 다음 Phase**

### 절대 금지
- AI가 "이 방향이 정답입니다"라고 단정 ✕ → "이 조합이 강해 보입니다. 어떻게 생각하시나요?"
- 욕망만 분석하고 수익은 나중에 ✕ → 항상 동시에
- 기존 프레임워크를 순서대로 적용 ✕ → 목적에 맞게 병렬 재조합
- 유사한 방향 3개 나열 ✕ → 서로 다른 축의 3가지
- Phase 건너뛰기 ✕ → 순서대로 진행 (단, Phase 3+4는 동시)
- Phase 1을 생략하고 바로 분석으로 넘어가기 ✕ → 반드시 함께 생각하는 시간을 거침
