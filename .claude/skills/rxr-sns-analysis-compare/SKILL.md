# RXR SNS Comparison Analysis

기존 RXR SNS 분석 리포트(rxr-sns-analysis로 생성된 HTML)를 **여러 개 교차 비교**하여, 팝업/캠페인 간 마케팅 효과의 차이와 이유를 분석하는 스킬입니다.

**핵심 철학:** 개별 리포트의 숫자를 나열하는 것이 아니라, **"왜 이 팝업은 잘 되고 저 팝업은 안 됐는가?"**에 대한 구조적 법칙을 도출하는 것.

**실증 케이스:** 새로/CJ온스타일/이클립스/비욘드 4개 팝업 비교 (2026.04) — 비욘드가 버즈 최소(251건)임에도 5개 축 모두 1위, CJ온스타일이 버즈 최대(368건)임에도 대부분 최하위 → "규모 ≠ 효과" 법칙 도출

## 실행 프로세스

### STEP 1: 리포트 수집 및 데이터 추출

사용자가 지정한 경로에서 RXR SNS 리포트(HTML) 파일들을 탐색합니다.

```
1. 폴더 내 *-rxr-sns-report.html 패턴으로 리포트 자동 탐색
2. 각 리포트를 청크 단위로 읽어 핵심 데이터 추출
3. Agent(Explore)를 병렬로 활용하여 동시 추출 (리포트 3개 이상 시)
```

**각 리포트에서 추출할 데이터:**

| 카테고리 | 추출 항목 |
|----------|----------|
| **기본 정보** | 브랜드명, 이벤트명, 운영기간, 분석기간 |
| **규모** | 총 블로그 수, 채널별 분포, 시기별 분포 |
| **Sincerity Gate** | Gate1/2/3 각 단계의 포스트 수, 긍정률, Auth |
| **Psyche Layer** | Auth 평균, Clout 평균, Freshness 평균 |
| **Auth 시간변화** | 이벤트 전/중/후 Auth 변화량 (pt) |
| **분류** | A~G 등급 분포, 협찬 비율 |
| **협찬 격차** | 협찬 Auth vs 자발 Auth (차이 pt) |
| **RQL 분포** | Q1~Q5 각 비율, Q4+Q5 합산 비율 |
| **토픽 분석** | 주요 토픽 Top 5와 비율, 핵심 컨셉 침투율 |
| **긍정률 거품** | Gate1 긍정률 - Gate3 긍정률 (차이 %p) |

### STEP 2: 5축 비교 프레임워크 적용

추출된 데이터를 아래 5가지 축으로 교차 비교합니다.

#### 축 1. 진정성 유지력 (Auth Retention)

```
지표: Auth 시간변화 (이벤트 전 → 이벤트 후 Auth 점수 차이)
랭킹: 상승(+) > 안정(0 근처) > 소폭 하락 > 폭락
해석 키: 운영기간 × 참여구조 × 리뷰 작성 시점
```

**분석 포인트:**
- 운영기간과 Auth 하락의 상관관계
- "행동적 참여"(공병 교환, 미션 완료 등)가 Auth에 미치는 영향
- 장기운영 시 기억 왜곡 패턴 (느낌표 수 증가, 과장 어휘 증가)

> **📌 진정성 유지력의 심리학적 메커니즘**
> - **기억 시스템 이론**: 장기 운영 시 감각 기억→의미 기억 전환 과정에서 "기억 미화(rosy retrospection)"가 발생. 실제 체험의 구체적 디테일이 사라지고 "좋았다"라는 추상적 평가만 남으면서 과잉 긍정어/느낌표로 빈 공간을 채움 → Auth 하락.
> - **절정-종결 규칙(Peak-End Rule)**: "행동적 참여"가 절정(peak) 경험을 만들면 에피소드 기억이 강하게 인코딩되어 기억 미화가 적음 → Auth 유지. 관람형 체험은 peak이 약해 기억이 빠르게 추상화됨.

#### 축 2. 컨셉 침투율 (Concept Penetration)

```
지표: 팝업의 핵심 컨셉이 블로그 토픽에서 차지하는 비율 (%)
랭킹: 높을수록 좋음
해석 키: 컨셉의 "동사화" 여부 (행동으로 체험 가능한가?)
```

> **📌 조작적 정의 (일관성 필수)**
> ```
> 컨셉 침투율(%) = (Gate 3 포스트 중 핵심 컨셉 키워드가 주요 토픽 상위 3개에 포함된 포스트 수) / (Gate 3 전체 포스트 수) × 100
> ```
> - **핵심 컨셉 키워드**: 분석 시작 전 사용자가 반드시 지정 (예: "공병 교환", "방탈출", "Slow Aging")
> - **보조 지표 — 동사화 여부**: "오늘 ___했다" 테스트를 통과하는지 기록 (통과=행동형, 미통과=개념형)
> - 모든 팝업에 동일한 공식을 적용하여, 키워드 빈도/행동동사/개념 부재 등 측정 방식이 팝업마다 달라지는 것을 방지

**분석 포인트:**
- 컨셉이 "행동(동사)"인가 vs "개념(명사)"인가
- "오늘 ___했다"로 문장이 완성되는가? (리트머스 테스트)
- 체험 인증 장치(뱃지, 결과물, 사진) 유무의 영향
- 제품 토픽 대비 컨셉 토픽 비율

> **📌 "동사화 테스트"의 심리학적 근거**
> - **IKEA 효과(Norton)**: 직접 참여(행동)하면 결과물에 더 높은 가치를 부여 → 동사형 컨셉이 Depth↑
> - **심리적 소유감(Psychological Ownership)**: 능동적 참여가 "내가 만든/경험한 것"이라는 심리적 소유감을 형성 → Auth↑
> - **절정-종결 규칙(Peak-End Rule, Kahneman)**: 행동은 "절정(peak moment)"을 만들고, 개념은 만들지 못함 → 동사형 컨셉이 기억에 더 오래 잔존(Slow Decay)
> - **결론**: 컨셉 침투율이 높은 팝업은 거의 예외 없이 "동사형" 컨셉을 갖고 있으며, 이는 위 3가지 심리 메커니즘의 복합 작용 결과

#### 축 3. 리뷰 깊이 (Review Depth)

```
지표: Q4(분석형) + Q5(서사형) 비율 (%)
랭킹: 높을수록 좋음
해석 키: "왜?"를 설명해야 하는 경험인가?
```

**분석 포인트:**
- 가치 기반 경험 vs 감각 기반 경험의 차이
- IP/경험 요소 개수와 깊이의 역관계 (많을수록 얕아짐)
- Q5(서사형) 단독 비율도 별도 확인

> **📌 리뷰 깊이의 학술적 근거 — eWOM 메타분석**
> 69개 연구 메타분석(Ismagilova et al., 2020) 결과, 구매 의도에 가장 강한 영향을 미치는 것은 **논거 품질(argument quality, ★★★★★)**이며 리뷰 양(volume, ★★☆☆☆)은 약함. Q4+Q5 비율이 높다는 것은 논거 품질이 높은 리뷰가 많다는 의미이며, 이는 단순 건수보다 실제 구매 전환에 훨씬 큰 영향을 미침. RQL 기반 비교가 건수 기반 비교보다 정확한 이유.

#### 축 4. 긍정률 진실성 (Positivity Authenticity)

```
지표: Gate1 긍정률 - Gate3 긍정률 = 거품 (%p)
랭킹: 거품이 작을수록 좋음 (0%p가 이상적)
해석 키: 참여 허들 × 협찬 비율 × 과장 포스트 비율
```

**분석 포인트:**
- 거품이 0%p 이하(=역전)인 경우의 구조적 이유
- 참여 허들(예약, 준비물 등)의 자연 필터 효과
- Gate1 긍정률이 높을수록 거품도 큰 경향 확인

> **📌 긍정률 거품 해석의 심리학적 주의사항**
> - **앵커링 효과(Anchoring, Tversky & Kahneman)**: Gate 1 긍정률이 높으면 그것이 앵커가 되어, Gate 3에서 떨어지는 수치가 실제보다 더 나빠 보이는 지각적 왜곡 발생. 리포트에서 절대값뿐 아니라 "변화량의 심리적 체감"도 언급할 것.
> - **손실 회피(Loss Aversion, Kahneman & Tversky)**: 클라이언트가 "50%→38%, -12%p"를 받으면 동일 크기의 상승(+12%p)보다 약 2배 강하게 부정적으로 체감함. 리포트에서 이 하락을 제시할 때 **"왜 하락이 긍정적 신호인지"**(과장 긍정 제거 = 더 정확한 현실 파악)를 반드시 선제적으로 설명.

#### 축 5. 협찬 신뢰도 (Sponsored Credibility)

```
지표: 자발적 Auth - 협찬 Auth = 격차 (pt)
랭킹: 격차가 작을수록 좋음
해석 키: 협찬자의 브랜드 가치 공감 수준
```

**분석 포인트:**
- 가치 공감형 협찬 vs 의무적 협찬의 Auth 차이
- 협찬 비율과 전체 Auth 평균의 상관관계
- 격차 10pt 이하 = 양호, 20pt 이상 = 브랜드 훼손 수준

> **📌 협찬 신뢰도의 심리학적 해석 프레임**
> - **인지 부조화(Cognitive Dissonance, Festinger)**: 가치 공감형 협찬자는 "내가 진짜 좋아하는 것을 소개하는 것"이라는 일관성이 유지되어 Auth 감소가 적음. 의무적 협찬자는 "좋다고 말해야 하지만 진심은 아님"이라는 부조화 상태에서 과잉 긍정어·느낌표로 보상 → Auth 급감.
> - **사회적 정체성 이론(Social Identity, Tajfel)**: 협찬자가 해당 브랜드를 자신의 정체성 일부로 인식하면("나는 ___러버") Auth 격차가 현저히 줄어듦. 격차 10pt 이하의 사례에서 정체성 선언 패턴을 확인할 것.

> **📌 Clout 미포함 근거**
> Clout은 개별 리포트(rxr-sns-analysis)의 Psyche Layer에서 Auth/Freshness와 함께 3개 게이지로 분석된다.
> 비교 프레임워크 5축에서 Clout을 독립 축으로 분리하지 않는 이유:
> - BEI Advocacy 축에 이미 `Clout×0.6 + P-01 비율×0.4`로 반영됨
> - 독립 축으로 추가 시 Advocacy와 이중 계산되어 Clout 과대 반영 위험
> - 비교 리포트에서 Clout 데이터가 필요한 경우, 각 팝업의 Psyche Layer 요약 테이블에 포함하여 참조 가능하게 한다

### STEP 3: 팝업별 잘한 점 / 못한 점 / 시사점 분석

각 팝업에 대해 아래 구조로 분석합니다:

```
■ 잘한 점 (3~5개)
  - [지표 기반 근거] + [왜 잘 된 건지 구조적 이유]

■ 못한 점 (3~5개)  
  - [지표 기반 근거] + [왜 못 된 건지 구조적 이유]

■ 시사점 (2~3개)
  - [발견] → [이유(왜?)] → [향후 적용 방법]
```

**분석 원칙:**
- 숫자만 나열하지 않고 반드시 **"왜?"**를 설명
- 다른 팝업과의 대비를 통해 차이의 원인을 구조적으로 설명
- "잘한 점"에서도 한계를 언급 (예: "자발적 비율 98%이지만 Clout 49.8로 추천 전환은 미흡")

---

### STEP 4: 교차 비교에서 법칙 도출

5축 비교 결과를 종합하여 **재사용 가능한 법칙**을 도출합니다.

```
법칙 구조:
1. [법칙명] (한 문장)
2. [근거] — 어떤 팝업 vs 어떤 팝업의 어떤 수치 비교
3. [메커니즘] — 왜 이런 차이가 발생하는지
4. [적용법] — 향후 팝업 기획 시 어떻게 적용할 것인지
```

**도출 기준:**
- 최소 2개 이상의 팝업에서 패턴이 확인될 때만 법칙으로 격상
- 단일 팝업에서만 발견되는 특이점은 "시사점"으로 분류
- 역설(Paradox)이 있으면 별도 강조 (예: "규모 1위 ≠ 효과 1위")

---

### STEP 5: 종합 랭킹 및 핵심 역설

```
■ 축별 순위 테이블 (1위~N위)
■ 종합 순위 (모든 축 합산)
■ 핵심 역설 (Paradox) — 직관과 다른 결과들
■ 최종 공식: 팝업 마케팅 효과 = [핵심 변수1] × [핵심 변수2] × [핵심 변수3]
```

#### 디자인 가이드 (R Design Guide 적용 — 필수)

> **⚠️ 디자인 일관성 강제 규정**
> 비교 리포트도 반드시 R Design Guide를 적용한다. McKinsey 블루(`#003A70`), 컨설팅 스타일 색상, 또는 기타 외부 색상 체계 사용을 금지한다.
> 모든 RXR 리포트(개별/비교/follow-up/value)는 동일한 시각적 아이덴티티(`--primary: #6666FF`)를 유지해야 클라이언트가 복수 리포트를 받았을 때 "같은 분석 체계"임을 즉시 인식할 수 있다.

```css
:root {

#### Insight Box 필수 규칙

- 좌측 보더: `4px solid var(--primary)`
- 반드시 "왜?"에 대한 답을 포함해야 함

## 분석 원칙

### 1. "왜?" 우선 원칙
숫자를 나열하지 않는다. 모든 비교에 **"왜 이런 차이가 나는가?"**를 반드시 설명한다.

### 2. 구조적 해석 원칙
개별 팝업의 고유 사정(브랜드 인지도, 예산 등)이 아니라, **재현 가능한 구조적 원인**을 찾는다.
- 좋음: "공병 교환이라는 참여 허들이 자연 필터 역할을 하여 거품 0%"
- 나쁨: "비욘드는 LG 계열이라 충성 고객이 많아서"

### 3. 역설 강조 원칙
직관과 다른 결과(규모 최대 ≠ 효과 최고 등)를 적극적으로 발굴하고 강조한다.

### 4. 법칙 도출 원칙
- 2개 이상 팝업에서 확인된 패턴 → **법칙**
- 1개 팝업에서만 발견 → **시사점**
- 법칙은 "근거 → 메커니즘 → 적용법" 3단 구조

### 5. 공정 비교 원칙
규모(블로그 수)가 작다고 폄하하지 않고, 크다고 칭찬하지 않는다. 질적 지표 기준으로 평가한다.

### 6. 한국 소비자 맥락 해석 원칙
> **전체적 사고(Holistic Thinking, Nisbett & Masuda)**: 한국 소비자는 제품 자체가 아니라 "그 제품이 놓인 맥락 전체"를 본다. 토픽 분석에서 "공간/분위기"가 "제품/맛"보다 높은 비율을 차지하는 것은 한국 소비자의 정상적 인지 패턴이며 실패가 아니다. 맥락(장면·분위기·동행자)이 먼저 인식되고, 제품은 그 맥락 안에서 평가됨.
>
> **정체성 시그널**: 한국 소비자가 SNS에 후기를 공유할 때 묻는 핵심 질문은 "이걸 공유하면 나를 어떻게 볼까?". 취향 시그널("나는 이런 감각의 사람"), 라이프스타일 시그널("나는 이렇게 사는 사람") 기능이 강한 팝업일수록 Q5(서사형) 비율과 Advocacy가 높아짐. 이것이 "체험 인증 장치"가 Clout에 영향을 미치는 이유.

### 비밀번호 잠금 파일 사양

- 외부용 HTML 전체를 base64 인코딩 → JS chunks로 분할 저장
- SHA-256 해시로 비밀번호 검증 (기본 비밀번호: `RXR-projectrent`, 사용자 요청 시 변경)
- 잠금 화면: 퍼플 자물쇠 아이콘 + 비밀번호 입력 + 에러 시 흔들림 애니메이션
- 올바른 비밀번호 입력 시 → document.write()로 전체 리포트 렌더링
- 파일 하나로 완결 (외부 서버/CDN 불필요, 오프라인 작동)
