# 비즈니스 컨텍스트 추론 보강 — Augmentation

미팅에서 논의됐지만 입력에 명시되지 않은 비즈니스 정보를 도출해 포함한다.
"사실만 깔끔하게 나열"을 막는 단계. **전 항목 [검토] 표기 필수.**

## A. 기존 6카테고리 (운영/콘셉트/정기보고/협상/킥오프/일반)

- **A. 매출·객단가** — 목표 수치의 달성 메커니즘 (회전율, 구성 등)
- **B. 벤치마크** — 직전 임차인·인접·동급 매장 비교
- **C. 수수료·비용** — 별도 항목까지 합산한 실질 부담
- **D. 일정** — 개인·외부 일정과의 충돌 (휴가·승인·행사)
- **E. 자금·계약** — 권한 구조 (감리·결정 권한이 어디에 있나)
- **F. 레퍼런스** — 유사 사례의 성공/실패 패턴

## B. 기획업무 6카테고리 (유형 = 기획일 때)

- **A. 방향성/디렉션 (WHY)** — 콘셉트, 비전, 전략 방향, 왜 이걸 하는가
- **B. 프로젝트 골/핵심 요구 (WHAT)** — 성공 기준, KPI, 핵심 요구사항 정의, 스코프
- **C. 경쟁/포지셔닝 (WHERE)** — 마켓 포지셔닝, 경쟁사, 유사 서비스·제품, 해외 사례
- **D. 실행 가능성 (CAN)** — 이해관계자, 보유 콘텐츠·재원, 역량, 제약 조건
- **E. 실행 방안 (HOW)** — 구현 경로, 파트너·조달, 계약 구조, 일정·의존관계
- **F. 리스크/기회 (IF)** — 사업 리스크, 기회 윈도우, 평판 영향, 법률·규제

## 표기 규칙

- 추론 보강 항목은 인라인 **[검토]** 마커 또는 검토 박스로 표시
- 문서 하단에 "[검토] 표시 안내" 자동 삽입
- 외부 공유 전 [검토] 항목 일괄 점검

## 보강 금지선 (어떤 경우에도 [검토] 없이 쓰지 않는다)

1. 법적·재무적 수치 단정
2. 인물·기업에 대한 부정 평가
3. 입력에 없는 사실의 단정 서술
