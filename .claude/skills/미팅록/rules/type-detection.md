# 미팅 유형 판별 — Type Detection

## 제1원칙

입력에 이미 헤딩이 있으면 그대로 쓴다. 화자가 잡아둔 안건 구조가 있으면 새로 추측하지 않고 그 구조를 존중한다. 추측은 헤딩이 없을 때만.

## 키워드 → 유형 매핑

| 감지 키워드 | 유형 | 본문 카테고리 가이드 |
|---|---|---|
| 시공 · 도면 · 공정 · 평수 · 오픈 | **운영** | 설계·시공 구조 / 타임라인 / 브랜드 방향성 / 운영·매출 |
| Pain Points · 가설 · 콘셉트 · 포지셔닝 | **콘셉트** | 시장·환경 인식 / Pain Points / 핵심 콘셉트 / 포지셔닝 |
| 진행 상황 · 보고 · 리뷰 | **정기 보고** | 진행 상황 / 이슈 / 다음 마일스톤 |
| 단가 · 견적 · 협상 · 조정 | **협상** | 양측 입장 / 합의점 / 보류 항목 |
| 킥오프 · 오리엔테이션 · 첫 미팅 | **킥오프** | 프로젝트 개요 / 역할 분담 / 일정 / 위험 요소 |
| 사업기획 · 해외진출 · 파트너십 · 투자 · 정부사업 · 수의계약 · MOU · 빌드업 · 파이프라인 · 영업 · 제안 · 입찰 · 법인 · 진출 | **기획** | 전략 프레임 / 안건별 논의 / 파이프라인 / 아이디어 뱅크 |
| 해당 없음 | **일반** | 안건 A / 안건 B / 안건 C — 입력 흐름 순서대로 |

## 확인 프로토콜

추정 결과는 한 줄로 확인받고 진행:

```
추정 미팅 유형: 운영 미팅. 이대로 진행? (Y/n)
```

## 유형별 추론 보강 카테고리 분기

- **그 외 유형** → `augmentation.md`의 기존 6카테고리 적용
