# 톤 규칙 — Tone Guide

## 보고서 톤 유지 (가장 중요)

| X 제안서 톤 (금지) | O 미팅록 보고서 톤 (사용) |
|---|---|
| 큰 KPI 밴드로 수치 강조 | 개요 키:값 테이블 + 본문 표·인사이트 박스 |
| 대형 카피라이팅 헤드 ("~합니다") | 차분한 번호 헤더 (01 · 02 · 03) |
| 마케팅 카드 그리드 | 키:값 테이블 + 번호 매김 + 2컬럼 |
| "~합니다 · ~필수입니다" | "~확정 · ~예정 · ~필요" 명사형 |
| 사실만 깔끔하게 나열 | 사실 + 컨텍스트 + 인사이트 + 미해결까지 |

## internal / external 스위치

| 스위치 | internal | external |
|---|---|---|
| 헤더 | 미팅명 + 날짜 | 클라이언트명 + 프로젝트명 + 미팅명 + 날짜 정식 표기 |
| 톤 | "~함, ~예정" 명사형 허용 | "~합니다 / ~예정입니다" 또는 정돈된 명사형 |
| 약어 | 그대로 (PM 등) | 첫 사용 시 풀네임 병기 |
| 풋터 | 내부공유용 | 회사명 정식 표기 |
| **콘텐츠** | **동일 — 정보 누락·요약 차이 없음 (CORE 3)** | **동일** |
