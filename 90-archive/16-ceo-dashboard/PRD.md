# CEO Daily Dashboard PRD

> **한 줄 정의**: 흩어진 경영 정보를 아침·저녁 한 화면에서 훑고, 오늘 해야 할 일을 30초 안에 판단하는 1인용 대시보드

---

## 1. 배경과 문제

| 현재 상황 | 문제 |
|-----------|------|
| 결제승인·근태 → Teams (직접 연동 어려움) | 월말에야 수치 파악, 실시간 모니터링 불가 |
| 업무 메일 → Gmail | 긴급 회신 건을 놓치거나 늦게 발견 |
| 일정 → Google Calendar | 오늘·금주 일정을 별도로 열어봐야 함 |
| 프로젝트 → 엑셀(주간회의자료) | 프로젝트별 진행률·지출을 한눈에 못 봄 |
| 매장 운영 → POS 엑셀 보고 | VOC·재고·매출 이슈를 파일 열어야 확인 |

**핵심 Pain**: 5개 이상의 소스를 각각 열어봐야 하고, "지금 내가 뭘 먼저 해야 하지?"를 매번 머릿속으로 조합해야 함.

## 2. 의사결정 맵 (Decision-First)

이 대시보드를 통해 내리는 **핵심 결정 5가지**:

| # | 결정 | 판단 기준 | 액션 |
|---|------|-----------|------|
| D1 | 지금 처리해야 할 긴급 건이 있는가? | 정기업무 D-day + 미회신 메일 우선순위 + 최근3일 메일 리뷰 | 즉시 처리/회신 or 위임 |
| D2 | 오늘·금주 놓치면 안 되는 일정은? | 캘린더 일정 + 마감 임박 태스크 | 일정 조정 or 준비 |
| D3 | 매장에 즉시 대응할 이슈가 있는가? | 일매출 전일비·전주비 이상, VOC 건수, 재고 경고 | 매장 연락 or 지시 |
| D4 | 프로젝트가 예산·일정 내에 있는가? | 진행률 vs 지출률 괴리, 오늘 논의 사항 | 회의 안건 조정 |
| D5 | 근태·인건비 이상 징후는? | 초과근무 비용 vs 예산, 연차 사용률, 잔여연차 | 근무 조정 지시 |

### 3.2 블록별 상세

#### [A] 긴급 액션 패널 — "지금 해야 할 것"
- **데이터 소스**: Gmail API (업무요청 메일), Google Calendar API (정기 업무), 수동 입력(승인 건)
- **표시 항목**:
  - **정기 업무 알림**: 임대료 송금, 세금 납부 등 캘린더에 등록된 정기 반복 업무 (D-day 카운트)
  - **업무요청 메일 리뷰 (최근 3일)**: 발신자, 제목, 수신일, 회신 여부 표시
  - **미회신 메일 우선순위**: 경과시간 + 발신자 중요도 기반 자동 정렬 (긴급→일반→참고)
  - 승인 대기 건: 건명, 금액, 요청일
- **우선순위 로직**:
  - 정기 업무 D-0 ~ D-1 → 빨간색 (즉시 처리)
  - 미회신 48h 초과 → 빨간색 / 24~48h → 노란색 / 24h 이내 → 초록색
  - 경영진·주요 거래처 발신 → 우선순위 1단계 상향
- **시각화**: 카드 리스트, 우선순위별 색상 에스컬레이션
- **인터랙션**: 메일 클릭 시 Gmail로 이동, 정기 업무 클릭 시 캘린더로 이동

#### [B] 오늘·금주 일정 — "오늘 어디 가야 하고 뭘 준비해야 하나"
- **데이터 소스**: Google Calendar API
- **표시 항목**:
  - 오늘 일정 타임라인 (시간순)
  - 금주 남은 주요 일정 (요약)
  - 마감 임박 태스크 (3일 이내)
- **시각화**: 타임라인 + 뱃지
- **인터랙션**: 클릭 시 Google Calendar로 이동

#### [C] 매장 현황 — "매장에 문제 있나?"
- **데이터 소스**: POS 엑셀 업로드 (일 1회), 매장별 업무요청 (메일/수동 입력)
- **표시 항목**:
  - **매출 종합 테이블** (매장별 한 행):
    | 매장명 | 전일 매출 | 전주 매출(동요일) | 지난달 매출(동일자) | 전일비 ▲▼ | 전주비 ▲▼ | 전월비 ▲▼ |
  - **전체 합산 매출** + 월 목표 대비 달성률
  - **매출 변동 알림**: 전일비·전주비·전월비 중 ±15% 이상 변동 시 자동 하이라이트 + 변동 사유 메모란
  - **이슈 사항**: VOC 건수(신규/미처리), 재고 경고(부족 품목), 매장 특이사항
  - **매장별 업무요청**: 각 매장에서 올라온 요청 사항 리스트 (요청일, 내용, 긴급도, 처리상태)
- **시각화**:
  - 매장별 카드: 3기간 매출 비교 바 + 스파크라인(7일 추이)
  - 변동 이상치 빨간 하이라이트, 업무요청 건수 뱃지
- **판단 로직**: 매출 ±15% 이상 변동 → 노란불, ±30% 이상 → 빨간불
- **비교 기준**: 전일비, 전주 동요일비, 전월 동일자비, 월 목표 대비

#### [D] 프로젝트 현황 — "프로젝트가 괜찮은가?"
- **데이터 소스**: 엑셀 업로드 (주간회의자료)
- **표시 항목**:
  - 프로젝트명, 진행률(%), 지출률(%), 상태 신호등
  - 오늘 논의 사항 리스트
  - 다음 마일스톤 및 D-day
- **시각화**: 프로그레스 바(진행률 vs 지출률 이중 바), 신호등 아이콘
- **판단 로직**: 지출률 > 진행률+10%p → 빨간불

#### [E] 근태 관리 — "근태·인건비 괜찮은가?"
- **데이터 소스**: 근태 엑셀 업로드 (월 1~2회, 또는 월말)
- **표시 항목**:
  - 월 누적 초과근무·야근·특근 시간 합계 + 예상 수당
  - 연차 사용 현황: 직원별 사용일수 / 총 부여일수 / **잔여연차**
  - 연차 사용률 (팀·매장별)
  - 예산 대비 초과근무 비용 게이지
  - 매장매출 대비 인건비 비율 (매장별)
- **시각화**: 게이지 차트(초과근무 예산) + 연차 잔여 바 차트(직원별) + 매장별 비율 바 차트
- **판단 로직**: 잔여연차 3일 이하 → 노란불, 초과근무 예산 90% 초과 → 빨간불
- **비교 기준**: 월 예산 대비, 전월 동기 대비

## 6. 기술 구현 방향

### 6.1 아키텍처
```
[엑셀 파일]  →  업로드/파싱  ─┐
[Gmail API]  →  메일 조회    ─┤→  JSON 데이터  →  HTML 대시보드
[Calendar API] → 일정 조회   ─┘
```

### 6.2 기술 스택
- **프론트엔드**: HTML + CSS + Vanilla JS (단일 파일, 경량)
- **데이터 처리**: Python 스크립트 (엑셀 파싱 → JSON 변환)
- **API 연동**: GWS CLI (Gmail, Calendar)
- **호스팅**: 로컬 파일 or 간단한 로컬 서버

### 6.3 MVP 범위 (Phase 1)
1. **Gmail 미회신 메일** 패널 (API 연동)
2. **Google Calendar 오늘 일정** 패널 (API 연동)
3. **매장 매출** 패널 (엑셀 업로드)
4. 전체 HTML 대시보드 레이아웃

### 6.4 Phase 2
5. 프로젝트 현황 패널 (엑셀 업로드)
6. 근태 관리 패널 — 초과근무 + 연차 현황 (엑셀 업로드)
7. 매장매출 vs 인건비 비교 뷰

### 6.5 Phase 3 (선택)
8. 승인 대기 수동 입력 기능
9. 알림/하이라이트 규칙 커스터마이징
10. 모바일 반응형
