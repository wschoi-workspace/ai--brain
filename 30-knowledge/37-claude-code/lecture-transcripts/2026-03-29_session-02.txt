## 2. 슬래시 커맨드 (Slash Commands) = 포스트잇

- 자주 쓰는 긴 지시를 파일 하나로 저장해두고 명령어로 즉시 호출
- 핵심: 슬래시(/)를 쳐야만 실행됨
- 예시:
  - /todo → 할 일 추가
  - /meeting → 미팅 노트 생성
  - /daily-review → 주간 정리 자동 생성
- MD 파일 하나에 Claude에게 시킬 말을 미리 저장해놓으면 끝

## 3. 스킬 (Skills) = 업무 매뉴얼 / SOP

- 스킬은 SOP(Standard Operating Procedure) = 표준화된 작업 지시서
- 슬래시 커맨드와의 차이: 스킬은 "알아서 판단 후 자동 호출"
- Model Invoked: 특정 단어만 감지되면 자동 실행 (예: "노션" 언급 시 노션 스킬 자동 호출)
- User Invoked: 사람이 직접 슬래시로 호출해야 실행

### 스킬 구조
- 폴더 이름 = 스킬 이름
- 핵심 파일: skill.md (어떤 상황에서 무엇을 하는지 명세)
- 참조 파일: 템플릿, 코딩 스크립트 등
- Description 부분만 Claude 컨텍스트에 올라감 (가벼움)

### 에스프레소 비유
- 포터필터 → 무게 → 도징 → 그램 체크 → 18.0g±0.1g → 탬핑 → 장착 → 추출
- 이처럼 절차가 순서대로 명세된 것이 스킬

## 4. 서브에이전트 (Sub-agents) = 전문 팀원

- 특정 역할만 하는 전문 에이전트
- 핵심: 메인 에이전트의 컨텍스트와 별도의 컨텍스트에서 작동
- 메인 에이전트에서 하지 않고 각각 다른 방을 만들어 조사 → 결과만 가져옴
- 병렬 처리할 때만 사용
  - 예: 성수동, 약수동, 오금동을 각각 별도 에이전트가 동시 조사
- 메인 컨텍스트 오염 방지 + 병렬 처리 가능

### 에이전트 폴더 구조
- .claude/agents/ 안에 MD 파일 하나만 있으면 됨
- research-worker: 외부 정보 수집/검증/의사결정용 팩트 전달

## 5. Sources of Truth (신뢰의 근거)

- AI는 정확한 룰 없이 만들면 허술하게 만들 때가 있음
- 신뢰할 만한 근거를 바탕으로 작업하라고 지시하는 것이 정석
- 공식 문서(skills-guide.md, subagents-guide.md 등)를 SOT로 활용
- 스킬/에이전트 만들 때: "@스킬가이드.md를 참고해서 만들어줘"

## 6. Git 연동 (GitHub)

- 세이브 포인트 개념: 게임처럼 중간 저장 → 문제 시 롤백 가능
- 비개발자는 매일 commit & push 하는 것이 좋음
- PC 간 동기화: push(올리기) → 다른 PC에서 pull(내려받기)
- GitHub CLI(gh) 설치하면 터미널에서 레포지토리 생성까지 가능
- 가입 시 퍼즐 보안이 매우 어려움 (봇 방지)

## 7. GWS CLI (Google Workspace CLI) 설치

- 지원 서비스: Gmail, 캘린더, 드라이브, 시트, 슬라이드, 챗 등
- 설치 과정이 가장 어려운 부분 (Google OAuth 설정)
- 핵심 난관: Google Cloud Console → 프로젝트 생성 → API 활성화 → OAuth 동의 화면
- 한 번 세팅하면 Claude Code에서 모든 구글 서비스 핸들링 가능
- 안 될 때마다 화면 긁어서 Claude에게 물어보면 해결 가능

## 8. 텔레그램 연결

- 카카오톡보다 개인 전용 용도로 텔레그램이 적합
- 파일 저장용 채널, 링크 수집용 채널(inbox collector) 등 활용
- PC 제한 없이 멀티 로그인 가능
