## 1. Claude AI 디자인 도구 (claude.ai/design)

### 기능 소개
- 디자인 시스템 자동 생성: 기존 웹사이트 코드를 분석하여 컬러 팔레트, 폰트, 컴포넌트 등 추출
- 골든 레이션 스케일(황금 비율)까지 자동 계산
- PRD 파일 업로드 → 디자인 자동 생성
- 슬라이드 덱, 프로토타입, 카드뉴스 제작 가능
- HTML 핸드오프: "Handoff to Claude Code" 버튼으로 코드 직접 전달

### 활용 사례
- 미팅 코치 앱: 발화자 목소리 실시간 타이핑 + 회의 맥락 분석 알림
- 브랜드 디자인 시스템 기반 캐러셀/슬라이드 자동 생성

## 2. 플레이라이트 (Playwright) 심화

### 핵심 개념
- 사람 없이 크롬을 조작하는 도구 (Microsoft 제작)
- Headless: 창 없이 백그라운드 동작
- 실제 크롬과 동일하게 웹 렌더링 → 픽셀 단위 스크린샷

### 3대 핵심
1. Headless 동작 (백그라운드)
2. Wait 명령 (페이지 로딩 시간 대기)
3. 스크린샷 캡처

### 활용 범위
- 크롤링: 페이지 텍스트/이미지/링크 추출
- 브라우저 자동화: 로그인, 폼 채우기, 버튼 클릭, 다운로드
  - 모두사인 계약서 발행 자동화 가능
  - ERP/내부 시스템 자동화 (API 없어도 가능)
- e2e 테스트: 웹앱 전체 동작 테스트 자동화
- 반복 작업: 100개 URL 순회하며 동일 작업
- 이미지 생성: HTML → 스크린샷 → PNG (캐러셀, 슬라이드, 명함 등)

### HTML 기반 OSMU (One Source Multi Use)
- HTML + CSS로 콘텐츠 제작 → 플레이라이트로 스크린샷
- 캐러셀(1080x1350px), 슬라이드, PDF 등 다양한 포맷 출력
- 템플릿 유형별로 만들어놓으면 텍스트+이미지만 바꿔서 자동 생산

### 명함 제작 사례
- HTML로 명함 디자인 → 인쇄용 PDF 생성
- 상원애드피아 가이드 스크린샷 → Claude에게 제공 → 재단선/도련/해상도 자동 처리
- 디자인 툴 없이 인쇄 가능한 명함 완성

## 3. Buffer를 통한 SNS 자동 배포

### Buffer란?
- SNS 통합 관리 도구: 인스타그램, 스레드, 링크드인, 페이스북, 유튜브, 틱톡, 트위터 등
- 하나의 플랫폼에서 모든 채널에 동시 배포
- 예약 배포 가능

### Claude Code와 연동
- Buffer API 키 발급 (developer API 사이트)
- API 키를 환경변수로 저장
- Claude Code에서 콘텐츠 작성 → Buffer를 통해 모든 SNS에 자동 배포
- PostKitty 앱에 "Buffer" 버튼 추가하여 원클릭 퍼블리싱

## 4. 센스메이킹 (Sensemaking) 스킬

### 목적
- 낯선 콘텐츠(플러그인, 논문, 프레임워크, 기술 문서)를 함께 해석
- AI와 협업할 수 있는 수준의 이해에 도달하는 도구
- "인지 외주화(Offloading)"가 아닌 "인지 증강(Augmentation)"

### 동작 원리
1. 사용자 맥락 수집: CLAUDE.md 읽고 사용자의 이해 수준 파악
2. 콘텐츠 분석: URL/파일/텍스트 제공
3. 갭 분석: 사용자 수준과 콘텐츠 요구 수준 사이의 차이 파악
4. 진입 장벽 낮추기: 비유로 설명, 전체 맥락 연결

### 실전 사례: Taste Skill 분석
- GitHub에서 화제가 된 프론트엔드 UI 디자인 스킬 분석
- 센스메이킹으로 뜯어보니: 코드 실행 로직 없이 프롬프트 규칙 모음이었음
- 발견한 설계 패턴:
  1. 금지 패턴 리스트업: "하지 마"를 명확하게 정의 (Hard/Medium/Soft)
  2. 다이얼 시스템: 1~10 변수로 행동 강도 조절
  3. Pre-flight 체크리스트: 작업 완료 전 AI가 스스로 검증
  4. 페르소나 정의
- 이 패턴들을 Skill Creator 스킬에 적용하여 업그레이드

## 5. 플러그인 & 마켓플레이스

### 플러그인 설치

## 6. 인바운드 리플라이 자동화 (강사 실전 Use Case)

### Reply Check 스킬
- Gmail에서 회신해야 할 메일 자동 분류
- "회신 필요" 빨간 라벨 자동 부착
- 이미 회신된 것은 자동 라벨 제거

### Inbound Reply 스킬 (가장 많이 깎인 스킬)
- 문의 유형별 템플릿:
  - 탐색형: "어떤 교육인가요?"
  - 구체적 요청형: "금액은 얼마인가요?"
  - 레퍼런스 경유형: "아는 분이 소개해줬는데"
  - 프리랜서/개인 모집형
  - 소규모 팀 문의
- 핵심 원칙: 상대의 톤에 맞추기 (미러링)
- CTA 변형: 유형별로 콜투액션 다르게 설정
- 캘린더 크로스체크 → 가능한 시간대 자동 제안
- 첨부 자료 경로 미리 지정 → 자동 첨부 발송

### 깎기 히스토리
- 3월 7일부터 지속적으로 업데이트
- 새로운 문의 유형이 나올 때마다 템플릿 추가
- 초안 확인 → 간단 수정 → 발송

## 7. 센스 익스트랙터 (Sense Extractor) - 최초 공개

### 목적
- 대화 세션에서 사용자의 구조적 사고 패턴을 추출
- 메타레이어 에이전트에 주입하여 "나처럼 판단"하게 만들기

### 추출 카테고리 (4가지)
1. 구조적 판단: 왜 이 방향으로 풀어야 하는가
2. 사고의 프레임워크: 어떤 틀로 생각하는가
3. 설계 원칙: 어떤 기준으로 만드는가
4. 누락 감지: 구조적으로 빠진 것은 무엇인가

### 핵심 관점
- 감정적 선호(좋다/싫다)가 아닌 구조적 사고를 추출
- 인지심리학 기반으로 메타인지 프레임 설계
- 궁극적 목표: 나의 판단 방식을 에이전트가 학습

## 8. 핵심 메시지

- 새로운 툴을 알게 되면 "기존 스킬에 반영할 만한 게 있을까"부터 생각하기
- 스킬 깎기: 사용할수록 점점 내 것이 되어가는 과정
- 인지 외주화(Offloading) vs 인지 증강(Augmentation): AI와 함께 이해하는 게 핵심
- 프로그레스 파일: 모든 프로젝트 폴더에 진행 상황 기록 → 맥락 유지의 핵심
