## 1. 오픈클로(OpenClaw) + ACP 연동

### 오픈클로란?
- 자율성 높은 AI 에이전트 플랫폼
- Claude Code와의 차이: 자율성의 차이
  - Claude Code: 내가 계속 컨텍스트를 주면서 함께 작업
  - 오픈클로: "알아서 해봐" 식으로 자율적 실행

### 현재 구조
- 오픈클로에 ChatGPT 구독 계정 연결 (유일하게 구독 계정으로 붙을 수 있는 모델)
- ChatGPT가 첫 쿠션 역할 → ACP(Agent Communication Protocol)로 로컬 Claude Code 호출
- `claude -p` 옵션으로 백그라운드에서 Claude 실행
- 보리(에이전트): ChatGPT 뇌로 알아듣고, Claude가 실행

### 중요 룰
- 오픈클로에게 PKM(핵심 작업 폴더) 직접 수정 금지 설정
- 대신 AIQ(큐) 채널을 통해 Claude Code에 전달 → Claude Code가 반영

### 제한사항
- Anthropic이 구독 계정으로의 오픈클로 접속을 차단 (트래픽 문제)
- API로 쓰면 비용 폭발 → 어쩔 수 없이 ChatGPT 경유
- Gemini도 차단됨

## 2. Claude Code 잘 쓰는 법 (가장 중요한 팁)

### Tip 1: 한 번에 다 하려고 하지 마라
- "매출 데이터 분석해서 대시보드 만들어줘" → 잘 안 됨
- 올바른 방법: 데이터 쪼개기 → 구조 파악 → 차트 생성 → 대시보드 조립
- 작은 단위로 쪼개서 시키는 게 훨씬 정확함

### Tip 2: 프로그레스 파일 만들기 (가장 큰 팁)
- 모든 프로젝트 폴더에 `[번호]-progress.md` 파일 생성
- 내용: 날짜별 작업 내역, 메일 송수신 기록, 액션 아이템 체크리스트
- 실제 사례: 일룸 프로젝트 프로그레스 파일 700줄 이상
  - 2025.11.29 첫 미팅 → 비용 발송 → 수신 확인 → 각 세션 기록
- 다음 세션에서 프로그레스 파일만 읽고 이어서 작업 가능
- 데일리 노트 + 프로그레스 파일 동시 업데이트 스킬 활용

### Tip 3: 클리어를 두려워하지 마라
- 대화가 길어질수록 품질이 확실히 떨어짐
- 3번 룰: 같은 문제를 3번 시도해도 안 되면 → 클리어
- 클리어 전: 프로그레스 파일에 "안 된 상황"만 기록
- 클리어 후: 새 세션에서 프로그레스 읽고 근본적 해결 시도
- "클리어는 리셋이 아니라 새 출발이다"

### Tip 4: "해줘" 다음에 "확인해줘"
- Boris Cherny(Claude Code 개발자)가 꼽은 최고 팁
- 해줘로 끝내지 말고, 반드시 확인 요청까지
- "원본 데이터와 합계가 일치하는지 확인해줘"
- "요구사항을 모두 충족하는지 빠진 거 없는지 체크해줘"
- 이것만으로도 성공 확률이 비약적으로 상승

### Tip 5: 엔트로픽 공식 초보자 5대 실수
1. Kitchen Sink 세션: 한 세션에서 여러 무관한 작업 혼합
2. 반복 도정: 같은 실수를 계속 고치려고 시도 (클리어가 답)
3. Claude Code 과부하: CLAUDE.md 과다 적재
4. 검증 부재: "해줘"만 하고 확인 안 함
5. 무한 동작: 끝없이 대화 이어가기

### Tip 6: 플랜 모드 습관화
- 새로운 작업, 여러 파일 동시 조정, 결과물 구조 잡을 때 필수
- 수정 작업에는 불필요하지만, 새 기능 추가 시에는 반드시 사용
- Shift+Tab으로 토글

## 4. 유즈케이스 상세

### Doc Updater 스킬
- 공식 문서(SOT)가 시간이 지나면 죽은 문서가 됨
- 주기적으로 공식 문서를 최신 버전으로 자동 업데이트

### Contract Docs 스킬 (견적서/계약서)
- HTML로 생성 → A4 PDF 인쇄 최적화
- 법인 인감 이미지 자동 삽입
- 세금계산서 자동 발행도 브라우저 자동화로 가능성 있음

### Proposal Draft 스킬 (제안서 초안)
- 포지셔닝에 따른 제안서 자동 생성
- 제안서 → 견적서 → 계약서 파이프라인

### Reply Check + Inbound Reply 스킬
- Gmail 라벨링으로 "회신 필요" 자동 표시
- 유형별 회신 템플릿 + 캘린더 연동 + 첨부파일 자동화

### Sense Extractor 스킬 (최초 공개)
- 대화에서 구조적 판단 패턴 추출 → 메타레이어 에이전트에 주입
- 4가지 추출: 구조적 판단, 사고 프레임워크, 설계 원칙, 누락 감지
- 궁극적 목표: "나처럼 판단하는" 에이전트 구축

## 5. Verifiable vs Non-verifiable

### 핵심 원칙
- AI에게 시키는 것: Verifiable(검증 가능한 것)만 시킴
  - 답이 있는 것, 되는지/안 되는지 확인 가능한 것
- AI에게 시키지 않는 것: Non-verifiable(취향, 감각, 주관적 판단)

### 왜 코딩이 가장 먼저 AI에 정복되었나?
- 코딩 = 가장 Verifiable한 영역 (되/안 돼, 통과/불통과)
- RLVF(Reinforcement Learning with Verifiable Rewards)
  - 기존 RLHF(Human Feedback)에서 진화
  - 검증 가능한 보상으로 AI 강화학습 → 못 풀던 것을 풀게 됨

### 다음 정복 영역: 과학
- 실리콘밸리 인재들이 CS에서 Science로 이동 중
- 재료공학 무인 실험실: AI가 소재 실험 → 전도체 여부 자동 검증
- AlphaFold: 단백질 접기 → 노벨 생물학상
- 미래: 생명공학 전공 없이도 신약 개발 가능한 시대

### Ralph Loop (랄프 루프)
- 답이 있지만 어떻게 풀지 모를 때 사용
- 목표 설정 → 검증 가능한 메트릭 만들기 → AI에게 시도 → 실패 기록 → 다른 방법 시도 → 반복
- 자기 전 토큰 남은 시간에 랄프 루프 돌리고 자는 문화

## 6. Learn과 Unlearn

### AI 시대의 무기력함
- 번역가: 인생이 송두리째 부정당하는 기분
- 기록학 전공자: 10년 커리어가 AI로 대체 가능해짐
- 텍스트 분석/분류: AI가 극소 비용으로 80-90% 수준 달성

### Unlearn의 중요성
- Learn: 새로운 것을 배우는 것
- Unlearn: 배웠던 것을 놓는 것 (버리는 것이 아닌 기회비용 관점)
- 도구가 바뀌어도 과정과 사고방식은 바뀌지 않음

### 일은 명사가 아니라 동사
- "커피와 디저트"가 아니라 "경험을 선물하는 일"
- 기록학 → 기록을 정리하는 일이 아닌, 그 감각과 센스의 동사적 표현
- 도메인 지식은 AI가 대체해도, 감각과 의도는 대체 불가

### "그거 AI 쓰면 되는 거 아니에요?"에 대한 답변
- "그러면 그냥 유저밖에 안 되는 거예요. 돈 내고 쓰세요."
- 직접 만드는 사람 vs 쓰기만 하는 사람의 차이는 기회 창출 능력
