[핵심 주제: AI 시대 철학, 메타레이어, 실제 스킬 활용(급여 자동화), 스킬 깎기]

## 1. 도망자연합과 AI 시대 철학

### 노정석 (빅팩토리 CEO) 발표 내용
- 프론티어랩(OpenAI, Claude, Gemini 등)에 의해 모든 것이 점령될 것 같은 위기감
- 인간에게 남는 것: 의지, 취향, 결정, 책임, 실행
- 능력으로 경쟁하려는 습관을 버려야 함
- AI가 모든 데이터를 분석해서 최적의 진단을 내려도, 결정은 인간만 할 수 있음
  → 책임은 지능에서 나오는 게 아니라 인간 장에서 나오기 때문

### 창발(Emergence) 현상
- 원자 → 분자: 수소와 산소는 물의 "축축함"을 이해할 수 없음
- 단백질 → 생명: 하위 차원이 상위 차원을 이해할 수 없음
- AI의 In-context Learning도 창발 현상일 수 있다는 가설
- 인간(N차원)이 AI(N+1차원)를 이해하기 어려울 수 있음

### 공생(Symbiosis) 관점
- 20억 년 전 미토콘드리아가 세포 안으로 들어와 에너지 폭발 → 다세포 진화
- AI를 타자로 두지 않고 "우리 안으로 품는 선택"을 할 수 있는 시대
- 결론: "받아들여. 인생의 주도권 갖고 살아."

### AI 시대 안전한 직업 = "사업가"
- 실패 확률 90%든 99%든 "그냥 하겠다"는 의지를 가진 사람
- "취직할 필요가 없는 사람들만 취직에 성공할 것"
- 의도를 전달하면 AI가 해결 경로를 추천 → 결정은 직접 내려야 함

## 2. 메타레이어 시스템 (내부 프로젝트)

- 의도를 전달하면 → 메타레이어가 분석 → 캐스케이드로 실행단에 전달
- 실행단이 작업 → 검증 레이어에서 체크 → 결과 보고서 상향 보고
- 목표: 디테일한 체크를 따로 하는 레이어를 만들어 의도만 전달하면 되는 시스템
- 밤새 루프 돌려서 인수 조건 분석, 루프 프루프 10회 실행, 적용 검토

## 3. 급여 관리 자동화 (Payroll 스킬) - 실전 데모

### 배경
- 오프라인 매장 4개, 직원 약 15명
- 매달 노무사한테 월 20만 원 지불 → 커뮤니케이션 리소스가 너무 큼
- 노무사 계약 해지 후 Claude Code로 전부 자동화

### Payroll 스킬 동작 과정
1. 점장 메일에서 급여 변동 내용 수집 (최근 5일간 검색)
2. 급여변동.md 작성 (메일 내용 + 투두 정리)
3. 급여 대장 엑셀 생성 (Python 스크립트 → 다중 시트, 합계, 일용직 포함)
4. Google 시트에 저장 (데이터베이스 유지)
5. 세무사 메일로 자동 발송 (첨부파일 포함)

### 핵심 포인트
- 점장들의 비정형 메일(카톡, 이메일)을 AI가 정리해서 MD 파일로 구조화
- 세무사 요청에 따라 시트 분리, 일용직 별도 관리 등 지속 업데이트
- 처음엔 오류 가능성 → 세무사에게 양해 구하고 → 점진적 정확도 향상

## 4. 스킬 설계 원칙

### 잘게 쪼개기 + 오케스트레이션
- 리서치 → 분석 → HTML 생성 → PDF 변환: 각각 별도 스킬로 만들기
- 총괄 스킬 하나가 개별 스킬들을 순차 호출하는 구조
- 한 번 만들었다고 끝이 아니라 계속 "깎아가는" 것이 핵심

### 스킬 업데이트 사이클
1. 스킬 사용 → 결과물 확인
2. 아쉬운 부분 발견 → 대화로 수정
3. "이 대화 과정을 반영해서 스킬 업데이트해" 지시
4. 스킬이 점점 내 요구에 맞게 깎여감

### 깎는다는 의미
- 처음 만든 스킬 ≠ 최종 스킬
- 반복 사용 → 피드백 → 업데이트의 반복
- 결과물을 만든 뒤 → 그 결과물이 나오도록 스킬을 업데이트하는 게 핵심

## 5. 참여자 Use Case 공유

- Thinking Partner를 활용한 아이디어 정리 (혼자 고민 → 놓치는 포인트 점검)
- 술 사진 → OCR 분석 → 웹 검색 → Slack 전송 → 확인 후 Notion 저장 자동화
- 제안서 작성: Thinking Partner로 3-4주 분량 디스커션을 수시간으로 압축
- 트랜스크립트 오거나이저로 녹음 텍스트 정리 → HTML 출력

## 6. 플레이라이트(Playwright) 소개

- Microsoft가 만든 웹 브라우저 자동화 도구
- 사람 없이 크롬을 조작: 페이지 열기, 클릭, 텍스트 선택, 스크린샷 등
- HTML → 플레이라이트로 스크린샷 → PNG 이미지 출력 가능
- 캐러셀(카드뉴스), 슬라이드, 명함 등 제작에 활용

## 7. 핵심 메시지

- AI에게 다 시키는 "인지 외주화"가 아닌, 함께 이해하는 "인지 증강"이 중요
- 극단적 필요성(노무법인 커뮤니케이션 스트레스)이 최고의 동기부여
- 스킬은 인턴을 가르치는 마인드로 만들면 된다
- 의도를 전달하는 시대: AI가 해결 경로를 제안, 인간이 결정
