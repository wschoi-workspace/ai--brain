# 하이브리드 프레젠테이션 가이드
## 클라이언트 제안 PT 실전 매뉴얼

> 7대 프레임워크 × 8개 전설적 사례에서 추출한 감성 설득 구조.
> 클라이언트 제안 PT에 특화. "설명"이 아닌 "설득"을 위한 슬라이드 설계법.

---

# 전체 구조: 10단계 하이브리드 공식

기존 8단계 하이브리드 공식에 **시장 문제 분석**과 **차별화 강점 어필**을 통합한 클라이언트 제안 PT 전용 구조.

```
Phase A — 공감과 맥락
  [1] WHY 선언          "우리는 ___을 믿는다"
  [2] Big Shift         "세상이 이렇게 변하고 있다"
  [3] 시장 문제 해부     현상 → 원인 → 구조적 한계 → 방치의 대가

Phase B — 비전과 해법
  [4] Promised Land     제품/서비스 없이 "도달해야 할 미래"
  [5] What Is ↔ Could Be  현실 고통 ↔ 이상 가능성 교차
  [6] Wow Moment        1개의 감정적 클라이맥스

Phase C — 우리만의 답
  [7] 차별화 강점 어필   왜 우리만 이것을 해결할 수 있는가
  [8] Magic Gifts       기능을 약속의 땅 도달 도구로 프레이밍

Phase D — 행동 유도
  [9] Success ↔ Failure  행동하면/안 하면
  [10] New Bliss         변화된 세계의 비전으로 마무리
```

---

# Phase A — 공감과 맥락

## Step 1. WHY 선언 (1장)

> 출처: Simon Sinek — Golden Circle

### 원칙
- 프레젠테이션의 첫 슬라이드는 "우리 회사 소개"가 **아니다**
- 우리의 신념, 존재 이유, 이 프로젝트에 임하는 철학을 선언
- 변연계(감정+의사결정)를 먼저 자극 → 이후 논리에 수용적 상태

### 실행 가이드
| 요소 | 내용 |
|------|------|
| 슬라이드 수 | 1장 |
| 텍스트 | 한 문장. 최대 15단어 |
| 비주얼 | 풀 블리드 이미지 또는 미니멀 타이포 |
| 톤 | 선언적. 겸손하되 확신 |

### 공식
```
"우리는 [신념/가치]를 믿습니다."
"[대상]이 [이상적 상태]에 도달할 수 있다고 확신합니다."
```

### 예시
- "우리는 모든 공간이 미디어가 될 수 있다고 믿습니다."
- "소비자의 경험이 브랜드의 유일한 자산이라고 확신합니다."

### 안티패턴
- ❌ "안녕하세요, 저희 회사는 2015년에 설립되어..."
- ❌ 회사 연혁 타임라인
- ❌ 조직도

---

## Step 2. Big Shift — 맥락 설정 (1~2장)

> 출처: Andy Raskin — 5-Part Strategic Narrative

### 원칙
- 우리 제품/서비스가 아닌 **세상의 변화**로 시작
- 클라이언트가 이미 느끼고 있는 변화를 명명
- "당신의 제품을 믿으라"가 아닌 "이 변화를 인정하라"

### 실행 가이드
| 요소 | 내용 |
|------|------|
| 슬라이드 수 | 1~2장 |
| 구조 | 변화의 명명 → 데이터/사례로 증명 |
| 데이터 | 1개의 충격적 수치 + 맥락화 |
| 톤 | 객관적이되 긴박한 |

### 공식
```
"[산업/시장]에 [거부할 수 없는 변화]가 일어나고 있습니다."
+ 이 변화를 증명하는 1개의 숫자/사례
```

### 예시
- "오프라인 리테일의 역할이 '판매'에서 '경험'으로 전환되고 있습니다."
- "소비자의 72%가 '브랜드 경험'이 구매 결정의 1순위라고 응답했습니다."

### 안티패턴
- ❌ 업계 상식의 반복 ("디지털 전환이 가속화되고 있습니다")
- ❌ 너무 많은 데이터 (1개면 충분. 3개 이상은 기억 안 됨)

---

## Step 3. 시장 문제 해부 (3~5장)

> 출처: Donald Miller SB7 (3중 Problem) + Andy Raskin (Winners & Losers) + Carmine Gallo (Villain First)

이 단계가 제안 PT의 **승부처**. 클라이언트의 문제를 그들보다 더 깊이 이해하고 있음을 증명하는 구간.

### 원칙
- 문제를 3개 층위로 해부: 현상 → 원인 → 구조적 한계
- 빌런(적)을 명확히 설정 — 경쟁사가 아닌 **구조적 문제/관행**이 빌런
- 방치의 대가를 보여줌 (Loss Aversion)
- 클라이언트가 "맞아, 바로 이게 우리 문제야"라고 느끼게

### 4단계 해부 구조

#### 3-1. 현상 진단 (1장)
```
"현재 [시장/업계]에서 일어나고 있는 일"
```
- 클라이언트가 이미 체감하는 표면적 증상
- 데이터 또는 사례로 객관화
- Seth Godin 방식: 통계표가 아닌 **감정적 증거** (사진, 스크린샷, 실제 사례)

#### 3-2. 원인 분석 (1장)
```
"왜 이런 일이 벌어지는가"
```
- 표면 아래의 구조적 원인 규명
- **3중 Problem 구조 적용:**

| 층위 | 질문 | 예시 (공간기획 제안) |
|------|------|---------------------|
| External | 표면적 문제? | 매장 방문객 감소, 체류시간 단축 |
| **Internal** | **감정적 문제?** | **"우리 브랜드가 잊혀지고 있다는 불안"** |
| Philosophical | 왜 이것이 부당한가? | "좋은 브랜드가 공간 경험 하나 때문에 저평가되어서는 안 된다" |

> **핵심**: External은 의식적이지만, **Internal(감정)이 실제 구매 동기**. 클라이언트의 감정적 고통을 정확히 짚어야 한다.

#### 3-3. 구조적 한계 (1장)
```
"기존 접근 방식으로는 해결할 수 없는 이유"
```
- 현재 업계의 일반적 해결책이 왜 불충분한지
- Raskin의 Winners & Losers: 변화에 적응한 자 vs 못한 자 대비
- **빌런 = 낡은 관행/방법론** (경쟁사를 직접 공격하지 않음)

#### 3-4. 방치의 대가 (1장)
```
"이 문제를 방치하면 어떤 일이 벌어지는가"
```
- Loss Aversion 활용: 얻는 것보다 잃는 것이 2배 강력
- Miller의 Failure 요소: 구체적이고 체감 가능한 부정적 미래
- **"소금처럼"** — 너무 많으면 공포 마케팅, 너무 적으면 긴장감 없음

### 슬라이드 설계 원칙
| 원칙 | 실행 |
|------|------|
| 감정적 증거 우선 | 데이터표 < 실제 고객 후기, 매장 사진, 소비자 반응 캡처 |
| 클라이언트의 언어 사용 | RFP/미팅에서 클라이언트가 쓴 단어를 그대로 인용 |
| 숫자의 인간화 | "매출 15% 하락" < "10명 중 1.5명이 다시 오지 않는다" |
| 대비 활용 | Before/After, 승자/패자, 이상/현실 나란히 |

### 안티패턴
- ❌ 문제를 클라이언트 탓으로 돌리기 (빌런은 구조/관행이지 클라이언트가 아님)
- ❌ 너무 많은 문제 나열 (3개 이하. Rule of Three)
- ❌ 문제 분석 없이 바로 솔루션으로 점프

---

# Phase B — 비전과 해법

## Step 4. Promised Land — 약속의 땅 (1~2장)

> 출처: Andy Raskin — 5-Part Narrative + Nancy Duarte — New Bliss

### 원칙
- 우리 제품/서비스를 **아직 언급하지 않고** 매력적 미래상 제시
- 클라이언트가 "저기 가고 싶다"고 느끼게
- 제품 소개를 지연시켜 갈망(desire)을 먼저 생성

### 실행 가이드
| 요소 | 내용 |
|------|------|
| 슬라이드 수 | 1~2장 |
| 구조 | "이런 세계가 가능하다면?" |
| 비주얼 | 열망적 이미지, 밝은 톤 (문제 파트와 대비) |
| 톤 | 희망적, 구체적, 감각적 |

### 공식
```
"만약 [이상적 상태]가 가능하다면?"
"[구체적 장면/경험]이 현실이 되는 세계"
```

### 예시
- "방문객이 매장을 나서며 SNS에 올리고 싶어지는 공간"
- "브랜드 경험 하나로 6개월간 재방문율이 유지되는 구조"

### 안티패턴
- ❌ "저희 솔루션은..." (아직 제품 소개 금지)
- ❌ 추상적 비전 ("혁신적인 미래") — 반드시 구체적 장면으로

---

## Step 5. What Is ↔ What Could Be 교차 (2~3장)

> 출처: Nancy Duarte — Sparkline

### 원칙
- 현실의 고통과 이상의 가능성을 반복 교차
- 교차할 때마다 "간극"을 체감 → 해결 욕구 증폭
- 프레젠테이션 중반부의 에너지를 유지하는 엔진

### 실행 가이드
| 구분 | What Is (현실) | What Could Be (이상) |
|------|---------------|---------------------|
| 톤 | 어둡고 무겁게 | 밝고 열망적으로 |
| 색상 | 모노톤, 회색 | 포인트 컬러, 선명 |
| 데이터 | 문제의 수치 | 가능성의 수치 |
| 이미지 | 현재 상태 사진 | 이상적 상태 레퍼런스 |

### 교차 패턴 (2~3회)
```
[현실] 현재 고객 체류시간 평균 12분
[이상] 체험형 매장 전환 후 평균 47분 (벤치마크)
[현실] 재방문율 8%
[이상] 감정적 경험 설계 후 34% (사례)
```

---

## Step 6. Wow Moment (1장)

> 출처: Carmine Gallo — Holy Smokes Moment + Nancy Duarte — STAR Moment

### 원칙
- 프레젠테이션 전체에서 **단 1개**의 감정적 클라이맥스
- 청중이 기억하고, 나가서 다른 사람에게 이야기할 순간
- 서프라이즈, 시연, 체험, 충격적 대비 중 택 1

### 5가지 유형 (택 1)

| 유형 | 방법 | 제안 PT 예시 |
|------|------|-------------|
| **극적 시연** | 실제로 보여주기 | 프로토타입 체험, 미니어처 모델 공개 |
| **충격 통계** | 맥락화된 수치 | "이 비용이면 매장 3개를 리뉴얼할 수 있습니다" |
| **강렬한 비주얼** | 한 장의 이미지 | 경쟁사 매장 vs 제안 매장 동일 앵글 비교 |
| **감성 스토리** | 실제 소비자 이야기 | 고객 인터뷰 영상 30초 |
| **반복 사운드바이트** | 기억에 남는 한 문장 | "공간이 미디어가 되는 순간" |

### 안티패턴
- ❌ Wow Moment 2개 이상 (1개만. 분산되면 효과 제로)
- ❌ 준비 없는 즉흥 (잡스는 며칠간 리허설)

# Phase C — 우리만의 답

## Step 7. 차별화 강점 어필 (2~3장)

> 출처: Sinek (WHY→HOW), Miller (Guide = Empathy + Authority), Gallo (Messianic Purpose)

이 단계에서 비로소 **"우리"**가 등장한다. 하지만 여전히 기능 나열이 아니라 **"왜 우리만 이것을 해결할 수 있는가"**를 증명하는 구조.

### 원칙
- 강점을 "기능"이 아닌 **"자격"**으로 프레이밍
- StoryBrand의 Guide 원칙: **공감(Empathy) + 권위(Authority)** 동시 필요
- 공감만 있으면 약해 보이고, 권위만 있으면 거만해 보임

### 3단계 어필 구조

#### 7-1. 공감 증명 (1장)
```
"우리는 이 문제를 깊이 이해합니다"
```
- 클라이언트와 같은 고민을 해본 경험
- 업계 인사이트, 소비자 심리에 대한 이해
- **클라이언트의 Internal Problem을 정확히 짚는 것** 자체가 공감 증명

| 공감 표현 | 예시 |
|----------|------|
| 경험 공유 | "저희도 동일한 상황에서 이 문제와 씨름했습니다" |
| 인사이트 제시 | "이 문제의 진짜 원인은 대부분이 놓치는 ___에 있습니다" |
| 클라이언트 언어 인용 | "말씀하신 '브랜드 피로감'이라는 표현이 정확합니다" |

#### 7-2. 권위 증명 (1장)
```
"그리고 이것을 해결할 역량이 있습니다"
```
- 숫자로 증명: 실적, 성과 지표, 레퍼런스
- **잡스의 숫자 인간화 적용**: 원시 데이터가 아닌 맥락화된 수치
- 로고 월(logo wall)보다 **1개의 깊은 케이스**가 강력

| 권위 표현 | 예시 |
|----------|------|
| 성과 수치 | "이 접근법으로 재방문율을 12% → 34%로 끌어올렸습니다" |
| 인정/수상 | 외부 검증 (수상, 미디어 보도, 파트너십) |
| 방법론 | "저희만의 [프레임워크명]으로 이 문제에 접근합니다" |

#### 7-3. 관점의 차별화 (1장)
```
"우리는 이 문제를 다르게 봅니다"
```
- 경쟁사와의 차이를 **접근 방식/관점**으로 설명
- 직접 비교보다 **프레임 전환**이 효과적
- Sinek의 WHY→HOW: "무엇을 하느냐"가 아닌 "어떻게 다르게 하느냐"

| 차별화 유형 | 방법 | 예시 |
|------------|------|------|
| **관점 전환** | 같은 문제를 다른 렌즈로 | "매장이 아닌 미디어로 봅니다" |
| **방법론 차별** | 독자적 프로세스/프레임워크 | "RXR 분석으로 소비자 감정을 수치화합니다" |
| **경험 차별** | 축적된 노하우 | "47개 공간 프로젝트에서 발견한 패턴" |
| **구조 차별** | 팀/협업 방식 | "기획-디자인-운영이 분리되지 않는 원팀 구조" |

### 슬라이드 설계
- 공감 → 권위 → 관점의 순서 (감정 먼저, 증거 다음, 차별화 마지막)
- 로고 월은 최대 1장, 1초 이상 설명하지 않음
- **1개의 깊은 케이스 스터디 > 10개의 로고 나열**

### 안티패턴
- ❌ 회사 연혁부터 시작 (연혁은 부록)
- ❌ 기능/서비스 목록 나열 (Step 8에서 처리)
- ❌ 자화자찬 톤 ("업계 최고", "혁신적") — 수치로 보여줘라
- ❌ 경쟁사 직접 비하 — 빌런은 "관행"이지 "경쟁사"가 아님

---

## Step 8. Magic Gifts — 기능/서비스 프레이밍 (2~3장)

> 출처: Andy Raskin — Magic Gifts

### 원칙
- 기능을 나열하지 않고, **약속의 땅(Step 4)에 도달하는 도구**로 프레이밍
- "이것이 우리 서비스입니다" ❌ → "이것이 당신을 약속의 땅으로 데려갑니다" ✓
- 각 기능/서비스 = 영웅(클라이언트)에게 주어지는 마법의 도구

### 실행 가이드

| 나열식 (❌) | Magic Gifts (✓) |
|------------|-----------------|
| "공간 디자인 서비스" | "브랜드 메시지를 공간 언어로 번역하는 도구" |
| "소비자 분석 리포트" | "보이지 않는 소비자 감정을 수치로 만드는 렌즈" |
| "운영 컨설팅" | "오픈 후 6개월간 성과를 보장하는 가드레일" |

### 프레이밍 공식
```
[약속의 땅의 특정 측면]을 실현하기 위해
[우리의 기능/서비스]가 [구체적 방법]으로 작동합니다.
```

### Rule of Three 적용
- 핵심 서비스/기능은 **3개**로 묶어라
- 4개 이상이면 기억하지 못함 (단기기억 용량)
- 3개 각각에 한 문장 헤드라인 부여 (잡스의 Headline Technique)

# Phase D — 행동 유도

## Step 9. Success ↔ Failure 대비 (1~2장)

> 출처: Donald Miller — StoryBrand SB7

### 원칙
- **행동하면**: 구체적이고 열망할 만한 미래
- **안 하면**: 지속되는 고통 (현재 문제가 악화)
- 두 미래를 나란히 비교 → 선택의 명확화

### 실행 가이드
| 구분 | Success | Failure |
|------|---------|---------|
| 톤 | 밝고 구체적 | 어둡지만 과하지 않게 |
| 시간 | 6개월/1년 후 | 현재 상태 지속 시 |
| 표현 | 수치 + 장면 | 기회비용 + 리스크 |
| 비중 | 70% | 30% (소금처럼) |

### 안티패턴
- ❌ 공포 마케팅 ("망합니다") — 과하면 반감
- ❌ Success만 있고 Failure 없음 — 긴장감 제로

---

## Step 10. New Bliss — 마무리 (1장)

> 출처: Nancy Duarte — Sparkline

### 원칙
- 프레젠테이션 시작점보다 **높은 차원**에서 마무리
- 제안 내용 요약이 아닌, **변화된 세계의 비전**
- 클라이언트가 이 프로젝트를 통해 도달할 미래를 한 장면으로

### 공식
```
"[프로젝트 완료 후]의 세계는 이런 모습입니다."
— 구체적 장면 1개 + 감정적 울림
```

### 예시
- "매장을 나서는 고객이 '여기 또 와야지'가 아니라 '여기 꼭 보여줘야 할 사람이 있어'라고 말하는 순간."
- "다음 분기 보고에서 '공간이 최고의 마케팅 채널'이라는 한 줄이 적히는 날."

# 부록 A: 슬라이드 설계 7원칙

| # | 원칙 | 근거 | 실행 |
|---|------|------|------|
| 1 | **슬라이드당 1아이디어** | Gallo, Godin, Airbnb | 2개 메시지가 들어가면 0개가 전달됨 |

| 6 | **10~12장 최적** | DocSend 데이터 | 더 긴 덱 대비 22% 높은 인게이지먼트 |
| 7 | **불릿 포인트 금지** | Godin | 불릿 1개 추가 = 감정 1개 제거 |

# 부록 C: 자가 검수 체크리스트

### 구조 검수
- [ ] 첫 슬라이드가 회사 소개가 아닌 WHY/신념 선언인가?
- [ ] 우리 제품/서비스가 전체의 절반 이후에 처음 등장하는가?
- [ ] 핵심 메시지가 3개 이하인가? (Rule of Three)
- [ ] 10분(또는 10장)마다 형식 전환이 있는가?
- [ ] Wow Moment가 정확히 1개 설계되어 있는가?

### 시장 문제 분석 검수
- [ ] 3중 Problem (External + Internal + Philosophical)이 모두 있는가?
- [ ] 빌런이 "경쟁사"가 아닌 "구조적 문제/관행"인가?
- [ ] 방치의 대가가 구체적이되 과하지 않은가?
- [ ] 클라이언트의 실제 언어/표현을 인용했는가?

### 차별화 강점 검수
- [ ] 공감(Empathy) + 권위(Authority) 둘 다 있는가?
- [ ] 강점을 "기능"이 아닌 "자격/관점"으로 프레이밍했는가?
- [ ] 로고 월이 1장 이하인가?
- [ ] 1개의 깊은 케이스가 10개의 로고보다 앞에 있는가?

### 슬라이드 검수
- [ ] 모든 슬라이드가 1아이디어인가?
- [ ] 불릿 포인트가 0개인가?
- [ ] 모든 숫자가 "삶의 맥락"으로 번역되어 있는가?
- [ ] 감정적 증거(이미지/사례)가 데이터표보다 많은가?

### 마무리 검수
- [ ] 마지막 슬라이드가 "감사합니다"가 아닌 New Bliss인가?
- [ ] 청중이 나가서 다른 사람에게 전할 수 있는 한 문장이 있는가?
- [ ] Success와 Failure가 나란히 대비되는 순간이 있는가?

# 부록 E: 뇌과학 기반 설득 원리

| 원리 | 메커니즘 | 프레젠테이션 적용 |
|------|---------|-----------------|
| **Loss Aversion** | 손실이 이득보다 2배 강력 (Kahneman) | Step 3 방치의 대가, Step 9 Failure |
| **Oxytocin Release** | 캐릭터 이야기 → 신뢰+공감 호르몬 (Zak) | 모든 단계에 인물 스토리 삽입 |
| **Neural Coupling** | 스토리 시 화자-청자 뇌 동기화 (Hasson) | 데이터 → 스토리로 포장 |
| **Peak-End Rule** | 정점과 끝만 기억 (Kahneman) | Step 6 Wow + Step 10 New Bliss |
| **Dual System** | System 1(감정) → 2(논리) 순서 최적 | 감정 먼저(A,B) → 논리 나중(C) |
| **Chunking** | 단기기억 3~4개 (Miller's Law) | Rule of Three 전체 적용 |
| **Visual Superiority** | 시각 = 텍스트 × 60,000배 속도 | 이미지 > 텍스트 |
| **Story Recall** | 스토리 기억 63% vs 데이터 5% | 모든 수치를 스토리 안에 |
