**핵심 내용**:

## Part 3: 클로드 코드의 내부 구조 - 본사와 지사

### 숨김 파일 표시 방법

실습을 위해 먼저 숨김 파일을 표시하는 방법을 안내했다.

| OS | 방법 |
|------|------|
| **Mac** | Finder에서 `Cmd + Shift + .` (점) |
| **Windows** | 탐색기 > 보기 > 숨김 항목 체크 |

### `.claude` 폴더 - 본사(전역) 개념

Claude Code를 설치하면 사용자 홈 디렉토리에 `.claude`라는 숨김 폴더가 생성된다. 이것이 Claude Code의 **"본사"**에 해당한다.

**경로**:
- Mac: `/Users/[사용자명]/.claude/`
- Windows: `C:\Users\[사용자명]\.claude\`

> "여러분들은 클로드 코드를 설치를 했잖아요. 근데 그 클로드 코드가 어디에 설치되어 있는지도 아직 몰랐던 거예요"

> "이 사용자 이름 안에 `.claude`라는 폴더가 설치된 겁니다"

### 본사와 지사 구조

| 구분 | 위치 | 역할 | 적용 범위 |
|------|------|------|----------|
| **본사 (전역)** | `~/.claude/` | 모든 Claude Code 세션에 적용 | 어디서든 적용 |
| **지사 (지역)** | `[워크스페이스]/.claude/` | 해당 워크스페이스에서만 적용 | 해당 폴더 내에서만 |

> "큰 클로드 코드에 설치를 했어. 그럼 여기가 내 클로드 코드를 제어하는 모든 클로드 코드를 제어할 때는 본사라고 생각하시면 돼요"

> "요번에 제가 세팅해서 클론해드린 Do Better 워크스페이스라는 곳은 지점입니다. 지사"

### CLAUDE.md의 본사/지사 적용

- **본사 CLAUDE.md** (`~/.claude/CLAUDE.md`): 어떤 폴더에서 작업하든 항상 읽힘
- **지사 CLAUDE.md** (`[워크스페이스]/CLAUDE.md`): 해당 워크스페이스에서만 읽힘

> "근데 만약에 여러분들이 Do Better 워크스페이스라고 하는 곳이 아닌 그 웹 폴더에서 클로드 작업을 할 거면은, 그때는 얘가 이 CLAUDE.md를 읽지 않기 때문에 내가 누군지 몰라"

> "그렇기 때문에 더 좋은 거는 본사에 있는 CLAUDE.md를 잘 만드는 게 굉장히 좋다"

### 전역 CLAUDE.md 만드는 방법

기존 지사(워크스페이스) CLAUDE.md를 전역으로 변환하는 것도 Claude에게 요청하면 된다.

> "지금 내 Do Better 워크스페이스 안에 있는 CLAUDE.md를 전역, 즉 전 지역에서 사용하는 전역 CLAUDE.md로 바꿔줘 하면은 알아서 바꿔줄 거구요"

## Part 4: 전역 스킬 / 에이전트 / 커맨드

### 지역 스킬의 한계

Do Better 워크스페이스 안의 `.claude/` 폴더에 있는 스킬, 에이전트, 커맨드는 **해당 워크스페이스 안에서만** 사용 가능하다.

> "이거는 이 Do Better 워크스페이스 안에 있을 때에만 사용을 할 수 있는 스킬과 에이전트와 커맨드였던 거예요"

### 전역으로 전환하기

모든 곳에서 사용하고 싶다면 전역으로 전환 가능하다.

> "전역 스킬로 바꿔줘, 전역 커맨드로 바꿔줘, 전역 에이전트로 바꿔줘 하면은 얘가 이 내용을 그대로 [전역 폴더로 이동시켜줌]"

### 지역 스킬이 유리한 경우

모든 스킬을 전역으로 올리면 컨텍스트를 소비하므로, 특정 프로젝트에서만 필요한 스킬은 지역에 두는 것이 효율적이다.

> "만약에 내가 하는 프로젝트가 굳이 그 전역에는 다 컨텍스트를 먹고, 정말 어떠한 폴더 안에 어떤 프로젝트 안에서만 필요한 스킬이라면 해당 지점에 스킬로 둬도 좋다는 거예요"

### 중첩 CLAUDE.md 구조

CLAUDE.md는 중첩(nested)이 가능하다:
1. **전역** (`~/.claude/CLAUDE.md`) - 전체 적용
2. **워크스페이스** (`[workspace]/CLAUDE.md`) - 워크스페이스 적용
3. **하위 폴더** (`[workspace]/[project]/CLAUDE.md`) - 프로젝트 적용

> "전역에 있는 CLAUDE.md가 있고, 그 지점 안에 있는 CLAUDE.md가 있고, 그 지점 안에 더 분점으로 CLAUDE.md가 있을 때 있습니다. 그러면 여기는 전체 다 읽고 있고, 없는 내용을 이렇게 추가 추가 하면 되는 거예요"

## Part 5: 웹소설 프로젝트 - 지역 스킬 활용 사례

### 프로젝트 배경

강사가 개인 취미로 진행하는 웹소설(Web Novel) 프로젝트를 예시로 지역 스킬의 실제 활용을 보여주었다.

> "제 취미 중에 하나가 웹소설 읽는 거거든요? 스트레스가 너무 받으면은 아예 아무것도 안 하고 유료 결제 딱 한 다음에 계속 그것만 본다"

### 웹소설 프로젝트 전용 스킬 구조

| 스킬/에이전트 | 역할 |
|--------------|------|
| **챕터 에디터** | 챕터 단위 편집 |
| **캐릭터 보이스 키퍼** | 캐릭터 말투/성격 일관성 유지 |
| **컨티뉴이티 체커** | 스토리 일관성 체크 |
| **플로 트래커** | 이야기 흐름 추적 |
| **세계관 스킬** | 세계관 설정 관리 |
| **캐릭터 스킬** | 캐릭터 프로파일 관리 |
| **섹션 스킬** | 섹션 구조 관리 |

> "근데 이거를 굳이 전역 스킬로 만들어서 다 로드 시켜놓을 위에 올려놓을 필요가 없잖아요"

### 멀티 에이전트 활용

에이전트를 3개로 나누어 한쪽에서는 계속 집필을 하면서 동시 작업이 가능하다.

> "이거를 섹션을 3개로, 에이전트를 3개로 나누면 한쪽에서는 계속 집필을 쓰고 있어도 됩니다"

> "그러면 얘는 작업할 때 '아 얘가 이런 블로그를 하고 있고 어떤 사이트가 있다'라는 걸 알고 있는 거죠. 그러면 너무 쉬워요. 일이 너무 쉬워진다"

#### 4. 연동 서비스 명시

GWS CLI를 통한 Google 연결, Notion, Telegram 등 사용 중인 서비스를 명시해야 한다.

> "나 GWS CLI로 구글 다 연결을 하는데, 이 내용 그냥 CLAUDE.md에 업데이트해놔라고만 해보면 한 줄 들어가요. 그거 때문에 '아, 구글 쓸 때 꼭 GWS CLI를 써야겠구나'라는 생각을 얘가 하게 되잖아"

> "경선이 같은 경우에는 '나는 지금 Notion, Telegram 이것들을 지금 사용 중이야, 다 API를 연결해서 사용 중이라고 하면' 얘가 딴짓할 수가 없어요. 다 연결이 되어 있다가 사실 알고 있는 거예요. 그 한 줄이 들어가서 중요합니다"

#### 5. 폴더 구조

PKM 폴더 구조를 명시하면 Claude가 파일 저장 위치를 자동으로 판단한다.

> "아예 안 읽어도 얘가 폴더 구조를 이해하고 있으면 '아 이렇게 해서 그냥 넣으면 되겠네'라는 생각을 하는 거죠"

#### 6. 파일 명명 규칙

`YYYY-MM-DD` 형식을 명시하지 않으면 Claude가 임의의 형식을 사용할 수 있다.

## 핵심 개념 정리

### 1. `.claude` 폴더 (본사/지사 구조)

- **정의**: Claude Code의 설정, 스킬, 에이전트, 커맨드가 저장되는 숨김 폴더
- **본사 (전역)**: `~/.claude/` - 모든 세션에 적용
- **지사 (지역)**: `[workspace]/.claude/` - 해당 워크스페이스에서만 적용
- **핵심**: 전역에서 사용할 것과 지역에서만 쓸 것을 구분하여 관리

### 2. CLAUDE.md (전역 설정 파일)

- **정의**: Claude Code에게 "나는 누구이고, 어떻게 작업하는지"를 알려주는 핵심 설정 파일
- **필수 포함 항목**: 사용자 프로필, 시스템 환경, 연동 서비스, 폴더 구조, 작업 규칙, 문제 해결 방식
- **적용**: 항상 살아있는 문서로 지속적으로 업데이트 필요

### 3. 전역 vs 지역 스킬/에이전트

- **전역 스킬**: 모든 워크스페이스에서 사용 가능, 항상 컨텍스트 소비
- **지역 스킬**: 특정 프로젝트에서만 로드, 컨텍스트 절약
- **판단 기준**: "이 스킬이 어디서든 필요한가, 아니면 특정 프로젝트에서만 필요한가?"

### 4. Source of Truth (공식 가이드 문서)

- **정의**: Claude Code 공식 문서에서 추출한 가이드 파일들
- **용도**: 스킬, 에이전트, CLAUDE.md 등을 만들 때 기준점으로 참조
- **활용**: "이 가이드를 참고해서 만들어줘"로 정확한 결과물 확보

### 5. 중첩 CLAUDE.md

- **정의**: 전역 > 워크스페이스 > 프로젝트 단위로 CLAUDE.md를 중첩 배치
- **동작 방식**: 상위 CLAUDE.md를 모두 읽고, 하위로 갈수록 추가 정보를 적용
- **장점**: 공통 규칙은 상위에, 프로젝트 특화 규칙은 하위에 배치
