# Perception Redefine Framework (인식 재정의 프레임워크)

> "무엇을 만들 것인가"가 아니라 "무엇을 다시 보게 할 것인가"
> — 생산자 중심이 아닌 소비자 중심, 기능·물성이 아닌 의미·가치 중심의 사고

## 핵심 정의

**Perception Redefine**은 기존 인식을 재정의하는 전략 프로세스다.

- 새로운 것을 심는 것(Design)이 아니라, 이미 있는 것의 의미를 바꾸는 것(Redefine)
- 설득이 아니라 공감. 새로운 것이 아니라 재발견.
- 느끼게 하고 싶은 것을 먼저 정하고, 그 인식을 만들기 위해 모든 것을 역으로 최적화한다.

## 기존 접근과의 차이

| | 일반적 설계 | Perception Redefine |
|---|---|---|
| 출발점 | "무엇을 만들 것인가" | "무엇을 느끼게 할 것인가" |
| 관점 | 생산자 중심 | 소비자/경험자 중심 |
| 대상 | 기능, 물성, 스펙 | 의미, 가치, 감각 |
| 방향 | 완성 → 전달 | 인식 목표 → 역설계 → 전체 최적화 |
| 평가 | "잘 만들었는가" | "의도한 대로 느꼈는가" |

| | 경험 디자인 (Experience Design) | Perception Redefine |
|---|---|---|
| 출발 | 터치포인트 설계 | 인식 목표 설정 |
| 방향 | 아래→위 (요소를 쌓아서 경험) | 위→아래 (인식을 정하고 역산) |
| 최적화 | 개별 접점의 퀄리티 | 전체의 총체적 인식 |
| 기준 | "잘 만들었는가" | "의도한 대로 다시 보게 되었는가" |

## 4-Stage 프로세스

### Stage 1: Decode — 진짜 욕구 해독

**"사람들이 말하는 것" 너머 "진짜 원하는 것"을 찾는다.**

- 표면 요구 → 심층 욕구 → 핵심 동기
- 클라이언트가 "되고 싶은 포지션"과 "실제 인식 좌표"의 갭을 먼저 진단
- 질문의 깊이가 결과의 차원을 결정 (드릴→구멍→액자→분위기→감정)
- **핵심 원칙: 같은 대상에게 다른 프레임으로 질문하면 진짜 인식이 드러난다**
  - 자기 업계 안에서 물으면 → "되고 싶은 포지션"이 나옴
  - 다른 카테고리로 재정의하게 하면 → 스스로 인정하는 실제 위치가 나옴
  - 이 두 답의 간극 = Perception Redefine의 출발점

**핵심 진단:**
```
[클라이언트가 되고 싶은 포지션]  ≠  [클라이언트 스스로가 인정하는 실제 위치]
         ↓
    이 갭이 Perception Redefine의 출발점
    → 클라이언트 조차도 되고 싶은 인식과 실제 인식 사이에 간극이 있다
```

**사례 (비욘드):**
- 질문 1: "뷰티 업계에서 본인 브랜드의 포지션을 골라보세요" → **아로마티카** (되고 싶은 포지션)
- 질문 2: "커피 브랜드로 본인 브랜드를 재정의한다면?" → **할리스** (스스로 인정하는 실제 인식)
- 핵심: 같은 담당자가 두 질문에 다른 답을 한다 — 되고 싶은 인식과 실제 인식의 간극을 본인도 알고 있다
- 이 갭의 발견이 전체 전략의 방향을 결정

**AI 역할:** 데이터 기반 욕구 패턴 분석, RXR 분석으로 실제 인식 좌표 도출
**인간 역할:** 행간 읽기, 비언어적 단서 포착, "진짜 문제"의 재정의

---

### Stage 2: Calibrate — 적절함의 밸런스

**내가 하고 싶은 이야기 × 상대가 받아들일 수 있는 형태 = 적절한 지점**

- 경쟁 환경 내 빈 포지션 탐색
- 브랜드의 강점 × 소비자 욕구 × 시장 기회의 교차점
- 과하지도 부족하지도 않은 "적절함(Appropriateness)"을 찾는 감각

**사례 (비욘드):**
- 20개 경쟁 브랜드 분석 → 4축 포지셔닝 맵
- 발견: 뷰티 브랜드는 "소극적/자체실천"에 몰려 있음
- 빈 자리: "적극적/양방향/소비자참여" 영역 = 비욘드의 기회
- "행동하는 자연주의" 포지셔닝 방향 확정

**AI 역할:** 시장·소비자 데이터 교차 검증, 경쟁사 포지셔닝 매핑
**인간 역할:** "이 정도가 맞다"는 감각적 판단, 빈 포지션의 전략적 의미 해석

---

### Stage 3: Scene — 하나의 강력한 씬

**문화적 메타포를 활용해, 납득 가능하고 공감 가능한 하나의 장면으로 압축한다.**

이 단계가 Perception Redefine의 핵심이자 가장 설명하기 어려운 부분.

- 사람들의 문화적 경험과 성장 속에서 형성된 사회적 메타포를 활용
- 듣는 사람, 보는 사람이 **납득 가능하고 공감 가능한가**가 최우선
- 설득 가능한 경험, 공감할 수 있는 담론
- 맥락 정보가 충분히 쌓이면 → 임계점 → 직관으로 완성

```
맥락 정보 축적 (양)  →  임계점 도달  →  직관적 수렴 (질)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
클라이언트 대화          "아, 이거다"        하나의 씬
시장 데이터              순간이 온다         = 프로젝트의 북극성
현장 감각
소비자 심리
문화적 레퍼런스
경쟁사 분석
```

**사례 (비욘드):**

인식 재정의의 순간:
> "우리는 10%의 화장품을 사는 걸까, 90%의 이쁜 쓰레기를 사는 걸까?"

이 한 문장이 "화장품을 산다"는 행위의 의미를 완전히 재정의:
```
[Before the Scene]  화장품을 산다 = 좋은 성분의 제품을 고르는 행위
[The Scene]         90%는 이쁜 플라스틱 쓰레기
[After the Scene]   화장품을 산다 = 쓰레기를 사는 행위 → 리필만 사면 된다
```

**→ "DON'T BUY PLASTIC, TAKE JUST COSMETIC"**

비욘드의 인식이 재정의됨:
- Before: "자연주의 화장품 브랜드" (수십 개 중 하나)
- After: "플라스틱을 팔지 않는 유일한 브랜드" (카테고리 재정의)

**AI 역할:** 메타포 후보 탐색, 문화적 레퍼런스 수집, 아이데이션 보조
**인간 역할:** "이 씬이다" 결정 — 95%의 비언어적 세계가 작동하는 순간

---

### Stage 4: Orchestrate — 순간의 설계

**씬 하나를 여러 체험 순간(moment)으로 쪼개어, 각각 무엇을 느끼게 할지 최적화한다.**

- 하나의 강력한 씬 → 체험으로 분해 → 각 순간의 인식 목표 설정 → 요소별 최적화
- 비언어적 요소(향, 동선, 조명, 타이밍, 말투)까지 의도적으로 설계

**사례 (비욘드):**

| 순간 | 체험 | 느끼게 할 것 |
|------|------|-------------|
| 도착 | Plastic Forest 파사드 | 충격 + 호기심 ("이게 뭐지?") |
| 진입 | 리필 제품만 판매하는 매장 | 발견 ("새 걸 안 사도 되는구나") |
| 체험 | 화장품 분리수거 워크숍 | 행동 ("나도 할 수 있다") |
| 참여 | 공병→리필 교환 캠페인 | 실천하는 소비자가 되는 경험 |
| 확산 | 반려식물 인증 이벤트 | 지속적 관계 + 자발적 공유 |
| 기념 | 쓰레기→에디션 기념품 | 가치 전환 ("쓰레기도 가치 있다") |

**AI 역할:** 요소별 옵션 생성, 일관성 검증, 사례 수집
**인간 역할:** 비언어적 요소 판단, 순간별 감정 흐름 설계

## AI 시대에서의 위치

```
┌──────────────────────────────────────────────┐
│          Perception Redefine × AI             │
├──────────────────────────────────────────────┤
│                                              │
│  AI가 가속하는 영역:                          │
│  · Stage 1: 브랜드 실제 인식 좌표 진단 (RXR)  │
│  · Stage 2: 경쟁사 분석 + 포지셔닝 맵         │
│  · Stage 4: 체험 요소 옵션 생성               │
│  · 전체: 맥락 정보 축적의 속도                 │
│                                              │
│  인간만의 영역:                               │
│  · Stage 1: "진짜 문제" 재정의 (행간 읽기)     │
│  · Stage 2: "적절함"의 감각적 판단            │
│  · Stage 3: 씬 메이킹 (직관적 수렴)           │
│  · Stage 4: 비언어적 요소 판단                │
│  · 전체: 납득·공감 가능성의 최종 판단          │
│                                              │
│  핵심:                                       │
│  AI가 Stage 3의 "임계점"까지의 도달 시간을     │
│  압축적으로 단축. 그러나 임계점 이후의          │
│  직관적 수렴은 인간의 영역.                    │
│                                              │
└──────────────────────────────────────────────┘
```
