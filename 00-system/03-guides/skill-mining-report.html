Basket 일일보고 시스템과 AX 교육 시스템을 개선 관점에서 검토하고, GitHub의 Claude 스킬·에이전트 생태계에서 바로 쓸 수 있는 것만 골라냈다. 핵심 기준: 우리가 이미 잘 만든 건 대체하지 않고, 빈 곳만 채굴해 얹는다.

1한눈에 — 결론 3줄

대체 아닌 보강

우리 텔레그램 봇·커스텀 HTML 슬라이드는 GitHub 유사 프로젝트보다 이미 정교하다. 통째 교체 대상이 아니라, 빠진 레이어만 스킬로 얹는다.

즉시 레버리지

Anthropic 공식 스킬(xlsx·pptx·pdf·docx) + CSV Data Summarizer. 산출물 편집성↑ + 쌓이기만 하던 보고 데이터를 인사이트로.

채굴 우선순위

① 데이터 활용(분석·대시보드) ② 편집형 산출물(PPTX/DOCX) ③ 콘텐츠 리서치·검증. LMS·n8n·Metabase는 현 규모엔 과함 → 보류.

검토 대상 현황: 보고 = 봇(/jot·/todo·일일보고)+Apps Script 웹앱+OpenAI 12섹션+구글시트. 갭 = 데이터 미활용·봇 슬립·PIN 평문·Executive 뷰 부재. 교육 = 제안서·데모3종·에세이4부작·HTML 강의슬라이드·7회 텔레그램 뉴스레터·직무 케이스북. 갭 = 콘텐츠 버전 산만·학습 추적 부재·편집형 PPTX 부재.

5권장 액션 — 우선순위 × 마찰도

  • Anthropic 공식 스킬 4종 활성화 (xlsx·pptx·pdf·docx)즉시·저마찰
    산출물 품질·편집성이 즉시 상승. 교육(PPTX/DOCX)·보고(XLSX 집계)·컨설팅(PDF) 전반에 공통 효과. 도입 전 skill-security-auditor로 점검.
  • 데이터 활용 파일럿 1개파일럿
    구글시트 일일보고 → CSV export → CSV Data Summarizer로 "이번 주 매장 인사이트 1장"(매출 추이·승인 병목·반복 블로커) 시연. "쌓이기만 하던 데이터"의 가치 증명 PoC.
  • Executive 대시보드 설계패턴 차용
    analytics-reporter / Data Visualization 패턴으로 기존 getWeeklySummary()를 매장간 비교·KPI 추이 뷰로 확장(멀티매장 대비).
  • 봇 복원력 보강보강
    PicklePersistence(상태 영속) + 맥미니 이관(슬립 단절 근절). 이미 적용한 caffeinate·drop_pending_updates=False와 합쳐 3중 안전장치.
  • 보류·조건부보류
    LMS·n8n·Metabase·Listmonk — 현 규모(매장 1·직원 9·뉴스레터 7회)엔 과투자. 멀티매장 확장 또는 AX 교육의 판매형 제품화 시점에 재검토(그때 Metabase=대시보드, LMS=수료·추적, Listmonk=이메일 전환).
