## 3. 타이포그래피

### 서체: Pretendard

모든 텍스트에 **Pretendard** 사용. 웹/앱/문서/프레젠테이션 통일.

### 타이포 계층 (프레젠테이션 기준 - 16:9)

| 레벨 | 크기 | 굵기 | 행간 | 용도 | 예시 |
|------|------|------|------|------|------|
| **Display** | 60-96pt | ExtraBold (800) | 1.0 | 커버, 섹션 타이틀 숫자 | "01", "%" |
| **H1** | 36pt | Bold (700) | 1.2 | 슬라이드 메인 타이틀 | "온라인시대, 오프라인의 재발견" |
| **H2** | 32pt | SemiBold (600) | 1.3 | 서브 타이틀, 강조 문구 | "리테일미디어 시장의 변화" |
| **H3** | 24pt | SemiBold (600) | 1.3 | 소제목 | "브랜드의 고민" |
| **Body L** | 18pt | Regular (400) | 1.5 | 주요 설명 텍스트 | 본문 내용 |
| **Body** | 14pt | Regular (400) | 1.6 | 일반 본문 | 상세 설명 |
| **Caption** | 11-12pt | Medium (500) | 1.4 | 라벨, 캡션, 출처 | "CONFIDENTIAL DOCUMENT" |
| **Small** | 9-10pt | Regular (400) | 1.4 | 각주, 부가 정보 | 데이터 출처, 연도 |

### 타이포 계층 (문서/웹 기준)

| 레벨 | 크기 | 굵기 | 용도 |
|------|------|------|------|
| H1 | 28-32px | Bold | 페이지/섹션 타이틀 |
| H2 | 22-24px | SemiBold | 서브 타이틀 |
| H3 | 18-20px | SemiBold | 소제목 |
| Body | 15-16px | Regular | 본문 |
| Caption | 12-13px | Medium | 라벨, 보조 |
| Small | 11px | Regular | 각주 |

### 타이포 규칙

- **제목**: 청보라(`#6C5CE7`) 또는 블랙(`#1A1A2E`)
- **본문**: 블랙(`#1A1A2E`) 기본, 보조는 다크 그레이(`#4A4A68`)
- **강조**: Bold + 청보라 컬러 조합. 밑줄 대신 컬러로 강조
- **영문**: Pretendard 그대로 사용 (영문도 깔끔하게 처리됨)
- **숫자 강조**: Display 크기 + ExtraBold로 임팩트 부여 (IR 덱 패턴)

## 4. 레이아웃

### 슬라이드 (프레젠테이션)

- **비율**: 16:9 (960 x 540pt / 1920 x 1080px)
- **여백**: 상하좌우 40-60px
- **그리드**: 12컬럼, 거터 20px
- **정렬**: 좌측 정렬 기본. 숫자/KPI는 중앙 정렬 허용

### 섹션 패턴 (IR 덱에서 추출)

| 패턴 | 구성 | 사용 |
|------|------|------|
| **커버** | 풀블리드 이미지 + 오버레이 + 타이틀 | 표지, 섹션 구분 |
| **문제 정의** | H1 타이틀 + 아이콘 그리드 + 설명 | Market Problem Define |
| **데이터 임팩트** | 대형 숫자(Display) + 컨텍스트 텍스트 | KPI, 시장 규모 |
| **비교/대조** | 2-3컬럼 레이아웃 + 화살표 흐름 | Before/After, 비교 |
| **프로세스** | 번호 + 단계별 설명 + 아이콘 | 서비스 플로우 |
| **케이스 스터디** | 이미지 + 결과 데이터 + 인용 | 성과 사례 |

## 5. 컴포넌트

### KPI 카드

```
┌─────────────────────────────┐
│  라벨 (Caption, #7F7F9E)    │
│  1,234  (Display, #1A1A2E)  │
│  ▲ 12.5% (Body, #00B894)   │
└─────────────────────────────┘
배경: #FFFFFF
테두리: 1px #E8E8F0
라운딩: 12px
패딩: 24px
그림자: 0 2px 8px rgba(108,92,231,0.08)
```

### 버튼

| 타입 | 배경 | 텍스트 | 테두리 |
|------|------|--------|--------|
| Primary | `#6C5CE7` | `#FFFFFF` | 없음 |
| Secondary | `#FFFFFF` | `#6C5CE7` | 1px `#6C5CE7` |
| Ghost | 투명 | `#6C5CE7` | 없음 |

라운딩: 8px, 패딩: 12px 24px, 폰트: Body SemiBold

### 태그/뱃지

```
배경: #EDE9FE (Primary Subtle)
텍스트: #6C5CE7
라운딩: 6px
패딩: 4px 12px
폰트: Caption Medium
```

### 구분선

```
색상: #E8E8F0
두께: 1px
여백: 상하 24px
```

## 6. 아이콘 & 그래픽

- **스타일**: 라인 아이콘 (2px stroke), 라운드 캡
- **크기**: 24px (기본), 20px (소), 32px (대)
- **색상**: 청보라(`#6C5CE7`) 또는 다크그레이(`#4A4A68`)
- **세트**: Lucide Icons 또는 Phosphor Icons 권장 (Pretendard와 조화)

---

## 7. 이미지 & 사진

- **스타일**: 도시/공간/리테일 중심. 밝고 현대적인 톤
- **오버레이**: 어두운 배경 위 텍스트 시 반투명 오버레이 사용
  - 다크: `rgba(26,26,46,0.6)`
  - 브랜드: `rgba(108,92,231,0.15)` (미묘한 퍼플 틴트)
- **라운딩**: 카드 내 이미지 8-12px, 풀블리드 이미지 0px

## 8. 톤 & 보이스

IR 덱에서 추출한 커뮤니케이션 패턴:

| 요소 | 스타일 |
|------|--------|
| **헤드라인** | 질문형 또는 선언형. 짧고 임팩트 있게 |
| **서브카피** | 문제→해결 구조. 데이터로 뒷받침 |
| **숫자 강조** | 대형 숫자 + 단위 + 맥락 설명 |
| **영문 혼용** | 키워드/브랜드명은 영문 그대로. 본문은 한글 |

예시:
- "누구나 필요할 때, 필요한 곳에 자신만의 매장과 미디어를 가질 수 있다면?"
- "300회 이상의 팝업서비스 이력"
- "Retail Media Store Platform"

## 9. Do & Don't

### Do
- 청보라를 포인트로 절제하여 사용 (전체 면적의 10-15%)
- 여백을 충분히 확보하여 고급스러운 인상
- 데이터/숫자를 크게, 설명을 작게 (정보 계층)
- Pretendard의 다양한 웨이트 활용 (Regular~ExtraBold)

### Don't
- 청보라를 배경 전체에 깔지 않기 (포인트로만)
- 3개 이상의 컬러를 동시에 강조하지 않기
- Display 크기를 본문에 사용하지 않기
- 순수 검정(`#000000`) 사용 금지 → `#1A1A2E` 사용
