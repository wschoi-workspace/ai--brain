1기존 맥 — 시크릿 번들 생성

기존 맥북

깃에 없는 것(시크릿·인증·plist)을 한 묶음으로

bash ~/do-better-workspace/00-system/05-macmini-migration/transfer-bundle.sh
# → ~/Desktop/basket-migration-bundle.tar.gz 생성
  • 생성된 번들을 맥미니로 AirDrop
  • ⚠️ API키·봇토큰 포함 — 이관 끝나면 양쪽에서 삭제

2맥미니 — 부트스트랩

맥미니

런타임·레포·venv·인증·plist 일괄

git clone https://github.com/wschoi-workspace/ai--brain.git ~/do-better-workspace
bash ~/do-better-workspace/00-system/05-macmini-migration/macmini-bootstrap.sh ~/Downloads/basket-migration-bundle.tar.gz
  • Xcode CLT·Homebrew·node·uv → npm CLI → 레포2개 → venv재생성 → 번들(.env·인증·gws·plist) 배치까지 자동
  • 중간에 Homebrew/Xcode 설치 팝업이 뜨면 승인

3맥미니 — 슬립 방지(상시가동 핵심)

맥미니 
sudo pmset -a sleep 0 disablesleep 1 displaysleep 0 womp 1
# sleep 0=시스템 잠들지 않음 / womp 1=Wake on LAN / 뚜껑 없는 맥미니라 클램쉘 이슈 없음
pmset -g | grep -iE "sleep|womp"   # sleep 0 확인
이게 이관의 진짜 목적. 맥미니는 뚜껑이 없어 caffeinate 없이도 pmset만으로 24/7 가동. 봇·주간인사이트가 더는 안 끊긴다.

4맥미니 — 인증 검증/재로그인

맥미니

번들로 옮긴 인증이 살아있는지 확인, 만료면 재로그인

gws auth status            # 구글 — 만료면: gws auth login
tgcli auth status          # 텔레그램 — 만료면: tgcli 재인증(api_id/hash+코드)
clasp login                # ⚠️ Apps Script — invalid_rapt였으니 ws.choi로 새로 로그인 권장
clasp는 반드시 ws.choi@project-rent.com(스크립트 소유자)로. 그래야 다음 Phase 6 웹앱 배포가 권한오류 없이 된다.

5컷오버 — 기존 맥 정지 → 맥미니 로드

기존 맥북

① 먼저 기존 맥 잡 전부 내리기(봇 중복 방지)

bash ~/do-better-workspace/00-system/05-macmini-migration/oldmac-teardown.sh
# 16잡 unload + 남은 봇 프로세스 확인
맥미니

② 그다음 맥미니에서 로드

for L in $(ls ~/Library/LaunchAgents/com.{basket,arisa,dobetter,projectrent}.*.plist); do launchctl load "$L"; done
launchctl list | grep -iE "basket|arisa|dobetter|projectrent"
순서 엄수: 텔레그램 봇은 한 기기에서만 폴링해야 한다(두 곳 동시 = getUpdates 충돌·중복발송). 기존 맥 teardown → 맥미니 load 순서로.

clasp push --force
clasp create-deployment -i AKfycbxagxA-BXfGAYQEK7Ye4lUQb4MWDZvH3vOqGlIi_5uY6S8iSKiV12sU_afbusKuVwV7GA -d "v3 경영뷰+빈제출가드"
