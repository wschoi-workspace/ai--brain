# EDCD Framework
## Evolutionary Desire & Commercial Design

> "사람들의 욕망과 구매 의사결정이 어떻게 형성되는가?"라는 하나의 목적 아래,
> 파편화된 전략 프레임들을 재조합한 전략 시뮬레이션 시스템.

---

## 0. 이 프레임워크의 정체

### 무엇을 재조합하는가

기존에 따로 존재하던 사고 프레임들:

| 기존 프레임 | 원래 용도 | EDCD에서의 재조합 용도 |
|------------|----------|---------------------|
| **브랜드 전략** (Aaker, Keller, Kapferer) | 브랜드 정체성/자산 관리 | → 욕망 점유 구조 설계 |
| **포지셔닝** (Ries & Trout) | 카테고리 내 차별화 | → 감정/장면 기반 재포지셔닝 |
| **카테고리 디자인** (Play Bigger) | 새 카테고리 창조 | → 빈 욕망 공간 탐지 |
| **JTBD** (Christensen) | 고객이 고용하는 이유 | → 욕망의 기능적/사회적/감정적 층위 |
| **디자인씽킹** (Stanford d.school) | 공감→정의→아이디어→프로토→테스트 | → 욕망 탐색의 구조화된 프로세스 |
| **서비스 디자인** (Customer Journey, Blueprint) | 경험 설계 | → 장면 설계 + 터치포인트별 감정 설계 |
| **행동경제학** (Kahneman, Thaler) | 비합리적 의사결정 이해 | → 구매 결정 메커니즘 시뮬레이션 |
| **비즈니스 모델 캔버스** (Osterwalder) | 비즈니스 구조 시각화 | → 수익 구조 시뮬레이션 |
| **블루오션** (Kim & Mauborgne) | 경쟁 없는 시장 창출 | → 욕망은 있지만 구조화 안 된 기회 탐지 |
| **Porter 5 Forces / Value Chain** | 산업 구조 / 가치 사슬 분석 | → 돈이 실제로 흐르는 구조 파악 |
| **린 캔버스** (Maurya) | 스타트업 가설 검증 | → 빠른 가설 생성 + 핵심 리스크 식별 |
| **플라이휠** (Collins/Bezos) | 성장의 자기강화 루프 | → 진화 경로의 선순환 구조 설계 |
| **CDJ** (McKinsey Consumer Decision Journey) | 구매 여정 분석 | → 욕망 형성→선택→체험→공유 전체 흐름 |
| **피라미드 원칙** (Minto) | 구조적 커뮤니케이션 | → 전략 시뮬레이션 결과의 전달 구조 |
| **Cultural Strategy** (Holt) | 문화적 브랜딩 | → 시대적 흐름 위에 브랜드를 위치시키기 |
| **Ethnography / 기호학** (Semiotic Analysis) | 문화 기호 해석 | → 행동·표현·사진·말투에서 욕망 신호 읽기 |
| **Brand Worldbuilding** | 게임/영화 세계관 구축 | → 공간·언어·음식·음악·오브제까지 포함한 세계 설계 |

### 왜 재조합하는가

이 프레임들은 각각 강력하지만:
- **따로 쓰면 파편적**이다 — 브랜딩 팀은 감성만, 사업팀은 숫자만
- **순차적으로 쓰면 느리다** — 분석이 끝났을 때 시장이 바뀜
- **정답 지향적이다** — 하나의 전략을 도출하려 함

EDCD는 이것들을 **"욕망과 수익의 동시 시뮬레이션"**이라는 렌즈로 재조합한다.

### 핵심 전환

```
기존: 프레임워크를 순서대로 적용 → 정답 도출 → 문서 제출
EDCD: 프레임워크를 동시에 병렬 적용 → 가능성 공간 탐색 → 토론 → 진화
```

## 1. 핵심 철학

### 6대 원칙

| 원칙 | 설명 | 반대되는 기존 방식 |
|------|------|------------------|
| **시뮬레이션 > 정답** | 여러 가능성을 생성·비교·하이브리드 | 하나의 전략을 정답처럼 제출 |
| **속도 > 완성도** | 빠르고 반복 가능한 사고 증폭 | 완벽한 분석 보고서 |
| **병렬 > 순차** | 욕망과 수익을 동시에 | 브랜딩 먼저 → 사업 계획 나중에 |
| **질문 > 답변** | AI는 질문 생성기 | AI가 답을 내려줌 |
| **진화 > 고정** | 시간에 따라 변하는 시스템 | 한 번 정하면 3년 유지 |
| **창출 > 분석** | 초기 브랜드는 욕망을 만들어야 함 | 기존 니즈를 분석해서 충족 |

### 욕망 분석 vs 욕망 창출

> "초기 브랜드는 욕망을 분석하는 게 아니라 만들어야 한다.
> 사람들이 원하던 것이 아니라, 사람들이 원하게 될 것을 제안하는 것."

| 상황 | 접근 | 사례 |
|------|------|------|
| **성숙 브랜드/시장** | 욕망 분석 → 충족되지 않은 욕망 탐지 → 구조화 | 기존 카테고리 리뉴얼, 리브랜딩 |
| **초기 브랜드/새 카테고리** | 욕망 창출 → 아직 존재하지 않는 욕망을 제안 → 장면으로 시연 | Supreme, Aesop, Blue Bottle |

초기 브랜드가 욕망을 창출한 패턴:
- **Supreme**: 아무도 "한정판 스트릿웨어에 줄 서고 싶다"고 말하지 않았지만, 그 장면을 만들어냄
- **Aesop**: "약국 같은 스킨케어 매장에서 상담받고 싶다"는 욕망은 없었지만, 그 경험을 제안
- **Blue Bottle**: "커피 한 잔을 15분 기다리고 싶다"는 사람은 없었지만, 그 기다림이 취향이 됨

**핵심**: 데이터에서 답을 찾으려 하지 말 것. 데이터는 "이미 있는 욕망"만 보여준다.
초기 브랜드에게는 **"이것이 당신이 원하게 될 것"을 보여주는 장면 설계**가 먼저다.

### 역할 분리

| AI가 하는 것 | 인간이 하는 것 |
|-------------|--------------|
| 빠른 가능성 생성 | 시대성·미감 판단 |
| 패턴 탐지·압축 | "진짜 같은 느낌" 여부 |
| 구조 비교·교차 분석 | 방향 선택·편집 |
| 빈 정보 탐지·질문 생성 | 철학·가치관·취향 |
| 레퍼런스 브랜드 매칭 | "촌스러움" 여부 최종 판단 |
| 수치·시장 데이터 압축 | 장기 지속 가능성 직감 |

## 2. Double Layer Structure

> 브랜드 전략과 비즈니스 전략은 따로 존재하지 않는다.
> Aesop은 감성 브랜드 같지만 객단가·리테일효율·SKU전략이 정교하고,
> Apple은 세계관처럼 보이지만 마진구조·업셀링·생태계락인이 핵심이다.

### ENGINE A — Desire Engine (욕망 시뮬레이션)

> "사람들이 왜 반응하는가?"
> 브랜드는 고객을 끌어온다.

#### 재조합된 프레임워크들

**① JTBD → 욕망의 3층위 분해**

| 층위 | 질문 | 원천 |
|------|------|------|
| Functional | 이 제품/서비스가 해결하는 실용적 문제는? | Christensen JTBD |
| Social | 이걸 사용/소유하면 어떤 사람으로 보이는가? | Keller Brand Resonance |
| Emotional | 이것이 만들어주는 감정 상태는? | Kahneman 경험 자아 |

**② 카테고리 디자인 → 감정 점유 지도**

기존 "카테고리를 만들어라" (Play Bigger)를 확장:
- 기능 카테고리가 아니라 **감정 카테고리**를 점유
- "커피를 파는가" → "아침의 작은 사치를 파는가"
- "샤퀴테리를 파는가" → "유럽 감성의 홈파티 장면을 파는가"

| 차원 | 질문 |
|------|------|
| 감정 점유 | 어떤 감정을 점유할 것인가? |
| 장면 설계 | 사람들이 반복적으로 소비하고 싶은 장면은? |
| 세계관 | 어떤 세계에 들어가고 싶어하는가? |
| 라이프스타일 | 어떤 삶의 방식을 제안하는가? |
| 동경 구조 | 무엇을 동경하는가? (상태/능력/관계/정체성) |
| 공유 욕구 | 어떤 순간을 SNS에 공유하고 싶은가? |

**③ Scene Analysis — 페르소나가 아닌 "장면" 단위 분석**

> 사람 자체보다 "어떤 순간이 매력적인가?"를 분석한다.
> 브랜드는 설명보다 **"반복되는 장면"으로 기억**된다.
> (Starbucks의 테이블, MUJI의 정리된 생활, Nike의 자기극복)

| 장면 유형 | 분석 질문 |
|----------|----------|
| 혼자의 시간 | 혼자 먹는 밤, 혼자의 루틴 — 어떤 감정이 흐르는가? |
| 관계의 순간 | 친구 초대, 연인과의 저녁 — 어떤 관계를 연출하는가? |
| 취향 표현 | 취향 자랑, SNS 업로드 — 어떤 정체성을 보여주는가? |
| 발견의 순간 | 처음 알게 된 경험 — 어떤 놀라움이 작동하는가? |
| 반복의 습관 | 다시 찾는 이유 — 습관인가, 동경인가, 커뮤니티인가? |

**핵심 전환:**
- 기존 Design Thinking: 문제 발견 → 공감 → 정의 → 해결 (기능적)
- Desire-Oriented Design: 동경하는 것 → 따라하고 싶은 것 → 들어가고 싶은 관계 → 되고 싶은 취향의 사람 (감정 기반)

**④ 3-Layer Brand Intelligence**

단순 관찰이 아니라, 의미 해석의 3단계를 거친다:

```
Layer 1: Observation (관찰)
    "무엇이 벌어지고 있는가?" — SNS, 리뷰, 행동, 소비 패턴
    ↓
Layer 2: Meaning (의미 해석)
    "왜 그런 반응이 나오는가?" — 욕망, 정체성, 문화적 맥락
    ↓
Layer 3: Strategic Narrative (전략적 서사)
    "이것을 어떤 이야기로 만들 것인가?" — 브랜드가 점유할 이야기
```

**⑤ 디자인씽킹 공감 단계 → 욕망 탐색 프로세스**

Stanford d.school의 공감 단계를 욕망 탐색용으로 전환:

```
관찰: 사람들이 실제로 어디서 돈과 시간을 쓰는가?
    ↓
해석: 그 행동 뒤에 숨은 욕망은 무엇인가? (말하는 것 ≠ 원하는 것)
    ↓
패턴: 반복되는 욕망의 패턴은? (SNS, 리뷰, 검색량에서)
    ↓
빈 공간: 욕망은 있지만 아직 구조화되지 않은 영역은?
```

> **직관은 비논리가 아니라 "압축된 패턴 인식"이다.**
> 직감적으로 "이건 아닌데"라고 느끼는 순간, 그 직관을 언어화하는 것이 Thinking Partner Phase의 핵심 역할.

**⑥ 행동경제학 → 선택 메커니즘 시뮬레이션**

| 원리 | 브랜드 설계에서의 적용 |
|------|---------------------|
| System 1/2 (Kahneman) | 이 브랜드는 직관적으로 반응하게 만드는가, 분석적 비교를 유도하는가? |
| 손실 회피 | "이걸 안 하면 놓치는 것"이 "이걸 하면 얻는 것"보다 강한가? |
| 앵커링 | 가격/가치의 기준점을 어디에 설정하는가? |
| 사회적 증거 | "다른 사람도 하는가"가 이 카테고리에서 얼마나 중요한가? |
| 현상유지 편향 | 기존 습관에서 전환하게 만드는 계기는 무엇인가? |
| 스키마 | 기존 카테고리 인식을 활용하는가, 의도적으로 깨뜨리는가? |
| 희소성 | 한정성이 가치를 만드는 구조인가? |
| 프레이밍 | 같은 것을 어떤 프레임으로 보여주면 반응이 달라지는가? |

**⑦ CDJ → 욕망 형성 여정**

McKinsey CDJ를 욕망 관점으로 재해석:

```
[인지] 어떻게 처음 알게 되는가? → 어떤 욕망이 촉발되는가?
   ↓
[탐색] 무엇을 비교하는가? → 어떤 기준으로 동경하는가?
   ↓
[선택] 왜 이것을 고르는가? → 어떤 감정이 결정을 만드는가?
   ↓
[체험] 어떤 장면을 경험하는가? → 기대 vs 현실의 갭은?
   ↓
[공유] 무엇을 남에게 말하는가? → 어떤 장면/감정을 전파하는가?
   ↓
[재방문] 왜 다시 오는가? → 습관인가, 동경인가, 커뮤니티인가?
```

**⑧ Decision Architecture — 디자인의 재정의**

> 디자인 = "예쁜 것"이 아니라 **"구매와 동경의 흐름을 설계하는 것"**
> 공간, 동선, 음식, 패키지, 언어, 가격, 사진, SNS, 음악, 조명, 커뮤니티,
> 희소성, 참여 구조 전체를 통해 **의사결정의 관점을 설계**한다.

모든 터치포인트는 다음 질문에 답해야 한다:
- 이 요소가 **어떤 감정**을 만드는가?
- 이 요소가 **어떤 선택**을 유도하는가?
- 이 요소가 **어떤 장면**을 만드는가? (공유 가능한가?)

**⑨ 보조 프레임 3종**

| 프레임 | 용도 | 적용 |
|--------|------|------|
| **Cultural Positioning** | 브랜드를 "문화적 위치"로 해석 | 어떤 시대적 흐름 위에 있는가? 한국 사회에서 왜 지금 필요한가? |
| **Relationship Mapping** | 브랜드-고객 관계를 연애처럼 설계 | 첫인상→썸→관계진입→반복방문→자랑→커뮤니티화 |
| **Brand Worldbuilding** | 게임/영화 세계관 구축 방식 적용 | 공간, 언어, 음식, 음악, 오브제, 복장, 사진톤, 대화방식까지 하나의 세계 |

---

### ENGINE B — Commercial Engine (수익 시뮬레이션)

> "어떻게 돈이 되는가?"
> 비즈니스 구조는 감정의 수익화 구조를 만든다.

#### 재조합된 프레임워크들

**① 비즈니스 모델 캔버스 → 수익 시뮬레이션 캔버스**

Osterwalder를 Desire Engine과 연결:

```
[고객 세그먼트]  ← 누가 이 욕망을 가장 강하게 느끼는가?
[가치 제안]      ← Desire Engine의 감정 점유와 직결
[채널]          ← 어디서 욕망과 만나는가? (온/오프/SNS/커뮤니티)
[고객 관계]      ← 일회성인가, 관계 기반인가, 커뮤니티인가?
[수익 흐름]      ← 상품/서비스/구독/라이선스/수수료/광고
[핵심 자원]      ← 브랜드 자산, 콘텐츠, 커뮤니티, 데이터, 공간
[핵심 활동]      ← 반드시 잘해야 하는 것
[핵심 파트너]    ← 혼자 못 하는 것
[비용 구조]      ← 무엇이 가장 비싼가
```

**② Porter Value Chain → 돈이 실제로 흐르는 구조**

| 질문 | 분석 |
|------|------|
| 이 산업에서 마진이 가장 높은 단계는? | 원재료→제조→유통→소매→경험 |
| 우리가 점유할 수 있는 가치 사슬 위치는? | 직접 생산 vs 큐레이션 vs 플랫폼 |
| 기존 구조의 비효율/과잉은 어디인가? | 중간 유통 과다, 정보 비대칭 등 |

**③ 린 캔버스 → 핵심 가설과 리스크**

| 영역 | 질문 |
|------|------|
| 핵심 가설 | 이 비즈니스가 작동하려면 반드시 참이어야 하는 것은? |
| 가장 큰 리스크 | 이것이 거짓이면 전체가 무너지는 것은? |
| 불공정 우위 | 쉽게 복제할 수 없는 것은? (브랜드, 커뮤니티, 데이터, 위치) |
| 핵심 지표 | 첫 3개월 내 확인해야 하는 숫자는? |

**④ 수익 구조 시뮬레이션 질문**

| 차원 | 핵심 질문 |
|------|----------|
| 수익원 | 어디서 돈이 생기는가? |
| 객단가 | 무엇이 객단가를 높이는가? (업셀, 크로스셀, 번들, 프리미엄) |
| 반복성 | 어떤 구조가 반복 구매를 만드는가? (습관, 소모품, 시즌, 커뮤니티) |
| 마진 구조 | 고마진인가 박리다매인가? 무엇이 마진을 결정하는가? |
| 확장성 | 어떤 방향이 확장 가능한가? (지역, 카테고리, 채널, 라이선스) |
| 롱테일 | 시간이 지날수록 강해지는 구조인가? |
| 경험 프라이싱 | 어떤 경험이 높은 가격을 정당화하는가? |

**⑤ 플라이휠 → 성장의 선순환 설계**

```
예시: Aesop 플라이휠
공간 경험 → 브랜드 인식 상승 → 입소문/SNS → 신규 고객
  ↑                                              ↓
높은 리테일 효율 ← 반복 구매 ← 제품 신뢰 ← 첫 구매 경험
```

모든 시뮬레이션에서 "이 모델의 플라이휠은 무엇인가?"를 반드시 질문.

## 4. Evolution Strategy Layer (진화 전략)

> 브랜드는 현재 모습이 아니라 "예상되는 미래"로 소비된다.
> 초기 행동은 미래 포지션을 위한 투자다.
> Supreme은 초기엔 돈이 안 됐지만 문화 자산을 쌓았고,
> Red Bull은 음료보다 미디어 자산을 먼저 구축했다.

### 3단계 진화 설계

| 차원 | 초기 (0~1년) | 중기 (1~3년) | 후기 (3년~) |
|------|-------------|-------------|------------|
| 브랜드 | 감도/콘텐츠/팬 형성 | 커뮤니티/신뢰/재방문 | IP화/문화 브랜드/확장 |
| 수익 | 핵심 상품 검증 | 반복 구매 구조 확립 | 다각화/라이선스/플랫폼 |
| 자산 | 콘텐츠/취향 자산 | 신뢰/데이터/커뮤니티 자산 | 브랜드 자산/생태계 |
| 핵심 활동 | 장면 만들기/공유 유도 | 관계 깊이/습관 형성 | 카테고리 점유/확장 |
| KPI | 인지도/팔로워/초기매출 | 재구매율/LTV/NPS | 마진율/카테고리점유율 |

### 진화 패턴 레퍼런스

| 패턴 | 진화 경로 | 사례 | 플라이휠 |
|------|----------|------|---------|
| 콘텐츠 → 커머스 | 정보→취향큐레이션→커뮤니티→자체상품 | 무신사, 29cm | 콘텐츠→트래픽→구매→데이터→더좋은큐레이션 |
| 제품 → 세계관 | 핵심제품→라이프스타일→문화브랜드 | Apple, Aesop | 제품신뢰→세계관확장→프리미엄프라이싱→R&D투자 |
| 공간 → 미디어 | 체험공간→콘텐츠→미디어플랫폼 | 젠틀몬스터, 탬버린즈 | 공간화제→방문→SNS콘텐츠→인지도→더강한공간 |
| 커뮤니티 → 플랫폼 | 동호회→마켓플레이스→플랫폼 | 오늘의집, 번개장터 | 유저콘텐츠→트래픽→셀러유입→더많은콘텐츠 |
| 서비스 → 데이터 | 서비스운영→데이터축적→인사이트 | 배달의민족 | 서비스이용→데이터→최적화→더좋은서비스 |
| 영향력 → 수익화 | 팬형성→신뢰→큐레이션→자체상품 | 1인 미디어, 블로그 | 콘텐츠→신뢰→구매전환→수익→더좋은콘텐츠 |

### 진화 시뮬레이션 핵심 질문

```
1. 우리는 최종적으로 어디서 돈을 벌 것인가? (End-state Revenue)
2. 현재 행동은 미래 어떤 자산을 축적하는가? (Asset Building)
3. 이 모델의 플라이휠은 무엇인가? (Self-reinforcing Loop)
4. 각 단계의 전환 트리거는 무엇인가? (Stage Gate)
5. 이 진화 경로의 구조적 리스크는? (Path Risk)
```

## 5. Opportunity Gap Detection (빈 기회 탐지)

> 빈 기회 = "사람들은 원하지만 아직 구조화 안 된 욕망"
> 단순히 "아무도 안 하는 것"이 아니라, "욕망은 있지만 브랜드화되지 않은 것"

### 블루오션 전략의 재조합

Kim & Mauborgne의 4가지 액션을 욕망 관점으로:

| 액션 | 원래 질문 | EDCD 재해석 |
|------|----------|-----------|
| 제거 (Eliminate) | 업계가 당연시하지만 제거할 요소는? | 사람들이 사실 원하지 않는데 관성으로 유지되는 것은? |
| 감소 (Reduce) | 업계 기준 이하로 줄일 요소는? | 과잉 투자되고 있지만 욕망과 무관한 것은? |
| 증가 (Raise) | 업계 기준 이상으로 높일 요소는? | 욕망은 강한데 충족이 부족한 것은? |
| 창조 (Create) | 업계에 없지만 제공해야 할 요소는? | 욕망은 있지만 아직 구조화되지 않은 것은? |

### 빈 기회 탐지 사례 패턴

```
혼술 문화는 있었지만 → 혼술 브랜드는 부족했던 시기
캠핑은 있었지만 → 캠핑 라이프스타일 브랜드는 부족했던 시기
홈카페는 있었지만 → 홈카페 큐레이션은 부족했던 시기
```

**공식: [욕망/행동은 존재] + [구조화된 브랜드/상품이 부재] = 기회**

### 빈 기회 스캔 프로세스

```
1. Signal Scan — SNS, 리뷰, 검색량, 밈, 콘텐츠 트렌드에서 욕망 신호 수집
2. Pattern Match — 반복되는 욕망/행동 패턴 압축
3. Supply Check — 해당 욕망을 구조적으로 충족하는 브랜드/서비스가 있는가?
4. Gap Identify — 욕망은 강한데 공급이 약한 영역 = 기회
5. Viability Test — 이 기회가 수익 구조로 전환 가능한가? (Commercial Engine 교차)
```

|------|------|---------------------|

| 비전 | 대표 철학, 창업 동기, 장기 비전 | Evolution: 진화 경로 방향 |

## 7. 시뮬레이션 프로세스

### 전체 흐름

```
Phase 0: INTAKE (최소 입력 수용)
    ↓
Phase 1: REALITY SCAN (현실 파악)
    ↓
Phase 2: DESIRE SIMULATION (욕망 시뮬레이션)
    ↓  ← 동시 진행 →
Phase 3: COMMERCIAL SIMULATION (수익 시뮬레이션)
    ↓
Phase 4: CROSS-MATRIX (교차 비교 + 빈 기회 탐지)
    ↓  ★ 인간 디스커션 게이트
Phase 5: DEEP DIVE (선택된 방향 심화)
    ↓
Phase 6: EVOLUTION PATH (진화 경로 설계)
    ↓
Phase 7: OUTPUT (산출물 생성)
```

### Phase별 상세

#### Phase 0: INTAKE

입력 가능 형태:
- 인터뷰 녹취 / 러프 메모 / 매출 자료 / 브랜드 소개
- 경쟁사 이름 / 현재 고민 / 링크 / SNS / 이미지 / 운영 자료

AI 수행:
1. 자료 구조화 (무엇이 있는가)
2. 빈 정보 탐지 (무엇이 없는가) → 🔴항목 질문
3. 초기 가설 생성 (이 브랜드는 아마 ~한 상황일 것이다)

#### Phase 1: REALITY SCAN

| 축 | 질문 | 방법 |
|----|------|------|
| 시장 현실 | 이 카테고리의 규모, 성장, 트렌드는? | 웹 리서치 + 데이터 |
| 경쟁 구조 | 누가 있고, 각각 어떤 포지션인가? | Porter 5 Forces 간소화 |
| 고객 현실 | 실제 어떻게 구매하고 소비하는가? | CDJ + 리뷰/SNS 분석 |
| 내부 현실 | 현재 강점, 약점, 자원 상태는? | 자료 기반 + 질문 |

#### Phase 2: DESIRE SIMULATION

**동시에 여러 프레임 적용:**

1. JTBD 3층위 분해 → 기능적/사회적/감정적 욕망
2. 감정 점유 지도 → 점유 가능한 감정 영역
3. 장면 설계 → 반복 소비할 장면
4. 행동경제학 렌즈 → 선택 메커니즘
5. CDJ 재해석 → 욕망 형성 여정

**산출물:** Brand Direction A/B/C (각각 다른 욕망 축)

#### Phase 3: COMMERCIAL SIMULATION

**동시에 여러 프레임 적용:**

1. 수익 시뮬레이션 캔버스 → 수익 구조
2. Value Chain → 돈이 흐르는 구조
3. 린 캔버스 → 핵심 가설과 리스크
4. 플라이휠 → 성장 선순환

**산출물:** Business Model A/B/C (각각 다른 수익 축)

#### Phase 4: CROSS-MATRIX + GAP DETECTION

1. Brand A/B/C × Business A/B/C 교차 비교 (9가지 조합)
2. 각 조합의 정합성/지속가능성/실행가능성 평가
3. 블루오션 4 Actions → 빈 기회 탐지
4. **★ 인간에게 토론 질문 제시**

```
토론 질문 예시:
- "이 중에서 뭐가 더 우리답나?"
- "이 조합이 3년 후에도 유효할까?"
- "이걸 하려면 반드시 필요한 건 뭔가?"
- "경쟁사가 따라하기 어려운 건 어느 방향인가?"
```

#### Phase 5: DEEP DIVE

선택된 방향에 대해서만 깊이 연구:
- 해당 방향의 레퍼런스 브랜드 상세 분석
- 구체적 상품/서비스 구조 설계
- 가격/마진 시뮬레이션
- 구체적 고객 세그먼트 정의
- 필요 역량/자원 정의

#### Phase 6: EVOLUTION PATH

- 3단계 진화 로드맵 (초기/중기/후기)
- 각 단계별 KPI, 핵심 활동, 자산 축적
- 단계 전환 트리거 정의
- 플라이휠 설계
- 리스크/피봇 시나리오

#### Phase 7: OUTPUT

**Mode B — 클라이언트 전략 제안서 (HTML)**

| 섹션 | 내용 |

## 8. A/B/C 방향 생성 원칙

### 생성 규칙

1. **3가지 방향은 서로 다른 축**이어야 함 (유사 방향 3개 ✕)
2. **각 방향에 실존 레퍼런스 브랜드** 반드시 포함 (추상적 방향 ✕)
3. **욕망 구조 + 수익 구조** 동시에 명시
4. **리스크와 전제 조건** 함께 제시
5. **하이브리드 가능성** 표시 (A+B, B+C 등 조합)
6. **"이 방향의 Aesop은 누구인가"** 반드시 답변 (감성+수익 모두 강한 레퍼런스)
