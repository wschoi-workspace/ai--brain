# Step 1. 수집 (Collect) - 책 리스트 + 짧은 써머리

## 목표
**토큰 최소로** 카테고리별 베스트 도서 리스트를 CSV로 정리한다.
각 책에는 **200자 이내 짧은 써머리**만 붙인다 (상세 추천사는 Step 3에서).

## 입력 정보 (사용자가 지정)

- **대상 독자**: 예) 2030 MZ, 전체 고객, 직장인 등
- **카테고리 목록**: 예) 자기계발, 에세이, 경제경영, 인문, 소설
- **소스**: 예) 교보문고 판매순위, 북스톤 신간, 알라딘 베스트 등
- **카테고리당 권수**: 예) 20권 또는 30권

## Claude에게 요청하는 방법

```
01-collect.md 기반 Step 1 실행해줘.
- 대상: [독자]
- 카테고리: [목록]
- 소스: [출처]
- 카테고리당: [N]권
- 써머리 글자수: 200자 이내
```

## 출력 형식 (book-list-template.csv)

각 행 = 책 1권, 컬럼:

| 컬럼 | 설명 | 예시 |
|------|------|------|
| category | 카테고리명 | 자기계발 |
| rank | 카테고리 내 순위 | 1 |
| title | 제목 | 제목 |
| author | 저자 | 저자 |
| publisher | 출판사 | 출판사 |
| price | 정가 | 15000 |
| summary | 200자 이내 요약 | 책의 핵심 메시지 1~2문장 |
| hook | 한줄 훅 (선택) | 독자를 끌어당기는 한 문장 |
| selected | 빈 칸 (사용자 채움) | ★ or 빈 칸 |

## 토큰 절감 규칙

- ❌ 1000자 매력글 X
- ❌ 추천사 X
- ❌ 목차 전체 X
- ✅ 200자 써머리만
- ✅ 메타데이터 중심 (제목/저자/가격)
- ✅ 한 번에 1~2 카테고리만 요청

## 블록 관리

| 작업량 | 권장 실행 |
|--------|-----------|
| 1 카테고리 × 20권 | 1회 요청 |
| 3 카테고리 × 20권 | 1블록 내 가능 |
| 5+ 카테고리 × 30권 | 2~3블록 분할 |

## 크롤링 대안 (토큰 0)

메타데이터는 크롤링으로 수집하면 **Claude 토큰을 전혀 쓰지 않는다**:

```bash
# Playwright로 교보문고/알라딘 판매순위 스크래핑
# → title, author, publisher, price만 CSV로 저장
# → summary는 Claude가 나중에 채워주거나, 웹에 이미 있는 소개 복사
```
