# Step 3. 상세 작성 (Enrich) - 선정된 책 추천사/매력글

## 목표
Step 2에서 선정된 책만 **1000자 내외 상세 추천사/매력글**을 작성한다.
**블록을 나눠** 실행하여 5시간 한도 내에서 안전하게 진행.

## 블록 분할 전략

| 작업 방식 | 한 블록 권장 |
|-----------|--------------|
| 1000자 매력글 | **15~20권** |
| 500자 매력글 | 30~40권 |
| 300자 매력글 | 50~60권 |

**안전 마진**: 위 수치의 70%에서 시작, 토큰 여유 보면서 증량.

## 실행 방법

### 1블록 단위로 요청하기

```
02-select.md로 만든 선정 리스트 중
priority 1~20번 책만 상세 추천사 작성해줘.
- 글자수: 각 1000자
- 구조: [독자의 고민] → [책의 해답] → [삶의 변화]
- 출력: final-recommendations-[YYYY-MM]-block1.md
```

### 다음 블록 (5시간 후 또는 다음날)

```
priority 21~40번 책 이어서 작성해줘.
- 같은 형식 유지
- block2.md로 저장
```

## 글 구조 템플릿 (1000자)

```markdown
### [책 제목]
**저자**: [저자명] | **출판사**: [출판사]

[도입 - 독자의 상황/고민] (200자)
"이런 고민 있지 않나요?" 형식으로 독자 공감 유도.

[본문 - 책이 주는 해답] (500자)
핵심 메시지 2~3가지.
저자의 차별화된 관점.
실제 사례 1~2개.

[결론 - 삶에의 적용] (300자)
이 책을 읽으면 무엇이 달라지는가?
어떤 독자에게 특히 추천하는가?
```

## 토큰 절감 팁

### 1. 배치 크기 관리
```
❌ "선정된 60권 전부 1000자로 써줘" (블록 초과)
✅ "20권씩 3번에 나눠서 쓸게. 이번엔 1~20번만" (안전)
```

### 2. 글자수 조절
- 마케팅 카피용: **300~500자**로 충분
- 콘텐츠 포스팅용: **800~1000자**
- 카탈로그 소개: **200~300자**

### 3. 구조화된 출력
```
JSON 형식으로 요청하면 일관성↑, 후처리 쉬움:
{
  "title": "...",
  "hook": "...",
  "body": "...",
  "closing": "..."
}
```

### 4. 중간 저장
각 블록 완료 시 즉시 파일 저장 → 한도 초과 시 복구 가능.
