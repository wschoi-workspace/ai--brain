특장점 — 5개의 축

02

이 워크스페이스의 가치는 폴더 정리에 있지 않다. 반복 업무를 스킬로 굳히고, 규율을 자동화·검증으로 체화하고, 맥락을 메모리에 남기는 "시스템으로서의 자기증강"에 있다.

01

스킬 생태계 — 56종

자주 하는 지적 노동을 커맨드로 응축했다. 분석부터 산출물 제작, 일상 자동화, 문체·회의까지 — 매번 프롬프트를 새로 짜지 않고 검증된 파이프라인을 호출한다.

분석 rxr-* / brand-analysis / psychology-critic 산출물 consulting-report / gov-proposal / pptx·docx·pdf 자동화 morning / 보드등록 / gmail-project-note 문체·회의 문체 / 회의분석 / transcript-organizer
02

자동화 인프라 — launchd 17종

사람이 기억할 일을 기계가 대신한다. 매일 아침 텔레그램 체크리스트, 일일보고봇(ARISA), Basket 운영봇, 대시보드 서버가 상시 가동. 53개 파이썬/셸 스크립트와 shared/ 공유코어가 복붙 없는 재사용을 지탱한다.

morning-telegram 매일 08:00 브리핑 daily-report-bot ARISA 채점 SSOT weekly-report-aggregate 주간 집계 dashboard-server 통합 대시보드
03

메모리·MOC 이중 레이어

세션이 끝나도 맥락이 사라지지 않는다. .claude auto-memory 100개 항목이 사용자 선호·피드백·프로젝트 상태를 누적하고, 99-moc Obsidian 그래프가 지식을 상호연결한다. 프로젝트별 progress.md는 세 번째 기억 장치.

auto-memory user / feedback / project / reference MOC projects · topics 그래프 progress.md 세션 연속성
04

진화형 운영 시스템 — 자체 프로덕트화

워크스페이스가 도구를 넘어 제품을 낳았다. 회의→6종 리포트(ARISA), 개인 사고 OS(Second Brain), 보고체계(Reporting OS), 현장 운영(REM OS)까지 — 20-operations는 사실상 사내 SaaS 인큐베이터.

ARISA 회의 인텔리전스 Second Brain 텔레그램→마크다운 Reporting OS 6-Layer 보고 REM OS 현장 운영
05

검증 내장 원칙 — 규율의 체화

퀄리티가 우연이 아니라 프로세스다. 숫자 3중 검증(추출→웹대조→재대조), "해줘 후 확인해줘", 3번 룰, 슬라이드=HTML+Playwright — CLAUDE.md의 규율이 파일이 아니라 매 작업의 절차로 박혀 있다.

숫자 3중 검증 단위·귀속·맥락·출처 확인해줘 누락 체크 습관 3번 룰 실패 시 기록→재시작

솔직한 자기점검 — 개선 여지

03

강점의 이면. 빠르게 축적한 시스템일수록 정리 부채가 쌓인다. 지금 손보면 검색·유지보수 비용을 크게 줄일 수 있는 지점들.

⚠루트 산재 PNG 64개

제안서·슬라이드 미리보기 캡처가 워크스페이스 루트에 그대로 노출. 구조의 첫인상을 해치고 git status를 오염시킨다.

→ 50-resources 하위 previews/ 또는 프로젝트별 assets/로 이동, PNG는 gitignore

⚠_to-delete · .bak 잔재

_to-delete 폴더와 다수 .bak/.bak-* 백업 파일이 활성 트리에 남아 실제 SSOT를 헷갈리게 한다.

→ 백업은 90-archive 또는 git 히스토리로 일원화, _to-delete 실제 삭제

⚠활성/아카이브 경계 모호

10-projects 26개 중 사실상 종료된 건이 섞여 있을 가능성. progress.md 최종일 기준 라이프사이클 판정이 없다.

→ Team Ops의 아카이브 라이프사이클(P1) 규칙을 프로젝트에도 적용

⚠node_modules 24K 검색 노이즈

의존성 24,384개 파일이 전체의 60%. 탐색·grep 시 매번 배제 플래그가 필요하다.

→ 루트 도구 체인을 00-system 하위로 격리, .claudeignore 정비

⚠MOC 최신성 유지 부담

99-moc는 생성기(idempotent)로 만들지만, 신규 프로젝트가 자동 편입되지 않으면 그래프가 실제보다 뒤처진다.

→ _generate-moc.py를 주간 launchd에 편입해 자동 갱신

⚠스킬 카탈로그 표류

56개 스킬이 강력하지만, 트리거 문구가 겹치거나 중복 기능(회의록/회의분석/회의록정리)이 존재해 호출 혼선 여지.

→ 스킬 인벤토리 정기 점검 + 별칭/역할 경계 문서화

CONCLUSION

구현된 상태.
