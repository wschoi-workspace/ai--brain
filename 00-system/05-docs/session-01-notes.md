## 1. AI 에이전트 개념 설명

### LLM vs AI 에이전트 차이점

**LLM (Large Language Model)**

- 기본적으로 "뇌"만 있는 상태
- 똑똑한 텍스트 생성기
- 사용자가 구체적인 명령을 내릴 때까지 대기
- 한 번에 한 답변 제공
- 예: "마케팅 전략 짜줘" → 전략은 짜주지만 실제 실행은 못함

**AI 에이전트**

- LLM에 "손과 발"을 달아준 형태
- 스스로 판단하고 움직임
- 연속적인 작업 수행 가능
- 외부 도구(파일, 앱 등) 활용 가능
- 목표를 주면 스스로 처리

### 에이전트의 3가지 핵심 특성

1. **대리성(Agency)**: 사용자를 대신해서 작업 수행
2. **자율성(Autonomy)**: 범위 내에서 스스로 판단
3. **목표 지향성(Goal-oriented)**: 명확한 목표를 가지고 달성을 위해 움직임

> "가장 쉽게 이해할 수 있는 에이전트는 부동산 중개인이에요. 나 대신해서(대리성) 내가 정한 가격대 선에서 네가 판단해서(자율성) 좋은 집을 구해줘(목표 지향성)"

---

### 에이전트 작동 방식 예시

"신제품 출시 캠페인 분석해줘"라는 요청에 대해:

1. **목표 해석 및 의도 추론**
    - 업무: 캠페인 분석, 목적: 성과 파악
    - 추론된 의도: 마케팅 전략 개선을 위한 리서치
2. **태스크 분해**
    - 캠페인 데이터 수집
    - 채널별 성과 분석
    - 경쟁사 비교
    - 인사이트 도출
    - 개선점 제안
3. **각 태스크별 도구를 활용하여 실행**
    - 각각의 에이전트가 검색, 분석 도구로 작업
4. **결과 통합**
    - 모든 태스크 결과를 하나로 합침
5. **최종 결과물 생성**
    - 핵심 인사이트, 채널 전략, 개선 방안까지

> "이게 바로 에이전트입니다. 제가 개떡같이 얘기해도 얘가 알아서 태스크를 쫙 나누고 그거를 정리해서 결과물 갖고 오는 게 AI 에이전트예요."

## 2. 인간 본성과 AI 워크스페이스

### 인간이 일을 잘 못하는 4가지 기제

1. **인지적 구두쇠**
    - 뇌가 에너지를 아끼려 함
    - 새로운 것보다 익숙한 것, 복잡한 것보다 단순한 것 선호
2. **현재 편향**
    - 즉각적인 피드백에 반응
    - 먼 미래보다 가까운 지금의 만족 선택
    - 작은 확실한 피드백이 일을 하게 하는 힘이 됨
3. **사회적 연결 본능**
    - 분산되기 쉬운 주의력
    - 웹창, 카카오톡, YouTube 배너 등으로 주의력 분산
4. **멀티태스킹 환경**
    - 창을 여러 개 쓰는 것 자체가 주의력 분산의 원인

### 해결책: 환경으로 우회

> "의지력은 소모품이니까 의지력에 의지하지 않고 환경으로 우회한다."

- 까만 터미널/VS Code 환경에서 작업
- 자극 없는 환경에서 집중력 유지
- 이 검은 화면이 "얼마나 일하기 좋은 환경인지"

## 3. VS Code + Claude Code 설치 및 설정

### 설치 단계

1. **VS Code 실행**
2. **Extension 설치**
    - 좌측 Extension 아이콘 클릭
    - "Claude Code for VS Code" 검색
    - Install 클릭
3. **Claude 계정 연결**
    - 좌측에 생긴 Claude 아이콘 클릭
    - 우측 상단 Claude 버튼 클릭
    - "Claude AI Subscription" 클릭
    - 구글 계정으로 로그인 및 승인
4. **연결 확인**
    - "Build Something Great" 메시지 확인
    - VS Code로 돌아와서 Claude 창 다시 열기

### Git Clone 실습 (워크스페이스 복제)

```bash
git clone https://github.com/Rhim80/do-better-workspace.git
```

**클론 후 작업:**

1. 클론된 폴더 위치 확인 (보통 C드라이브)
2. VS Code에서 File > Open Folder
3. do-better-workspace 폴더 선택
4. 좌측에 폴더 구조가 나타남

> "이 창은 코딩을 하는 게 아니고 뭐든지 그냥 AI랑 얘기하는 것처럼 뭐든지 얘기하면 돼요. '안녕. 나 누구게?' 하면 그냥 AI라고 생각하시면 됩니다."

## 4. do-better-workspace 폴더 구조 (Johnny Decimal)

### 폴더 구조 설명

```
do-better-workspace/
├── 00-inbox/          # 임시 저장, 미분류
├── 00-system/         # 템플릿, 가이드, 설정
│   ├── 01-templates/
│   ├── 02-scripts/
│   └── 03-guides/
├── 10-projects/       # 진행 중인 프로젝트
├── 20-operations/     # 운영 업무 (HR, 행정)
├── 30-knowledge/      # 축적된 지식, 인사이트
├── 40-personal/       # 개인 기록 (daily, weekly, ideas)
│   ├── 41-daily/
│   ├── 42-weekly/
│   └── 46-todos/
├── 50-resources/      # 참고 자료, 데이터
└── 90-archive/        # 완료/보관 자료
```

| 번호 | 폴더명 | 용도 |
| --- | --- | --- |
| 00 | inbox, system | 임시 저장, 시스템 설정 |
| 10 | projects | 시작일/종료일이 있는 프로젝트 (캠페인, 신제품 출시 등) |
| 20 | operations | 지속적인 운영 관리 (모니터링, 리포트 등) |
| 30 | knowledge | 지식 아카이브 |
| 40 | personal | 개인 노트 (데일리, 위클리 등) |
| 50 | resources | 참고 자료, 샘플 데이터 |
| 90 | archive | 완료/중단 프로젝트 |

### 핵심 원칙

> "10번대는 다 시작일이 있고 종료일이 있는 것들만 관리하는 거예요. 20번대는 지속적인 운영을 하는 거. 회사를 떠나기 전까지 계속 관리하는 것들."

- 폴더 앞에 숫자를 붙여 마킹
- 도서관에서 사용하는 시스템 기반
- AI가 직관적으로 이해하기 좋은 구조
- 팀별로 개인화하여 수정 가능

## 5. 다운로드 폴더 정리 실습

### 실습 과정

**1단계: 현황 파악**

```
다운로드 폴더 확인해줘
```

→ 파일 개수, 용량, 유형별 분류 확인

**2단계: 계획 수립**

```
이 파일들을 유형별로 정리하는 계획을 세워줘
```

→ 설치 파일, 문서, 이미지, 프로젝트 등으로 분류 계획

**3단계: 실행**

```
계획대로 정리해줘
```

→ 실시간으로 폴더 생성 및 파일 이동

### 실습 결과

- 설치 파일 이동
- 문서 파일 이동
- 이미지 파일 분류
- 프로젝트 파일 이동
- 결과 리포트 자동 생성

> "너무 신기하죠? 이미 폴더들이 쫙 생겨져 있고 지가 만들었고 하나씩 막 옮기는 거 보면... 이게 바로 에이전트입니다."

**핵심 인사이트:**

> "내 파일을 핸들링할 수 있다라는 게 얼마나 강력한 거구나라는 거를 아시면 좋겠다는 거예요."

> **상세 실습 가이드**: `00-system/claude-code-practice-guide.md` 참조

### 상황 설정

**금요일 오후 3시.**

사무실에서 기획 업무를 마무리하고 있다. 성수 매장에서 외국인 관광객들이 줄을 서고 있다는 슬랙이 올라왔다.

슬랙 알림이 울린다.

> **규림님**: 원님, 대표님이 월요일 오전에 2월 실적 리뷰하고 3월 벚꽃 시즌 전략 논의하자고 하셨어요
>
> 성수뿐 아니라 북촌이랑 온라인도 같이 봐주세요
> 전채널 매출 데이터랑 재고 현황 분석해서 인사이트 정리해주실 수 있어요?
>
> 대표님 스타일 아시잖아요. 숫자 나열하면 "그래서 뭐?" 하시고
> 단순 보고서는 넘기시고 스토리가 있어야 넘어가요
>
> 특히 외국인 관광객 데이터 중요해요
> 벚꽃 시즌에 인바운드 어떻게 대응할지 근거가 필요하거든요

### 골뱅이(@) 명령어 활용

- `@` + 파일명으로 특정 파일 참조
- Tab 키로 자동완성
- 예: `@newmix_sales_seongsu_2602.csv 분석해서 2월 매출 현황 알려줘`

### 미션 1: 성수 매출 현황 + 1월 대비 성장

```
"@newmix_sales_seongsu_2602.csv 랑 @newmix_sales_seongsu_2601.csv 두 파일 분석해줘.
2월 매출 현황 전체 정리하고, 1월 대비 성장률도 비교해줘.
주차별 추이, 카테고리별 매출, 베스트셀러 TOP 5까지 한 번에 봐줘"
```

### 미션 2: 고객 국적별 분석

```
"@newmix_sales_seongsu_2602.csv 에서 고객 국적별 분석해줘.
국적별 매출 비중, 인기 상품 차이, 평균 객단가 비교하고,
외국인이 주로 오는 시간대가 한국 고객이랑 다른지도 봐줘"
```

### 미션 3: 재고 관리 & 벚꽃 시즌 준비

```
"@newmix_inventory_data.csv 재고 현황 전체 분석해줘.
재고일수 7일 미만 긴급 품목 우선으로 정리하고,
3월 벚꽃 시즌에 관광객 30% 증가 가정하면 지금 재고로 며칠 버틸 수 있는지도"
```

### 미션 4: 대표 보고용 1페이지 요약

```
"지금까지 분석한 내용으로 대표님 보고용 1페이지 요약본을 만들어줘"
```

> "프레임워크나 템플릿화시켜 놓으면 자료만 들어왔을 때, 그걸로 긁어서 바로 붙여서 '이거대로 분석해 봐' 하는 게 되는 거예요."

## 7. 토큰과 컨텍스트 윈도우

### 토큰(Token)

- AI가 텍스트를 처리하기 위해 나누는 최소 단위
- 한 단어보다 작고, 글자보다 큰 조각
- AI가 이해하는 의미의 최소 단위

**예시:**

- "I don't like flying" = 5 토큰 (I, don, 't, like, flying)
- "darkness" = dark + ness (2 토큰)

### 컨텍스트 윈도우(Context Window)

- AI가 한 번에 기억하고 처리할 수 있는 최대 용량
- AI의 최대 기억력
- Claude Code: 약 20만 토큰 (약 500페이지 분량)

**중요한 개념:**

- 입력값 + 출력값 합계가 컨텍스트 윈도우
- 내가 정보를 많이 주면 → 얘가 생각할 수 있는 컨텍스트가 줄어듦

> "채팅 더 이상 못 친다거나, 애가 멍청해져 있을 때가 있어요. 그거는 컨텍스트 윈도우 범위를 넘어서서 그런 거예요."

### 컨텍스트 관리 명령어

| 명령어 | 용도 | 사용 시점 |
| --- | --- | --- |
| `/context` | 현재 컨텍스트 사용량 확인 | 수시로 체크 |
| `/compact` | 맥락 압축 | 50% 넘어가면 실행 권장 |
| `/clear` | 맥락 초기화 | 새 프로젝트 시작 시 |

> "작업을 하다가 연속적인 작업을 할 때는 /compact를 중간중간 해주시는 게 좋아요. 맥락을 잃지 않고 계속 일을 하려면."

## 8. 커스텀 커맨드 활용

- 워크스페이스 초기 설정

- CLAUDE.md 파일 자동 생성

> "이 사람은 이렇고 이런 폴더 구조를 사용하고 있고 이런 커맨드들을 갖고 있다라는 거를 인지하고 있는 상태로 되는 거예요."

### /daily-note

- 데일리 노트 생성/업데이트
- "오늘 작업한 내용 업데이트해줘"로 자동 기록
- 업무일지가 자동으로 생성

**사용 예:**

```
/daily-note 오늘 작업한 내용 업데이트해줘
```

> "얘랑 만약 다 같이 일을 한다면은 얘한테 물어본 내용, 정리한 내용 이런 것들 다 알아서 정리가 되는 템플릿을 만들어 놓은 거예요."

### /daily-review

- 어제 작업 리뷰 및 오늘 할 일 정리
- 아침에 실행하면 좋음

### /todo, /todos

- 할 일 추가 및 확인
- 액티브 투두 관리
- 해야 될 것들을 계속 업데이트

### /thinking-partner (추천!)

- 답변 생성 없이 질문만 하는 사고 파트너
- 회고, 고민 정리에 활용
- 의도적으로 결과물을 내려고 하지 않음

> "보통의 AI는 자꾸 무슨 파일을 생성하거나 결과물을 내려고 하는데, 얘는 의도적으로 절대 답변을 생성하지 말고 내 고민을 계속 물어보면서 생각의 파트너로 만들어 놓은 거예요."

**실제 사용 사례:**

```
/thinking-partner 신제품 캠페인 방향에 대해 고민 중이야
```

→ 답변 대신 질문으로 사고를 확장

## 9. 노션(Notion) 연동 설정

### API 설정 단계

1. **notion.so/my-integrations 접속**
2. **"새 API 통합" 클릭**
3. **설정:**
    - 통합 이름: "Claude" 또는 원하는 이름
    - 워크스페이스 선택
    - 유형: Internal (기본값)
4. **저장 후 "통합 시크릿(토큰)" 복사**

### Claude Code 연동

```
노션 핸들러 스킬 세팅하려고 해. 토큰 줄게 [토큰 붙여넣기] 보안 설정 잘 해줘
```

→ 필요한 패키지 설치 (PIP, 파이썬 등)
→ 환경 변수로 토큰 안전하게 저장

### 노션 페이지 권한 설정

1. 노션에서 새 페이지 생성
2. 점점점(...) 메뉴 > 연결 > Claude 선택
3. 해당 페이지 및 하위 페이지에 Claude 접근 권한 부여

> "메인 페이지에다가 권한을 연결해 놓으면 제 노션 안에 있는 모든 페이지를 클로드가 연결 가능해요."

### 데이터베이스 생성 실습

**페이지 ID 복사:**

- 노션 페이지 URL 뒤의 숫자가 페이지 ID
- 예: notion.so/페이지이름-**abc123def456**

**명령어:**

```
[페이지ID] 이 노션 페이지에 SNS 콘텐츠 캘린더 데이터베이스 만들어줘. 노션 스킬로 만들어.
```

**자동 생성된 속성:**

- 제목, 채널
- 상태 (작성 전/작성 중/완료/발행)
- 카테고리, 담당자
- 발행일, 인게이지먼트

> "데이터베이스를 속성값 다 지정해 가지고 만든다는 거는 진짜 어려운 일인데 그게 자연어로 처리가 된다는 것도 보여드리고 싶었어요."
