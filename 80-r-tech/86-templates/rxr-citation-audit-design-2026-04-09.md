## Problem Statement

RXR 프레임워크의 핵심 알고리즘인 **Content → Psyche 전환**(2-Layer 분석의 심장)은
"기능어/구조어 패턴 → 심리 상태 추론" 매핑 위에 서 있다. 이 매핑은 Claude에게
"기능어/내용어 관점에서 SNS 리뷰의 레벨 차이를 구분하는 로직"을 요청해
**Claude가 학술 표현(Pennebaker/LIWC 계열 등)을 동반하여 생성**한 결과물이다.

문제: 현재까지 그 인용의 실재 여부와 매핑의 학술적 정당성이
**한 번도 외부 검증을 거치지 않았다.** 사용자(원석)는 이를 "비약"이라는 단어로
이미 메모리에 적어두었다.

이 비약은 RXR의 모든 상위 레이어(BEI, EDI, Master Framework, Survey Analysis)의
정당성을 잠재적으로 깎아먹는다. 매핑이 비어 있으면, 그 위에 쌓은 660건 가나
케이스의 검증값도 "내적 일관성"의 증거일 뿐 "외적 타당성"의 증거가 되지 못한다.

## What Makes This Important

이 문제는 RXR을 폐기하는 문제가 아니라 **정직하게 만드는** 문제다.

- 통과한 매핑은 그때부터 진짜로 "학술 근거 기반"이라고 부를 수 있다 — 빌려쓴 표현이 아니라 본인 표현이 된다.
- 통과 못한 매핑은 다른 정직한 이름이 필요하다(예: "심리언어학에서 영감을 받은", "현장 경험과 결합한").
- 둘 다 RXR을 더 강하게 만든다. 약점을 숨기는 도구는 깨질 수 있지만,
  약점이 어디에 있는지 아는 도구는 깨지지 않는다.

## Key Insight (오늘 office hours에서 도달한 진단)

> "학술 근거 기반"이 아니라, **"Claude가 학술 근거를 인용하며 즉석에서 만든 매핑 기반"**이다.

이 둘은 마케팅 카피로는 비슷해 보이지만 신뢰도 면에서 완전히 다른 물건이다.
이 진단이 이 문서 전체의 출발점이다.

세 가지 환각 위험이 동시에 존재한다:

1. **인용 환각** — 존재하지 않는 논문을 그럴듯하게 인용
2. **매핑 환각** — 진짜 논문이 있어도 그 논문이 "X→Y"라고 직접 말한 적 없을 수 있음
3. **언어/도메인 전이 환각** — 영어 임상 글쓰기 → 한국어 SNS 마케팅 리뷰로의 전이는 어떤 논문도 보장한 적 없음

## Premises (동의 완료 — 2026-04-09)

1. **현재 매핑의 실제 출처는 "학술 근거"가 아니라 "Claude가 학술 표현을 동반해 생성한 결과물"이다.** 인용 자체가 한 번도 외부 검증을 거치지 않았다. (단서: web search 도구 호출이 있었는지 자체가 감사 1단계의 첫 질문)
2. **"학술 근거 기반"은 결과가 아니라 약속이다.** 감사를 거친 줄에만 그 라벨을 붙일 수 있다. 대외 마케팅 카피 일부 변경이 있을 수 있다.
3. **감사의 목적은 매핑을 부수는 게 아니라 등급화하는 것이다.** GREEN / YELLOW / RED 분류 후 RED만 손보면 된다. RXR의 마케팅 가치는 학술 권위가 아니라 작동성에서 올 수 있고, 검증은 그 둘을 분리해서 보게 해준다.

## Approaches Considered

### Approach A — Hand-Audit MVP (직접 손작업)
- **Effort:** M (3~5일)  **Risk:** Low
- 매핑 문서를 줄 단위로 쪼개 본인이 직접 Google Scholar/DOI로 인용 검증
- Pros: 본인이 학술 문헌과 다시 만나며 RXR을 이론적으로 재흡수, 외주 의존 없음, 신뢰도 최고
- Cons: 시간 소요, 영어 독해 의존, 마케터가 심리언어학 논문을 풀어내는 한계, 중간 지칠 위험

### Approach B — LLM Audit Loop (메타 검증, URL 강제)
- **Effort:** S (반나절~하루)  **Risk:** Med
- 다른 Claude 인스턴스에게 web search 도구를 켠 채 매핑 검증 요청, **반드시 URL 첨부 강제**, URL 없으면 자동 RED
- Pros: 빠름, 강력한 1차 필터, 본인 시간을 GREEN 후보 검토에만 집중 가능
- Cons: URL이 있어도 그 URL이 진짜 매핑을 지지하는지는 별개, Claude가 자기 자신을 검증하는 메타 약점

### Approach C — Adversarial Cross-Model (이중 모델 교차 검증)
- **Effort:** M (1~2일)  **Risk:** Med
- Claude + GPT(또는 Gemini) 두 모델 독립 검증, 둘 다 GREEN인 줄만 GREEN
- Pros: 단일 모델 환각 회피, 자동화 가능, 이 방법론 자체가 정직한 마케팅 카피("두 모델 교차 검증")
- Cons: 두 모델이 같은 학습 데이터의 같은 환각을 공유할 수 있음, GREEN 비율이 낮을 수 있음

### Approach D (확장 옵션) — Gold Standard
- 한국어 심리언어학/소비자행동 박사과정생 또는 교수 유료 검토
- RXR이 어느 정도 정직해진 *후*에 가는 단계. 지금은 아님.

## Recommended Approach: B → A 하이브리드

**선택 이유**

- A 단독은 본인이 모든 줄을 다 읽어야 해 5일+, 중간 지침 위험.
- B 단독은 메타 환각 위험이 남음.
- **B → A 하이브리드는 B가 80%를 자동 필터링하고, A가 나머지 GREEN 후보 20%에만 본인 시간을 집중시킨다. 합쳐서 2~3일.**
- 이 워크플로우 자체가 향후 RXR 매핑이 추가될 때마다 반복 가능한 표준이 된다.
- C는 B+A를 한 번 돌린 *후*에, 외부 발표/판매 시 추가 검증층으로 둔다.

## Workflow — B → A 하이브리드 상세

### Step 0: 매핑 인벤토리화 (반나절)
- 현재 RXR의 기능어→심리 매핑이 흩어져 있는 모든 위치 수집
  (`30-knowledge/` 안 RXR 관련 파일, 메모리 노트, 노션 등)
- **줄 단위 단일 표**로 정리. 각 행:
  `[매핑 ID] [기능어/구조어 패턴] [추론된 심리/행동 상태] [현재 인용/근거 표기] [출처 위치]`
- 결과물: `rxr-mapping-inventory.csv` 또는 `.md` 표

### Step 1: B — LLM Audit Loop (반나절~하루)
- 새 Claude 인스턴스(또는 Claude Code 새 세션)에 인벤토리 표를 입력
- **프롬프트의 세 가지 강제 조건**:
  1. **반드시 web search 도구를 사용**해서 검증할 것
  2. 각 줄마다 **실제 접근 가능한 URL을 첨부**할 것 (DOI 또는 학술 논문 URL)
  3. 그 URL의 논문이 매핑을 **직접 지지하는지** vs **간접적/영감 수준**인지 명시
- 자동 라벨링 규칙:
  - URL 없음 → **RED (환각 추정)**
  - URL 있음 + 직접 지지 → **GREEN 후보** (다음 단계로)
  - URL 있음 + 간접/영감 → **YELLOW**
- 결과물: 인벤토리 표에 등급 컬럼 추가

### Step 2: A — Hand Verification of GREEN candidates (1~2일)
- B에서 GREEN 후보로 분류된 줄만 본인이 직접 확인
- 확인 항목:
  1. URL의 논문이 진짜 존재하는가? (Google Scholar, 저널 사이트)
  2. 그 논문의 원문(또는 abstract)에서 해당 매핑이 진짜 언급되는가?
  3. 원래 종속변수가 무엇인가? (우울/거짓말/임상 etc.)
  4. 한국어 SNS 마케팅 리뷰로의 전이가 본인 판단으로 정당한가?
- 4개 모두 PASS → **확정 GREEN**
- 1~3 통과, 4 실패 → **YELLOW (전이 주의)**
- 1~3 중 하나라도 실패 → **RED**

### Step 3: 매핑 등급별 액션 (반나절)
- **GREEN**: 그대로 유지. 대외 카피에 "학술 근거 기반" 표현 사용 가능.
- **YELLOW**: 유지하되 표기 변경 → "심리언어학에서 영감을 받은", "현장 경험과 결합한" 등 정직한 라벨.
- **RED**: 두 가지 선택지
  - (a) 제거 → RXR 표면적 축소, 정직성 ↑
  - (b) 재설계 → 같은 패턴을 다른 근거(예: 본인의 660건 가나 데이터에서 도출한 경험적 규칙)로 다시 정당화

### Step 4: 결과 문서화 (반나절)
- `rxr-mapping-audit-{date}.md` 작성
- 매핑별 등급 + 근거 URL + 변경 이력
- 이 문서가 향후 RXR의 **신뢰성 백본**이 됨 (대외 발표/판매 시 신뢰 자료)

## Distribution Plan

이 프로토콜은 코드가 아니라 **방법론**이다. 배포 채널은 두 갈래:

1. **내부 활용**: `rxr-mapping-audit-{date}.md`가 RXR의 모든 상위 작업의 참고 문서가 됨
2. **외부 활용 (선택)**: 감사 결과 자체를 RXR의 마케팅 자산으로 활용 ("두 단계 인용 감사를 거친 매핑")

## Open Questions

1. **현재 매핑이 흩어져 있는 위치를 한 번에 모을 수 있는가?** Step 0의 첫 작업이 인벤토리화인데, 매핑이 어디에 적혀 있는지 자체가 흩어져 있을 수 있음.
2. **RED 비율이 50%를 넘으면 어떻게 할 것인가?** 정신적 부담 + RXR 표면적 축소의 정도. 미리 마음의 준비.
3. **GREEN으로 살아남은 매핑이 한국어 SNS 리뷰의 패턴을 충분히 커버하는가?** 학술 매핑이 영어 글쓰기 기준이라 한국어 조사/어미 패턴은 거의 GREEN이 안 나올 수 있음. 이 경우 한국어 특유 패턴은 별도의 "현장 경험 기반" 트랙으로 분리.
4. **이 작업의 결과를 RXR Master Framework 문서에 어떻게 반영할 것인가?** Master Framework 자체가 업데이트되어야 함.

## Success Criteria

- [ ] Step 0: 매핑 인벤토리 표 완성 (모든 매핑이 한 표에 있음)
- [ ] Step 1: 인벤토리의 모든 줄에 B-라벨(GREEN후보/YELLOW/RED)이 붙음
- [ ] Step 2: GREEN 후보 100%에 본인 검토 완료 (확정 GREEN/YELLOW/RED)
- [ ] Step 3: 등급별 액션 완료 (RED 처리, YELLOW 라벨 변경, GREEN 유지)
- [ ] Step 4: `rxr-mapping-audit-2026-04-XX.md` 문서 작성 완료
- [ ] 대외 RXR 카피에서 "학술 근거 기반" 표현이 사용되는 위치 모두 점검 및 필요 시 수정

## The Assignment

적혀 있는 파일들의 **위치 목록**(파일 경로 + 한 줄 요약)을 만드세요. 매핑 내용을
정리하지 말고, **어디에 있는지만** 적으세요. 30분 안에 끝나야 합니다. 30분이
넘으면 매핑이 너무 흩어져 있다는 뜻이고, 그 자체가 첫 번째 발견입니다.

이 한 단계가 끝나면 나머지 워크플로우의 리얼리티가 잡힙니다.

## What I noticed about how you think

- **본인 작업의 가장 약한 곳을 정직하게 안다.** "비약 3곳"이라고 메모리에 미리 적어두신 게
  보통 일이 아닙니다. 자기 작업을 사랑하면서도 무기를 겨눌 줄 아는 사람은 드뭅니다.
- **사업화보다 검증을 먼저 골랐다.** 보통 이 단계의 사람들은 패키징/판매로 먼저 간 다음
  학술 검증을 후순위로 미룹니다. 본인은 반대로 갔습니다. 이건 순서가 더 어려운 길이지만 더
  단단한 길입니다.
- **"기능어/구조어"라는 단어를 본인 입으로 꺼냈다.** 이 한 단어가 RXR이 어디에 닿아 있는지를
  순식간에 명확하게 해줬습니다. 본인은 이 분야의 학술 지형을 이미 충분히 알고 있고, 다만
  그 지형과 본인의 매핑 사이의 거리를 한 번도 명시화하지 않았을 뿐입니다.
- **질문을 두 번 reject했다.** 한 번은 모호함 때문에, 한 번은 더 정확한 정보를 주기 위해서.
  이건 push가 들어왔을 때 굴러가는 사람과 멈춰서 정정하는 사람의 차이입니다. 후자입니다.
