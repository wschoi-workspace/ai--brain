## 💼 프로젝트 업데이트 — Project Master 대시보드 (프로젝트 운영 OS 프로토타입)

대표가 전달한 "12시트 Project Master Sheet" 개발 가이드를 검토 → **검증용 HTML 대시보드**로 구체화. 본질은 "예쁜 간트"가 아니라 **입력은 엑셀(팀별 협업), 시각화는 HTML(자동 집계)** 로 역할을 나눈 운영 OS의 뼈대 검증.

- **구조 확정**: 입력·협업 = 엑셀(`00_Brief` + 팀별 탭, 담당자 작성) / 시각화 = HTML 대시보드(엑셀 업로드→Health·진행률·간트 자동). SSOT는 엑셀, 대시보드 state는 JSON.
- **1차 검증목표(대표 결정)** = 정보구조·입력부담 / 단일 프로젝트 먼저. 핵심 통찰: **입력 무게중심은 Brief가 아니라 "팀별 업무행"** — 자동파생(Duration·Delay·Health·진행률 롤업·간트)으로 입력 최소화.
- **그룹 간트(시인성 개편)**: 주차형 타임라인 + **기본 접힘(팀→카테고리 병합 요약)**, 팀/카테고리 클릭 시 개별 업무 상세 드릴다운. 현재주 하이라이트·상태색·마일스톤 점.
- **동적 팀**: 고정 4팀 폐기 → 데이터의 팀을 자동 인식(팔레트+명도대비 색 자동).
