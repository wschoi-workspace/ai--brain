# REM Education System — 교육 체계 설계서

> 목적: REM 현장팀의 체계적 성장을 위한 교육 시스템
> 형태: 레벨별 모듈 + 프로젝트 연동 하이브리드
> 이론 전달: 가이드에 핵심 내장 + 도서 과제 병행
> 시작일: 2026-06-09

---

## 시스템 개요

### 두 개의 축

```
축 0: WHY LAYER (공감 단계)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
L0: "왜 다르게 일해야 하는가?" — 변화의 이유에 대한 공감
모든 레벨의 전제 조건. 납득 없이는 깊이 움직이지 않는다.

축 1: 레벨별 모듈 (Level Track)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
L0(공감) → L1 → L2 → L3 → L4 → L5 → L6 → L7
개인 성장 속도에 맞춰 순차 진행

축 2: 프로젝트 연동 (Project Track)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
프로젝트 시작 시 → 인식 브리프 기반 맞춤 교육
프로젝트 중 → 관찰/실험 미션
프로젝트 종료 → 프로젝트 회고
```

### 교차 작동 방식

| 상황 | Level Track | Project Track |
|------|-------------|---------------|
| 신규 입사 | **L0 WHY LAYER 먼저** → L1 모듈 | 진행 중인 프로젝트에 배정 |
| 프로젝트 시작 | 현재 레벨에 맞는 관찰 미션 | 인식 브리프 + 브랜드 해석 교육 |
| 프로젝트 중 | 일일 복기로 레벨 성장 | 관찰/실험 기록 |
| 프로젝트 종료 | 레벨 평가 (다음 레벨 해금?) | 프로젝트 회고 리포트 |
| 프로젝트 없는 기간 | 이론 학습 + 도서 읽기 | — |

## 축 0: WHY LAYER (공감 단계)

### L0 — 왜 다르게 일해야 하는가 "납득"

**목표:** "왜 관찰까지 해야 하지?", "왜 커피만 만들면 안 되지?"에 대한 답을 스스로 찾는다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L0-01 | 왜 변화가 필요한가 — WHY LAYER | HTML 가이드 (✅ 완성: `L0-why-we-change.html`) |

**핵심 메시지:**
- 이건 "더 열심히"가 아니라 "더 깊게" — 노동 증가가 아닌 해상도 증가
- 기능 경쟁에서 인식 경쟁으로 세상이 바뀌고 있다 (통신사×카카오톡, 은행×토스)
- AI가 대체하는 건 단순 기능. 해석·관찰·감정 전달은 인간이 더 중요해진다
- REM 3년 경력 = 경험 설계자 역량 (일반 F&B 3년과 완전히 다른 이력서)
- "내 행동이 결과를 바꿨다"를 체감할 때 변화가 시작된다

**진행 기준:**
- [ ] L0 가이드 읽기 완료
- [ ] "왜 다르게 일해야 하는지"를 자기 말로 설명할 수 있음

**핵심 설계 원칙 (Master Frame §9):**
1. 평가 시스템처럼 느껴지면 안 된다 — 감시가 아니라 사고 파트너
2. 업무 증가처럼 느껴지면 실패한다 — 더 많이가 아니라 더 깊게
3. 답변보다 사고 유도가 중요하다 — 정답보다 질문
4. 반복 가능한 질문 구조가 중요하다 — 같은 질문을 반복하는 조직

## 축 1: 레벨별 모듈 (Level Track)

### L1 — 매장 단독 운영 "뭘 봤나?"

**목표:** 운영 체크리스트를 따라 안정적으로 오픈/마감할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L1-01 | 온보딩: 프로젝트렌트는 어떤 회사인가 | HTML 가이드 (완성) |
| L1-02 | 오픈/마감 체크리스트 | 매뉴얼 (A 현장 자료 후 제작) |
| L1-03 | 이슈 대응 플로우 | 매뉴얼 |
| L1-04 | 정산 프로세스 | 매뉴얼 |
| L1-05 | REM Bot 사용법 | 실습 가이드 |

**이론 학습:**
- 핵심 내장: "매장 = 미디어" 개념 (온보딩 가이드에 포함)
- 도서 과제: 『지적 자본론』 — 마스다 무네아키

**레벨업 기준:**
- [ ] 체크리스트 없이 오픈/마감 가능
- [ ] 이슈 발생 시 대응 순서를 알고 있음
- [ ] REM Bot에 일일 복기 3회 이상 완료

---

### L2 — 고객 반응 관찰 "왜 그랬을까?"

**목표:** 고객이 어디서 멈추고, 뭘 말하는지 기록할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L2-01 | 관찰의 기술: 뭘 봐야 하는가 | HTML 가이드 |
| L2-02 | 관찰 기록법 (체류/단어/촬영/반응) | 실습 + 템플릿 |
| L2-03 | "왜?"를 묻는 습관 — 공감(Empathy) | HTML 가이드 |

**이론 학습 — 가이드 내장 개념:**
- 파코 언더힐의 관찰법: "입구에서 오른쪽으로 도는 경향", "손 닿는 높이의 상품이 팔린다"
- 디자인 싱킹 Empathy: 행동 관찰 → 이유 추론 → 인사이트 발견
- 가설 검증으로서의 관찰: "인식 목표 → 예상 반응 → 실제 관찰 → 조정"

**도서 과제:**
- 필수: 『나는 왜 이 가게에서 사는가』 — 파코 언더힐
- 선택: 『디자인 씽킹』 1~4장 — 팀 브라운

**읽기 가이드 질문 (도서 읽으면서 생각할 것):**
1. 파코 언더힐이 발견한 패턴 중, 우리 매장에서도 관찰할 수 있는 것 3가지는?
2. 이번 프로젝트에서 가장 관찰하고 싶은 고객 행동 1가지는?
3. "왜?"를 3번 연속 물었을 때 도달한 인사이트가 있다면?

**레벨업 기준:**
- [ ] 관찰 기록 10건 이상 (REM Bot 경유)
- [ ] "왜?" 추론이 포함된 관찰 기록 5건 이상
- [ ] 관찰에서 발견한 패턴 1건 이상 팀 공유

---

### L3 — 브랜드/콘텐츠 설명 "어떤 인식이?"

**목표:** 브랜드의 매력을 자기 말로 전달할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L3-01 | 인식 설계 개념: 정보 vs 인식 | HTML 가이드 (완성: perception-brief-guide) |
| L3-02 | 기억되는 메시지의 6원칙 (SUCCESs) | HTML 가이드 |
| L3-03 | 설득의 6원칙 — 현장 적용법 | HTML 가이드 |
| L3-04 | 브랜드 해석 훈련: 질문 기반 | 실습 (REM Bot 활용) |

**이론 학습 — 가이드 내장 개념:**
- 칩 히스의 SUCCESs 모델: 단순성, 의외성, 구체성, 신뢰성, 감성, 스토리
  - 현장 적용: "이 멘트는 6가지 중 뭘 충족하는가?" 체크
  - 예시: "독일식 살라미" (구체성만) vs "10년 수련, 2개월 숙성" (구체성+신뢰성+스토리)
- 치알디니 6원칙: 상호성, 일관성, 사회적 증거, 호감, 권위, 희소성
  - 현장 적용: 시음=상호성, 셰프 스토리=권위, "이번 시즌만"=희소성
- 인식 목표의 "카톡 테스트": 친구한테 카톡으로 보낼 수 있는 말인가?

**도서 과제:**
- 필수: 『스틱!』 — 칩 히스, 댄 히스
- 선택: 『설득의 심리학』 — 로버트 치알디니

**읽기 가이드 질문:**
1. 우리 매장에서 가장 "스티키(기억에 남는)" 멘트는 무엇이고, SUCCESs 중 어떤 원칙을 충족하는가?
2. 이번 프로젝트의 인식 목표를 SUCCESs로 검증하면 몇 개를 충족하는가?
3. 치알디니 6원칙 중, 이번 프로젝트에서 가장 활용할 수 있는 원칙은?

**레벨업 기준:**
- [ ] 브랜드 해석을 자기 말로 설명할 수 있음 (REM Bot 테스트)
- [ ] 멘트 실험 3건 이상 (변경 전후 반응 비교 기록)
- [ ] "이 멘트가 먹힌 이유"를 이론으로 설명할 수 있음 1건 이상

---

### L4 — 작은 실험 기획 "다음엔 어떻게?"

**목표:** 멘트/진열/동선 변경을 기획하고, 반응을 비교할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L4-01 | 실험 설계의 기본: 하나만 바꾸고, 비교하고, 기록한다 | HTML 가이드 |
| L4-02 | A/B 실험 템플릿: 가설→실행→관찰→결론 | 템플릿 + 실습 |
| L4-03 | 경험 경제: 가치의 사다리 이해 | HTML 가이드 |

**이론 학습 — 가이드 내장 개념:**
- 경험 경제 (조셉 파인): 원자재→상품→서비스→경험→변환
  - 현장 적용: "내 설명 한마디가 가치의 사다리를 한 칸 올린다"
  - 경험의 4영역: 오락·교육·탈출·미학 → 우리 프로젝트는 어느 영역인가?
- 실험의 3원칙: ①하나만 바꾼다 ②비교 가능하게 ③기록한다

**도서 과제:**
- 필수: 『경험 경제』 1~3장 — 조셉 파인, 제임스 길모어
- 선택: 『관찰의 힘』 — 맥스 베이저먼

**읽기 가이드 질문:**
1. 우리 프로젝트의 고객 경험은 "오락/교육/탈출/미학" 중 어디에 해당하는가?
2. 가치의 사다리에서, 우리 매장은 "서비스"와 "경험" 중 어디에 있는가?
3. 이번 프로젝트에서 직접 설계한 실험 1건의 가설-결과-교훈은?

**레벨업 기준:**
- [ ] 실험 기획서(가설→실행→결과) 3건 이상
- [ ] 실험 결과에서 패턴 발견 1건 이상
- [ ] "이 실험을 왜 이렇게 설계했는가" 설명 가능

### L5 — 프로젝트 회고 "패턴을 발견하다"

**목표:** 프로젝트 전체를 복기하고 패턴을 발견할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L5-01 | 프로젝트 회고 프레임워크 | HTML 가이드 + 템플릿 |
| L5-02 | 반성적 실천: 경험→복기→패턴화 | HTML 가이드 |
| L5-03 | 데이터 읽기: 관찰 데이터에서 인사이트 추출 | 실습 |

**이론 학습 — 가이드 내장 개념:**
- 반성적 실천 (도널드 쇤): "전문가는 경험이 아니라 경험을 복기하는 능력으로 성장한다"
- 실천 공동체 (에티엔 웽거): "좋은 관찰을 공유하면 조직 전체가 성장한다"
- 센스메이킹: 데이터에서 숫자가 아니라 "의미"를 읽는 능력

**도서 과제:**
- 필수: 『일의 감각』 — 크리스티안 마두스베르그
- 선택: 『피드백 이야기』 — 더글러스 스톤, 쉴라 힌

**레벨업 기준:**
- [ ] 프로젝트 회고 리포트 1건 이상 작성
- [ ] 회고에서 도출한 "다음 프로젝트 적용 포인트" 3개 이상
- [ ] 팀 주간 회고에서 인사이트 공유 3회 이상

### L6 — 매출/경험 개선 제안 "연결하다"

**목표:** 매출과 경험을 연결해서 구체적 개선안을 제시할 수 있다

**교육 내용:**
| 모듈 | 내용 | 형태 |
|------|------|------|
| L6-01 | 매출 데이터와 현장 관찰의 교차 분석 | 실습 |
| L6-02 | 인식 KPI: "인식이 매출로 연결되는 구조" | HTML 가이드 |
| L6-03 | 개선 제안서 작성법 | 템플릿 + 실습 |

**이론 학습 — 가이드 내장 개념:**
- RXR 분석의 기본: 온라인 멘션에서 인식 패턴 읽기
- 인식→행동→매출 연결 구조

**레벨업 기준:**
- [ ] "이 관찰이 매출에 이렇게 연결된다" 분석 1건
- [ ] 구체적 개선 제안 → 실행 → 결과 확인 1건

## 축 2: 프로젝트 연동 (Project Track)

### 프로젝트 시작 시 — 킥오프 교육

| 제공 항목 | 내용 | 누구에게 |
|----------|------|---------|
| 인식 브리프 | 인식 목표 + 관찰 포인트 + 실험 포인트 | 전원 |
| 브랜드 해석 가이드 | 참여 브랜드별 매력 포인트 + 해석 질문 | 전원 |
| 레벨별 미션 | L1: 체크리스트 숙지 / L2: 관찰 기록 5건 / L3: 멘트 실험 1건 / L4: 실험 기획 1건 | 개인별 |
| 관련 이론 요약 | 이번 프로젝트와 연결되는 개념 1~2개 | 전원 |

### 프로젝트 중 — 일일/주간 루프

| 주기 | 활동 | 도구 |
|------|------|------|
| 매일 | 일일 복기 (REM Bot 3개 질문) | REM Bot |
| 매일 | 관찰 기록 | REM Bot 구조화 |
| 주간 | 주간 회고 (팀 전체) | 회고 템플릿 |
| 주간 | 이번 주 이론 한 토막 (REM Bot이 제공) | REM Bot |

### 프로젝트 종료 — 회고 + 레벨 체크

| 활동 | 내용 |
|------|------|
| 프로젝트 회고 | 인식 목표 달성 여부 + 관찰 패턴 + 실험 결과 + 다음 적용 포인트 |
| 레벨 체크 | 이번 프로젝트에서 레벨업 기준 충족 확인 |
| 성공 아카이브 | 축적된 성공 사례를 팀 자산으로 정리 |
| 도서 과제 확인 | 이번 레벨 도서 읽었는지 + 읽기 가이드 질문 토론 |

## 이론 전달 체계

### 1. 가이드 내장 (최소한 보장)

각 레벨 모듈의 HTML 가이드에 핵심 개념을 풀어서 설명한다.
책을 안 읽어도 핵심은 이해할 수 있도록.

**형태:**
- "핵심 개념 XX" 블록에 이론 배경 + 현장 적용 예시
- 원래 이론가/저자 이름 명시 (학습 동기 유발)
- "더 알고 싶다면" → 도서 안내

### 2. 도서 과제 (깊이 확장)

각 레벨에 필수 1권 + 선택 1권.
읽기 가이드 질문 3개를 함께 제공 (현장 경험과 연결하는 질문).

**형태:**
- 필수 도서: 레벨업 전에 읽기 권장 (강제는 아님)
- 읽기 가이드 질문: "이 책에서 배운 것 중 우리 매장에서 바로 쓸 수 있는 것은?"
- 주간 회고에서 자연스럽게 토론 (강제 발표 아님)

### 3. REM Bot "이번 주 이론 한 토막"

주 1회, REM Bot이 짧은 이론 조각을 던진다.
부담 없이, 호기심을 자극하는 형태.

**예시:**
```
REM: 이번 주 한 토막 💡

파코 언더힐이라는 사람이 20년 동안 매장에서 고객을 관찰했는데,
발견한 것 중 하나가 "시음 후 설명을 들은 고객은 그냥 시음한 고객보다
구매율이 3배 높다"는 거야.

이번 주에 시음 타이밍을 바꿔보면서 확인해볼 수 있을까?
```

## 설계 원칙

### 교육 전달 원칙
1. **책을 안 읽어도 핵심은 이해** — 모든 이론은 가이드에 풀어서 내장
2. **책을 읽으면 깊이가 붙음** — 도서 과제 + 읽기 가이드 질문으로 현장 연결
3. **강제가 아니라 호기심** — REM Bot "한 토막"으로 자연스럽게 이론 노출
4. **레벨은 자연스럽게** — 평가 느낌이 아니라 "이제 이것도 할 수 있게 됐네" 느낌
5. **프로젝트가 최고의 교실** — 이론만으로는 안 된다. 실제 프로젝트 안에서 체험해야

### WHY LAYER 원칙 (Master Frame §6, §9)
6. **납득이 먼저** — "왜 해야 하는지" 공감 없이는 깊이 움직이지 않는다
7. **더 열심히가 아니라 더 깊게** — REM은 노동량 증가가 아닌 관찰 해상도 증가
8. **회사 비전보다 개인 미래** — "회사 때문에 억지로"가 아니라 "이건 내 성장에도 도움"
9. **작은 성공이 철학보다 강하다** — "내 행동이 결과를 바꿨다"를 체감할 때 변화 시작
10. **감시가 아니라 사고 파트너** — 평가 시스템처럼 느껴지면 실패
