Section 02 — Pipeline

7단계 파이프라인 흐름

02

1
Phase 1 — RFP 분석

/rfp-analysis 호출

3개 에이전트 병렬(문서분석 + 리서치 + 평가배점). 자동 모드 감지(일반 / 정부).

Output: rfp-analysis-{name}.md + GO/PASS 판단
2
Phase 2 — 배경 리서치

★ 질문 게이트: 팩트 부족 시 진행 중단하고 질문

3축 리서치(지난행사 + 사회문제 + 경쟁사). 완료 조건: 팩트무기 5개+, 3중Problem 재료, 차별화 2개+.

3
Phase 3 — 전략 브리프

R가이드 7대 설득원칙 적용

SCQA 정의, HIRE 확정, 3중Problem, Promised Land, Wow Moment(정확히 1개), Sparkline(감정곡선 3피크).

Phase 4 — 페이지네이션

분량별 레시피 (50P / 40P / 30P / 20P)

슬라이드별 명세(제목, 핵심메시지, R패턴, 감정강도). 3피크 위치 확정.

5
Phase 5 — 내러티브 아크

/proposal-architect 호출

Phase 0: 탐색 모드(대화형). Phase 1: 10단계 구조화 — WHY → BigShift → 문제해부 → PromisedLand → WhatIs↔CouldBe → Wow → 차별화 → MagicGifts → Success↔Failure → NewBliss.

6
Phase 6 — HTML 제작

design-guide 적용. 1920×1080px, 16:9 FullHD

감정피크 디자인 규칙(Peak1 어두운색, Peak2 극적, Peak3 밝은색). 1개 HTML 파일로 완성.

7
Phase 7 — QA

R가이드 체크리스트 + 발표 설계

제목, 메시지, 수치, 비주얼, HIRE 연결 검토. 발표 설계(15분 / 30분 시간배분). s01부터 순차 검토 → 수정 → 최종 완성.

Section 03 — RFP Analysis

진단 — "발주처의 진짜 의도"를 파악한다

03

2가지 모드를 자동 감지하여 다른 프레임워크를 적용합니다.

Mode A — 일반기업 RFP

감지 조건
민간 기업 RFP, 경쟁 PT
적용 프레임워크
analysis-framework.md
핵심 로직
클라이언트 상황 → 요구사항 정의 → 4관점 차별화(디자인 / CX / 비즈니스 / 시너지) → Winning Point
특징
경쟁 관점: "다른 팀은 어떻게 접근할까?"

Mode B — 정부사업

감지 조건
나라장터, 조달청, 공고번호 키워드
적용 프레임워크
gov-analysis-framework.md
핵심 로직
평가배점 역추론 ★ → 양식×RFP×배점 3자교차 → GO/PASS 판단
특징
"배점 높은 곳에 전략 집중"

핵심 차이: Mode A는 "클라이언트 이해"가 중심, Mode B는 "평가배점"이 중심입니다. 정부사업은 아무리 좋은 제안서도 배점 구조를 못 맞추면 탈락합니다.

Mode B 양식 심층분석 (Phase 5-GOV)

양식 구조 ↔ 평가항목 매핑

숨은 요구 추출 — 세부항목이 많을수록 구체적 답 요구. 자유섹션 = 차별화 기회. 배점 높은 곳에 지면 60% 이상 투자.

Mode B 초안 검토 (Phase 6-GOV)

4단계 평가 + 평가위원 시뮬레이션

✅ 충족 / ⚠️ 부분 / ❌ 미충족 / 🔍 확인. 배점 대비 지면 균형 체크(Under / Over-invested). 평가위원 관점: 첫인상, 차별성, 실행가능성, 증빙.

Section 04 — Strategy Framework

4국면 × 12Step — 전략 설계의 뼈대

04

RFP 분석 후 "GO" 판정 시 진행하는 2단계 전략제언입니다.

1
국면 1 — 진단 심화 / Step 1

Pain Point 발굴

과거 사업 딥리서치. 심사위원이 가려운 지점 정의.

"작년 약점이 올해의 무기."
2
국면 1 — 진단 심화 / Step 2

관통 메시지

Pain을 뒤집은 한 줄. 제안서 전체의 방향타.

3
국면 1 — 진단 심화 / Step 3

Why Us

Pain ↔ 우리 강점 1:1 매칭. 제안서의 뼈대.

4
국면 2 — 전략 설계 / Step 4

JTBD 재정의

Functional / Emotional / Social 3층. 발주처가 진짜 고용하려는 것.

5
국면 2 — 전략 설계 / Step 5

마케팅 전략

STP + OMO + Flywheel. 사업 결과물을 "상품"으로 설계.

6
국면 2 — 전략 설계 / Step 6

컨셉 3안

A(안정형) / B(차별형) / C(혁신형). 의사결정자에게 선택지 제공.

7
국면 2 — 전략 설계 / Step 7

예산 설계

투찰가 · 마진 · 집행배분. 제출용 / 내부실질 2층 구조.

8
국면 3 — 효과 구조화 / Step 8 ★ 핵심

4-Layer Impact + ROI

L1: 직접성과 / L2: 행동변화 / L3: 파급효과 / L4: 지속가능성. ROI 3종(직접 / 종합 / SROI).

막연한 기대효과 ❌ → 구조적 근거가 있는 효과 설계 ✅
9
국면 4 — 검증·전달 / Step 9

평가위원 설득전략

SCQA + Cialdini. 중심경로(논리) + 주변경로(신뢰).

10
국면 4 — 검증·전달 / Step 10

자체평가

배점표로 보수적 채점. 감점 이유까지 명시.

11
국면 4 — 검증·전달 / Step 11

의사결정 리뷰

3안 나란히 비교 + 결정포인트 5개.

12
국면 4 — 검증·전달 / Step 12

제안서 반영 가이드

4-Layer를 제안서 어느 슬라이드에 어떻게 녹일지 매핑.

가장 중요한 원칙: "~할 것으로 기대됨" ❌ → "이 메커니즘으로 이 가치가 생성된다" ✅. 4-Layer Impact는 막연한 기대효과가 아니라, 구조적 근거가 있는 효과 설계입니다.

Section 05 — Narrative Architecture

설득의 구조 — 10단계 하이브리드 PT

05

7대 프레임워크(Sinek, Raskin, Duarte, Gallo, Miller, Godin)와 8개 전설적 사례(Jobs, Nike, Airbnb 등)에서 추출한 통합 공식입니다.

Phase A

공감과 맥락

40% / 5~8장 / Step 1~3

WHY 선언 → Big Shift → 시장문제 해부(현상→원인→구조적한계→방치의대가)

Phase B

비전과 해법

30% / 4~6장 / Step 4~6

Promised Land → What Is↔Could Be → Wow Moment (★정확히 1개)

Phase C

우리만의 답

25% / 4~6장 / Step 7~8

차별화 강점(공감+권위+관점) → Magic Gifts(Rule of Three)

Phase D

행동 유도

5% / 2~3장 / Step 9~10

Success↔Failure → New Bliss

Phase A에 40%를 투자하는 이유: 문제를 충분히 해부하지 않으면 솔루션에 대한 갈망이 생기지 않습니다. "왜 필요한가?"가 납득되어야 "뭘 하겠다"가 먹힙니다.

3중 Problem 구조 — 문제를 3층으로 쌓아야 설득력이 생깁니다.

External

표면적 문제

"매장 방문객 감소" — 누구나 보는 것. 도입부에서 공통 인식 형성.

Internal ★

감정적 문제

"브랜드가 잊혀진다는 불안" — 실제 구매 동기. 이걸 짚어야 설득력이 생깁니다.

Philosophical

철학적 문제

"좋은 브랜드가 공간 하나 때문에 저평가" — 부당함 호소. 심사위원의 사명감을 자극.

뇌과학 설득원리 핵심 3개

Loss Aversion

손실이 이득의 2배 강력

Step 3 방치의대가, Step 9 Failure 슬라이드에 적용. "하지 않으면 잃는 것"을 먼저.

Peak-End Rule

정점과 끝만 기억

Step 6 Wow Moment + Step 10 New Bliss에 적용. 감정 설계의 핵심 원리.

Story Recall

스토리 기억 63% vs 데이터 5%

모든 수치를 스토리 안에 넣기. 숫자 단독 제시는 기억되지 않습니다.

Section 06 — Quality Gate

검수 — 답을 주지 않고 질문으로 깊이를 높인다

06

제안서 초안이 나오면 /proposal-critic이 질문형 코칭으로 사고의 깊이를 검증합니다.

1

1단계: WHY — 왜 해야 하는가

목적/이유가 명확한가? "왜 지금 이걸 해야 하는가"에 답할 수 있는가? 근거가 있는가?

막연한 "좋을 것 같아서" ❌ → 근거 있는 WHY ✅
2

2단계: WHAT/HOW — 그래서 뭘 어떻게

이유와 실행방향의 논리적 연결. "이유 A → 그래서 실행방향 B"의 관계가 명확한가? 빠진 논리 고리는?

3

3단계: 통합적 사고 — 실행의 현실성

일정 · 예산 · 난이도 · 리스크를 인식하고 있는가? 누가 실행하는지 명확한가?

절대 하지 않는 것

직원 대신 내용 채워주기
"이렇게 고치면 됩니다" 답 주기
기획서 대신 써주기

반드시 하는 것

질문만 던진다
판단은 직원이 한다
스스로 생각하게 만든다
좋은 포인트는 즉시 인정

핵심: 3가지 필수 조건 — WHY 명확+근거 ✓ / 연결성 논리적 ✓ / 실행방안 현실성 ✓ — 을 모두 충족하면 "제출해도 됩니다." 하나라도 부족하면 추가 질문으로 보완합니다.
