Section 05 — Connections

솔루션과 스킬이 어떻게 연결되는가

05

6개 솔루션은 독립 시스템이 아니다. 데이터와 인사이트가 위에서 아래로 흐르며, 모닝 루틴이 전체를 매일 연결한다.

전체 의존관계 맵
6개 솔루션의 상호 연결
┌─────────────────────────────────────────────────┐
│              ARISA (경영진 레이어)                │
│  GPT 회의 → 전략 논의 → 프로젝트 방향 설정       │
└──────────────────────┬──────────────────────────┘
                       │ 인식 브리프 ↓  │ 패턴 ↑
┌──────────────────────▼──────────────────────────┐
│              REM OS (현장 레이어)                 │
│  관찰 → 복기 → 실험 → 회고 → 데이터 축적         │
└──────────────────────┬──────────────────────────┘
                       │ 현장 데이터 ↓
┌──────────────────────▼──────────────────────────┐
│           RXR 분석 체계 (데이터 레이어)            │
│  크롤링 → 감성분석 → 언어학 → EMV → 추적          │
└──────────────────────┬──────────────────────────┘
                       │ 분석 결과 ↓
┌──────────────────────▼──────────────────────────┐
│      정부사업 파이프라인 + 브랜드 분석              │
│  /brand-analysis → /rfp-analysis → /gov-proposal │
└─────────────────────────────────────────────────┘

횡단 지원:
  모닝 루틴      ←→  전체 (매일 아침 브리핑)
  인건비 시뮬    ←→  REM OS (운영 비용 설계)
  /psychology-critic  ←→  REM OS + 제안서 (심리학 크리틱)
  /humanize-writing   ←→  모든 문서 산출물
핵심 흐름: ARISA(전략) → REM OS(현장) → RXR(데이터) → 제안서/브랜드 분석(산출물). 모닝 루틴이 전체를 매일 연결하는 접착제 역할.

수직 흐름
전략(ARISA) → 현장(REM OS) → 데이터(RXR) → 산출물(제안서). 위에서 내려온 방향 설정이 아래 레이어의 관찰 기준을 정한다.

역방향 피드백
현장 데이터(RXR 분석)가 경영진 논의(ARISA)로 역류. 패턴이 쌓이면 전략 방향을 수정할 근거가 된다.

횡단 지원 레이어
모닝 루틴, 인건비 시뮬, 심리학 크리틱, 휴머나이저는 특정 솔루션이 아니라 전체 시스템을 가로질러 지원한다.
