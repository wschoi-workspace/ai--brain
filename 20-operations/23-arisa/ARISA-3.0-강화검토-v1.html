3.0의 정의는 끝났다. 비어 있는 건 '무엇을 모아 무엇을 잴까'다

ARISA 3.0 = 조직의 학습 속도를 측정·가속하는 Organizational Learning OS. KPI는 업무 완료율이 아니라 조직의 학습 속도. — 이 정의는 SSOT에 확립됐다(DEFINITION.md:287–309). 남은 과제는 단 하나: 멀티채널 수집 → 사고 트래킹 → 측정·성장의 실행 구조.

진단 ③ — 저장 구조 미정05

"ARISA의 진짜 자산"이라 부른 데이터가 시트의 행(行)에 갇혀 있다

Thinking Data Log = ARISA의 진짜 자산(DEFINITION.md:345). 그러나 실제 저장은 구글시트 행과 mem0 보조 인덱스뿐 — 시간축도 관계도 추적할 수 없는 평면 구조.

지금 · 구글시트
행 단위 누적. 검색·집계는 되지만 "누가 언제 무엇을 어떻게 이해했나"의 관계가 없다
지금 · mem0
직원별 재부상 보조 인덱스. 의미 검색은 되나 시간 변화·오해를 못 잡는다
3.0이 필요로 하는 것
시간축(언제) + 관계(누가·무엇과)를 가진 저장 구조 = 지식그래프

→ "오해지도·학습속도"는 본질적으로 관계·시간 질문이다. 평면 시트로는 구조적으로 답할 수 없다.

강화 프레임06

세 갭을 수집 → 트래킹 → 성장 한 줄기로 잇는다

진단의 세 공백(채널·측정·저장)은 사실 하나의 파이프라인의 세 단계다. 시장에서 검증된 패턴을 각 레이어에 매핑한다.

레이어 1 — 수집

흩어진 채널을 한 포맷으로

텔레그램·Gmail·회의록 → ingest 어댑터

레퍼런스: Matterbridge · Unipile
레이어 2 — 트래킹

사고를 시간·관계로 저장

지식그래프로 오해지도 · 집단지성

레퍼런스: Graphiti · ONA(NetworkX)
레이어 3 — 성장

질문으로 측정·코칭

AI 코치 루프 + 학습 속도 KPI

레퍼런스: BetterUp · 코치 루프

각 레이어는 독립 도입 가능 — 레이어1만으로도 즉시 가치(채널 통합), 레이어2·3은 측정 기능을 점진 활성화.

레이어 2 — 조직 사고 트래킹08

오해지도와 집단지성은 지식그래프에서만 그려진다

Graphiti(MIT) = bi-temporal 지식그래프. 각 사실에 valid_at / invalid_at 시간축 → "예전엔 A로 이해했는데 지금은 B로 배웠다"를 추적. mem0(단기·개인) 위에 Graphiti(장기·조직)를 얹는 2층 구조. MCP 서버 제공으로 Claude와 직접 연동.

③ 오해지도 (Define Cost)
같은 지시 → 직원별 이해를 노드로, 차이를 엣지로. "어디서 어긋나는가"가 그래프로 보인다
④ 집단지성 지도
회의·보고·기획을 엮어 "이번 달 가장 많이 고민한 문제"를 관계망으로 집계
+ 조직망 분석(ONA)

기술 노선 비교 — 의사결정 항목11

그래프를 보조로 둘까, 본체로 올릴까 — fidelity가 가른다

핵심 긴장: 지식그래프의 LLM 자동 추출은 ARISA의 원문 보존·추측 금지 철학과 충돌할 수 있다(과거 Letta를 같은 이유로 기각). 두 노선의 전제를 병기한다.

노선 1 — 보존 우선 보수 · 저위험
원문은 파일·시트에 그대로, Graphiti는 재부상·관계탐색 보조 인덱스로만.
+ fidelity 철학 유지 · 즉시 도입 · 롤백 쉬움
− 오해지도·학습속도 본격 구현엔 한계
노선 2 — Graphiti 본격 강력 · 검증 필요
bi-temporal 그래프를 조직 메모리 1차 저장소로 격상, 오해지도·학습속도 본격 구현.
+ 측정 기능의 잠재력 최대
− LLM 추출이 추측금지와 충돌 → fidelity 게이트 필수

권고: 노선1로 시작 → 측정 본격화 시 노선2 부분 도입(hybrid). 전제 = 원문 raw는 어느 노선이든 항상 보존. 최종 노선 = 이번 결정 대상
