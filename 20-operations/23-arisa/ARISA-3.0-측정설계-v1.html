ARISA 3.0 · 측정 설계 v1 · 2026-06-20

'조직 학습 속도'를 실제로 잴 수 있는 숫자로 내린다

3.0의 KPI 4지표는 지금까지 "초안 산식" 한 줄이었다. 이 문서는 각 지표를 현재 봇이 실제로 수집하는 필드에 매핑해 계산 가능한 스펙으로 만든다. 핵심 발견 — 4지표 중 둘은 지금 데이터로 즉시 측정되지만, 둘은 새로운 수집 흐름이 선행돼야 한다. 즉 "무엇을 잴지"가 "무엇을 모을지(=멀티채널 수집 L1의 우선순위)"를 역산한다.

0 · 데이터 가용성 — 봇이 지금 주는 것 vs 지표가 필요한 것현황 매핑

일일보고봇(daily-report-bot.py)이 구글시트 3탭에 저장하는 실제 필드를 기준으로, 4지표의 측정 가능성을 판정한다.

메타 탭tomorrow · improvement · blockers · decision_needed · support_needed · reflection(good/missing/next) · raw(원문)
mem0직원별 핵심업무 누적 (재부상 인덱스)
KPI 지표필요 입력지금 가능?판정

핵심 인사이트: ③·④가 "추가 수집 필요"라는 사실이 곧 멀티채널 수집(L1)의 우선 수집 대상을 정해준다 — 정식 지시-이해 흐름과 질문 섹션. 측정 설계가 수집 설계를 역산하는 지점이다.

1 · Thinking Resolution — 사고 해상도즉시 측정

지표 ①

같은 사실을 얼마나 또렷이 보는가

즉시 측정
입력
핵심업무 process·output·outcome·issues + reflection — 채점 기준은 항상 raw(원문), 정리본 아님
산식
LLM 루브릭 채점으로 6지표 각 0~3점 (현재 JSON 구조만 있고 채점 로직 미구현)
resolution = mean(clarity, evidence, structure, output_q, outcome_q, decision_q) // 0~3
루브릭 예
(clarity)
0모호어만 — "열심히 했다 / 잘 진행 중"
1대상은 있으나 추상 — "경쟁사를 봤다"
2구체 대상+행위 — "29CM 3개 브랜드 가격대 정리"
3대상+근거+함의 — "…정리, 우리가 중가에 빈자리 확인"
정규화
직원별 baseline 대비 변화로 본다(절대점수 비교 금지) — 주간 이동평균, 4주 기울기 = 성장률
출력
직원별 6지표 레이더 + 해상도 시계열. 대표는 추세(기울기)만, 일별 점수 비노출
fidelity: 점수는 평가가 아니라 관찰. 직원에게는 점수 대신 거울(정리본)만 비추고, 추측 채점 금지(근거 불충분 시 null). 채점은 원문 기준 — 정리본을 채점하면 ARISA가 ARISA를 채점하는 순환오류.

2 · 조직 학습 속도 — 반복하지 않는 속도즉시 측정

지표 ②

같은 실패를 다시 하지 않게 되는 속도

즉시 측정
입력
issues · blockers + 시계열 + reflection.next(다음 보완점) + mem0 누적
산식
이슈를 임베딩 후 클러스터링 → 동일 유형 클러스터의 재발 간격(일). 간격↑ = 학습됨
learning_velocity = Δ(동일유형 이슈 재발 간격) · 신규/반복 이슈 비율
보조축
약속–이행 추적: reflection.next에 적은 보완점이 다음 보고에서 실제 해소됐는가 (의도→행동 폐루프)
윈도우
월 단위 클러스터 재발맵 — 조직 전체 "반복 실패 영역" 히트맵
출력
조직 반복실패 히트맵 + 직원별 재발간격 추세. 집단지성 지도(3.0 진화 ④)의 1차 입력
주의: 이슈 표현이 매번 달라 클러스터링 정확도가 관건 — 동일유형 판정 코사인 임계값을 캘리브레이션(§5)에서 튜닝. 표현만 다른 같은 문제를 놓치면 "학습됐다"는 거짓 신호.

3 · Define Cost — 오해 거리추가 수집 필요

지표 ③

지시가 전달되며 벌어지는 의미의 거리

추가 수집 필요
측정 공백
현재 봇은 직원의 '한 일 보고'만 받는다. 대표 지시 ↔ 그 지시를 받은 직원의 이해, 이 쌍(pair)이 수집되지 않는다. → 오해 거리를 잴 원재료 자체가 없음
필요한
새 흐름
① 대표/팀장이 지시 등록(텍스트) → ② 지시 받은 직원에게 "이 지시를 어떻게 이해했나요?" 1문 → ③ 두 문장의 의미 거리 측정
산식
define_cost = 1 − cos( emb(지시문), emb(이해문) ) // 0=완전일치 … 1=완전오해 누적거리 = Σ 단계별 거리 (대표→팀장→직원, 전파 감쇠)
출력
지시별 오해 점수 + 조직 오해지도 — 어느 단계·누구에서 거리가 벌어지는가(3.0 진화 ③)
단계 도입
1단계(약): 일일보고의 decision_needed·지시 언급에서 약하게 추출 → 2단계(정식): 지시-이해 전용 흐름. 멀티채널(메일·회의 지시)과 직결
fidelity: 거리가 크다고 직원 탓이 아니다 — 지시 자체의 모호성일 수 있다. 오해지도는 양방향 관찰(지시 명료성 ↔ 이해 정확성)로 읽어야 하며, 책임 귀속 도구가 아니라 커뮤니케이션 비용 가시화 도구다.

4 · 질문 수준 — 답에서 질문으로질문보고 전환 선행

지표 ④

보고가 '한 일'이 아니라 '질문'이 되는 정도

부분 가능
측정 공백
현재 일일보고는 "한 일" 중심이라 질문 데이터가 빈약. raw에서 의문문·불확실 표현 비율은 약하게 가능하나 신호가 약함
선행 작업
보고 양식에 질문 섹션 추가 — DEFINITION 진화 ①: "오늘 발견한 가장 중요한 질문 / 잘못 이해했던 것 / 생각이 바뀐 것"
산식
질문 수 / 보고 단위 + 질문 추상도 등급 (LLM 분류)
사실확인 질문 < 해석 질문 < 재정의 질문 // 등급이 올라갈수록 사고 성숙
출력
직원별 질문 등급 분포 + 조직 질문 성숙도 추세 (시간이 갈수록 상위 등급 비율↑가 목표)
설계 메모: 이 지표는 측정이자 동시에 개입이다 — 질문 섹션을 두는 순간 직원은 "질문하는 보고"를 학습한다. 측정 도구가 곧 교육 도구가 되는 3.0의 전형.

5 · 수집 요건 역산 — 측정이 정의한 수집 스펙L1로 연결

위 4지표가 요구하는 데이터를 거꾸로 모으면, 그것이 곧 멀티채널 수집(L1) 어댑터가 채워야 할 필드가 된다.

지표필요 데이터지금 있나추가 수집 액션
① 해상도process·output·outcome·raw있음없음 — 채점 로직만 구현
② 학습속도issues 시계열·reflection.next있음없음 — 클러스터링만 구현
③ Define Cost지시문 ↔ 이해문 쌍없음지시 등록 + 이해 확인 흐름 신설 (메일·회의 지시 포함)
④ 질문수준보고 내 질문빈약보고 양식에 질문 섹션 추가

→ 다음 수집(L1)의 우선순위가 자동으로 정해진다

  • 최우선: 보고 양식 질문 섹션 — 봇 프롬프트 한 단계 추가, 비용 거의 0, 즉시 ④ 활성화
  • 그다음: 지시-이해 쌍 흐름 — 대표/팀장 지시를 텔레그램·Gmail·회의록에서 받아 등록 → ③ 오해지도의 원재료. 멀티채널 수집의 첫 정당화 사유

6 · 캘리브레이션 플랜 — 12명 × 1~2주실데이터 튜닝

산식은 설계 초안이다. 직원 12명의 실제 보고에 돌려 임계값·정규점을 잡아야 비로소 '잴 수 있는' 지표가 된다.

주 0 · 앵커링

R이사 이상본 7건(before=구어 raw / after=정리본)으로 6지표 루브릭 0~3 기준점 고정. fidelity 1차 구현 때 쓴 데이터 재활용

주 1~2 · 채점

12명 실보고에 ①해상도 ②학습속도 채점 적용 → 점수 분포·이상치 확인, 직원별 baseline 산정

튜닝

이슈 클러스터 코사인 임계값, 해상도 정상범위, 채점 신뢰도(LLM 일관성) 보정

산출

지표별 정상범위·이상치 기준 + 직원 baseline 표 → 측정 엔진 v1 확정

fidelity 게이트(전 단계 공통): 모든 채점·거리 계산은 raw 원문 기준. 근거 불충분 시 추측 금지(null 처리). 점수는 직원 통제 수단이 아니라 거울의 해상도 눈금 — 노선2(Graphiti 본격) 채택 시 이 게이트가 LLM 자동추출의 안전장치가 된다.

이 설계가 확정하는 것 · 남기는 결정

  • 즉시 착수 가능: ①해상도 ②학습속도 — 데이터가 이미 있다. 채점·클러스터링 로직 구현 + 주0~2 캘리브레이션
  • 수집 선행 필요: ④질문 섹션(저비용·최우선) → ③지시-이해 흐름(멀티채널 L1의 첫 정당화)
  • 남는 결정 1: 캘리브레이션을 지금 12명으로 착수할지(2.0 데이터 위에서) — Yes면 다음 산출물 = 채점 루브릭 6지표 전체 정의서
  • 남는 결정 2: 질문 섹션을 일일보고 양식에 바로 넣을지(봇 프롬프트 수정) — 측정 ④와 교육을 동시에 켜는 가장 싼 한 수
