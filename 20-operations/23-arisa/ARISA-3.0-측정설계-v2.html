ARISA 3.0 · 측정설계 v2 · 2026-06-20

해상도는 '요소를 얼마나·얼마나 깊게' 채웠는가다 — 업무의 무게에 맞게

v1은 해상도를 추상 6지표 평균으로 봤다. v2는 사용자 관점으로 다시 정의한다: 좋은 보고 = 그 업무가 요구하는 구성요소가, 업무의 중요성·난이도에 맞게, 구체적이고 핵심이 명확하게 채워진 것. 즉 해상도 = 커버리지(요소 충족) × 깊이(구체성)를 중요도·난이도로 가중해 기대 대비 상대값으로 잰다.

v1 대비 변경: 지표 ①(Thinking Resolution) 전면 재작성. 요소 사전·유형별 기대셋·가중 모델 신설. 기존 6지표는 폐기가 아니라 흡수(요소별 깊이 채점의 하위 렌즈로 재배치). 지표 ②③④·캘리브레이션은 v1 유지·정합 패치. 산출물 비교 보존: ARISA-3.0-측정설계-v1.html.

3 · 중요도·난이도 가중 — 기대 깊이를 동적으로신설 · 직원표시+ARISA보정

유형이 "어떤 요소"를 정하면, 중요도·난이도가 "얼마나 깊게"를 정한다. 같은 project라도 사소한 건과 회사 명운이 걸린 건의 기대 깊이는 다르다.

입력 — 직원 자가표시
보고 시 업무에 중요도 H/M/L · 난이도 H/M/L 라이트 태그(2탭). 현재 core_indices(GPT가 복잡/시간 추천 :224)를 명시 태그로 승격.
보정 — ARISA
내용 근거로 상·하향. 불일치 시 되묻기: "이 업무를 핵심으로 봐도 될까요?" fidelity 사실확인 게이트와 동일 메커니즘 — 추측 단정 금지.
expected_depth(요소) = base(유형) × w(중요도) × w(난이도) // 기대 깊이 w(H)=1.0 · w(M)=0.7 · w(L)=0.4 // 중요·고난도일수록 더 깊이 기대, 더 많은 요소 활성 저중요·저난도 → 기대셋 자동 축소(선택요소 비활성) = 과잉보고 2차 방지

fidelity 균형: 가중은 "더 캐묻기"를 정당화하는 장치가 아니다. 중요·고난도일 때만 깊이를 더 기대하고, 사소한 업무엔 기대를 낮춰 질문을 줄인다(질문 폭탄 금지·Mirror 80% 보존). 가중의 1차 목적은 채점 정밀화가 아니라 "어디에 사고를 더 쓸지" 신호.

4 · 해상도 2축 산식 — 지표 ① 전면 재작성6지표 흡수

지표 ① · Thinking Resolution

기대한 만큼, 기대한 깊이로 채워졌는가

즉시 측정(태그 제외)
Coverage
(완전성)
해당 유형·중요도의 기대 요소 중 실제 채워진 비율(가중)
coverage = Σ(채워진 요소 × 요소가중) / Σ(기대 요소 × 요소가중)
Depth
(구체성·핵심)
채워진 요소별 구체성 0~3 평균. §1 요소 사전의 "0→3 갈림"이 앵커. 핵심 요소(리스크·의사결정 등) 가중
depth = Σ(요소깊이 0~3 × 요소가중) / Σ(채워진 요소 × 요소가중)
해상도
기대 대비 상대값 — 절대점 비교 금지
resolution = (coverage × depth) / expected_depth // §3 expected_depth로 정규화
6지표 흡수
기존 clarity/evidence/structure/output_q/outcome_q/decision_q는 폐기 아님 → 요소별 Depth를 채점하는 하위 렌즈로 재배치:
전역clarity·structure·evidence = 모든 요소의 깊이 0~3 판정 기준
국소output_q→산출물 요소 · outcome_q→의미/기대효과 · decision_q→의사결정 요소
출력
직원별 요소 커버리지 맵 + 깊이 레이더 + 4주 해상도 기울기(=성장률). 대표는 추세만, 일별 점수 비노출
fidelity: 채점은 항상 raw 원문 기준(정리본 채점 금지=순환오류). 근거 불충분 시 추측 채점 금지(null). 점수는 평가가 아니라 관찰 — 직원에겐 점수 대신 거울(정리본)만.

5 · Engine B(Completion) 통합 — 측정 = 교육신설

같은 요소 사전을 채점(측정)과 질문생성(교육)이 공유한다. 측정에서 비거나 얕은 요소가 곧 ARISA가 물을 항목이 된다.

현재(이진)v2 추가 차원
공란·더미·동어반복·추상어 탐지(_rule_based_gaps:543 / completion_check:559)요소 누락 × 중요도 가중 — 중요·고난도인데 리스크·일정·의사결정이 비면 우선 질문. 저중요 업무엔 묻지 않음
충분/부족 단정얕은 요소(깊이 0~1)를 "한 단계 더" 유도하는 질문(해당 요소의 0→3 갈림 사용)

제약 유지: 최대 3질문 · Mirror 80% · 추측 금지. 가중 덕에 질문이 중요 업무에 집중되어, 캐묻되 폭탄이 되지 않는다. — 측정 루브릭과 질문이 한 사전을 공유하므로 "재는 것"이 곧 "가르치는 것".

v2가 확정하는 것 · 남기는 결정

  • 해상도 재정의 완료: 추상 6지표 평균 → 커버리지 × 깊이 ÷ 기대깊이, 유형별 기대셋 + 중요도·난이도 가중. 6지표는 깊이 채점 렌즈로 흡수

  • 남는 결정 1: 요소 사전·유형 기대셋을 봇 프롬프트(STRUCTURE/COMPLETION)에 실제 반영할지 → Yes면 다음 산출물 = 프롬프트 개정 스펙 + 자가표시 UI 1탭
