# Basket 운영 DB 시트 스키마 v1

> 일일·주간 업무관리 매뉴얼 / 표준 일일보고 양식을 **구글시트·노션 입력 테이블**로 떨군 스키마.
> 지금은 손으로 채워 쓰고, 이후 RX Operations Bot이 이 테이블을 읽어 "오늘 할 일"·리포트를 만든다.
> 출처: 양은정 운영자료 6종(현행 충실 통합). 2026-06-21.

## 1. Routine Task (루틴 업무 마스터)

매장의 모든 반복 업무 1행 = 1업무. 봇이 주기·시점으로 "오늘 할 일"을 생성하는 근거.

| 컬럼 | 타입 | 값 / 예시 |
|------|------|-----------|
| `task_id` | text | RT-001 … |
| `task_name` | text | 그룹헤드별 샷 1회 추출 |
| `biz` | enum | 카페 / 편의점 / 대관 / 공간·시설 |
| `cycle` | enum | 일 / 주 / 격주 / 월 / 상시 / 비정기 |
| `weekday` | enum? | (주간일 때) 화·수·목·금·토 / (없으면 공란) |
| `time_slot` | enum | AM(오픈) / 영업중 / PM(마감) / 상시 |
| `owner` | text | A / B / 1인근무자 / 점장 |
| `done_criteria` | text | 완료 판단 기준(됨/아직) |
| `linked_sop` | text? | 연결 SOP 부록 항목 |
| `note` | text? | 비고 |

**시드 데이터(발췌, 매뉴얼에서 추출)**

## 2. Daily Report (일일 보고)

표준 일일보고 양식 1건 = 1행. 배열형 컬럼은 시트에선 줄바꿈 텍스트, 노션/DB에선 멀티라인.

| 컬럼 | 타입 | 비고 |
|------|------|------|
| `date` | date | YYMMDD |
| `sales_ours` | number | 매출 — 우리매장 |
| `sales_royal` | number | 매출 — 바스켓 로얄 |
| `sales_gs` | number | 매출 — GS타워 |
| `expense_total` | number | 지출 총합 |
| `approval_needed` | list | ③ 송금·승인 급한 건(1차 승인자) |
| `notes` | list | ④ 특이사항 |
| `equipment` | list | ⑤ 장비 |
| `work_log` | list | ⑥ 업무 보고(완료/진행/예정) |
| `rental` | list | ⑦ 대관 |
| `staff` | list | ⑧ 스태프 |
| `purchase` | list | ⑨ 구매(고위드/법인카드) |
| `tenant_proposal` | list | ⑩ 입점 제안 |
| `reflection` | text? | ⑪ 복기(REM, 선택) |
| `author` | text | 작성자 |
| `uploaded_at` | datetime | 16:30 운영방 업로드 시각 |

**파생/집계(대시보드용)**
- 주간 매출 = `sales_ours/royal/gs` 요일 합(토요일 주간보고 자동 채움)
- `approval_needed` 미해결 건 → Executive 대시보드 Decision Queue
- `equipment` 반복 → 장비 교체·구매 의사결정 신호

## 3. (참고) Vendor · Product 컬럼 초안

향후 발주·유통기한 자동화를 위한 최소 스키마.

**Vendor**

| 컬럼 | 예시 |
|---|---|
| `vendor_id` / `name` | 쿠팡 로켓프레시 / 블러썸(커피머신) |
| `items` | 발주 가능 품목 |
| `order_method` | 쿠팡앱 / 전화 / 메일 |
| `order_day` | 화요일 |
| `lead_time` | 1일 |
| `contact` | 담당자·연락처 |

**Product (편의점·카페 재고)**

| 컬럼 | 예시 |
|---|---|
| `product_id` / `name` | — |
| `category` | 음료 / 스낵 / 원두 / 소모품 |
| `vendor_id` | → Vendor |
| `min_stock` | 최소 재고량 |
| `expiry` | 유통기한 |
| `discount_rule` | D-7 할인 |
| `dispose_rule` | 폐기 분류·등록 |

> 유통기한 관리(D-30 / D-14 / D-7 할인 / 폐기)는 `expiry` 기준 자동 알림으로 연결 예정.
