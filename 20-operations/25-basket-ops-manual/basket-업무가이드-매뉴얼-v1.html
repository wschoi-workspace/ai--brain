Part 0

왜 이렇게 일하는가

일을 시작하기 전에 "무엇을 하는가"보다 "왜 다르게 하는가"를 먼저 안다. 납득하지 않으면 깊게 움직이지 않는다.

0-1철학 (L0)

"우리는 운영하지 않는다 — 경험을 설계한다."

Basket은 커피·상품을 파는 곳이 아니라 사람의 인식을 설계하는 매체다. 같은 시간을 일해도, 우리는 "사람들이 왜 멈추고, 왜 찍고, 왜 다시 오는가"를 현장에서 본다. 그래서 우리의 약속은 "더 열심히"가 아니라 "더 깊게"다. 일을 늘리는 게 아니라 눈을 바꾼다.

스타벅스에서 가져온 것 — 실행의 정밀함

  • 이름 붙은 루틴 — 모든 반복 작업을 'OO 루틴'으로 표준화
  • "왜"까지 — 단계 + 핵심포인트 + 이유
  • 됨 / 아직 — 점수가 아니라 관찰형 이분법으로 평가
  • 무한 변수 → N스텝 — 컴플레인은 공식으로 압축

REM에서 더한 것 — 학습의 깊이

  • 운영 = 경험 설계 — 현장 반응을 읽는다
  • 운영 = 학습 루프 — 실행→관찰→기록→복기→개선
  • 봇은 감시가 아니라 사고 파트너 — "다음엔 뭘 바꿔볼까?"
  • 경험 설계자 육성 — 단순 숙련공이 아니라 해석가

0-2운영 원칙 (L1) — 스스로 판단하는 기준

모든 업무는 이 우선순위로 판단한다. 위 순위가 아래 순위에 앞선다.

  • 1순위 · 고객 응대 및 고객 경험 — 모든 업무보다 우선. 입장 시 즉시 인사, 주문·문의·요청 신속 처리, 매장 상태 수시 확인.
  • 2순위 · 대표·매니저가 전달한 당일 업무 — 당일 업무 확인 → 우선순위 파악 → 운영 지장 없는 범위에서 수행 → 완료 후 공유·보고.
  • 3순위 · 메뉴 및 매장 개선(디벨롭) — 메뉴 퀄리티 개선점 발굴, 고객 피드백 수집·공유, 운영 효율·환경 개선 제안.
  • 4순위 · 정리정돈 및 기타 — 청결 유지, 정리정돈 생활화, 기타 운영 업무.

판단이 애매할 때의 기준선

  • 고객 경험 > 규정 — 규정이 경험을 해치면, 멈추고 매니저에게 묻는다.
  • 기록되지 않은 노하우는 자산이 아니다 — "나는 이렇게 해요"가 아니라 "우리의 표준은 무엇인가"로 참여한다.
  • 컴플레인 즉시 보상 — 객단가 이내는 직원 재량 YES YES 한도 금액 확인 필요. 초과 시 매니저.
절대 원칙 — 커피 머신은 전원을 절대 끄지 않는다. 마감 시 머신을 제외한 전원(조명·에어컨·DID·핫 디스펜서)만 OFF. 그라인더만 OFF.

3-3REM 복기 & SOP↔루틴↔봇 연결

SOP = "왜·어떻게"의 표준 지식. 평소엔 안 펼쳐 봐도 된다. 막히거나 신입일 때 펼친다.

루틴 카드 = 매 시프트 '오늘 할 일'. SOP의 압축 실행판. 체크박스 + SOP 링크 + REM 질문.

봇 = 둘의 인터페이스. 시각 트리거로 루틴 카드를 띄우고, 질문엔 SOP로 답하고, 마감엔 복기 3문을 물어 Daily Report를 만든다.
