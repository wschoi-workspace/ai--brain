# IR Deck V8 설계 — Sequoia + Hybrid 통합

> 설계일: 2026-06-07
> 원칙: Core 15장(VC가 2분 안에 스캔) + Appendix(관심 시 추가 열람)
> 기반: Sequoia 10-Slide + YC Seed Deck + Hybrid 10-Step + V7.2 데이터

---

## 설계 원칙

1. **VC는 2분 18초 안에 판단한다** — 15장이 한계. 모든 장이 1메시지.
2. **첫 3장에서 "이게 뭐고 왜 중요한지" 완결** — WHY + WHAT + WHY NOW
3. **Airbnb 공식**: 한 줄 정의 + 감정적 Problem + 간결한 Solution + Traction
4. **하이브리드 공식은 발표 시에만** — 덱 자체는 Sequoia 골격, 발표할 때 감정 곡선 적용

## Core Deck 15장

### s01 — Cover + 한 줄 정의

**메시지**: WHY + WHAT 동시에. Airbnb의 "Book rooms with locals, rather than hotels" 공식.

```
PROJECT RENT

"모든 매장이 미디어가 될 수 있다고 믿습니다."

오프라인 경험 데이터 플랫폼
— 팝업을 미디어 채널로, 경험을 데이터 자산으로

IR DECK 2026 · CONFIDENTIAL
```

**설계 포인트**: V7.2는 WHY만 있고 WHAT이 없었음. 투자자는 3초 안에 "뭐하는 회사인지" 알아야 함. WHY 1줄 + WHAT 1줄 동시 배치.

### s02 — Problem: 오프라인은 블랙박스다

**메시지**: 투자자가 "아, 이 문제 진짜 크다"고 느끼는 순간.

```
소비자 지출의 80%는 오프라인에서 일어난다.
마케팅 성과 측정의 90%는 온라인에서만 가능하다.

이 불일치 = 10조원 시장이 '감'으로 집행되고 있다는 뜻.

┌─────────────┬─────────────┐
│ 브랜드       │ 광고주       │
│ 팝업에 수억   │ 오프라인 ROI │
│ 쓰지만 ROI를  │ 정량화 방법  │
│ 모른다       │ 없음         │
├─────────────┼─────────────┤
│ 소비자       │ 부동산       │
│ 매력적 경험이 │ 건물은 있는데│
│ 없어서 안 나옴│ 상권이 죽어감│
└─────────────┴─────────────┘

OOH 4.3조 + 카페 6조 = 10조원 (성과 측정 0)
```

**설계 포인트**: V7.2의 s04~s05(2장)를 1장으로 압축. "80% vs 90%" 한 줄이 이 슬라이드의 전부여도 됨. Airbnb의 Problem 슬라이드처럼 감정적으로.

### s03 — Market: TAM/SAM/SOM

**메시지**: "이 시장은 충분히 크다."

```
글로벌 리테일 미디어 시장
$3,120억 (2030, Forrester)         ← TAM

한국 오프라인 마케팅 + 부동산 시장
OOH 4.3조 + 카페 6조 = 10조+       ← SAM

PR이 3년 내 접근 가능한 시장
리테일미디어 서비스 + RXR SaaS
= 100~300억                        ← SOM

[참고] Amazon Ads: $450억 (온라인 데이터의 힘)
       오프라인에는 아직 Amazon Ads가 없다.
```

**설계 포인트**: V7.2에는 독립 시장 규모 슬라이드가 없었음. TAM/SAM/SOM을 1장에 보여주는 것이 VC의 기본 기대.

**⚠️ 내용 필요**: SOM(100~300억)의 근거. 현재 40억 매출에서 3년 내 300억까지 가는 논리적 경로. → 매장 확장(10개소 × 10억) + RXR SaaS(구독 N사) + 커머스로 설명 가능. 숫자를 확정해야 함.

```
3가지가 동시에 일어나고 있다:

① 팝업 폭발: 2024년 3,077개 (+80% YoY)
   → 오프라인 마케팅 수요가 사상 최대

② 리테일 미디어 부상: 글로벌 $7,090억
   → BUT 90%가 온라인. 오프라인 RMN = 아직 0

③ AI 자체개발 가능 시대

   → 과거에는 수십억 개발비가 필요했던 것을
      지금은 CEO가 직접 만든다

선점자가 데이터를 먼저 쌓으면, 후발주자는 따라잡을 수 없다.

### s12 — Unit Economics: 매장 1개소 = 10억

**메시지**: "스케일의 단위가 명확하다."

```
매장 1개소당 연 매출 모델 (성수동 검증 기반)

공간 서비스     4억  ████████████████
팝업 RaaS      2억  ████████
F&B/커머스     3억  ████████████
RXR/데이터     1억  ████
─────────────────────────
합계           ~10억

2028 목표:
매장 10개소 × 10억 = 100억
+ OMO 커머스      =  80억
+ 부동산/시공      =  80억
+ Retail Tech     =  50억
────────────────────────
합계              = 310억
```

**설계 포인트**: V7.2의 s28을 유지. 투자자가 가장 좋아하는 슬라이드 = "N × M = 매출" 공식이 명확한 것.

**⚠️ 내용 필요**: "성수동 검증 기반"이라고 했는데, 실제 성수 매장의 연 매출이 10억인지 확인 필요. 2024년 매출 40억 / 2개소 = 20억/개소? 아니면 다른 수입이 포함? 여기 숫자가 투자자 DD(Due Diligence)에서 첫 번째로 검증할 포인트.

### s14 — The Ask: 투자 제안

**메시지**: "얼마를 원하고, 어디에 쓰고, 얼마를 만들 것인가."

```

─────────────────────

투자 안전장치:
