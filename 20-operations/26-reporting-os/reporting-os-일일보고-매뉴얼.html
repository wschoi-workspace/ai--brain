028항목 체크리스트

#영역항목왜 쓰는가
1맥락프로젝트명어떤 일 이야기인지 즉시 파악
2맥락오늘 가장 중요한 업무 + 왜맥락 복원 한 줄이면 끝
3진행오늘 얻은 결과 (산출물)"했다"가 아니라 "뭐가 나왔다" — 명사
4판단그래서 어디까지 온 건가 (의미)산출물이 만든 변화, 목표 대비 위치
5판단사실인가, 내 의견인가근거 구분이 있어야 판단 가능
6리스크발견한 문제사실 + 영향 + 대응. "없음"도 OK
7우선순위내일 가장 중요한 일10개 나열 말고, 최중요 1개에 ★
8요청관리자에게 필요한 것선택지 + 추천안 + 기한. "없음"도 OK

03기억할 공식 3개

#1~2 맥락 쓰기

[프로젝트명] : [무엇을] — [왜 / 무엇을 위해]

Before

"도면 작업 중"

After

"세스크멘슬: 주방가구 도면 정리 — 내일 주방업체 미팅 자료"

#3~4 산출물 + 의미

산출물 이름 + 수량 (명사) → 이제 [무엇]이 가능

Before

"리서치 진행함"

After

"업체별 단가·제작기간 비교표 1건 → 목요일 내부 논의 근거 확보"

Before

"회의 참석함"

After

"클라이언트 미팅 → 합의사항 3건 + 수정요청 리스트 → 다음 시안 반영 가능"

#8 요청 쓰기

판단 요청: 선택지(2개 이상) + 추천안(내 의견+근거) + 기한(언제까지)

지원 요청: 무엇이(사실) + 언제까지(기한) + 왜(안 되면 무슨 영향)

Before

"대표님 컨펌 필요."

After

"공간 규모 100평 vs 255평 · 총용역비 배분 확정 필요 — 수요일까지 (설치·물류 일정 직결)"

04사실과 의견 분리

한 문장에 사실과 추측이 섞여 있으면, 읽는 사람이 판단할 수 없습니다.

Before — 섞인 문장

"클로드가 막히면서 늦게 시작했고, 토큰이 모자라서 마지막 자료를 잘 정리하지 못했어"

사실

클로드 토큰 소진 → 약 2시간 작업 중단

의견

토큰 한도 안 늘면 이번 주 반복될 것

요청

팀 공용 AI 토큰 확충 검토 (이번 주 내)

05리스크 3요소

사실 + 예상 영향 + 대응

공란 = 생각 안 함  |  "없음" = 생각한 결과, 리스크가 없다는 판단

06제출 전 셀프체크 (1분)

  • 프로젝트명과 "왜 오늘 이게 중요한지"가 적혀 있는가?
  • 산출물이 명사로 되어 있는가? ("~했다"가 아니라 "~표", "~안", "~건")
  • 내가 쓴 문장에서 사실과 의견이 구분되는가?
  • 리스크·요청 항목에 "없음"이라도 적었는가? (공란이 아닌지)
  • 관리자가 이걸 읽고 추가 질문 없이 판단할 수 있는가?

08실전 사례 비교 — 같은 업무, 다른 보고

같은 업무를 어떻게 보고하느냐에 따라, 읽는 사람의 판단 가능 여부가 완전히 달라집니다.

사례 1 — 제작 견적 리서치

해상도 낮은 보고

"VIP 패키지 제작 관련 리서치 진행함. 견적 받아봄. 정리 중."

→ 무엇을 리서치했는지, 어디까지 왔는지, 다음에 무엇을 해야 하는지 판단 불가

해상도 높은 보고 (실제 사례 기반)

"VIP 패키지 제작 기획안 — 제작방식 비교 정리본, 오동나무/허니콤보드 상세 견적 비교 자료, ALT별 예상 제작 예산표 정리.
각 자료에 업체별 단가·제작기간·샘플비·장단점·리스크·7/30 납품 기준 타임라인 정리.
여러 제작안을 넓게 검토하기보다 상하뚜껑형 베이직 기준으로 재견적, 허니콤보드 샘플 먼저 검증하는 방향으로 좁힘."

→ 산출물(4건) + 의미(방향이 좁혀짐) + 해결방향(베이직 기준 재견적) 모두 있어 즉시 판단 가능

사례 2 — 현장 설치 업무

해상도 낮은 보고

"브루비 설치 → 현장 인수인계 / 설치 팔로업
가구 이슈 정리 / 가구 내부 배선 타공 팔로업
주방가구 도면 정리"

→ 한 일 나열만. 문제가 있었는지, 다음에 뭘 해야 하는지, 대표가 정해줄 것이 있는지 모름

이렇게 쓰면 판단 가능

"브루비 설치 완료 — 현장 인수인계 마침. 배선 타공 위치 2곳 클라이언트 확인 필요 (목요까지).
주방가구 도면 1차 완료 → 내일 주방업체 미팅 자료로 사용 예정.
문제: 가구 내부 배선 타공 규격이 기존 도면과 불일치 — 재실측 후 금요일까지 수정본 전달."

→ 산출물 + 의미 + 문제(구체적) + 일정이 있어 추가 질문 불필요

사례 3 — 운영/관리 업무

해상도 낮은 보고

"정리 중 ??
대관 일정 재방문 및 점검
기획안 예산 수정안 전달 완료
네이버 페이 폼 서칭"

→ "정리 중 ??" — 무엇을 정리 중인지 불명. 예산이 어떻게 바뀌었는지, 폼 서칭 결과가 뭔지 판단 불가

이렇게 쓰면 판단 가능

"기획안 예산 수정안 전달 완료 — 기존 350만→280만으로 조정(인쇄비 삭제, 디지털 전환).
네이버 페이 폼 3개 서비스 비교 → 무료 플랜으로 충분, 내일 세팅 예정.
요청: 런칭 일정 딜레이 여부 — 현재 기준(7/20) vs 1주 연기(7/27) 중 결정 필요 (컬쳐마켓 섭외 일정 직결)"

→ 의미(금액 변화) + 해결방향(무료 플랜) + 의사결정 요청(선택지+기한+영향) 완비

좋은 보고의 공통점

1. 산출물 다음에 "그래서"가 있다 — 뭐가 달라졌는지, 어디까지 온 건지
2. 문제가 구체적이다 — "이슈 있음"이 아니라 "무엇이 왜 문제이고 언제까지 해결"
3. 대표가 정해줄 것이 명확하다 — 선택지 + 추천 + 기한 + 늦으면 어떤 영향
